წვლილის სახელმძღვანელო¶

მივესალმებით esp-idf პროექტში შემოწირულობებს!

როგორ შევიტანოთ წვლილი¶

esp-idf-ში წვლილის შეტანა - შეცდომების გამოსწორება, ფუნქციების დამატება, დოკუმენტაციის დამატება - მისასალმებელია. ჩვენ ვიღებთ წვლილის შეტანას შემდეგი გზით: Github-ის Pull-ის მოთხოვნები.

წვლილის შეტანამდე¶

Pull Request-ის გამოგზავნამდე, გთხოვთ, გაითვალისწინოთ შემდეგი პუნქტების სია:

  • წვლილი მთლიანად თქვენი ნამუშევარია თუ უკვე ლიცენზირებულია Apache License 2.0-თან თავსებადი ღია კოდის ლიცენზიით? თუ არა, სამწუხაროდ, ჩვენ მას ვერ მივიღებთ.
  • რაიმე ახალი კოდი შეესაბამება esp-idf-ს? სტილის სახელმძღვანელო?
  • კოდის დოკუმენტაცია შეესაბამება თუ არა მოთხოვნებს? კოდის დოკუმენტირება?
  • კოდი საკმარისად არის კომენტირებული, რათა ადამიანებმა გაიგონ, თუ როგორ არის სტრუქტურირებული?
  • არსებობს თუ არა დოკუმენტაცია ან მაგალითები, რომლებიც კოდში შეტანილ წვლილს მოჰყვება? არსებობს დამატებითი რჩევები კარგი მაგალითების დასაწერად. მაგალითები წაიკითხე.
  • კომენტარები და დოკუმენტაცია დაწერილია გასაგებ ინგლისურ ენაზე, ორთოგრაფიული და გრამატიკული შეცდომების გარეშე?
  • ასევე მისასალმებელია წვლილის ნიმუშები. გთხოვთ, შეამოწმოთ მაგალითების შექმნა სახელმძღვანელო ამათთვის.
  • თუ წვლილი შეიცავს რამდენიმე კომიტს, დაჯგუფებულია თუ არა ისინი ლოგიკურ ცვლილებებად (თითო pull request-ზე ერთი მნიშვნელოვანი ცვლილება)? არის თუ არა რაიმე კომიტი ისეთი სახელწოდებით, როგორიცაა „შეცდომის გამოსწორება“. წინა კომიტებში ჩაჭედილი?
  • თუ რომელიმე პუნქტთან დაკავშირებით არ ხართ დარწმუნებული, გთხოვთ, მაინც გახსნათ Pull Request და შემდეგ გვთხოვოთ გამოხმაურება.

მოთხოვნის გატანის პროცესი¶

Pull Request-ის გახსნის შემდეგ, სავარაუდოდ, თავად მოთხოვნის კომენტარების ველში იქნება გარკვეული განხილვა.

როგორც კი Pull Request მზად იქნება გაერთიანებისთვის, ის თავდაპირველად გაერთიანდება ჩვენს შიდა git სისტემაში შიდა ავტომატიზირებული ტესტირებისთვის.

თუ ეს პროცესი წარმატებით დასრულდება, ის გაერთიანდება საჯარო github საცავში.

იურიდიული ნაწილი¶

წვლილის მიღებამდე, თქვენ უნდა მოაწეროთ ხელი ჩვენს კონტრიბუტორის შეთანხმებაამის მოთხოვნა ავტომატურად მოგეთხოვებათ Pull Request პროცესის ფარგლებში.

დოკუმენტაცია, დამატებები და გაფართოებები, მითითება¶

ეს დოკუმენტაცია შექმნილია გამოყენებით სფინქსი აპლიკაცია, რომელიც ტექსტის წყაროს ფაილებს ამუშავებს გადასტრუქტურირებული ტექსტი (.rst) ფორმატი, რომელიც მდებარეობს დოკუმენტები დირექტორია. ამ პროცესის შესახებ დამატებითი ინფორმაციისთვის იხილეთ განყოფილება კოდის დოკუმენტირება.

Sphinx-ის გარდა, არსებობს კიდევ რამდენიმე აპლიკაცია, რომლებიც ხელს უწყობენ კარგად ფორმატირებული და მარტივად ნავიგაციისთვის განკუთვნილი დოკუმენტაციის უზრუნველყოფას. ეს აპლიკაციები ჩამოთვლილია განყოფილებაში შენობის დოკუმენტაციის ადგილობრივად დაყენება ფაილში მოცემული დაინსტალირებული ვერსიის ნომრებით docs/requirements.txt.

ამასთანავე, ჩვენ შევქმენით რამდენიმე მორგებული დამატება და გაფართოება, რათა დაგვეხმაროს დოკუმენტაციის ხაზგასმასთან ინტეგრაციაში. ESP-IDF საცავის შექმნა და ნავიგაციისა და დოკუმენტაციის მოვლა-პატრონობის შემდგომი გაუმჯობესება.

ამ განყოფილების მიზანია დამატებებისა და გაფართოებების შესახებ სწრაფი ინფორმაციის მიწოდება.

დოკუმენტაციის საქაღალდის სტრუქტურა¶

  • ESP-IDF საცავი შეიცავს სპეციალურ დოკუმენტაციის საქაღალდეს დოკუმენტები ფესვში.
  • ის docs საქაღალდე შეიცავს ლოკალიზებულ დოკუმენტაციას დოკუმენტები/en (ინგლისური) და დოკუმენტები/zh_CN (გამარტივებული ჩინური) ქვესაქაღალდეები.
  • ლოკალიზებული დოკუმენტაციისთვის საერთო გრაფიკული ფაილები და შრიფტები შეიცავს დოკუმენტები/_სტატიკური ქვესაქაღალდე
  • დარჩენილი ფაილები root-ში docs ასევე docs/en და docs/zh_CN უზრუნველყოს კონფიგურაცია და სკრიპტები, რომლებიც გამოიყენება დოკუმენტაციის დამუშავების ავტომატიზაციისთვის, მათ შორის დამატებებისა და გაფართოებების ჩათვლით.
  • დოკუმენტაციის შექმნისას დინამიურად გენერირდება რამდენიმე საქაღალდე და ფაილი, რომლებიც ძირითადად განთავსებულია docs/[lang]/_build საქაღალდეები. ეს საქაღალდეები დროებითია და არ ჩანს ESP-IDF საცავი,

დამატებებისა და გაფართოებების მითითება¶

docs/conf_common.py
This file contains configuration common to each localized documentation (e.g. English, Chinese). The contents of this file is imported to standard Sphinx configuration file conf.py located in respective language folders (e.g. docs/en, docs/zh_CN) during build for each language.
docs/check_doc_warnings.sh
If there are any warnings reported during documentation build, then the build is failed. The warnings should be resolved before merging any documentation updates. This script is doing check for warnings in respective log file to fail the build. See also description of sphinx-known-warnings.txt below.
docs/check_lang_folder_sync.sh

პარალელური ენის ვერსიის შენარჩუნებისას პოტენციური შეუსაბამობების შესამცირებლად, ენის საქაღალდეების სტრუქტურა და ფაილების სახელები docs/en და docs/zh_CN საქაღალდეები იდენტური უნდა იყოს. სკრიპტი check_lang_folder_sync.sh მუშაობს თითოეულ დოკუმენტაციაზე, რათა დადასტურდეს, დაკმაყოფილებულია თუ არა ეს პირობა.

შენიშვნა

თუ ახალი შინაარსი მოწოდებულია, მაგალითად, ინგლისურ ენაზე და ჯერ არ არის თარგმანი, მაშინ შესაბამისი ფაილი zh_CN საქაღალდე უნდა შეიცავდეს .. include:: დირექტივა, რომელიც მიუთითებს წყაროს ფაილზე ინგლისურ ენაზე. ეს ავტომატურად მოიცავს ჩინელი მკითხველებისთვის ხილულ ინგლისურ ვერსიას. მაგალითად, თუ ფაილი docs/zh_CN/contribute/documenting-code.rst თუ ჩინური თარგმანი არ აქვს, მაშინ ის უნდა შეიცავდეს .. include:: ../../en/contribute/documenting-code.rst სამაგიეროდ.

docs/docs_common.mk
It contains the common code which is included into the language-specific Makefiles. Note that this file contains couple of customizations comparing to what is provided within standard Sphinx installation, e.g. gh-linkcheck command has been added.
docs/gen-dxd.py
A Python script that generates API reference files based on Doxygen xml output. The files have an inc extension and are located in docs/[lang]/_build/inc directory created dynamically when documentation is build. Please refer to კოდის დოკუმენტირება and API დოკუმენტაციის შაბლონი, section API მითითება for additional details on this process.
docs/ ინსტრუმენტების ნაკრები
There couple of places in documentation that provide links to download the toolchain. To provide one source of this information and reduce effort to manually update several files, this script generates toolchain download links and toolchain unpacking code snippets based on information found in ინსტრუმენტები/ ინსტრუმენტების ნაკრები _versions.mk.
docs/gen-version-specific-includes.py
Another Python script to automatically generate reStructuredText Text .inc snippets with version-based content for this ESP-IDF version.
docs/html_redirects.py
During documentation lifetime some source files are moved between folders or renamed. This Python script is adding a mechanism to redirect documentation pages that have changed URL by generating in the Sphinx output static HTML redirect pages. The script is used together with a redirection list html_redirect_pages defined in file docs/conf_common.py.
docs/link-roles.py
This is an implementation of a custom სფინქსის როლები to help linking from documentation to specific files and folders in ESP-IDF. For description of implemented roles please see დაკავშირების მაგალითები and ენობრივი ვერსიების დაკავშირება.
docs/local_util.py
A collection of utility functions useful primarily when building documentation locally (see შენობის დოკუმენტაციის ადგილობრივად დაყენება) to reduce the time to generate documentation on a second and subsequent builds. The utility functions check what Doxygen xml input files have been changed and copy these files to destination folders, so only the changed files are used during build process.
docs/sphinx-known-warnings.txt
There are couple of spurious Sphinx warnings that cannot be resolved without doing update to the Sphinx source code itself. For such specific cases respective warnings are documented in sphinx-known-warnings.txt file, that is checked during documentation build, to ignore the spurious warnings.
ინსტრუმენტები/gen_esp_err_to_name.py
This script is traversing the ESP-IDF directory structure looking for error codes and messages in source code header files to generate an .inc file to include in documentation under შეცდომის კოდების მითითება.
ინსტრუმენტები/kconfig_new/confgen.py
Options to configure ESP-IDF’s კომპონენტი სემესტრი are contained in Kconfig files located inside directories of individual components, e.g. კომპონენტი s/bt/Kconfig. This script is traversing the component directories to collect configuration options and generate an .inc file to include in documentation under კონფიგურაციის პარამეტრების მითითება.

ESP32-Ethernet-Kit V1.0-ის დაწყების სახელმძღვანელო¶

[中文]

ეს სახელმძღვანელო გაჩვენებთ, თუ როგორ დაიწყოთ ESP32-Ethernet-Kit-ის დეველოპერული დაფის გამოყენება და ასევე გთავაზობთ ინფორმაციას მისი ფუნქციონალურობისა და კონფიგურაციის ვარიანტების შესახებ.

ის ESP32-Ethernet-Kit არის Ethernet-to- Wi-Fi განვითარების დაფა, რომელიც საშუალებას იძლევა Ethernet მოწყობილობები ერთმანეთთან იყოს დაკავშირებული Wi-Fi საშუალებით. ამავდროულად, უფრო მოქნილი კვების წყაროს ვარიანტების უზრუნველსაყოფად, ESP32-Ethernet-Kit ასევე მხარს უჭერს Ethernet-ის საშუალებით კვების მიწოდებას (PoE).

რა გჭირდებათ¶

  • ESP32-Ethernet-Kit V1.0 დაფა
  • USB 2.0 A-დან მიკრო B-მდე კაბელი
  • კომპიუტერი, რომელიც მუშაობს Windows-ზე, Linux-ზე ან macOS-ზე

შეგიძლიათ გამოტოვოთ შესავალი ნაწილები და პირდაპირ გადახვიდეთ განყოფილებაზე აპლიკაციის შემუშავების დაწყება.

მიმოხილვა¶

ESP32-Ethernet-Kit არის ESP32-ზე დაფუძნებული განვითარების დაფა, რომელიც წარმოებულია ესპრესო.

ის შედგება ორი განვითარების დაფისგან, Ethernet დაფისგან A და PoE დაფისგან B. Ethernet დაფა (A) შეიცავს Bluetooth / Wi-Fi ორრეჟიმიან ESP32-WROVER-B მოდულს და IP101GRI-ს, ერთპორტიან 10/100 სწრაფ Ethernet გადამცემ-მიმღებს (PHY). PoE დაფა (B) უზრუნველყოფს Ethernet-ის საშუალებით კვების ფუნქციონალს. A დაფას შეუძლია დამოუკიდებლად იმუშაოს, B დაფის დაყენების გარეშე.

ESP32-Ethernet-Kit V1.0

აპლიკაციების ჩატვირთვისა და მონიტორინგისთვის, Ethernet დაფა (A) ასევე აღჭურვილია FTDI FT2232H ჩიპით - გაუმჯობესებული მრავალინტერფეისიანი USB ხიდით. ეს ჩიპი საშუალებას იძლევა გამოიყენოთ JTAG ESP32-ის პირდაპირი გამართვა USB ინტერფეისის მეშვეობით, ცალკე JTAG გამართვა გარეშე.

ფუნქციონალურობის მიმოხილვა¶

ქვემოთ მოცემულ ბლოკ-დიაგრამაზე ნაჩვენებია ESP32-Ethernet-Kit-ის ძირითადი კომპონენტი 19 და მათი ურთიერთკავშირები.

ESP32-Ethernet-Kit ბლოკ-სქემა (გადიდებისთვის დააწკაპუნეთ)

ფუნქციური აღწერა¶

ქვემოთ მოცემულ ორ ფიგურასა და ცხრილში აღწერილია ESP32-Ethernet-Kit-ის ძირითადი კომპონენტი , ინტერფეისები და მართვის საშუალებები.

Ethernet დაფა (A)¶

ESP32-Ethernet-Kit - Ethernet დაფის (A) განლაგება (დააწკაპუნეთ გასადიდებლად)

ქვემოთ მოცემულ ცხრილში მოცემულია აღწერა სურათის ზედა მარჯვენა კუთხიდან დაწყებული საათის ისრის მიმართულებით.

ძირითადი კომპონენტი აღწერა
ESP32-WROVER-B ეს ESP32 მოდული აღჭურვილია 64 მბიტიანი PSRAM-ით, რაც უზრუნველყოფს მონაცემთა შენახვისა და დამუშავების მოქნილ და გაფართოებულ შესაძლებლობებს.
GPIO სათაური 2 ხუთი დაუსახლებელი ნახვრეტიანი შედუღების ბალიში ESP32-ის არჩეულ GPIO წვდომის უზრუნველსაყოფად. დეტალებისთვის იხილეთ GPIO სათაური 2.
ნაკადის კონტროლი ჯამპერი ჰედერი დაფის სიგნალებზე წვდომით. დეტალებისთვის იხილეთ ნაკადის კონტროლი.
ფუნქციის გადამრთველი DIP გადამრთველი, რომელიც გამოიყენება ESP32-ის არჩეული GPIO 7 ფუნქციონალურობის კონფიგურაციისთვის. დეტალებისთვის იხილეთ ფუნქციის გადამრთველი.
გადაცემის/მიღების LED-ები ორი LED ინდიკატორი UART გადაცემის სტატუსის საჩვენებლად.
GPIO სათაური 3 უზრუნველყოფს წვდომას ESP32-ის ზოგიერთ GPIO 7, რომელთა გამოყენება შესაძლებელია პოზიციის მიხედვით. ფუნქციის გადამრთველი.
FT2232H FT2232H ჩიპი მრავალპროტოკოლიანი USB-სერიული ხიდის ფუნქციას ასრულებს, რომლის დაპროგრამება და კონტროლი შესაძლებელია USB-ის საშუალებით, რათა უზრუნველყოს ESP32-თან კომუნიკაცია. FT2232H-ს ასევე აქვს USB-ს JTAG ინტერფეისი, რომელიც ხელმისაწვდომია ჩიპის A არხზე, ხოლო USB-სერიული ინტერფეისი B არხზეა. FT2232H ჩიპი აუმჯობესებს მომხმარებლისთვის მოხერხებულობას აპლიკაციების შემუშავებისა და გამართვა კომუნიკაციის თვალსაზრისით. იხილეთ ESP32-Ethernet-Kit V1.0 Ethernet დაფის (A) სქემატური სქემები.
USB პორტი USB ინტერფეისი. დაფის კვების წყარო, ასევე კომპიუტერსა და დაფას შორის საკომუნიკაციო ინტერფეისი.
დენის გადამრთველი ჩართვა/გამორთვის გადამრთველი. გადართვა მიმართულებით ჩექმა ღილაკი ჩართავს დაფას, გამორთვით ჩექმა თიშავს დაფას.
5 ვოლტიანი შეყვანა 5 ვოლტიანი კვების წყაროს ინტერფეისი შეიძლება უფრო მოსახერხებელი იყოს, როდესაც დაფა ავტონომიურად მუშაობს (კომპიუტერთან არ არის დაკავშირებული).
5 ვოლტიანი ჩართვის LED ინდიკატორი ეს წითელი LED ინდიკატორი ინთება, როდესაც დაფას მიეწოდება კვება, USB-დან ან 5V შეყვანიდან.
DC/DC გადამყვანი უზრუნველყოფილია DC 5 V-დან 3.3 V-მდე კონვერტაცია, გამომავალი დენი 2A-მდე.
დაფა B კონექტორები წყვილი მამრობითი კონექტორით დასამონტაჟებლად PoE დაფა (B).
IP101GRI (ფიზიკური) ფიზიკური ფენის (PHY) კავშირი Ethernet კაბელთან ხორციელდება შემდეგი მეთოდის გამოყენებით: IP101GRI ჩიპი. PHY-სა და ESP32-ს შორის კავშირი ხორციელდება შემცირებული მედია-დამოუკიდებელი ინტერფეისის (RMII) მეშვეობით, რომელიც მედია-დამოუკიდებელი ინტერფეისის ვარიანტია. (მიუხედავად იმისა, რომ ეს აფეთქება მოხდა) სტანდარტი. PHY მხარს უჭერს IEEE 802.3 / 802.3u სტანდარტს 10/100 Mbps.
RJ45 პორტი Ethernet ქსელის მონაცემთა გადაცემის პორტი.
მაგნიტის მოდული მაგნიტები Ethernet სპეციფიკაციის ნაწილია, რომელიც იცავს მას გაუმართაობისა და გარდამავალი მოვლენებისგან, მათ შორის გადამცემ-მიმღების ინტეგრალურ სქემასა და კაბელს შორის საერთო რეჟიმის სიგნალების უარყოფისგან. მაგნიტები ასევე უზრუნველყოფს გალვანურ იზოლაციას გადამცემ-მიმღებსა და Ethernet მოწყობილობას შორის.
კავშირის/აქტივობის LED-ები ორი LED ინდიკატორი (მწვანე და წითელი), რომლებიც შესაბამისად მიუთითებენ PHY-ის „კავშირის“ და „აქტივობის“ სტატუსებზე.
ჩატვირთვის ღილაკი ჩამოტვირთვის ღილაკი. დაჭერისას დააჭირეთ. ჩატვირთვა და შემდეგ დაჭერით CH_PU იწყებს Firmware-ის ჩამოტვირთვის რეჟიმს პროგრამული უზრუნველყოფა სერიული პორტი ჩამოსატვირთად.
CH_PU ღილაკი გადატვირთვის ღილაკი.
GPIO სათაური 1 ეს კოლექტორი უზრუნველყოფს ექვს დაუსახლებელ გამჭოლ ნახვრეტში შესადუღებელ ბალიშს, რომლებიც დაკავშირებულია ESP32-ის სათადარიგო GPIO 7-თან. დეტალებისთვის იხილეთ GPIO სათაური 1.

PoE დაფა (B)¶

ეს დაფა იყენებს Ethernet კაბელის (PoE) მეშვეობით მიწოდებულ ენერგიას, რათა უზრუნველყოს Ethernet დაფის (A) კვების წყარო. PoE დაფის (B) ძირითადი კომპონენტი ნაჩვენებია ბლოკ-დიაგრამაზე ქვემოთ მოცემულ სურათზე. ფუნქციონალურობის მიმოხილვა.

PoE დაფას (B) აქვს შემდეგი მახასიათებლები:

  • IEEE 802.3at-ის მხარდაჭერა
  • გამომავალი სიმძლავრე: 5 ვ, 1.4 ა

PoE ფუნქციონალურობის გამოსაყენებლად, RJ45 პორტი Ethernet დაფის (A) კაბელით უნდა იყოს დაკავშირებული PoE-ს მხარდამჭერ კომუტატორთან. როდესაც Ethernet დაფა (A) აღმოაჩენს PoE დაფიდან (B) გამომავალ 5 ვოლტ სიმძლავრეს, USB კვება ავტომატურად გაითიშება.

ESP32-Ethernet-Kit - PoE დაფის (B) განლაგება (დააწკაპუნეთ გასადიდებლად)

ძირითადი კომპონენტი აღწერა
დაფის A კონექტორი ოთხი დედალი კონექტორის ქინძისთავი ამ დაფაზე დასამაგრებლად Ethernet დაფა (A).
გარე კვების ტერმინალები PoE დაფის (B) დამატებითი კვების წყარო.

დაყენების პარამეტრები¶

ეს განყოფილება აღწერს ESP32-Ethernet-Kit აპარატურის კონფიგურაციის ვარიანტებს.

ფუნქციის გადამრთველი¶

კონკრეტული GPIO პინების ფუნქციების არჩევა შესაძლებელია ფუნქციის გადამრთველის გამოყენებით.

1 GPIO 14-ე კურსი დაკავშირებულია FT2232H-თან JTAG ფუნქციონირების უზრუნველსაყოფად
2 GPIO 12-ე კურსი დაკავშირებულია FT2232H-თან JTAG ფუნქციონირების უზრუნველსაყოფად
3 GPIO 13-ე კურსი დაკავშირებულია FT2232H-თან JTAG ფუნქციონირების უზრუნველსაყოფად
4 GPIO 15-ე კურსი დაკავშირებულია FT2232H-თან JTAG ფუნქციონირების უზრუნველსაყოფად
5 GPIO 4 დაკავშირებულია FT2232H-თან JTAG ფუნქციონირების უზრუნველსაყოფად

ნაკადის კონტროლი¶

ეს არის 2 x 2 ჯუმპერის პინის ჰედერი, რომელიც განკუთვნილია UART ნაკადის კონტროლისთვის.

. სიგნალი კომენტარი
1 MTDO GPIO 7 13, იხილეთ აგრეთვე ფუნქციის გადამრთველი
2 MTCK GPIO 7 15, იხილეთ აგრეთვე ფუნქციის გადამრთველი
3 RTS FT2232H-ის RTS სიგნალი
4 CTS FT2232H-ის CTS სიგნალი

GPIO განაწილება¶

ეს ნაწილი აღწერს ESP32 GPIO ის განაწილებას ESP32-Ethernet-Kit-ის კონკრეტულ ინტერფეისებზე ან ფუნქციებზე.

IP101GRI (PHY) ინტერფეისი¶

ESP32 (MAC) პინების IP101GRI (PHY)-ზე განაწილება ნაჩვენებია ქვემოთ მოცემულ ცხრილში. ESP32-Ethernet-Kit-ის იმპლემენტაციისთვის ნაგულისხმევად გამოიყენება შემცირებული მედია-დამოუკიდებელი ინტერფეისი (RMII).

. ESP32 პინი (MAC) IP101GRI (ფიზიკური)
RMII ინტერფეისი
1 GPIO TX_EN
2 GPIO 19-ე კურსი ტრანზაქცია[0]
3 GPIO ტექსდართვა[1]

6 GPIO 27-ე კურსი CRS_DV

სერიული მართვის ინტერფეისი

PHY-ის გადატვირთვა

შენიშვნა

REF_CLK-ის გარდა, ყველა პინის განაწილება RMII ინტერფეისი ფიქსირებულია და მისი შეცვლა არც IOMUX-ის და არც GPIO მატრიცის მეშვეობით შეუძლებელია.

GPIO სათაური 2¶

ეს სათაური შეიცავს GPIO ს სპეციფიკური MII ფუნქციონალით (გარდა GPIO 2-ისა), განსხვავებით ESP32-Ethernet-Kit დაფაზე ნაგულისხმევად დანერგილი შემცირებული მედია-დამოუკიდებელი ინტერფეისის (RMII) ფუნქციონალისგან, იხ. IP101GRI (PHY) ინტერფეისისიტუაციიდან გამომდინარე, თუ MMI გამოიყენება, კონკრეტულ Ethernet აპლიკაციებს შეიძლება დასჭირდეთ ეს ფუნქციონირება.

შენიშვნა

  1. ESP32 პინები, GPIO 16 და GPIO 7 17, არ არის გამოყოფილი ESP32-WROVER-B მოდულთან და შესაბამისად, გამოსაყენებლად მიუწვდომელია. თუ ამ პინების გამოყენება გჭირდებათ, გთხოვთ, მიამაგრეთ მოდული SPIRAM მეხსიერების გარეშე, მაგალითად, ESP32-WROOM-32D ან ESP32-SOLO-1.
  2. ფუნქციონალურობა დამოკიდებულია პარამეტრებზე ფუნქციის გადამრთველი.

შენიშვნა

  1. GPIO 2 გამოიყენება PHY-ის გარე ოსცილატორის ჩასართავად.
  2. GPIO 0 არის PHY-სთვის 50 MHz საცნობარო საათის წყარო. საათის სიგნალი თავდაპირველად ინვერსირდება გადამცემი ხაზის დაყოვნების გათვალისწინებით და შემდეგ მიეწოდება PHY-ს.
  3. იმისათვის, რომ PHY მხარეს საათის გამომავალი სიგნალმა არ იმოქმედოს GPIO 7 0-ის ჩართვის მდგომარეობაზე, PHY გარე ოსცილატორი აქტიურდება GPIO 2-ის გამოყენებით ESP32-ის ჩართვის შემდეგ.
  4. ESP32 პინები, GPIO 16 და GPIO 7 17, არ არის გამოყოფილი ESP32-WROVER-B მოდულთან და შესაბამისად, გამოსაყენებლად მიუწვდომელია. თუ ამ პინების გამოყენება გჭირდებათ, გთხოვთ, მიამაგრეთ მოდული SPIRAM მეხსიერების გარეშე, მაგალითად, ESP32-WROOM-32D ან ESP32-SOLO-1.

აპლიკაციის შემუშავების დაწყება¶

ESP32-Ethernet-Kit-ის ჩართვამდე დარწმუნდით, რომ დაფა კარგ მდგომარეობაშია და დაზიანების აშკარა ნიშნები არ აქვს.

საწყისი დაყენება¶

  1. დააყენეთ ფუნქციის გადამრთველი ზე Ethernet დაფა (A) ნაგულისხმევ პოზიციაზე გადასასვლელად ყველა გადამრთველის ჩართვით ჩართულია.
  2. ფლეშ მეხსიერება აპლიკაციის ტესტირებისა და მუშაობის გასამარტივებლად, არ დააინსტალიროთ მხტუნავები და არ დააკავშიროთ სიგნალები დაფის ჰედერებთან.
  3. ის PoE დაფა (B) ახლა მისი შეერთება შესაძლებელია, მაგრამ გარე კვების წყარო არ შეაერთოთ.
  4. დააკავშირეთ Ethernet დაფა (A) კომპიუტერთან USB კაბელით.
  5. ჩართეთ დენის გადამრთველი GND-დან 5V0 პოზიციამდე, 5 ვოლტიანი ჩართვის LED ინდიკატორი უნდა განათდეს.

ახლა კი განვითარებისკენ¶

გააგრძელეთ დაწყება, სადაც სექცია ინსტალაცია ეტაპობრივად სწრაფად დაგეხმარებათ განვითარების გარემოს დაყენებაში და შემდეგ ფლეშ მეხსიერება თქვენს დაფაზე პროექტის მაგალითის განთავსებაში.

შემდეგ სექციაზე გადადით მხოლოდ იმ შემთხვევაში, თუ წარმატებით დაასრულეთ ყველა ზემოთ ჩამოთვლილი ნაბიჯი.

Ethernet-ის მაგალითის კონფიგურაცია და ჩატვირთვა¶

განვითარების გარემოს დაყენებისა და დაფის ტესტირების შემდეგ, შეგიძლიათ კონფიგურაცია და ფლეშ მეხსიერება ის გამოყენება. ეთერნეტი/ეთერნეტი მაგალითი. ეს მაგალითი შეიქმნა Ethernet-ის ფუნქციონალურობის შესამოწმებლად. ის მხარს უჭერს სხვადასხვა PHY-ს, მათ შორის IP101GRI დაინსტალირებულია ESP32-Ethernet-Kit V1.0 დაფა.

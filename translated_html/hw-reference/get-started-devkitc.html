ESP32-DevKitC V4 დაწყების სახელმძღვანელო¶

[中文]

ეს სახელმძღვანელო გვიჩვენებს, თუ როგორ დაიწყოთ ESP32-DevKitC V4 დეველოპერული დაფის გამოყენება. ESP32-DevKitC-ის სხვა ვერსიების აღწერილობისთვის იხილეთ ESP32 აპარატურის ცნობარი.

რა გჭირდებათ¶

  • ESP32-DevKitC V4 დაფა
  • USB A / მიკრო USB B კაბელი
  • კომპიუტერი, რომელიც მუშაობს Windows-ზე, Linux-ზე ან macOS-ზე

შეგიძლიათ გამოტოვოთ შესავალი ნაწილები და პირდაპირ გადახვიდეთ განყოფილებაზე აპლიკაციის შემუშავების დაწყება.

მიმოხილვა¶

ESP32-DevKitC V4 არის მცირე ზომის ESP32-ზე დაფუძნებული დეველოპერის დაფა, რომელიც წარმოებულია ესპრესოშეყვანის/გამოყვანის პინების უმეტესობა ორივე მხარეს გაყოფილია პინების ჰედერებად მარტივი ინტერფეისისთვის. დეველოპერებს შეუძლიათ პერიფერიული მოწყობილობების მიერთება ჯამპერებით ან ESP32-DevKitC V4-ის დამონტაჟება პუდბორდზე.

მომხმარებლის ფართო სპექტრის მოთხოვნების დასაკმაყოფილებლად, ხელმისაწვდომია ESP32-DevKitC V4-ის შემდეგი ვერსიები:

  • სხვადასხვა ESP32 მოდული

    • ESP32-WROOM-32
    • ESP32-WROOM-32D
    • ESP32-WROOM-32U
    • ESP32-SOLO-1
    • ESP32-WROVER
    • ESP32-WROVER-B
    • ESP32-WROVER-I
    • ESP32-WROVER-B (IPEX)

  • მამრობითი ან მდედრობითი სქესის პინის ჰედერები.

დეტალებისთვის იხილეთ Espressif-ის პროდუქტის შეკვეთის ინფორმაცია.

ფუნქციური აღწერა¶

ქვემოთ მოცემულ ფიგურასა და ცხრილში აღწერილია ESP32-DevKitC V4 დაფის ძირითადი კომპონენტი 19, ინტერფეისები და მართვის საშუალებები.

ESP32-DevKitC V4 ESP32-WROOM-32 მოდულით შედუღებული

ძირითადი კომპონენტი აღწერა
ESP32-WROOM-32 მოდული, რომლის ბირთვიც ESP32-ია.
ინგლისური გადატვირთვის ღილაკი.
ჩექმა ჩამოტვირთვის ღილაკი. დაჭერისას დააჭირეთ. ჩექმა და შემდეგ დაჭერით ინგლისური იწყებს Firmware-ის ჩამოტვირთვის რეჟიმს პროგრამული უზრუნველყოფა სერიული პორტი ჩამოსატვირთად.
USB-დან UART-მდე ხიდი ერთი USB-UART ხიდის ჩიპი უზრუნველყოფს 3 Mbps-მდე გადაცემის სიჩქარეს.
მიკრო USB პორტი USB ინტერფეისი. დაფის კვების წყარო, ასევე კომპიუტერსა და ESP32 მოდულს შორის საკომუნიკაციო ინტერფეისი.
5 ვოლტიანი ჩართვის LED ინდიკატორი ირთვება, როდესაც USB ან გარე 5V კვების წყარო დაფას უერთდება. დეტალებისთვის იხილეთ სქემები დაკავშირებული დოკუმენტები.
შეყვანა/გამოსვლა ESP მოდულის პინების უმეტესობა დაფაზე პინების ჰედერებად არის გაყოფილი. თქვენ შეგიძლიათ დაპროგრამოთ ESP32 ისეთი ფუნქციების ჩასართავად, როგორიცაა PWM, ADC, DAC, I2C, I2S, SPI და ა.შ.

შენიშვნა

ESP32-სა და SPI ფლეშ მეხსიერება მეხსიერებას შორის კომუნიკაციისთვის შიდად გამოიყენება D0, D1, D2, D3, CMD და CLK პინები. ისინი დაჯგუფებულია ორივე მხარეს USB კონექტორის მახლობლად. მოერიდეთ ამ პინების გამოყენებას, რადგან ამან შეიძლება ხელი შეუშალოს SPI ფლეშ მეხსიერება მეხსიერებაზე / SPI RAM-ზე წვდომას.

შენიშვნა

პინები GPIO 16 და GPIO 17 გამოიყენება მხოლოდ ESP32-WROOM და ESP32-SOLO-1 მოდულების მქონე პლატებზე. ESP32-WROVER მოდულების მქონე პლატებზე პინები განკუთვნილია შიდა გამოყენებისთვის.

კვების წყაროს პარამეტრები¶

დაფისთვის ენერგიით მომარაგების სამი ურთიერთგამომრიცხავი გზა არსებობს:

  • მიკრო USB პორტი, სტანდარტული კვების წყარო
  • 5V / GND ჰედერის პინები
  • 3V3 / GND ჰედერის პინები

გაფრთხილება

ელექტროენერგიის მიწოდება უნდა მოხდეს გამოყენებით ზემოთ ჩამოთვლილი ვარიანტებიდან მხოლოდ ერთი, წინააღმდეგ შემთხვევაში შეიძლება დაზიანდეს დაფა და/ან კვების წყარო.

შენიშვნა C15-ზე¶

კომპონენტი C15-მა შეიძლება გამოიწვიოს შემდეგი პრობლემები ESP32-DevKitC V4 დაფების ადრინდელ ვერსიებზე:

  • დაფა შეიძლება ჩაიტვირთოს ჩამოტვირთვის რეჟიმში
  • თუ გამომავალ საათს GPIO 7 0-ზე გამოავლენთ, C15-მა შეიძლება გავლენა მოახდინოს სიგნალზე.

თუ ასეთი პრობლემები წარმოიქმნება, გთხოვთ, წაშალოთ კომპონენტი ქვემოთ მოცემულ ფიგურაზე ყვითლად მონიშნულია C15.

C15-ის (ყვითელი) მდებარეობა ESP32-DevKitC V4 დაფაზე

აპლიკაციის შემუშავების დაწყება¶

ESP32-DevKitC V4-ის ჩართვამდე დარწმუნდით, რომ დაფა კარგ მდგომარეობაშია და დაზიანების აშკარა ნიშნები არ აქვს.

ამის შემდეგ, გააგრძელეთ ESP32 აპარატურის ცნობარი, სადაც სექცია ინსტალაცია ეტაპობრივად სწრაფად დაგეხმარებათ განვითარების გარემოს დაყენებაში და შემდეგ ფლეშ მეხსიერება თქვენს დაფაზე პროექტის მაგალითის განთავსებაში.

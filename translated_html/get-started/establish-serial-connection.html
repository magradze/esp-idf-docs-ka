სერიული კავშირის დამყარება ESP32-თან¶

[中文]

ეს განყოფილება გთავაზობთ ინსტრუქციებს, თუ როგორ უნდა დაამყაროთ სერიული კავშირი ESP32-სა და კომპიუტერს შორის.

ESP32-ის კომპიუტერთან დაკავშირება¶

USB კაბელის გამოყენებით, შეაერთეთ ESP32 დაფა კომპიუტერთან. თუ მოწყობილობის დრაივერი ავტომატურად არ დაინსტალირდება, იპოვეთ USB-დან სერიულ გადამყვან ჩიპი თქვენს ESP32 დაფაზე (ან გარე გადამყვან დონგლზე), მოძებნეთ დრაივერები ინტერნეტში და დააინსტალირეთ ისინი.

ქვემოთ მოცემულია Espressif-ის მიერ წარმოებული ESP32 დაფების დრაივერების ბმულები:

განვითარების საბჭო USB დრაივერი შენიშვნები
ESP32-DevKitC CP210x  
ESP32-LyraT CP210x  
ESP32-LyraTD-MSC CP210x  
ESP32-PICO-KIT CP210x  
ESP-WROVER-KIT FTDI  
ESP32 დემო დაფა FTDI  
ESP-პროგ FTDI პროგრამირების დაფა (ESP32-ის გარეშე)
ESP32-MeshKit-Sense არ არის ხელმისაწვდომი გამოყენება ESP-პროგ
ESP32-Sense კომპლექტი არ არის ხელმისაწვდომი გამოყენება ESP-პროგ
  • CP210x: CP210x USB-დან UART ხიდზე გადამყვანი VCP დრაივერები
  • FTDI: FTDI ვირტუალური COM პორტის დრაივერები

ზემოთ მოცემული დრაივერები ძირითადად საცნობაროა. ნორმალურ პირობებში, დრაივერები უნდა იყოს ოპერაციულ სისტემასთან ერთად და ავტომატურად უნდა დაინსტალირდეს ჩამოთვლილი პლატებიდან ერთ-ერთის კომპიუტერთან შეერთებისას.

შეამოწმეთ პორტი Windows-ზე¶

შეამოწმეთ იდენტიფიცირებული COM პორტების სია Windows Device Manager-ში. გათიშეთ ESP32 და ხელახლა შეაერთეთ, რათა დაადასტუროთ, რომელი პორტი ქრება სიიდან და შემდეგ ისევ ჩნდება.

ქვემოთ მოცემულ ფიგურებში ნაჩვენებია სერიული პორტი ESP32 DevKitC-ისა და ESP32 WROVER KIT-ისთვის.

ESP32-DevKitC-ის USB-დან UART-მდე ხიდი Windows Device Manager-ში

ESP-WROVER-KIT-ის ორი USB სერიული პორტი Windows Device Manager-ში

პორტის შემოწმება Linux-სა და MacOS-ზე¶

თქვენი ESP32 დაფის (ან გარე გადამყვანი დონგლის) სერიული პორტი ის მოწყობილობის სახელის შესამოწმებლად, ორჯერ შეასრულეთ ეს ბრძანება, ჯერ დაფა/დონგლი გამორთული, შემდეგ კი შეერთებული. მეორედ გამოჩენილი პორტი არის ის, რაც გჭირდებათ:

Linux

ls /dev/tty*

MacOS

ls /dev/cu.*

შენიშვნა

MacOS-ის მომხმარებლები: თუ ვერ ხედავთ სერიული პორტი , შეამოწმეთ, რომ დაინსტალირებული გაქვთ USB/სერიული დრაივერები, როგორც ეს მითითებულია თქვენი კონკრეტული დეველოპერული დაფისთვის „დაწყების სახელმძღვანელოში“. MacOS High Sierra-ს (10.13) შემთხვევაში, შესაძლოა, დრაივერების ჩატვირთვის ნებართვაც მოგიწიოთ. გახსენით „სისტემის პარამეტრები“ -&gt; „უსაფრთხოება და კონფიდენციალურობა“ -&gt; „ზოგადი“ და შეამოწმეთ, ხომ არ არის აქ ნაჩვენები შეტყობინება „სისტემის პროგრამული უზრუნველყოფა დეველოპერისგან...“-ს შესახებ, სადაც დეველოპერის სახელია Silicon Labs ან FTDI.

მომხმარებლის დამატება dialout Linux-ზე¶

ამჟამად შესულ მომხმარებელს უნდა ჰქონდეს სერიული პორტი წაკითხვისა და ჩაწერის წვდომა USB-ის საშუალებით. Linux-ის დისტრიბუციების უმეტესობაზე ეს ხდება მომხმარებლის დამატებით dialout დააჯგუფეთ შემდეგი ბრძანებით:

sudo usermod -a -G dialout $USER

Arch Linux-ზე ეს ხდება მომხმარებლის დამატებით uucp დააჯგუფეთ შემდეგი ბრძანებით:

sudo usermod -a -G uucp $USER

დარწმუნდით, რომ ხელახლა შეხვალთ, რათა ჩაართოთ წაკითხვისა და ჩაწერის ნებართვები სერიული პორტი ისთვის.

სერიული კავშირის შემოწმება¶

ახლა გადაამოწმეთ, რომ სერიული კავშირი ფუნქციონირებს. ამის გაკეთება შეგიძლიათ სერიული ტერმინალის პროგრამის გამოყენებით. ამ მაგალითში ჩვენ გამოვიყენებთ PuTTY SSH კლიენტი რომელიც ხელმისაწვდომია როგორც Windows-ისთვის, ასევე Linux-ისთვის. შეგიძლიათ გამოიყენოთ სხვა სერიული პროგრამა და დააყენოთ კომუნიკაციის პარამეტრები, როგორც ქვემოთ არის მოცემული.

გაუშვით ტერმინალი, დააყენეთ იდენტიფიცირებული სერიული პორტი , ბოდის სიჩქარე = 115200, მონაცემთა ბიტები = 8, გაჩერების ბიტები = 1 და პარიტეტი = N. ქვემოთ მოცემულია პორტის და ასეთი გადაცემის პარამეტრების (მოკლედ აღწერილი, როგორც 115200-8-1-N) დაყენების მაგალითები Windows-სა და Linux-ზე. არ დაგავიწყდეთ ზუსტად იგივე სერიული პორტი არჩევა, რომელიც ზემოთ მოცემულ ნაბიჯებში ამოიცანით.

შემდეგ გახსენით სერიული პორტი ტერმინალში და შეამოწმეთ, ხედავთ თუ არა ESP32-ის მიერ დაბეჭდილ რაიმე ჟურნალს. ჟურნალის შინაარსი დამოკიდებული იქნება ESP32-ზე ჩატვირთულ აპლიკაციაზე. ESP32-ის მიერ ჟურნალის მაგალითი ნაჩვენებია ქვემოთ.

თუ ხედავთ წასაკითხ ჟურნალის გამომავალს, ეს ნიშნავს, რომ სერიული კავშირი მუშაობს და თქვენ მზად ხართ ინსტალაციის გასაგრძელებლად და ბოლოს, ESP32-ზე აპლიკაციის ატვირთვა .

შენიშვნა

ზოგიერთი სერიული პორტი გაყვანილობის კონფიგურაციისთვის, სერიული RTS და DTR პინები უნდა გამორთოთ ტერმინალის პროგრამაში, სანამ ESP32 ჩაიტვირთება და სერიული გამომავალი გამოჩნდება. ეს დამოკიდებულია თავად აპარატურაზე, უმეტესი დეველოპერული დაფები (მათ შორის ყველა Espressif დაფა) არ ეს პრობლემა არსებობს. პრობლემა არსებობს, თუ RTS და DTR პირდაპირ EN და GPIO 0 ქინძისთავებთან არის დაკავშირებული. იხილეთ esptool-ის დოკუმენტაცია დამატებითი დეტალებისთვის.

შენიშვნა

კომუნიკაციის მუშაობის დადასტურების შემდეგ დახურეთ სერიული ტერმინალი. შემდეგ ეტაპზე ატვირთვა ისთვის სხვა აპლიკაციას გამოვიყენებთ, ESP32-ისთვის კი ახალ პროგრამული უზრუნველყოფა . ეს აპლიკაცია ვერ შეძლებს სერიული პორტი წვდომას, სანამ ის ტერმინალში გახსნილია.

თუ აქიდან მოხვედით ნაბიჯი 6. თქვენი მოწყობილობის დაკავშირება ESP32 შემუშავებისთვის პროგრამული უზრუნველყოფის ინსტალაციისას, შეგიძლიათ გააგრძელოთ ნაბიჯი 7. კონფიგურაცია.

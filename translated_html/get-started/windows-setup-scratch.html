Windows Toolchain-ის ნულიდან დაყენება¶

[中文]

ეს არის ეტაპობრივი ალტერნატივა გაშვებისა ESP-IDF ინსტრუმენტების ინსტალატორი CMake-ზე დაფუძნებული აწყობის სისტემისთვის. ყველა ინსტრუმენტის ხელით ინსტალაცია პროცესის უკეთ კონტროლის საშუალებას იძლევა და ასევე მოწინავე მომხმარებლებს აძლევს ინფორმაციას ინსტალაციის პერსონალიზაციისთვის.

ინსტრუმენტების ნაკრები ის და სხვა ინსტრუმენტების სტანდარტული გზით სწრაფად დასაყენებლად, ESP-IDF ინსტრუმენტების ინსტალატორის გამოყენებით, გადადით განყოფილებაში Windows-ისთვის Toolchain-ის სტანდარტული დაყენება.

შენიშვნა

GNU Make-ზე დაფუძნებული აწყობის სისტემა მოითხოვს MSYS2 Unix-თან თავსებადობის გარემო Windows-ზე. CMake-ზე დაფუძნებულ აწყობის სისტემას ეს გარემო არ სჭირდება.

მიიღეთ ESP-IDF¶

შენიშვნა

ESP-IDF ის წინა ვერსიები იყენებდა MSYS2 bash ტერმინალი ბრძანების ხაზი. მიმდინარე cmake-ზე დაფუძნებული აწყობის სისტემას შეუძლია ჩვეულებრივ რეჟიმში მუშაობა. Windows-ის ბრძანების ხაზი რომელიც აქ გამოიყენება.

თუ იყენებთ bash-ზე დაფუძნებულ ტერმინალს ან PowerShell-ს, გაითვალისწინეთ, რომ ზოგიერთი ბრძანების სინტაქსი განსხვავდება ქვემოთ ნაჩვენებისგან.

გახსენით Command Prompt და შეასრულეთ შემდეგი ბრძანებები:

mkdir %userprofile%\esp
cd %userprofile%\esp
git clone --recursive https://github.com/espressif/esp-idf.git

ESP-IDF გადმოწერილი იქნება %userprofile%\esp\esp-idf.

კონსულტაცია ESP-IDF ვერსიები ინფორმაციისთვის, თუ რომელი ESP-IDF ვერსია უნდა იქნას გამოყენებული მოცემულ სიტუაციაში.

შენიშვნა

ეს ბრძანება დააკლონირებს მასტერ ტოტს, რომელსაც აქვს ESP-IDF ის უახლესი („გაუმჯობესებული“) ვერსია. ის სრულად ფუნქციონალურია და ყოველკვირეულად განახლდება უახლესი ფუნქციებითა და შეცდომების გამოსწორებით.

შენიშვნა

GitHub-ის „zip ფაილის ჩამოტვირთვის“ ფუნქცია არ მუშაობს ESP-IDF თან, a-სთან. git clone აუცილებელია. სარეზერვო ასლის სახით, სტაბილური ვერსია შეიძლება დაინსტალირდეს Git-ის გარეშე.

შენიშვნა

არ გამოტოვოთ --recursive ვარიანტი. თუ უკვე დაკლონირეთ ESP-IDF ამ ვარიანტის გარეშე, შეასრულეთ სხვა ბრძანება ყველა ქვემოდულის მისაღებად:

cd esp-idf
git submodule update --init

ხელსაწყოები¶

სმეიქი¶

ჩამოტვირთეთ უახლესი სტაბილური ვერსია CMake Windows-ისთვის და გაუშვით ინსტალერი.

როდესაც ინსტალერი მოგთხოვთ ინსტალაციის პარამეტრებს, აირჩიეთ „CMake-ის დამატება სისტემის გზაზე ყველა მომხმარებლისთვის“ ან „CMake-ის დამატება სისტემის გზაზე მიმდინარე მომხმარებლისთვის“.

ნინძას კონსტრუქცია¶

შენიშვნა

Ninja ამჟამად მხოლოდ 64-ბიტიანი Windows-ისთვის გვთავაზობს ბინარულ ფაილებს. შესაძლებელია CMake-ის და idf.py სხვა აწყობის ხელსაწყოებთან ერთად, როგორიცაა mingw-make, 32-ბიტიან ფანჯრებზე. თუმცა, ეს ამჟამად დაუდოკუმენტირებელია.

ჩამოტვირთეთ ნინძა Windows-ის უახლესი სტაბილური ვერსია ()-დანჩამოტვირთვის გვერდი).

Windows-ისთვის Ninja-ს ჩამოტვირთვა არის .zip ფაილი, რომელიც შეიცავს ერთ ninja.exe ფაილი, რომლის გახსნაც საჭიროა დირექტორიაში, რომელიც შემდეგ არის დაემატა თქვენს გზას (ან შეგიძლიათ აირჩიოთ დირექტორია, რომელიც უკვე თქვენს გზაზეა).

Python 2.x¶

ჩამოტვირთეთ უახლესი პითონი 2.7 Windows-ის ინსტალატორისთვის და გაუშვით.

Python-ის ინსტალატორის „მორგების“ ნაბიჯი გთავაზობთ ვარიანტების სიას. ბოლო ვარიანტია „python.exe-ს დამატება გზაზე“. შეცვალეთ ეს ვარიანტი „დაინსტალირდება“-ს არჩევით.

Python-ის ინსტალაციის შემდეგ, გახსენით Windows Command Prompt Start მენიუდან და გაუშვით შემდეგი ბრძანება:

pip install --user pyserial

ხელსაწყოთა ჯაჭვის დაყენება¶

ჩამოტვირთეთ pre კომპილაცია d Windows ინსტრუმენტების ნაკრები :

https://dl.espressif.com/dl/xtensa-esp32-elf-gcc8_2_0-esp-2019r2-win32.zip

გახსენით zip ფაილი C:\Program Files (ან სხვა ადგილას). zip ფაილი შეიცავს ერთ დირექტორიას xtensa-esp32-elf.

შემდეგ, bin ამ დირექტორიის ქვედირექტორია უნდა იყოს დაემატა თქვენს გზასმაგალითად, დასამატებელი დირექტორია შეიძლება იყოს C:\Program Files\xtensa-esp32-elf\bin.

შენიშვნა

თუ უკვე დაინსტალირებული გაქვთ MSYS2 გარემო („GNU Make“-ის აწყობის სისტემასთან გამოსაყენებლად), შეგიძლიათ გამოტოვოთ ცალკე ჩამოტვირთვა და დაამატოთ დირექტორია. C:\msys32\opt\xtensa-esp32-elf\bin Path-ზე, რადგან ინსტრუმენტების ნაკრები შედის MSYS2 გარემოში.

დირექტორიის დამატება გზაზე¶

Windows Path გარემოს ცვლადში ნებისმიერი ახალი დირექტორიის დასამატებლად:

გახსენით სისტემის მართვის პანელი და გადადით გარემოს ცვლადების დიალოგურ ფანჯარაზე. (Windows 10-ზე ეს შეგიძლიათ იხილოთ სისტემის გაფართოებული პარამეტრების ქვეშ).

ორჯერ დააწკაპუნეთ Path ცვლადი (მომხმარებლის ან სისტემის გზა, იმისდა მიხედვით, გსურთ თუ არა, რომ სხვა მომხმარებლებს ჰქონდეთ ეს დირექტორია თავიანთ გზაზე.) გადადით მნიშვნელობის ბოლოში და დაამატეთ ;<new value>.

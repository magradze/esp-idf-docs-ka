Mac OS-ისთვის Toolchain-ის ნულიდან დაყენება¶

[中文]

პაკეტების მენეჯერი¶

ინსტრუმენტების ნაკრები ნულიდან დაყენება, იმის ნაცვლად, რომ ვადამდელი კომპილაცია დღის ინსტრუმენტების ნაკრები ჩამოტვირთვა, თქვენ დაგჭირდებათ ან მაკპორტები ან სახლში დამზადებული ლუდი პაკეტის მენეჯერი.

MacPorts-ს სჭირდება XCode-ის სრული ინსტალაცია, ხოლო homebrew-ს მხოლოდ XCode ბრძანების ხაზის ინსტრუმენტები სჭირდება.

იხილეთ Toolchain-ის მორგებული დაყენება განყოფილებაში მოცემულია რამდენიმე მიზეზი, თუ რატომ შეიძლება საჭირო გახდეს ინსტრუმენტების ნაკრები ნულიდან ინსტალაცია.

ინსტალაციის წინაპირობები¶

  • პიპის ინსტალაცია:

    sudo easy_install pip

  • დააინსტალირეთ pyserial:

    pip install --user pyserial

  • დააინსტალირეთ CMake &amp; Ninja ბილდი:

    • თუ თქვენ გაქვთ HomeBrew, შეგიძლიათ გაუშვათ:

      brew install cmake ninja

    • თუ თქვენ გაქვთ MacPorts, შეგიძლიათ გაუშვათ:

      sudo port install cmake ninja

წყაროდან ინსტრუმენტების ჯაჭვის კომპილაცია¶

  • ინსტალაციის დამოკიდებულებები:

    • MacPorts-ით:

      sudo port install gsed gawk binutils gperf grep gettext wget libtool autoconf automake make

    • სახლში დამზადებული ლუდით:

      brew install gnu-sed gawk binutils gperftools gettext wget help2man libtool autoconf automake make

შექმენით ფაილური სისტემის გამოსახულება, რომელიც რეგისტრის მიმართ მგრძნობიარე იქნება:

hdiutil create ~/esp/crosstool.dmg -volname "ctng" -size 10g -fs "Case-sensitive HFS+"

დაამონტაჟეთ:

hdiutil mount ~/esp/crosstool.dmg

შექმენით სიმბოლური ბმული თქვენს სამუშაო დირექტორიაში:

mkdir -p ~/esp
ln -s /Volumes/ctng ~/esp/ctng-volume

გადადით ახლად შექმნილ დირექტორიაში:

cd ~/esp/ctng-volume

ჩამოტვირთვა crosstool-NG და ააშენეთ იგი:

git clone https://github.com/espressif/crosstool-NG.git
cd crosstool-NG
git checkout esp-2019r2
git submodule update --init
./bootstrap && ./configure --enable-local && make

შექმენით ინსტრუმენტების ნაკრები :

./ct-ng xtensa-esp32-elf
./ct-ng build
chmod -R u+w builds/xtensa-esp32-elf

ხელსაწყოების ჯაჭვი აშენდება ~/esp/ctng-volume/crosstool-NG/builds/xtensa-esp32-elfმის გამოსაყენებლად, თქვენ უნდა დაამატოთ ~/esp/ctng-volume/crosstool-NG/builds/xtensa-esp32-elf/bin რომ PATH გარემოს ცვლადი.

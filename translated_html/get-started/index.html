დაწყება¶

[中文]

ეს დოკუმენტი შექმნილია იმისთვის, რომ დაგეხმაროთ Espressif-ის ESP32 ჩიპზე დაფუძნებული აპარატურისთვის პროგრამული უზრუნველყოფის შემუშავების გარემოს დაყენებაში.

ამის შემდეგ, მარტივი მაგალითი გაჩვენებთ, თუ როგორ გამოიყენოთ ESP-IDF (Espressif IoT Development Framework) მენიუს კონფიგურაციისთვის, შემდეგ კი ფლეშ მეხსიერება ასაგებად და პროგრამული უზრუნველყოფა ESP32 დაფაზე ჩასართავად.

შენიშვნა

ეს არის ESP-IDF ის მასტერ ფრონტის (უახლესი ვერსიის) დოკუმენტაცია. ეს ვერსია მუდმივი დამუშავების პროცესშია. სტაბილური ვერსია დოკუმენტაცია ხელმისაწვდომია, ისევე როგორც სხვა ESP-IDF ვერსიები.

შესავალი¶

ESP32 არის ჩიპზე დაფუძნებული სისტემა, რომელიც აერთიანებს შემდეგ მახასიათებლებს:

  • Wi-Fi (2.4 GHz დიაპაზონი)
  • Bluetooth 4.2
  • ორმაგი მაღალი ხარისხის ბირთვი
  • ულტრა დაბალი სიმძლავრის კოპროცესორი
  • რამდენიმე პერიფერიული მოწყობილობა

40 ნმ ტექნოლოგიით აღჭურვილი ESP32 უზრუნველყოფს ძლიერ, მაღალ ინტეგრირებულ პლატფორმას, რომელიც ხელს უწყობს ენერგიის ეფექტური გამოყენების, კომპაქტური დიზაინის, უსაფრთხოების, მაღალი შესრულებისა და საიმედოობის მუდმივი მოთხოვნების დაკმაყოფილებას.

Espressif უზრუნველყოფს საბაზისო აპარატურულ და პროგრამულ რესურსებს, რათა დაეხმაროს აპლიკაციების შემქმნელებს ESP32 სერიის აპარატურის გამოყენებით თავიანთი იდეების რეალიზებაში. Espressif-ის მიერ შექმნილი პროგრამული უზრუნველყოფის შემუშავების ჩარჩო განკუთვნილია ნივთების ინტერნეტის (IoT) აპლიკაციების შემუშავებისთვის Wi-Fi , Bluetooth , ენერგიის მართვისა და სხვა რამდენიმე სისტემური ფუნქციის გამოყენებით.

რა გჭირდებათ¶

აპარატურა:

  • ერთი ESP32 დაფა
  • USB კაბელი - USB A / მიკრო USB B
  • კომპიუტერი Windows-ის, Linux-ის ან macOS-ის გაშვებული

პროგრამული უზრუნველყოფა:

  • ხელსაწყოების ჯაჭვი ESP32-ისთვის კომპილაცია კოდი
  • აწყობის ხელსაწყოები - CMake და Ninja სრული ასაშენებლად აპლიკაცია ESP32-ისთვის
  • ESP-IDF რომელიც არსებითად შეიცავს API (პროგრამული უზრუნველყოფის ბიბლიოთეკები და საწყისი კოდი) ESP32-ისთვის და სკრიპტებს მისი მუშაობისთვის. ხელსაწყოების ჯაჭვი
  • ტექსტის რედაქტორი პროგრამების დასაწერად (პროექტები) C-ში, მაგ., დაბნელება

ESP32-ისთვის აპლიკაციების შემუშავება

  • ESP-WROVER-KIT

ნაბიჯი 1. წინაპირობების ინსტალაცია¶

შემდეგ ნაბიჯებზე გადასვლამდე კომპიუტერზე რამდენიმე ინსტრუმენტი უნდა დაინსტალიროთ. თქვენი ოპერაციული სისტემის ინსტრუქციებისთვის მიჰყევით ქვემოთ მოცემულ ბმულებს:

Windows Linux Mac OS

ნაბიჯი 2. მიიღეთ ESP-IDF¶

ESP32-ისთვის აპლიკაციების შესაქმნელად, დაგჭირდებათ Espressif-ის მიერ მოწოდებული პროგრამული ბიბლიოთეკები. ESP-IDF საცავი.

ESP-IDF მისაღებად, გადადით თქვენს ინსტალაციის დირექტორიაში და დააკლონირეთ რეპოზიტორია git cloneქვემოთ მოცემული ინსტრუქციების შესაბამისად, რომლებიც სპეციფიკურია თქვენი ოპერაციული სისტემისთვის.

შენიშვნა

ეს სახელმძღვანელო იყენებს დირექტორიას ~/esp Linux-სა და macOS-ზე ან %userprofile%\esp Windows-ზე, როგორც ESP-IDF ის ინსტალაციის საქაღალდე. შეგიძლიათ გამოიყენოთ ნებისმიერი დირექტორია, მაგრამ შესაბამისად, ბრძანებების ბილიკების კორექტირება დაგჭირდებათ. გაითვალისწინეთ, რომ ESP-IDF არ უჭერს მხარს ბილიკებში დაშორებებს.

Linux და macOS¶

გახსენით ტერმინალი და შეასრულეთ შემდეგი ბრძანებები:

cd ~/esp
git clone --recursive https://github.com/espressif/esp-idf.git

ESP-IDF გადმოწერილი იქნება ~/esp/esp-idf.

კონსულტაცია ESP-IDF ვერსიები ინფორმაციისთვის, თუ რომელი ESP-IDF ვერსია უნდა იქნას გამოყენებული მოცემულ სიტუაციაში.

Windows¶

ხელსაწყოების დამონტაჟების გარდა, ESP-IDF ინსტრუმენტების ინსტალატორი Windows-ისთვის, რომელიც პირველ ეტაპზე იყო წარმოდგენილი, ასევე შეგიძლიათ ESP-IDF ფაილის ასლის ჩამოტვირთვა.

კონსულტაცია ESP-IDF ვერსიები ინფორმაციისთვის, თუ რომელი ESP-IDF ვერსია უნდა იქნას გამოყენებული მოცემულ სიტუაციაში.

თუ გსურთ ESP-IDF ჩამოტვირთვა ESP-IDF Tools Installer-ის დახმარების გარეშე, იხილეთ ესენი ინსტრუქციები.

ნაბიჯი 3. ხელსაწყოების დაყენება¶

ESP-IDF გარდა, თქვენ ასევე უნდა დააინსტალიროთ ESP-IDF მიერ გამოყენებული ინსტრუმენტები, როგორიცაა კომპილაცია r, გამართვა ger, Python-ის პაკეტები და ა.შ.

Windows¶

ESP-IDF ინსტრუმენტების ინსტალატორი Windows-ისთვის, რომელიც პირველ ეტაპზე იყო წარმოდგენილი, ყველა საჭირო ინსტრუმენტს აყენებს.

თუ გსურთ ინსტრუმენტების ინსტალაცია ESP-IDF Tools Installer-ის დახმარების გარეშე, გახსენით Command Prompt და მიჰყევით ამ ნაბიჯებს:

cd %userprofile%\esp\esp-idf
install.bat

ან Windows PowerShell-ით

cd ~/esp/esp-idf
./install.ps1

Linux და macOS¶

cd ~/esp/esp-idf
./install.sh

ხელსაწყოების ინსტალაციის გზის პერსონალიზაცია¶

ამ ეტაპზე წარმოდგენილი სკრიპტები მომხმარებლის საშინაო დირექტორიაში ESP-IDF ის მიერ მოთხოვნილ კომპილაციის ინსტრუმენტებს აინსტალირებს: $HOME/.espressif Linux-სა და macOS-ზე, %USERPROFILE%\.espressif Windows-ზე. თუ გსურთ ინსტრუმენტების სხვა დირექტორიაში ინსტალაცია, დააყენეთ გარემოს ცვლადი IDF_TOOLS_PATH ინსტალაციის სკრიპტების გაშვებამდე დარწმუნდით, რომ თქვენს მომხმარებელს აქვს საკმარისი ნებართვები ამ გზის წასაკითხად და ჩასაწერად.

თუ შეცვლით IDF_TOOLS_PATH, დარწმუნდით, რომ ინსტალაციის სკრიპტის ყოველ ჯერზე ერთი და იგივე მნიშვნელობა აქვს დაყენებული (install.bat, install.ps1 ან install.sh) და ექსპორტის სკრიპტი (export.bat, export.ps1 ან export.sh) სიკვდილით დასაჯებიან.

ნაბიჯი 4. გარემოს ცვლადების დაყენება¶

დაინსტალირებული ინსტრუმენტები ჯერ არ არის დამატებული PATH გარემოს ცვლადში. იმისათვის, რომ ინსტრუმენტები ბრძანების ხაზიდან გამოსაყენებელი გახდეს, საჭიროა გარემოს ზოგიერთი ცვლადის დაყენება. ESP-IDF გთავაზობთ სხვა სკრიპტს, რომელიც ამას აკეთებს.

Windows¶

ESP-IDF ინსტრუმენტების ინსტალატორი Windows-ისთვის, Start მენიუში იქმნება „ ESP-IDF Command Prompt“-ის მალსახმობი. ეს მალსახმობი ხსნის Command Prompt-ს და აყენებს ყველა საჭირო გარემოს ცვლადს. შეგიძლიათ გახსნათ ეს მალსახმობი და გადახვიდეთ შემდეგ ეტაპზე.

ალტერნატიულად, თუ გსურთ გამოიყენოთ ESP-IDF არსებულ ბრძანების სტრიქონის ფანჯარაში, შეგიძლიათ გაუშვათ:

%userprofile%\esp\esp-idf\export.bat

ან Windows PowerShell-ით

.$HOME/esp/esp-idf/export.ps1

Linux და macOS¶

ტერმინალში, სადაც ESP-IDF გამოყენებას აპირებთ, გაუშვით:

. $HOME/esp/esp-idf/export.sh

ყურადღება მიაქციეთ მანძილს წამყვან წერტილსა და გზას შორის!

ასევე შეგიძლიათ ამ ნაბიჯის ავტომატიზაცია, ESP-IDF ინსტრუმენტების ყველა ტერმინალში ხელმისაწვდომობის მიცემით, თქვენს ტერმინალში ამ ხაზის დამატებით. .profile ან .bash_profile სცენარი.

ნაბიჯი 5. პროექტის დაწყება¶

ახლა თქვენ მზად ხართ მოამზადოთ თქვენი განაცხადი ESP32-ისთვის. შეგიძლიათ დაიწყოთ დაწყება/გამარჯობა_მსოფლიო პროექტი მაგალითები დირექტორია IDF-ში.

კოპირება დაწყება/გამარჯობა_მსოფლიო რომ ~/esp დირექტორია:

Linux და macOS¶

cd ~/esp
cp -r $IDF_PATH/examples/get-started/hello_world .

Windows¶

cd %userprofile%\esp
xcopy /e /i %IDF_PATH%\examples\get-started\hello_world hello_world

არსებობს პროექტების სამაგალითო მრავალფეროვნება, მაგალითები დირექტორია ESP-IDF ში. თქვენ შეგიძლიათ ნებისმიერი პროექტი დააკოპიროთ ზემოთ წარმოდგენილი მეთოდით და გაუშვათ.

ასევე შესაძლებელია მაგალითების ადგილზე შექმნა, მათი წინასწარი კოპირების გარეშე.

მნიშვნელოვანი

ESP-IDF აწყობის სისტემა არ უჭერს მხარს სივრცეებს არც ESP-IDF და არც პროექტებისკენ მიმავალ ბილიკებში.

ნაბიჯი 6. თქვენი მოწყობილობის დაკავშირება¶

ახლა შეაერთეთ თქვენი ESP32 დაფა კომპიუტერთან და შეამოწმეთ, რომელი სერიული პორტი ქვეშ ჩანს ის.

სერიულ პორტებს სახელწოდებაში შემდეგი შაბლონები აქვთ:

  • Windows: სახელები, როგორიცაა COM1
  • Linux: დაწყებული /dev/tty
  • macOS: დაწყებული /dev/cu.

თუ არ ხართ დარწმუნებული, თუ როგორ შეამოწმოთ სერიული პორტი სახელი, გთხოვთ, იხილოთ სერიული კავშირის დამყარება ESP32-თან სრული დეტალებისთვის.

შენიშვნა

პორტის სახელი ხელთ გქონდეთ, რადგან ის შემდეგ ნაბიჯებში დაგჭირდებათ.

ნაბიჯი 7. კონფიგურაცია¶

ნავიგაცია თქვენს hello_world დირექტორია ნაბიჯი 5. პროექტის დაწყება და გაუშვით პროექტის კონფიგურაცია პროგრამა menuconfig.

Linux და macOS¶

cd ~/esp/hello_world
idf.py menuconfig

თუ Python-ის თქვენი ნაგულისხმევი ვერსია 3.x არის, შეიძლება დაგჭირდეთ გაშვება python2 $(which idf.py) menuconfig სამაგიეროდ.

Windows¶

cd %userprofile%\esp\hello_world
idf.py menuconfig

თუ წინა ნაბიჯები სწორად შესრულდა, გამოჩნდება შემდეგი მენიუ:

პროექტის კონფიგურაცია - მთავარი ფანჯარა

შენიშვნა

მენიუს ფერები შეიძლება განსხვავდებოდეს თქვენს ტერმინალში. თქვენ შეგიძლიათ შეცვალოთ მისი გარეგნობა ოფციის გამოყენებით. --styleგთხოვთ, გაიქეცით idf.py menuconfig --help დამატებითი ინფორმაციისთვის.

ნავიგაციისთვის და გამოსაყენებლად menuconfig, დააჭირეთ შემდეგ ღილაკებს:

  • ნავიგაციისთვის ისრის ღილაკები
  • Enter ქვემენიუში გადასასვლელად
  • Esc ერთი დონით ასვლა ან გასვლა
  • ? დახმარების ეკრანის სანახავად. Enter ღილაკი დახმარების ეკრანიდან გასასვლელად გამოიყენება.
  • Space, ან Y და N კლავიშები კონფიგურაციის ელემენტების ჩასართავად (დიახ) და გამორთვისთვის (არა) მონიშვნის ველებით[*]„
  • ? კონფიგურაციის ელემენტის მონიშვნისას, ამ ელემენტის შესახებ დახმარების საჩვენებლად
  • / კონფიგურაციის ელემენტების მოსაძებნად

ყურადღება

თუ ESP32-DevKitC დაფას იყენებთ ESP32-SOLO-1 მოდული, ჩართეთ ერთბირთვიანი რეჟიმი (CONFIG_FREE RTOS _UNICORE) menuconfig ფლეშ მეხსიერება , მაგალითად.

ნაბიჯი 8. პროექტის შექმნა¶

ნაბიჯი 9. ფლეშის განთავსება მოწყობილობაზე¶

ESP32 დაფაზე ახლად აწყობილი ბინარული ფაილების ფლეშ დრაივი შემდეგი ბრძანებით:

idf.py -p PORT [-b BAUD] flash

შეცვალეთ PORT თქვენი ESP32 დაფის სერიული პორტი სახელით ნაბიჯი 6. თქვენი მოწყობილობის დაკავშირება.

ასევე შეგიძლიათ შეცვალოთ ფლეშ მეხსიერება ან ბოდის სიჩქარე BAUD-ის თქვენთვის საჭირო ბოდის სიჩქარე ით ჩანაცვლებით. ნაგულისხმევი ბოდის სიჩქარე არის 460800.

idf.py არგუმენტების შესახებ დამატებითი ინფორმაციისთვის იხილეთ idf.py.

შენიშვნა

ვარიანტი flash ავტომატურად აშენებს და ფლეშ მეხსიერება ასრულებს პროექტს, ამიტომ მუშაობს idf.py build აუცილებელი არ არის.

ნაბიჯი 10. მონიტორი¶

იმის შესამოწმებლად, ნამდვილად მუშაობს თუ არა „hello_world“, აკრიფეთ idf.py -p PORT monitor (არ დაგავიწყდეთ PORT-ის თქვენი სერიული პორტი სახელით ჩანაცვლება).

ეს ბრძანება იწყებს IDF მონიტორი აპლიკაცია:

თუ IDF მონიტორი ატვირთვა ის შემდეგ მალევე გაითიშება, ან თუ ზემოთ მოცემული შეტყობინებების ნაცვლად ქვემოთ მოცემულის მსგავს შემთხვევით ნაგავს ხედავთ, თქვენი დაფა, სავარაუდოდ, 26 MHz კრისტალს იყენებს. დეველოპერული დაფების უმეტესობა 40 MHz-ს იყენებს, ამიტომ ESP-IDF ამ სიხშირეს ნაგულისხმევ მნიშვნელობად იყენებს.

თუ ასეთი პრობლემა გაქვთ, გააკეთეთ შემდეგი:

  1. გამოდით მონიტორიდან.
  2. დაბრუნება menuconfig.
  3. გადადით კომპონენტის კონფიგურაციაზე –&gt; ESP32-სპეციფიკურზე –&gt; მთავარი XTAL სიხშირეზე, შემდეგ შეცვალეთ CONFIG_ESP32_XTAL_FREQ_SEL 26 MHz-მდე.
  4. ამის შემდეგ, აწყობა და ფლეშ მეხსიერება განაცხადი კვლავ.

შენიშვნა

თქვენ შეგიძლიათ გააერთიანოთ მშენებლობა, ფლეშ მეხსიერება სწავლება და მონიტორინგი ერთ ეტაპზე შემდეგი ნაბიჯების შესრულებით:

idf.py -p PORT flash monitor

აგრეთვე იხილეთ:

  • IDF მონიტორი მოსახერხებელი მალსახმობებისა და IDF მონიტორის გამოყენების შესახებ დამატებითი დეტალებისთვის.
  • idf.py სრული მითითებისთვის idf.py ბრძანებები და პარამეტრები.

ეს ყველაფერია, რაც გჭირდებათ ESP32-ის დასაწყებად!

ახლა თქვენ მზად ხართ სცადოთ სხვა მაგალითები, ან პირდაპირ გადადით საკუთარი აპლიკაციების შემუშავებაზე.

ESP-IDF განახლება¶

დროდადრო უნდა განაახლოთ ESP-IDF , რადგან ახალი ვერსიები ასწორებს შეცდომებს და გვთავაზობს ახალ ფუნქციებს. განახლების უმარტივესი გზა არსებულის წაშლაა. esp-idf საქაღალდე და ხელახლა დააკლონირეთ, თითქოს ასრულებთ აღწერილ საწყის ინსტალაციას ნაბიჯი 2. მიიღეთ ESP-IDF.

კიდევ ერთი გამოსავალია მხოლოდ შეცვლილი ინფორმაციის განახლება. განახლების პროცედურა დამოკიდებულია თქვენს მიერ გამოყენებულ ESP-IDF ვერსიაზე.

ESP-IDF განახლების შემდეგ, ხელახლა შეასრულეთ ინსტალაციის სკრიპტი, იმ შემთხვევაში, თუ ESP-IDF ის ახალი ვერსია მოითხოვს ინსტრუმენტების განსხვავებულ ვერსიებს. იხილეთ ინსტრუქციები ნაბიჯი 3. ხელსაწყოების დაყენება.

ახალი ხელსაწყოების ინსტალაციის შემდეგ, განაახლეთ გარემო ექსპორტის სკრიპტის გამოყენებით. ინსტრუქციები იხილეთ შემდეგ ბმულზე: ნაბიჯი 4. გარემოს ცვლადების დაყენება.

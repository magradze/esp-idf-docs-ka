Windows-ისთვის Toolchain-ის სტანდარტული დაყენება¶

[中文]

შენიშვნა

ამჟამად Windows-ის მხოლოდ 64-ბიტიანი ვერსიებია მხარდაჭერილი. 32-ბიტიან Windows-ს შეუძლია გამოიყენოს მემკვიდრეობით მიღებული GNU Make Build System.

შესავალი¶

ESP-IDF საჭიროა რამდენიმე წინაპირობადი ინსტრუმენტის ინსტალაცია, რათა შეძლოთ პროგრამული უზრუნველყოფა აწყობა მხარდაჭერილი ჩიპებისთვის. წინაპირობად ინსტრუმენტებს შორისაა Python, Git, cross- კომპილაცია rs, CMake და Ninja-ს აწყობის ინსტრუმენტები.

ამ „დასაწყისისთვის“ ჩვენ გამოვიყენებთ ბრძანების ხაზს, მაგრამ ESP-IDF ის ინსტალაციის შემდეგ შეგიძლიათ გამოიყენოთ დაბნელება ან სხვა გრაფიკული IDE CMake მხარდაჭერით.

შენიშვნა

ESP-IDF ის წინა ვერსიები იყენებდა მემკვიდრეობით მიღებული GNU Make Build System და MSYS2 Unix-თან თავსებადობის გარემო. ეს აღარ არის საჭირო, ESP-IDF გამოყენება შესაძლებელია Windows-ის ბრძანების ხაზიდან.

ESP-IDF ინსტრუმენტების ინსტალატორი¶

ESP-IDF ის წინაპირობების ინსტალაციის უმარტივესი გზაა ESP-IDF Tools ინსტალერის ჩამოტვირთვა ამ URL-დან:

https://dl.espressif.com/dl/esp-idf-tools-setup-2.2.exe

ინსტალერი მოიცავს cross- კომპილაცია rs-ს, OpenOCD-ს, სმეიქი და ნინძა აწყობის ინსტრუმენტი. ინსტალერს ასევე შეუძლია ინსტალატორების ჩამოტვირთვა და გაშვება პითონი 3.7 და Git Windows-ისთვის თუ ისინი უკვე არ არის დაინსტალირებული კომპიუტერზე.

ინსტალერი ასევე გთავაზობთ ESP-IDF ის ერთ-ერთი გამოშვების ვერსიის ჩამოტვირთვას.

ბრძანების ხაზის გამოყენება¶

დარჩენილი დაწყების ნაბიჯებისთვის, ჩვენ გამოვიყენებთ Windows-ის ბრძანების ხაზს.

ESP-IDF ინსტრუმენტების ინსტალატორი შექმნის მალსახმობს Start მენიუში ESP-IDF ბრძანების ხაზის გასაშვებად. ეს მალსახმობი ხსნის ბრძანების ხაზს (cmd.exe) და ამუშავებს export.bat სკრიპტი გარემოს ცვლადების დასაყენებლად (PATH, IDF_PATH და სხვა). ამ ბრძანების ხაზში ხელმისაწვდომია ყველა დაინსტალირებული ინსტრუმენტი.

გაითვალისწინეთ, რომ ეს მალსახმობი სპეციფიკურია ESP-IDF ინსტრუმენტების ინსტალერში არჩეულ ESP-IDF დირექტორიისთვის. თუ კომპიუტერში გაქვთ ESP-IDF დირექტორიების რამდენიმე ვარიანტი (მაგალითად, ESP-IDF ის სხვადასხვა ვერსიასთან სამუშაოდ), მათი გამოყენების ორი ვარიანტი გაქვთ:

  1. შექმენით ESP-IDF Tools Installer-ის მიერ შექმნილი მალსახმობის ასლი და შეცვალეთ ახალი მალსახმობის სამუშაო დირექტორია იმ ESP-IDF დირექტორიით, რომლის გამოყენებაც გსურთ.
  2. ალტერნატიულად, გაუშვით cmd.exe, შემდეგ გადადით ESP-IDF დირექტორიაში, რომლის გამოყენებაც გსურთ და გაუშვით export.batგაითვალისწინეთ, რომ წინა ვარიანტისგან განსხვავებით, ამ გზით საჭიროა Python-ის და Git-ის არსებობა. PATHთუ Python-ის ან Git-ის ვერ პოულობასთან დაკავშირებულ შეცდომებს მიიღებთ, გამოიყენეთ პირველი ვარიანტი.

შემდეგი ნაბიჯები¶

თუ ESP-IDF Tools Installer წარმატებით დასრულდა, მაშინ განვითარების გარემოს დაყენება დასრულებულია. გადადით პირდაპირ ნაბიჯი 5. პროექტის დაწყება.

დაკავშირებული დოკუმენტები¶

მოწინავე მომხმარებლებისთვის, რომელთაც სურთ ინსტალაციის პროცესის პერსონალიზება:

  • Windows Toolchain-ის ნულიდან დაყენება
  • ESP-IDF ინსტრუმენტების განახლება Windows-ზე

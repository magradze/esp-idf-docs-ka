ESP32-S2-ის წინასწარი გადახედვის მხარდაჭერა¶

ESP-IDF ის მიმდინარე მასტერ შტო (V4.2-მდე) შეიცავს ESP32-S2-ის წინასწარი ვერსიის მხარდაჭერას.

პროგრამული უზრუნველყოფის მხარდაჭერა ჯერ კიდევ შემუშავების პროცესშია ESP32-S2-ის საბოლოო გამოშვებამდე (იხ. შეზღუდვები და გზამკვლევი ქვემოთ მოცემული სექციები).

დოკუმენტაცია¶

გარდა იმ შემთხვევებისა, როდესაც კონკრეტულად არის ნახსენები ESP32-S2, ESP-IDF პროგრამირების სახელმძღვანელოს ეს ვერსია მხოლოდ ESP32-სთვისაა დაწერილი. პროგრამული უზრუნველყოფის API და კომპონენტი ვერსიების უმეტესობა ESP32-S2-ისთვის იგივე ან ძალიან ჰგავს ერთმანეთს, თუმცა განსხვავებები შესაძლოა ჯერ არ იყოს დოკუმენტირებული.

ESP32-S2-ის დაყენება¶

დასაწყებად, მიჰყევით დაწყება სახელმძღვანელო ESP32-ისთვის CMake-ზე დაფუძნებული build სისტემის კონფიგურაციისთვის.

სირბილის დროს install.sh ან install.bat „დაწყების სახელმძღვანელოს“ ფარგლებში, ESP32-S2 ინსტრუმენტების ნაკრები ავტომატურად ჩამოიტვირთება და დაემატება IDF Tools დირექტორიას. თუ უკვე გაუშვით install.sh ან install.bat ESP32-S2 მხარდაჭერის მქონე ვერსიაზე განახლებამდე, შეიძლება დაგჭირდეთ მისი ხელახლა გაშვება.

ინსტრუმენტების ნაკრები tuple არის xtensa-esp32s2-elf-, მაგალითად, ამ სამიზნისთვის GCC კომპილაცია r დასახელებულია xtensa-esp32s2-elf-gcc.

სახელმძღვანელო ინსტრუმენტების ნაკრები URL-ები¶

თუ არ იყენებთ install.sh ან install.bat, შეგიძლიათ ხელით ჩამოტვირთოთ სახელმძღვანელო ინსტრუმენტების ნაკრები აქედან:

  • Windows
  • macOS
  • Linux 64-ბიტიანი x86
  • Linux 32-ბიტიანი x86

თუ ინსტრუმენტების ნაკრები ხელით დააინსტალირებთ, სადმე გახსენით და დაამატეთ bin თქვენი ქვედირექტორია PATH.

ESP32-S2-ისთვის პროექტის შექმნა¶

მნიშვნელოვანი

ESP-IDF ყველა მაგალითი არ უჭერს მხარს ESP32-S2-ს.

ზოგიერთი მაგალითის შემთხვევაში ეს იმიტომ ხდება, რომ საჭირო აპარატურა არ შედის ESP32-S2-ში, ამიტომ მისი მხარდაჭერა შეუძლებელია. ზოგიერთი მაგალითის შემთხვევაში ეს იმიტომ ხდება, რომ მაგალითი ჯერ არ განახლებულა ESP32-S2-ისთვის.

თუ მაგალითს ქმნით, გთხოვთ, შეამოწმოთ CMakeLists.txt ფაილი პუნქტისთვის. SUPPORTED_TARGETS esp32თუ ეს არსებობს, მაგალითი არ იმუშავებს ESP32-S2-ზე.

ESP-IDF პროექტის შექმნამდე, შეასრულეთ შემდეგი ბრძანება, რათა სამიზნე ESP32-S2-ზე გადართოთ:

idf.py set-target esp32s2

მნიშვნელოვანი

სირბილი set-target გაასუფთავებს პროექტის კონფიგურაცია და შექმნის ახალ ცარიელ ველს sdkconfig პროექტის ფაილი. ძველი კონფიგურაცია შეინახება sdkconfig.old.

შემდეგ დააკონფიგურირეთ პროექტი:

idf.py menuconfig

მნიშვნელოვანი

ხელმისაწვდომია ESP32-S2-ის 3 ვერსია სხვადასხვა Wi-Fi PHY-ებით. მათი იდენტიფიცირება შესაძლებელია თავად ჩიპზე მონიშვნით. პროექტი უნდა იყოს კონფიგურირებული ჩიპთან შესაბამისობაში პროექტის პარამეტრის „ESP32-S2 ჩიპის ვერსია“ მეშვეობით (CONFIG_ESP32S2_CHIP_VERSION). ეს შეგიძლიათ იხილოთ Component Config -&gt; ESP32S2-სპეციფიკურ განყოფილებაში. თუ პროექტის პარამეტრი არ ემთხვევა ჩიპს, მაშინ Wi-Fi მუშაობა შეიძლება ძალიან ცუდი იყოს, ან ჩიპი შეიძლება გაითიშოს Wi-Fi ინიციალიზაციისას.

პროექტის აშენება და ფლეშ მეხსიერება ჩვეულებისამებრ, მაგალითად:

idf.py -p PORT flash monitor

გაიარეთ კონსულტაცია დაწყება სახელმძღვანელო ESP-IDF პროექტის შექმნისა და ფლეშ მეხსიერება დაგეგმვის შესახებ დამატებითი ინფორმაციისთვის.

შენიშვნა

შესაძლებელია პროექტის მიზნის ESP32-ზე დაბრუნება მისი გაშვებით. idf.py set-target esp32ეს ასევე ხსნის პროექტის კონფიგურაცია .

შეზღუდვები¶

ESP32-S2 მხარდაჭერა ამჟამად წინასწარი გადახედვის ეტაპზეა და ყველა ფუნქციას არ უჭერს მხარს. Espressif პროგრამული უზრუნველყოფა დეველოპერები დარჩენილ მხარდაჭერაზე ინტენსიურად მუშაობენ, ამიტომ გთხოვთ, განაახლოთ master რეგულარულად ეწვიეთ ფილიალს უახლესი კოდის მისაღებად.

  • ინტეგრირებული USB OTG პერიფერიული მოწყობილობა არ არის მხარდაჭერილი
  • ESP32-S2-ისთვის დოკუმენტაცია უმეტეს ადგილას არ განახლებულა
  • პერიფერიული დრაივერები დამუშავების პროცესშია
  • ენერგიის დაზოგვის რეჟიმები არ არის
  • არ აქვს აპარატურული უსაფრთხოების ფუნქციები ან კრიპტოგრაფიული ამაჩქარებლის მხარდაჭერა
  • ფრენის დროის (TOF) გაზომვები ნორმალური Wi-Fi პაკეტებით არ არის მხარდაჭერილი.

გზამკვლევი¶

  • ESP-IDF V4.1 მხარს დაუჭერს ESP32-S2beta ჩიპს წინასწარი ვერსიის სახით, გარკვეული შეზღუდვებით.
  • ESP-IDF V4.2 მხარს დაუჭერს ESP32-S2-ს (წინა ESP32-S2beta ჩიპი ვერ იმუშავებს V4.2-ზე).

ESP32-S2beta-ს მხარდაჭერა ESP-IDF დან გაუქმდება, როგორც კი ESP32-S2 ხელმისაწვდომი გახდება.

საკითხები¶

თუ აღმოაჩენთ პრობლემებს, რომლებიც არ არის ნახსენები შეზღუდვები, გთხოვთ გახსენით საკითხი GitHub-ზე.

გთხოვთ, აუცილებლად ახსენოთ, რომ იყენებთ ESP32-S2-ის წინასწარი გადახედვის მხარდაჭერას და მიუთითოთ ESP-IDF ის თქვენი ზუსტი ვერსია.

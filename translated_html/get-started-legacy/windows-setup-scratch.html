Windows Toolchain-ის ნულიდან დაყენება (Legacy GNU Make)¶

შენიშვნა

ESP-IDF V4.0-დან დაწყებული, ნაგულისხმევი აწყობის სისტემა დაფუძნებულია CMake-ზე. ეს დოკუმენტაცია განკუთვნილია GNU Make-ზე დაფუძნებული მემკვიდრეობითი აწყობის სისტემისთვის. ამ აწყობის სისტემის მხარდაჭერა შესაძლოა მომავალ ძირითად ვერსიებში გაუქმდეს.

გარემოს დაყენება გაძლევთ მეტ კონტროლს პროცესზე და ასევე უზრუნველყოფს ინფორმაციას მოწინავე მომხმარებლებისთვის ინსტალაციის პერსონალიზაციისთვის. წინასწარ შექმნილი გარემო, რომელიც ნაკლებად გამოცდილი მომხმარებლებისთვისაა განკუთვნილი, მომზადებულია ამ ნაბიჯების დაცვით.

სტანდარტული გზით, წინასწარ აგებული გარემოს გამოყენებით, ინსტრუმენტების ნაკრები სწრაფად დასაყენებლად, გადადით განყოფილებაზე Windows-ისთვის Toolchain-ის სტანდარტული დაყენება (Legacy GNU Make).

ნულიდან ინსტრუმენტთა ჯაჭვისა და გარემოს კონფიგურაცია¶

ეს პროცესი მოიცავს ინსტალაციას MSYS2, შემდეგ კი ინსტალირდება MSYS2 და Python-ის პაკეტები, რომლებსაც ESP-IDF იყენებს და ბოლოს, Xtensa ინსტრუმენტების ნაკრები ჩამოტვირთვა და ინსტალაცია.

  • ნავიგაცია შემდეგზე MSYS2 ინსტალერის გვერდი და ჩამოტვირთეთ msys2-i686-xxxxxxx.exe ინსტალერის შესრულებადი ფაილი (ჩვენ მხოლოდ 32-ბიტიან MSYS გარემოს ვუჭერთ მხარს, ის მუშაობს როგორც 32-ბიტიან, ასევე 64-ბიტიან Windows-ზე.) წერის მომენტისთვის, ინსტალერის უახლესი ვერსიაა msys2-i686-20161025.exe.

  • გაიარეთ ინსტალატორის ნაბიჯები. ბოლოში მოხსენით მონიშვნა „Run MSYS2 32-bit now“-ს ველიდან.

  • ინსტალატორის გამოსვლის შემდეგ, გახსენით „დაწყების“ მენიუ და იპოვეთ „MSYS2 MinGW 32-bit“ ტერმინალის გასაშვებად.

    (რატომ უნდა გავუშვათ ეს განსხვავებული ტერმინალი? MSYS2-ს აქვს სხვადასხვა სახის გარემოს კონცეფცია. ნაგულისხმევი „MSYS“ გარემო Cygwin-ის მსგავსია და იყენებს თარგმანის ფენას Windows API ყველა გამოძახებისთვის. ჩვენ გვჭირდება „MinGW“ გარემო, რათა გვქონდეს მშობლიური Python, რომელიც მხარს უჭერს COM პორტებს.)

  • github-ზე ESP-IDF საცავი შეიცავს სკრიპტს tools დირექტორიაში სახელწოდებით windows_install_prerequisites.shთუ ჯერ არ გაქვთ ESP-IDF ის ლოკალური ასლი, არა უშავს - შეგიძლიათ უბრალოდ ჩამოტვირთოთ ეს ერთი ფაილი Raw ფორმატში აქედან: tools/windows/windows_install_prerequisites.shშეინახეთ ის თქვენს კომპიუტერში სადმე.

  • MSYS2 ტერმინალის ფანჯარაში შეიყვანეთ shell სკრიპტის გზა. შეგიძლიათ აკრიფოთ ის ჩვეულებრივი Windows-ის გზად, მაგრამ უკუ-სლეშის ნაცვლად გამოიყენეთ წინ მიმართული სლეშები. მაგ.: C:/Users/myuser/Downloads/windows_install_prerequisites.shშეგიძლიათ წინასწარ წაიკითხოთ სკრიპტი, რათა ნახოთ, რას აკეთებს ის.

  • ის windows_install_prerequisites.sh სკრიპტი ჩამოტვირთავს და დააინსტალირებს პაკეტებს ESP-IDF მხარდაჭერისთვის და ESP32 ინსტრუმენტების ნაკრები .

პრობლემების მოგვარება¶

  • ინსტალაციის სკრიპტის გაშვებისას, MSYS-მა შეიძლება განაახლოს თავისი თავი ისეთ მდგომარეობაში, რომ ვეღარ იმუშავებს. შეიძლება შეგხვდეთ შემდეგი შეცდომები:

    *** fatal error - cygheap base mismatch detected - 0x612E5408/0x612E4408. This problem is probably due to using incompatible versions of the cygwin DLL.

    თუ მსგავს შეცდომებს შეამჩნევთ, მთლიანად დახურეთ ტერმინალის ფანჯარა (იქ მიმდინარე პროცესების შეწყვეტით) და შემდეგ ხელახლა გახსენით ახალი ტერმინალი. ხელახლა გაუშვით. windows_install_prerequisites.sh (რჩევა: ბოლო გაშვების ბრძანების სანახავად გამოიყენეთ ზემოთ მიმართული ისრის ღილაკი). განახლების პროცესი ამ ნაბიჯის შემდეგ განახლდება.

  • MSYS2 არის „მოძრავი“ დისტრიბუცია, ამიტომ ინსტალატორის სკრიპტის გაშვებამ შეიძლება დააინსტალიროს უფრო ახალი პაკეტები, ვიდრე ის, რაც გამოიყენება წინასწარ აგებულ გარემოში. თუ შეამჩნევთ რაიმე შეცდომას, რომელიც, როგორც ჩანს, დაკავშირებულია MSYS2 პაკეტების ინსტალაციასთან, გთხოვთ, შეამოწმოთ MSYS2-პაკეტების პრობლემების სია ცნობილი პრობლემებისთვის. თუ ვერ ხედავთ რაიმე შესაბამის პრობლემას, გთხოვთ, IDF-ის საკითხის წამოწევა.

MSYS2 სარკეები ჩინეთში¶

ჩინეთში არსებობს რამდენიმე (არაოფიციალური) MSYS2 სარკე, რაც მნიშვნელოვნად აუმჯობესებს ჩამოტვირთვის სიჩქარეს ჩინეთში.

ამ სარკეების დასამატებლად, დაყენების სკრიპტის გაშვებამდე შეცვალეთ შემდეგი ორი MSYS2 სარკეების სიის ფაილი. სარკეების სიის ფაილები შეგიძლიათ იხილოთ შემდეგში: /etc/pacman.d დირექტორია (მაგ. c:\msys2\etc\pacman.d).

დაამატეთ ეს ხაზები ზედა ნაწილში mirrorlist.mingw32:

Server = https://mirrors.ustc.edu.cn/msys2/mingw/i686/
Server = http://mirror.bit.edu.cn/msys2/REPOS/MINGW/i686

დაამატეთ ეს ხაზები ზედა ნაწილში mirrorlist.msys:

Server = http://mirrors.ustc.edu.cn/msys2/msys/$arch
Server = http://mirror.bit.edu.cn/msys2/REPOS/MSYS2/$arch

HTTP პროქსი¶

MSYS და PIP ჩამოტვირთვებისთვის HTTP პროქსის ჩართვა შეგიძლიათ შემდეგი პარამეტრის დაყენებით: http_proxy ცვლადი ტერმინალში დაყენების სკრიპტის გაშვებამდე:

export http_proxy='http://http.proxy.server:PORT'

ან რწმუნებათა სიგელებით:

export http_proxy='http://user:[email protected]:PORT'

დაამატეთ ეს ხაზი /etc/profile MSYS დირექტორიაში, რათა MSYS-ის გამოყენებისას პროქსი სამუდამოდ ჩაირთოს.

ალტერნატიული დაყენება: უბრალოდ ჩამოტვირთეთ ინსტრუმენტების ნაკრები¶

თუ უკვე გაქვთ MSYS2 ინსტალაცია ან გსურთ სხვაგვარად გააკეთოთ საქმეები, შეგიძლიათ ჩამოტვირთოთ მხოლოდ ინსტრუმენტების ნაკრები აქ:

https://dl.espressif.com/dl/xtensa-esp32-elf-gcc8_2_0-esp-2019r2-win32.zip

შენიშვნა

თუ ინსტრუქციებს მიჰყევით ნულიდან ინსტრუმენტთა ჯაჭვისა და გარემოს კონფიგურაცია, თქვენ უკვე გაქვთ ინსტრუმენტების ნაკრები და ეს ჩამოტვირთვა არ დაგჭირდებათ.

მნიშვნელოვანი

უბრალოდ ამ ინსტრუმენტების ნაკრები ქონა არის არ არის საკმარისი Windows-ზე ESP-IDF გამოსაყენებლად. თქვენ დაგჭირდებათ მინიმუმ GNU make, bash და sed. ზემოთ მოცემული გარემოები უზრუნველყოფს ამ ყველაფერს, პლუს კომპილაცია r ჰოსტს (საჭიროა menuconfig მხარდაჭერისთვის).

შემდეგი ნაბიჯები¶

განვითარების გარემოს დაყენების გასაგრძელებლად, გადადით განყოფილებაში ნაბიჯი 2. მიიღეთ ESP-IDF.

გარემოს განახლება¶

IDF-ის განახლებისას, ზოგჯერ საჭიროა ახალი ინსტრუმენტების ნაკრები ები ან Windows MSYS2 გარემოს ემატება ახალი სისტემური მოთხოვნები.

ახალი გარემოს შექმნის ნაცვლად, შეგიძლიათ განაახლოთ არსებული Windows გარემო და ინსტრუმენტების ნაკრები :

  • განაახლეთ IDF იმ ახალ ვერსიამდე, რომლის გამოყენებაც გსურთ.
  • გაუშვით tools/windows/windows_install_prerequisites.sh სკრიპტი IDF-ში. ეს დააინსტალირებს ნებისმიერ ახალ პროგრამულ პაკეტს, რომელიც ადრე არ იყო დაინსტალირებული და ჩამოტვირთავს და ჩაანაცვლებს ინსტრუმენტების ნაკრები უახლესი ვერსიით.

Windows-ისთვის Toolchain-ის სტანდარტული დაყენება (Legacy GNU Make)¶

[中文]

შენიშვნა

ESP-IDF V4.0-დან დაწყებული, ნაგულისხმევი აწყობის სისტემა დაფუძნებულია CMake-ზე. ეს დოკუმენტაცია განკუთვნილია GNU Make-ზე დაფუძნებული მემკვიდრეობითი აწყობის სისტემისთვის. ამ აწყობის სისტემის მხარდაჭერა შესაძლოა მომავალ ძირითად ვერსიებში გაუქმდეს.

შესავალი¶

Windows-ს არ აქვს ჩაშენებული „make“ გარემო, ამიტომ ინსტრუმენტების ნაკრები ის ინსტალაციის გარდა, დაგჭირდებათ GNU-თან თავსებადი გარემო. ჩვენ ვიყენებთ MSYS2 გარემო ამის უზრუნველსაყოფად. თქვენ არ გჭირდებათ ამ გარემოს მუდმივად გამოყენება (შეგიძლიათ გამოიყენოთ დაბნელება ან სხვა რაიმე ფრონტ-ენდი), მაგრამ ის კულისებში მუშაობს.

ხელსაწყოთა ჯაჭვის დაყენება¶

სწრაფი დაყენებაა Windows-ის ყველა-ერთში ინსტრუმენტების ნაკრები და MSYS2 zip ფაილის ჩამოტვირთვა dl.espressif.com-დან:

https://dl.espressif.com/dl/esp32_win32_msys2_environment_and_gcc8_ ინსტრუმენტების ნაკრები -20191231.zip

გახსენით zip ფაილი C:\ (ან სხვა ადგილას, მაგრამ ეს სახელმძღვანელო ვარაუდობს C:\) და ეს შექმნის msys32 დირექტორია წინასწარ მომზადებული გარემოთი.

მნიშვნელოვანი

თუ გამოიყენება ინსტრუმენტების ნაკრები სხვა მდებარეობა (ნაგულისხმევი მდებარეობისგან განსხვავებული) C:\msys32), გთხოვთ, დარწმუნდეთ, რომ გზა, სადაც ყოვლისმომცველი ინსტრუმენტების ნაკრები იხსნება, არის უბრალო ASCII კოდი, არ შეიცავს ინტერვალებს, სიმბოლურ ბმულებს ან აქცენტებს.

გაეცანით¶

გახსენით MSYS2 MINGW32 ტერმინალის ფანჯარა შემდეგი ბრძანების გაშვებით: C:\msys32\mingw32.exeამ ფანჯარაში გარემო bash shell-ია. შექმენით დირექტორია სახელწოდებით esp ეს არის ESP32 აპლიკაციების შემუშავების ნაგულისხმევი ადგილმდებარეობა. ამისათვის, გაუშვით შემდეგი shell ბრძანება:

mkdir -p ~/esp

აკრეფით cd ~/esp შემდეგ შეგიძლიათ ახლად შექმნილ დირექტორიაში გადახვიდეთ. თუ შეცდომის შეტყობინებები არ არის, ეს ნაბიჯი დასრულებულია.

MSYS2 MINGW32 გარსის ფანჯარა

გამოიყენეთ ეს ფანჯარა ESP32-ისთვის შემუშავების გარემოს დაყენების შემდეგ ნაბიჯებში.

შემდეგი ნაბიჯები¶

განვითარების გარემოს დაყენების გასაგრძელებლად, გადადით განყოფილებაში ნაბიჯი 2. მიიღეთ ESP-IDF.

გარემოს განახლება¶

IDF-ის განახლებისას, ზოგჯერ Windows MSYS2 გარემოში ახალი ინსტრუმენტების ნაკრები ფაილებია საჭირო ან ახალი მოთხოვნები ემატება. კომპილაცია d გარემოს წინა ვერსიიდან ახალში მონაცემების გადასატანად:

  • ავიღოთ ძველი MSYS2 გარემო (მაგ. C:\msys32) და გადაიტანეთ/შეცვალეთ სახელი სხვა დირექტორიაში (მაგ. C:\msys32_old).
  • ჩამოტვირთეთ ახალი წინასწარი კომპილაცია ე გამოცდის გარემო ზემოთ მოცემული ნაბიჯების გამოყენებით.
  • გახსენით ახალი MSYS2 გარემო zip-ში C:\msys32 (ან სხვა ადგილას).
  • იპოვეთ ძველი C:\msys32_old\home დირექტორია და გადაიტანეთ ეს C:\msys32.
  • ახლა შეგიძლიათ წაშალოთ C:\msys32_old დირექტორია, თუ ის აღარ გჭირდებათ.

თქვენს სისტემაში შეგიძლიათ გქონდეთ დამოუკიდებელი სხვადასხვა MSYS2 გარემო, იმ პირობით, რომ ისინი სხვადასხვა დირექტორიაში იქნებიან.

არსებობს ასევე ნაბიჯები არსებული გარემოს განახლებისთვის ახლის ჩამოტვირთვის გარეშე, თუმცა ეს უფრო რთულია.

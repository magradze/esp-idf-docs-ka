IDF_PATH-ის დამატება მომხმარებლის პროფილში (Legacy GNU Make)¶

[中文]

შენიშვნა

ESP-IDF V4.0-დან დაწყებული, ნაგულისხმევი აწყობის სისტემა დაფუძნებულია CMake-ზე. ეს დოკუმენტაცია განკუთვნილია GNU Make-ზე დაფუძნებული მემკვიდრეობითი აწყობის სისტემისთვის. ამ აწყობის სისტემის მხარდაჭერა შესაძლოა მომავალ ძირითად ვერსიებში გაუქმდეს.

პარამეტრის შესანარჩუნებლად IDF_PATH სისტემის გადატვირთვებს შორის გარემოს ცვლადის სახით, დაამატეთ იგი მომხმარებლის პროფილში ქვემოთ მოცემული ინსტრუქციების შესაბამისად.

Windows¶

მომხმარებლის პროფილის სკრიპტები მოთავსებულია C:/msys32/etc/profile.d/ დირექტორია. ისინი სრულდება ყოველ ჯერზე, როდესაც ხსნით MSYS2 ფანჯარას.

  1. შექმენით ახალი სკრიპტის ფაილი C:/msys32/etc/profile.d/ დირექტორია. დაარქვით სახელი. დაარქვით მას სახელი. export_idf_path.sh.

  2. ESP-IDF დირექტორიისკენ მიმავალი გზის იდენტიფიცირება. ის თქვენი სისტემის კონფიგურაციისთვისაა დამახასიათებელი და შეიძლება ასე გამოიყურებოდეს: C:\msys32\home\user-name\esp\esp-idf

  3. დაამატეთ export ბრძანება სკრიპტის ფაილზე, მაგ.:

    export IDF_PATH="C:/msys32/home/user-name/esp/esp-idf"

    გახსოვდეთ, რომ Windows-ის საწყის გზაზე უკუ-დახრილი ხაზები წინ-დახრილი ხაზებით უნდა შეცვალოთ.

  4. შეინახეთ სკრიპტის ფაილი.

  5. დახურეთ MSYS2 ფანჯარა და ხელახლა გახსენით. შეამოწმეთ, თუ IDF_PATH დაყენებულია აკრეფით:

    printenv IDF_PATH

    სკრიპტის ფაილში ადრე შეყვანილი გზა უნდა დაიბეჭდოს.

თუ არ მოგწონს ქონა IDF_PATH მომხმარებლის პროფილში მუდმივად დაყენების შემთხვევაში, MSYS2 ფანჯრის გახსნისას ის ხელით უნდა შეიყვანოთ:

export IDF_PATH="C:/msys32/home/user-name/esp/esp-idf"

თუ აქ სექციიდან მოხვდი ნაბიჯი 3. გარემოს ცვლადების დაყენება, ESP32 შემუშავებისთვის პროგრამული უზრუნველყოფის ინსტალაციისას, შემდეგ დაბრუნდით განყოფილებაში ნაბიჯი 5. პროექტის დაწყება.

Linux და MacOS¶

დაყენება IDF_PATH შემდეგი ხაზის დამატებით ~/.profile ფაილი:

export IDF_PATH=~/esp/esp-idf

ამ ცვლილების ძალაში შესასვლელად, გამოდით სისტემიდან და ხელახლა შედით სისტემაში.

შენიშვნა

თუ გაქვთ /bin/bash დაყენებულია როგორც შესვლის გარსი და ორივე .bash_profile და .profile არსებობს, შემდეგ განახლდება .bash_profile სამაგიეროდ.

შეამოწმე, თუ არა, შეასრულე შემდეგი ბრძანება IDF_PATH დაყენებულია:

printenv IDF_PATH

ადრე შეყვანილი გზა ~/.profile ფაილი (ან ხელით დაყენებული) უნდა დაიბეჭდოს.

თუ არ მოგწონს ქონა IDF_PATH მუდმივად დაყენების შემთხვევაში, ყოველი გადატვირთვის ან გასვლისას ის ხელით უნდა შეიყვანოთ ტერმინალის ფანჯარაში:

export IDF_PATH=~/esp/esp-idf

თუ აქ სექციიდან მოხვდი ნაბიჯი 3. გარემოს ცვლადების დაყენება, ESP32 შემუშავებისთვის პროგრამული უზრუნველყოფის ინსტალაციისას, შემდეგ დაბრუნდით განყოფილებაში ნაბიჯი 5. პროექტის დაწყება.

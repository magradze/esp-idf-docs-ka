Linux Toolchain-ის ნულიდან დაყენება (Legacy GNU Make)¶

[中文]

შენიშვნა

ESP-IDF V4.0-დან დაწყებული, ნაგულისხმევი აწყობის სისტემა დაფუძნებულია CMake-ზე. ეს დოკუმენტაცია განკუთვნილია GNU Make-ზე დაფუძნებული მემკვიდრეობითი აწყობის სისტემისთვის. ამ აწყობის სისტემის მხარდაჭერა შესაძლოა მომავალ ძირითად ვერსიებში გაუქმდეს.

შენიშვნა

აღწერილია ინსტრუმენტების ნაკრები ინსტალაციის სტანდარტული პროცესი აქ. იხილეთ Toolchain-ის მორგებული დაყენება განყოფილებაში მოცემულია რამდენიმე მიზეზი, თუ რატომ შეიძლება საჭირო გახდეს ინსტრუმენტების ნაკრები ნულიდან ინსტალაცია.

ინსტალაციის წინაპირობები¶

კომპილაცია ისთვის ESP-IDF ით თქვენ უნდა მიიღოთ შემდეგი პაკეტები:

  • Ubuntu და Debian:

    sudo apt-get install gcc git wget make libncurses-dev flex bison gperf python python-pip python-setuptools python-serial python-cryptography python-future python-pyparsing python-pyelftools libffi-dev libssl-dev

  • თაღი:

    sudo pacman -S --needed gcc git make ncurses flex bison gperf python2-pyserial python2-cryptography python2-future python2-pyparsing python2-pyelftools

შენიშვნა

ზოგიერთი ძველი (2014 წლამდე) Linux დისტრიბუცია შეიძლება იყენებდეს pyserial ვერსია 2.x, რომელიც არ არის მხარდაჭერილი ESP-IDF მიერ. ამ შემთხვევაში, გთხოვთ, დააინსტალიროთ მხარდაჭერილი ვერსია. pip როგორც აღწერილია ნაწილში ნაბიჯი 4. დააინსტალირეთ საჭირო Python პაკეტები.

წყაროდან ინსტრუმენტების ჯაჭვის კომპილაცია¶

  • ინსტალაციის დამოკიდებულებები:

    • CentOS 7:

      sudo yum install gawk gperf grep gettext ncurses-devel python python-devel automake bison flex texinfo help2man libtool

    • Ubuntu 16.04-მდე ვერსია:

      sudo apt-get install gawk gperf grep gettext libncurses-dev python python-dev automake bison flex texinfo help2man libtool

    • Ubuntu 16.04 ან უფრო ახალი ვერსია:

      sudo apt-get install gawk gperf grep gettext python python-dev automake bison flex texinfo help2man libtool libtool-bin

    • დებიანი 9:

      sudo apt-get install gawk gperf grep gettext libncurses-dev python python-dev automake bison flex texinfo help2man libtool libtool-bin

შექმენით სამუშაო დირექტორია და შედით მასში:

mkdir -p ~/esp
cd ~/esp

ჩამოტვირთვა crosstool-NG და ააშენეთ იგი:

git clone https://github.com/espressif/crosstool-NG.git
cd crosstool-NG
git checkout esp-2019r2
git submodule update --init
./bootstrap && ./configure --enable-local && make

შექმენით ინსტრუმენტების ნაკრები :

./ct-ng xtensa-esp32-elf
./ct-ng build
chmod -R u+w builds/xtensa-esp32-elf

ხელსაწყოების ჯაჭვი აშენდება ~/esp/crosstool-NG/builds/xtensa-esp32-elf. გაჰყევით სტანდარტული დაყენების ინსტრუქციები რომ დაამატოთ ინსტრუმენტების ნაკრები თქვენს PATH.

შემდეგი ნაბიჯები¶

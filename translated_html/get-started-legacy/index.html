დაწყება (Legacy GNU Make)¶

[中文]

შენიშვნა

ESP-IDF V4.0-დან დაწყებული, ნაგულისხმევი აწყობის სისტემა დაფუძნებულია CMake-ზე. ეს დოკუმენტაცია განკუთვნილია GNU Make-ზე დაფუძნებული მემკვიდრეობითი აწყობის სისტემისთვის. ამ აწყობის სისტემის მხარდაჭერა შესაძლოა მომავალ ძირითად ვერსიებში გაუქმდეს.

ეს დოკუმენტი შექმნილია იმისთვის, რომ დაგეხმაროთ Espressif ESP32-ზე დაფუძნებული აპარატურისთვის პროგრამული უზრუნველყოფის შემუშავების გარემოს დაყენებაში.

ამის შემდეგ, მარტივი მაგალითი გაჩვენებთ, თუ როგორ გამოიყენოთ ESP-IDF (Espressif IoT Development Framework) მენიუს კონფიგურაციისთვის, შემდეგ კი როგორ ააწყოთ ფლეშ მეხსიერება პროგრამული უზრუნველყოფა ESP32 დაფაზე.

შენიშვნა

ეს არის ESP-IDF ის მასტერ ფრონტის (უახლესი ვერსიის) დოკუმენტაცია. ეს ვერსია მუდმივი დამუშავების პროცესშია. სტაბილური ვერსია დოკუმენტაცია ხელმისაწვდომია, ისევე როგორც სხვა ESP-IDF ვერსიები.

შესავალი¶

ESP32 არის ჩიპზე დაფუძნებული სისტემა, რომელიც აერთიანებს შემდეგ მახასიათებლებს:

  • Wi-Fi (2.4 GHz დიაპაზონი)
  • Bluetooth 4.2
  • ორმაგი მაღალი ხარისხის ბირთვი
  • ულტრა დაბალი სიმძლავრის კოპროცესორი
  • რამდენიმე პერიფერიული მოწყობილობა

40 ნმ ტექნოლოგიით აღჭურვილი ESP32 უზრუნველყოფს ძლიერ, მაღალ ინტეგრირებულ პლატფორმას, რომელიც ხელს უწყობს ენერგიის ეფექტური გამოყენების, კომპაქტური დიზაინის, უსაფრთხოების, მაღალი შესრულებისა და საიმედოობის მუდმივი მოთხოვნების დაკმაყოფილებას.

Espressif უზრუნველყოფს საბაზისო აპარატურულ და პროგრამულ რესურსებს, რათა დაეხმაროს აპლიკაციების შემქმნელებს ESP32 სერიის აპარატურის გამოყენებით თავიანთი იდეების რეალიზებაში. Espressif-ის მიერ შექმნილი პროგრამული უზრუნველყოფის შემუშავების ჩარჩო განკუთვნილია ნივთების ინტერნეტის (IoT) აპლიკაციების შემუშავებისთვის Wi-Fi , Bluetooth , ენერგიის მართვისა და სხვა რამდენიმე სისტემური ფუნქციის გამოყენებით.

რა გჭირდებათ¶

აპარატურა:

  • ერთი ESP32 დაფა
  • USB კაბელი - USB A / მიკრო USB B
  • კომპიუტერი Windows-ის, Linux-ის ან macOS-ის გაშვებული

პროგრამული უზრუნველყოფა:

  • ხელსაწყოების ჯაჭვი ასაშენებლად აპლიკაცია ESP32-ისთვის
  • ESP-IDF რომელიც არსებითად შეიცავს API (პროგრამული უზრუნველყოფის ბიბლიოთეკები და საწყისი კოდი) ESP32-ისთვის და სკრიპტებს მისი მუშაობისთვის. ხელსაწყოების ჯაჭვი
  • ტექსტის რედაქტორი პროგრამების დასაწერად (პროექტები) C-ში, მაგ., დაბნელება

ESP32-ისთვის აპლიკაციების შემუშავება

განვითარების საბჭოს მიმოხილვები¶

თუ ქვემოთ ჩამოთვლილი ESP32 დეველოპერული დაფებიდან ერთ-ერთი გაქვთ, მისი აპარატურის შესახებ დამატებითი ინფორმაციის მისაღებად შეგიძლიათ დააჭიროთ ბმულს.

  • ESP32-DevKitC
  • ESP-WROVER-KIT
  • ESP32-PICO-KIT
  • ESP32-Ethernet-Kit

ნაბიჯი 1. ინსტრუმენტების ჯაჭვის დაყენება¶

ინსტრუმენტების ნაკრები არის პროგრამების ნაკრები კოდის კომპილაციისა და აპლიკაციების შესაქმნელად.

ESP32-ით შემუშავების დაწყების ყველაზე სწრაფი გზა წინასწარ აწყობილი ინსტრუმენტების ნაკრები ის ინსტალაციაა. აიღეთ თქვენი ოპერაციული სისტემა ქვემოთ და მიჰყევით მოცემულ ინსტრუქციებს.

Windows Linux macOS

შენიშვნა

ეს სახელმძღვანელო იყენებს დირექტორიას ~/esp Linux-სა და macOS-ზე ან %userprofile%\esp Windows-ზე, როგორც ESP-IDF ის ინსტალაციის საქაღალდე. შეგიძლიათ გამოიყენოთ ნებისმიერი დირექტორია, მაგრამ შესაბამისად, ბრძანებების ბილიკების კორექტირება დაგჭირდებათ. გაითვალისწინეთ, რომ ESP-IDF არ უჭერს მხარს ბილიკებში დაშორებებს.

თქვენი გამოცდილებიდან და პრეფერენციებიდან გამომდინარე, შეიძლება დაგჭირდეთ თქვენი გარემოს პერსონალიზაცია წინასწარ შექმნილი ინსტრუმენტების ნაკრები გამოყენების ნაცვლად10. სისტემის საკუთარი გზით დასაყენებლად გადადით განყოფილებაში Toolchain-ის მორგებული დაყენება (Legacy GNU Make).

ნაბიჯი 2. მიიღეთ ESP-IDF¶

ინსტრუმენტების ნაკრები გარდა, თქვენ ასევე გჭირდებათ ESP32-ისთვის სპეციფიკური API (პროგრამული უზრუნველყოფის ბიბლიოთეკები და საწყისი კოდი). მათ Espressif უზრუნველყოფს. ESP-IDF საცავი.

ESP-IDF ის ლოკალური ასლის მისაღებად, გადადით თქვენს ინსტალაციის დირექტორიაში და დააკლონირეთ რეპოზიტორია git clone.

გახსენით ტერმინალი და შეასრულეთ შემდეგი ბრძანებები:

cd ~/esp
git clone --recursive https://github.com/espressif/esp-idf.git

ESP-IDF გადმოწერილი იქნება ~/esp/esp-idf.

კონსულტაცია ESP-IDF ვერსიები ინფორმაციისთვის, თუ რომელი ESP-IDF ვერსია უნდა იქნას გამოყენებული მოცემულ სიტუაციაში.

შენიშვნა

ეს ბრძანება დააკლონირებს მასტერ ტოტს, რომელსაც აქვს ESP-IDF ის უახლესი („გაუმჯობესებული“) ვერსია. ის სრულად ფუნქციონალურია და ყოველკვირეულად განახლდება უახლესი ფუნქციებითა და შეცდომების გამოსწორებით.

შენიშვნა

GitHub-ის „zip ფაილის ჩამოტვირთვის“ ფუნქცია არ მუშაობს ESP-IDF თან, a-სთან. git clone აუცილებელია. სარეზერვო ასლის სახით, სტაბილური ვერსია შეიძლება დაინსტალირდეს Git-ის გარეშე.

შენიშვნა

არ გამოტოვოთ --recursive ვარიანტი. თუ უკვე დაკლონირეთ ESP-IDF ამ ვარიანტის გარეშე, შეასრულეთ სხვა ბრძანება ყველა ქვემოდულის მისაღებად:

cd esp-idf
git submodule update --init

ნაბიჯი 3. გარემოს ცვლადების დაყენება¶

ინსტრუმენტების ნაკრები იყენებს გარემოს ცვლადს IDF_PATH ESP-IDF დირექტორიაზე წვდომისთვის. ეს ცვლადი თქვენს კომპიუტერში უნდა იყოს დაყენებული, წინააღმდეგ შემთხვევაში პროექტები არ შეიქმნება.

ეს ცვლადები შეიძლება დაყენდეს დროებით (თითო სესიაზე) ან მუდმივად. გთხოვთ, მიჰყვეთ კონკრეტულ ინსტრუქციებს. Windows , Linux და macOS განყოფილებაში IDF_PATH-ის დამატება მომხმარებლის პროფილში (Legacy GNU Make).

ნაბიჯი 4. დააინსტალირეთ საჭირო Python პაკეტები¶

ESP-IDF მიერ მოთხოვნილი Python პაკეტები მდებარეობს IDF_PATH/requirements.txtმათი ინსტალაცია შეგიძლიათ შემდეგი ბრძანების გაშვებით:

python -m pip install --user -r $IDF_PATH/requirements.txt

შენიშვნა

გთხოვთ, შეამოწმოთ Python-ის ინტერპრეტატორის ვერსია, რომელსაც ESP-IDF თან ერთად გამოიყენებთ. ამისათვის გაუშვით ბრძანება python --version და შედეგიდან გამომდინარე, შეიძლება დაგჭირდეთ გამოყენება python2, python2.7 ან მსგავსი, უბრალოდ ნაცვლად python, მაგ.:

python2.7 -m pip install --user -r $IDF_PATH/requirements.txt

ნაბიჯი 5. პროექტის დაწყება¶

ახლა თქვენ მზად ხართ მოამზადოთ თქვენი განაცხადი ESP32-ისთვის. შეგიძლიათ დაიწყოთ დაწყება/გამარჯობა_მსოფლიო პროექტი მაგალითები დირექტორია IDF-ში.

კოპირება დაწყება/გამარჯობა_მსოფლიო რომ ~/esp დირექტორია:

Linux და macOS¶

cd ~/esp
cp -r $IDF_PATH/examples/get-started/hello_world .

Windows¶

cd %userprofile%\esp
xcopy /e /i %IDF_PATH%\examples\get-started\hello_world hello_world

არსებობს პროექტების სამაგალითო მრავალფეროვნება, მაგალითები დირექტორია ESP-IDF ში. თქვენ შეგიძლიათ ნებისმიერი პროექტი დააკოპიროთ ზემოთ წარმოდგენილი მეთოდით და გაუშვათ.

ასევე შესაძლებელია მაგალითების ადგილზე შექმნა, მათი წინასწარი კოპირების გარეშე.

მნიშვნელოვანი

Esp-idf-ის აწყობის სისტემა არ უჭერს მხარს სივრცეებს არც esp-idf-ის და არც პროექტებისკენ მიმავალ ბილიკებში.

ნაბიჯი 6. თქვენი მოწყობილობის დაკავშირება¶

ახლა შეაერთეთ თქვენი ESP32 დაფა კომპიუტერთან და შეამოწმეთ, რომელი სერიული პორტი ქვეშ ჩანს ის.

სერიულ პორტებს სახელწოდებაში შემდეგი შაბლონები აქვთ:

  • Windows: სახელები, როგორიცაა COM1
  • Linux: დაწყებული /dev/tty
  • macOS: დაწყებული /dev/cu.

თუ არ ხართ დარწმუნებული, თუ როგორ შეამოწმოთ სერიული პორტი სახელი, გთხოვთ, იხილოთ სერიული კავშირის დამყარება ESP32-ით (Legacy GNU Make) სრული დეტალებისთვის.

შენიშვნა

პორტის სახელი ხელთ გქონდეთ, რადგან ის შემდეგ ნაბიჯებში დაგჭირდებათ.

ნაბიჯი 7. კონფიგურაცია¶

ნავიგაცია თქვენს hello_world დირექტორია ნაბიჯი 5. პროექტის დაწყება და გაუშვით პროექტის კონფიგურაცია პროგრამა menuconfig.

Linux და macOS¶

cd ~/esp/hello_world
make menuconfig

Windows¶

cd %userprofile%\esp\hello_world
make menuconfig

თუ წინა ნაბიჯები სწორად შესრულდა, გამოჩნდება შემდეგი მენიუ:

პროექტის კონფიგურაცია - მთავარი ფანჯარა

მენიუში გადადით შემდეგზე: Serial flasher config &gt; Default serial port სერიული პორტი ის კონფიგურაციისთვის, სადაც პროექტი ჩაიტვირთება. დაადასტურეთ შერჩევა enter-ის დაჭერით, შეინახეთ კონფიგურაცია არჩევით < Save > და შემდეგ გასვლა menuconfig შერჩევით < Exit >.

ნავიგაციისთვის და გამოსაყენებლად menuconfig, დააჭირეთ შემდეგ ღილაკებს:

  • ნავიგაციისთვის ისრის ღილაკები
  • Enter ქვემენიუში გადასასვლელად
  • Esc ერთი დონით ასვლა ან გასვლა
  • ? დახმარების ეკრანის სანახავად. Enter ღილაკი დახმარების ეკრანიდან გასასვლელად გამოიყენება.
  • Space, ან Y და N კლავიშები კონფიგურაციის ელემენტების ჩასართავად (დიახ) და გამორთვისთვის (არა) მონიშვნის ველებით[*]„
  • ? კონფიგურაციის ელემენტის მონიშვნისას, ამ ელემენტის შესახებ დახმარების საჩვენებლად
  • / კონფიგურაციის ელემენტების მოსაძებნად

შენიშვნა

თუ თქვენ ხართ Arch Linux მომხმარებელზე, გადადით SDK tool configuration და სახელის შეცვლა Python 2 interpreter -დან python რომ python2.

ყურადღება

თუ ESP32-DevKitC დაფას იყენებთ ESP32-SOLO-1 მოდული, ჩართეთ ერთბირთვიანი რეჟიმი (CONFIG_FREE RTOS _UNICORE) menuconfig ფლეშ მეხსიერება , მაგალითად.

ნაბიჯი 8. აწყობა და ფლეშირება¶

პროექტის აწყობა და ფლეშ მეხსიერება დასრულება შემდეგი ბრძანებით:

make flash

ეს ბრძანება კომპილაცია აპლიკაციას და ყველა ESP-IDF კომპონენტი ს დააგენერირებს, შემდეგ კი ჩამტვირთავი , დანაყოფების ცხრილი და აპლიკაციის ბინარულ ფაილებს აგენერირებს. ამის შემდეგ, ეს ბინარული ფაილები ფლეშ მეხსიერება აიტვირთება თქვენს ESP32 დაფაზე.

თუ ფლეშ მეხსიერება პროცესის ბოლოს პრობლემები არ იქნება, ქვემოთ ნახავთ შეტყობინებებს, რომლებიც აღწერენ ჩატვირთვის პროცესის მიმდინარეობას. შემდეგ დაფა გადაიტვირთება და ჩაირთვება „hello_world“ აპლიკაცია.

ნაბიჯი 9. მონიტორი¶

იმის შესამოწმებლად, ნამდვილად მუშაობს თუ არა „hello_world“, აკრიფეთ make monitor.

ეს ბრძანება იწყებს IDF მონიტორი აპლიკაცია:

IDF მონიტორიდან გასასვლელად გამოიყენეთ მალსახმობი Ctrl+].

თუ IDF მონიტორი ატვირთვა შემდეგ მალევე გაითიშება, ან თუ ზემოთ მოცემული შეტყობინებების ნაცვლად ქვემოთ მოცემულის მსგავს შემთხვევით ნაგავს ხედავთ, თქვენი დაფა, სავარაუდოდ, 26 MHz კრისტალს იყენებს. დეველოპერული დაფების უმეტესობა 40 MHz-ს იყენებს, ამიტომ ESP-IDF ამ სიხშირეს ნაგულისხმევ მნიშვნელობად იყენებს.

თუ ასეთი პრობლემა გაქვთ, გააკეთეთ შემდეგი:

  1. გამოდით მონიტორიდან.
  2. დაბრუნება menuconfig.
  3. გადადით კომპონენტის კონფიგურაციაზე –&gt; ESP32-სპეციფიკურზე –&gt; მთავარი XTAL სიხშირეზე, შემდეგ შეცვალეთ CONFIG_ESP32_XTAL_FREQ_SEL 26 MHz-მდე.
  4. ამის შემდეგ, აწყობა და ფლეშ მეხსიერება განაცხადი კვლავ.

შენიშვნა

თქვენ შეგიძლიათ გააერთიანოთ მშენებლობა, ფლეშ მეხსიერება სწავლება და მონიტორინგი ერთ ეტაპზე შემდეგი ნაბიჯების შესრულებით:

make flash monitor

აგრეთვე იხილეთ IDF მონიტორი მოსახერხებელი მალსახმობებისა და IDF მონიტორის გამოყენების შესახებ დამატებითი დეტალებისთვის.

ეს ყველაფერია, რაც გჭირდებათ ESP32-ის დასაწყებად!

ახლა თქვენ მზად ხართ სცადოთ სხვა მაგალითები, ან პირდაპირ გადადით საკუთარი აპლიკაციების შემუშავებაზე.

გარემოს ცვლადები¶

გამოძახებისას შესაძლებელია გარემოს ზოგიერთი ცვლადის მითითება make მომხმარებლებს საშუალებას აძლევს არგუმენტების გადაფარვა მათი ხელახლა კონფიგურაციის საჭიროების გარეშე make menuconfig.

ცვლადები აღწერა და გამოყენება
ESPPORT

უგულებელყოფს სერიული პორტი რომელიც გამოყენებულია flash და monitor.

მაგალითები: make flash ESPPORT=/dev/ttyUSB1, make monitor ESPPORT=COM1
ESPBAUD

ESP32-ში ფლეშ მეხსიერება ჩართვისას, სერიული ბოდის სიჩქარე მნიშვნელობას უგულებელყოფს.

მაგალითი: make flash ESPBAUD=9600
MONITORBAUD

მონიტორინგის დროს გამოყენებულ სერიულ ბოდის სიჩქარე უგულებელყოფს.

მაგალითი: make monitor MONITORBAUD=9600

შენიშვნა

შეგიძლიათ გარემოს ცვლადების ექსპორტი (მაგ. export ESPPORT=/dev/ttyUSB1). ყველა შემდგომი ზარი make იმავე ტერმინალის სესიის ფარგლებში გამოყენებული იქნება ექსპორტირებული მნიშვნელობა იმ პირობით, რომ ცვლადი ერთდროულად არ გადაიხედება.

ESP-IDF განახლება¶

დროდადრო უნდა განაახლოთ ESP-IDF , რადგან ახალი ვერსიები ასწორებს შეცდომებს და გვთავაზობს ახალ ფუნქციებს. განახლების უმარტივესი გზა არსებულის წაშლაა. esp-idf საქაღალდე და ხელახლა დააკლონირეთ, თითქოს ასრულებთ აღწერილ საწყის ინსტალაციას ნაბიჯი 2. მიიღეთ ESP-IDF.

თუ ახალ გზაზე გადმოწერთ, გახსოვდეთ IDF_PATH-ის დამატება მომხმარებლის პროფილში (Legacy GNU Make) რათა ინსტრუმენტების ნაკრები სკრიპტებმა ESP-IDF იპოვონ მისი გამოშვებისთვის სპეციფიკურ ადგილას.

კიდევ ერთი გამოსავალია მხოლოდ შეცვლილი ინფორმაციის განახლება. განახლების პროცედურა დამოკიდებულია თქვენს მიერ გამოყენებულ ESP-IDF ვერსიაზე.

დაკავშირებული დოკუმენტები¶

  • IDF_PATH-ის დამატება მომხმარებლის პროფილში (Legacy GNU Make)
  • სერიული კავშირის დამყარება ESP32-ით (Legacy GNU Make)
  • აწყობა და ფლეშ-რეჟიმში Make-ის გამოყენებით (Legacy GNU Make)
  • Eclipse IDE-ს (Legacy GNU Make) გამოყენებით აწყობა და ფლეშ-პროგრამირება
  • IDF მონიტორი
  • Toolchain-ის მორგებული დაყენება (Legacy GNU Make)

Linux-ისთვის Toolchain-ის სტანდარტული დაყენება (Legacy GNU Make)¶

[中文]

შენიშვნა

ESP-IDF V4.0-დან დაწყებული, ნაგულისხმევი აწყობის სისტემა დაფუძნებულია CMake-ზე. ეს დოკუმენტაცია განკუთვნილია GNU Make-ზე დაფუძნებული მემკვიდრეობითი აწყობის სისტემისთვის. ამ აწყობის სისტემის მხარდაჭერა შესაძლოა მომავალ ძირითად ვერსიებში გაუქმდეს.

ინსტალაციის წინაპირობები¶

კომპილაცია ისთვის ESP-IDF ით თქვენ უნდა მიიღოთ შემდეგი პაკეტები:

  • CentOS 7:

    sudo yum install gcc git wget make flex bison gperf python python2-cryptography

  • Ubuntu და Debian:

    sudo apt-get install gcc git wget make flex bison gperf python python-pip python-setuptools python-serial python-cryptography python-future python-pyparsing python-pyelftools libffi-dev libssl-dev

  • თაღი:

    sudo pacman -S --needed gcc git make flex bison gperf python2-pyserial python2-cryptography python2-future python2-pyparsing python2-pyelftools

შენიშვნა

ზოგიერთ ძველ Linux დისტრიბუციას შესაძლოა აკლდეს ზემოთ ჩამოთვლილი Python-ის ზოგიერთი პაკეტი (ან შეიძლება იყენებდეს pyserial ვერსია 2.x, რომელიც არ არის მხარდაჭერილი ESP-IDF მიერ). ამ პაკეტების ინსტალაცია შესაძლებელია pip სამაგიეროდ - როგორც აღწერილია ნაწილში ნაბიჯი 4. დააინსტალირეთ საჭირო Python პაკეტები.

ხელსაწყოთა ჯაჭვის დაყენება¶

ESP32 ინსტრუმენტების ნაკრები Linux-ისთვის ხელმისაწვდომია Espressif-ის ვებსაიტიდან ჩამოსატვირთად:

  • 64-ბიტიანი Linux-ისთვის:

    https://dl.espressif.com/dl/xtensa-esp32-elf-gcc8_2_0-esp-2019r2-linux-amd64.tar.gz

  • 32-ბიტიანი Linux-ისთვის:

    https://dl.espressif.com/dl/xtensa-esp32-elf-gcc8_2_0-esp-2019r2-linux-i686.tar.gz

  1. გადმოწერეთ ეს ფაილი, შემდეგ კი ამოაარქივეთ ~/esp დირექტორია:

    • 64-ბიტიანი Linux-ისთვის:

      mkdir -p ~/esp
cd ~/esp
tar -xzf ~/Downloads/xtensa-esp32-elf-gcc8_2_0-esp-2019r2-linux-amd64.tar.gz

    • 32-ბიტიანი Linux-ისთვის:

      mkdir -p ~/esp
cd ~/esp
tar -xzf ~/Downloads/xtensa-esp32-elf-gcc8_2_0-esp-2019r2-linux-i686.tar.gz

  1. ინსტრუმენტების ნაკრები ამოღებული იქნება ~/esp/xtensa-esp32-elf/ დირექტორია.

    მის გამოსაყენებლად, თქვენ უნდა განაახლოთ თქვენი PATH გარემოს ცვლადი ~/.profile ფაილი. შესაქმნელად xtensa-esp32-elf ხელმისაწვდომია ყველა ტერმინალის სესიისთვის, დაამატეთ შემდეგი ხაზი თქვენს ~/.profile ფაილი:

    export PATH="$HOME/esp/xtensa-esp32-elf/bin:$PATH"

    ალტერნატიულად, შეგიძლიათ შექმნათ მეტსახელი ზემოთ მოცემული ბრძანებისთვის. ამ გზით თქვენ შეგიძლიათ მიიღოთ ინსტრუმენტების ნაკრები მხოლოდ მაშინ, როცა დაგჭირდებათ. ამისათვის დაამატეთ სხვა ხაზი თქვენს ბრძანებაში. ~/.profile ფაილი:

    alias get_esp32='export PATH="$HOME/esp/xtensa-esp32-elf/bin:$PATH"'

    შემდეგ, როდესაც დაგჭირდებათ ინსტრუმენტების ნაკრები , შეგიძლიათ აკრიფოთ get_esp32 ბრძანების ხაზზე და ინსტრუმენტების ნაკრები დაემატება თქვენს PATH.

    შენიშვნა

    თუ გაქვთ /bin/bash დაყენებულია როგორც შესვლის გარსი და ორივე .bash_profile და .profile არსებობს, შემდეგ განახლდება .bash_profile სამაგიეროდ. CentOS-ში, alias უნდა დაინერგოს .bashrc.

  2. გასაკეთებლად გამოდით და ხელახლა შედით სისტემაში .profile ცვლილებები ძალაშია. შეასრულეთ შემდეგი ბრძანება იმის გადასამოწმებლად, თუ PATH სწორად არის დაყენებული:

    printenv PATH

    თქვენ ეძებთ მსგავს შედეგს, რომელიც შეიცავს ინსტრუმენტების ნაკრები ის გზას ნაჩვენები სტრიქონის დასაწყისში:

    $ printenv PATH
/home/user-name/esp/xtensa-esp32-elf/bin:/home/user-name/bin:/home/user-name/.local/bin:/usr/local/sbin:/usr/local/bin:/usr/sbin:/usr/bin:/sbin:/bin:/usr/games:/usr/local/games:/snap/bin

    ნაცვლად /home/user-name თქვენი ინსტალაციისთვის სპეციფიკური სახლის გზა უნდა იყოს.

ნებართვის პრობლემები /dev/ttyUSB0¶

ზოგიერთი Linux დისტრიბუციის შემთხვევაში, შეიძლება მიიღოთ Failed to open port /dev/ttyUSB0 შეცდომის შეტყობინება ფლეშ მეხსიერება ESP32-ის დაყენებისას. ამის მოგვარება შესაძლებელია მიმდინარე მომხმარებლის dialout ჯგუფში დამატებით..

შემდეგი ნაბიჯები¶

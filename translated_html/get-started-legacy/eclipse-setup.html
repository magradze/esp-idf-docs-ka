Eclipse IDE-ს (Legacy GNU Make) გამოყენებით აწყობა და ფლეშ-პროგრამირება¶

[中文]

შენიშვნა

ESP-IDF V4.0-დან დაწყებული, ნაგულისხმევი აწყობის სისტემა დაფუძნებულია CMake-ზე. ეს დოკუმენტაცია განკუთვნილია GNU Make-ზე დაფუძნებული მემკვიდრეობითი აწყობის სისტემისთვის. ამ აწყობის სისტემის მხარდაჭერა შესაძლოა მომავალ ძირითად ვერსიებში გაუქმდეს.

Eclipse IDE-ს ინსტალაცია¶

Eclipse IDE გთავაზობთ გრაფიკულად ინტეგრირებულ განვითარების გარემოს ESP-IDF პროექტების წერის, კომპილაციისა და გამართვა .

  • დაიწყეთ თქვენი პლატფორმისთვის esp-idf ფაილის ინსტალაციით (იხილეთ ამ დირექტორიაში არსებული ფაილები Windows-ის, OS X-ის, Linux-ის ნაბიჯებით).
  • გირჩევთ, პროექტი თავდაპირველად ბრძანების ხაზიდან შექმნათ, რათა გაიგოთ, თუ როგორ მუშაობს ეს პროცესი. ასევე, ბრძანების ხაზი უნდა გამოიყენოთ თქვენი esp-idf პროექტის კონფიგურაციისთვის (via make menuconfig), ეს ამჟამად Eclipse-ში მხარდაჭერილი არ არის.
  • ჩამოტვირთეთ Eclipse-ის ინსტალერი თქვენი პლატფორმისთვის: eclipse.org.
  • Eclipse ინსტალატორის გაშვებისას აირჩიეთ „Eclipse for C/C++ Development“ (სხვა ადგილებში ამას CDT-ს უწოდებენ).

Eclipse-ის დაყენება¶

როგორც კი თქვენი ახალი Eclipse ინსტალაცია დაიწყება, მიჰყევით ამ ნაბიჯებს:

ახალი პროექტის იმპორტი¶

  • Eclipse იყენებს Makefile მხარდაჭერას ESP-IDF ში. ეს ნიშნავს, რომ თქვენ უნდა დაიწყოთ ESP-IDF პროექტის შექმნით. შეგიძლიათ გამოიყენოთ idf-template პროექტი github-დან ან გახსნათ ერთ-ერთი მაგალითი esp-idf examples ქვედირექტორიაში.
  • Eclipse-ის გაშვების შემდეგ აირჩიეთ ფაილი -&gt; იმპორტი…
  • გამოსულ დიალოგურ ფანჯარაში აირჩიეთ „C/C++“ -&gt; „არსებული კოდი, როგორც Makefile პროექტი“ და დააჭირეთ ღილაკს „შემდეგი“.
  • შემდეგ გვერდზე, თქვენი IDF პროექტის დირექტორიად შეიყვანეთ „არსებული კოდის ადგილმდებარეობა“. არ მიუთითოთ ESP-IDF დირექტორიისკენ მიმავალი გზა (რომელიც მოგვიანებით გამოჩნდება). თქვენს მიერ მითითებული დირექტორია უნდა შეიცავდეს ფაილს სახელწოდებით „Makefile“ (პროექტის Makefile).
  • იმავე გვერდზე, „ინდექსატორის პარამეტრებისთვის ხელსაწყოთა ჯაჭვის“ ქვეშ აირჩიეთ „Cross GCC“. შემდეგ დააჭირეთ ღილაკს „დასრულება“.

პროექტის თვისებები¶

  • ახალი პროექტი გამოჩნდება Project Explorer-ში. დააწკაპუნეთ მაუსის მარჯვენა ღილაკით პროექტზე და კონტექსტური მენიუდან აირჩიეთ Properties.
  • დააწკაპუნეთ „გარემოს“ თვისებების გვერდზე „C/C++ Build“-ის ქვეშ. დააწკაპუნეთ „დამატება…“-ზე და შეიყვანეთ სახელი. BATCH_BUILD და ღირებულება 1.
  • კვლავ დააჭირეთ ღილაკს „დამატება…“ და შეიყვანეთ სახელი IDF_PATHმნიშვნელობა უნდა იყოს სრული გზა, სადაც ESP-IDF არის დაინსტალირებული. Windows-ის მომხმარებლებს შეუძლიათ კოპირება IDF_PATH Windows Explorer-დან.
  • რედაქტირება PATH გარემოს ცვლადი. შეინახეთ მიმდინარე მნიშვნელობა და დაამატეთ გზა Xtensa ინსტრუმენტების ნაკრები ისკენ, რომელიც დაინსტალირებულია IDF დაყენების ნაწილად, თუ ის უკვე არ არის ჩამოთვლილი PATH-ში. ინსტრუმენტების ნაკრები ისკენ მიმავალი ტიპიური გზა ასე გამოიყურება: /home/user-name/esp/xtensa-esp32-elf/binგაითვალისწინეთ, რომ ორწერტილის დამატება გჭირდებათ : დამატებული გზის წინ. Windows-ის მომხმარებლებს დასჭირდებათ წინასწარი ჩასმა C:\msys32\mingw32\bin;C:\msys32\opt\xtensa-esp32-elf\bin;C:\msys32\usr\bin რომ PATH გარემოს ცვლადი (თუ msys32 სხვა დირექტორიაში დააინსტალირეთ, მაშინ ეს ბილიკები უნდა შეცვალოთ, რომ ემთხვეოდეს).
  • macOS-ზე დაამატეთ PYTHONPATH გარემოს ცვლადი და დააყენეთ ის /Library/Frameworks/Python.framework/Versions/2.7/lib/python2.7/site-packagesეს იმისთვის კეთდება, რომ სისტემურმა Python-მა, რომელშიც დაყენების ნაბიჯების ნაწილად pyserial-ია დაინსტალირებული, გადაფაროს ნებისმიერი ჩაშენებული Eclipse Python.

დამატებითი შენიშვნათუ IDF_PATH დირექტორია ან პროექტის დირექტორია გარეთ მდებარეობს C:\msys32\home დირექტორიაში, თქვენ უნდა მისცეთ მორგებული build ბრძანება C/C++ Build თვისებებში, როგორც: python ${IDF_PATH}/tools/windows/eclipse_make.py (გაითვალისწინეთ, რომ ამ მეთოდით შეიძლება მნიშვნელოვნად გაიზარდოს აწყობის დრო.)

გადადით „C/C++ General“ -&gt; „Preprocessor Include Paths“ თვისებების გვერდზე:

  • დააჭირეთ ღილაკს „პროვაიდერები“
  • პროვაიდერების სიაში დააწკაპუნეთ „CDT Cross GCC Built-in Compiler Settings“-ზე. შეცვალეთ „Command to get კომპილაცია r specs“-ით. xtensa-esp32-elf-gcc ${FLAGS} -std=c++11 -E -P -v -dD "${INPUTS}".
  • პროვაიდერების სიაში დააწკაპუნეთ „CDT GCC Build Output Parser“-ზე და შეცვალეთ „Compiler command pattern“-ით xtensa-esp32-elf-(gcc|g\+\+|c\+\+|cc|cpp|clang)

გადადით „C/C++ ზოგადი“ -&gt; „ინდექსატორის“ თვისებების გვერდზე:

  • ამ გვერდზე დანარჩენი პარამეტრების ჩასართავად მონიშნეთ „პროექტის სპეციფიკური პარამეტრების ჩართვა“.
  • მოხსენით მონიშვნა „ჩართვების ევრისტიკული გარჩევადობის დაშვება“. როდესაც ეს პარამეტრი ჩართულია, Eclipse ზოგჯერ ვერ პოულობს სწორ სათაურის დირექტორიებს.

გადადით „C/C++ Build“ -&gt; „Behavior“ თვისებების გვერდზე:

  • რამდენიმე აწყობის დავალების პარალელურად ჩასართავად მონიშნეთ „პარალელური აწყობის ჩართვა“.

შენობა Eclipse-ში¶

პროექტის აწყობამდე, Eclipse-მა შეიძლება აჩვენოს მრავალი შეცდომა და გაფრთხილება განუსაზღვრელი მნიშვნელობების შესახებ. ეს იმიტომ ხდება, რომ ზოგიერთი საწყისი ფაილი ავტომატურად გენერირდება esp-idf აწყობის პროცესის ფარგლებში. ეს შეცდომები და გაფრთხილებები გაქრება პროექტის აწყობის შემდეგ.

  • Eclipse-ში Properties დიალოგური ფანჯრის დასახურად დააჭირეთ ღილაკს OK.
  • Eclipse-ის გარეთ, გახსენით ბრძანების ხაზი. გადადით თქვენი პროექტის დირექტორიაში და გაუშვით make menuconfig თქვენი პროექტის esp-idf პარამეტრების კონფიგურაციისთვის. ეს ნაბიჯი ამჟამად Eclipse-ის გარეთ უნდა შესრულდეს.

თუ კონფიგურაციის ეტაპის გაშვების გარეშე ეცდებით აწყობას, esp-idf ბრძანების ხაზზე კონფიგურაციას მოგთხოვთ, თუმცა Eclipse-ს ამის მოგვარება არ შეუძლია, ამიტომ აწყობა გაიჭედება ან ვერ მოხერხდება.

  • დაბრუნდით Eclipse-ში, პროექტის ასაგებად აირჩიეთ Project -&gt; Build.

რჩევათუ თქვენი პროექტი უკვე შექმნილია Eclipse-ის გარეთ, შესაძლოა დაგჭირდეთ Project -&gt; Clean-ის გაკეთება Project -&gt; Build-ის არჩევამდე. ეს კეთდება იმისათვის, რომ Eclips-მა ნახოს კომპილაცია r არგუმენტები ყველა საწყისი ფაილისთვის. ის მათ იყენებს სათაურის ჩართვის გზების დასადგენად.

ფლეში Eclipse-დან¶

თქვენ შეგიძლიათ „make ფლეშ მეხსიერება “-ის სამიზნის ინტეგრირება თქვენს Eclipse პროექტში ფლეშ მეხსიერება თან Eclipse-ის ინტერფეისიდან esptool.py-ის გამოყენებით:

  • დააწკაპუნეთ მარჯვენა ღილაკით თქვენს პროექტზე Project Explorer-ში (მნიშვნელოვანია დარწმუნდეთ, რომ პროექტს არჩევთ და არა პროექტში არსებულ დირექტორიას, წინააღმდეგ შემთხვევაში Eclipse-მა შეიძლება არასწორი Makefile იპოვოს.)
  • კონტექსტური მენიუდან აირჩიეთ სამიზნეების შექმნა -&gt; შექმნა….
  • სამიზნე სახელად აკრიფეთ „ ფლეშ მეხსიერება “. სხვა პარამეტრები ნაგულისხმევად დატოვეთ.
  • ახლა შეგიძლიათ გამოიყენოთ Project -&gt; Build Target -&gt; Build (Shift+F9) ფლეშ მეხსიერება მორგებული სამიზნის ასაშენებლად, რომელიც კომპილაცია და ფლეშ მეხსიერება მოახდენს პროექტის ასაშენებლად.

გაითვალისწინეთ, რომ სერიული პორტი და ფლეშ მეხსიერება ის სხვა კონფიგურაციის პარამეტრების დასაყენებლად დაგჭირდებათ „make menuconfig “-ის გამოყენება. „make menuconfig “-ს მაინც სჭირდება ბრძანების ხაზის ტერმინალი (იხილეთ თქვენი პლატფორმის ინსტრუქციები).

დამატების მიზნით, მიჰყევით იგივე ნაბიჯებს bootloader და partition_table საჭიროების შემთხვევაში, სამიზნეები.

კონსოლი¶

[中文]

ESP-IDF უზრუნველყოფს console კომპონენტი , რომელიც მოიცავს ინტერაქტიული კონსოლის შესაქმნელად საჭირო საშენ ბლოკებს სერიული პორტი განმავლობაში. ეს კომპონენტი მოიცავს შემდეგ შესაძლებლობებს:

  • სტრიქონის რედაქტირება, მოწოდებულია ხაზის ხმაური ბიბლიოთეკა. ეს მოიცავს backspace და ისრის ღილაკების გამოყენებას, ბრძანებების ისტორიაში გადახვევას, ბრძანებების ავტომატურ შევსებას და არგუმენტების მინიშნებებს.
  • ბრძანების ხაზის არგუმენტებად დაყოფა.
  • არგუმენტის დამუშავება, მოწოდებულია argtable3 ბიბლიოთეკა. ეს ბიბლიოთეკა მოიცავს API ს, რომლებიც სასარგებლოა GNU სტილის ბრძანების ხაზის არგუმენტების დამუშავებისთვის.
  • ბრძანებების რეგისტრაციისა და გაგზავნის ფუნქციები.

ამ საშუალებების გამოყენება შესაძლებელია ერთად ან დამოუკიდებლად. მაგალითად, შესაძლებელია ხაზის რედაქტირებისა და ბრძანებების რეგისტრაციის ფუნქციების გამოყენება, მაგრამ გამოიყენეთ getopt ან არგუმენტის დამუშავების მორგებული კოდი, ნაცვლად argtable3ასევე, შესაძლებელია ბრძანების შეყვანის უფრო მარტივი საშუალებების გამოყენება (მაგალითად, fgets) ბრძანებების დაყოფისა და არგუმენტების დამუშავების დანარჩენ საშუალებებთან ერთად.

სტრიქონის რედაქტირება¶

ხაზების რედაქტირების ფუნქცია მომხმარებლებს საშუალებას აძლევს შექმნან ბრძანებები მათი აკრეფით, სიმბოლოების წაშლით „უკან ღილაკზე“ დაჭერით, ბრძანებაში ნავიგაციით მარცხენა/მარჯვენა ღილაკების გამოყენებით, ადრე აკრეფილ ბრძანებებზე გადასვლით ზემოთ/ქვემოთ ღილაკების გამოყენებით და ავტომატური შევსებით „ტაბ“ ღილაკზე დაჭერით.

შენიშვნა

ეს ფუნქცია ეყრდნობა ტერმინალის აპლიკაციაში ANSI escape sequence-ის მხარდაჭერას. შესაბამისად, სერიული მონიტორები, რომლებიც აჩვენებენ UART ნედლ მონაცემებს, არ შეიძლება გამოყენებულ იქნას ხაზის რედაქტირების ბიბლიოთეკასთან ერთად. თუ ხედავთ [6n ან მსგავსი გაქცევის თანმიმდევრობა, როდესაც ბრძანების ხაზის ნაცვლად get_started/console-ის მაგალითის გაშვება ხდება ([esp32]>), ეს ნიშნავს, რომ სერიული მონიტორი არ უჭერს მხარს გაქცევის თანმიმდევრობებს. ცნობილია, რომ პროგრამები, რომლებიც მუშაობენ, არის GNU screen, minicom და idf_monitor.py (რომლის გამოძახებაც შესაძლებელია idf.py monitor პროექტის დირექტორიიდან).

აქ მოცემულია ფუნქციების მიმოხილვა, რომელსაც უზრუნველყოფს ხაზის ხმაური ბიბლიოთეკა.

კონფიგურაცია¶

Linenoise ბიბლიოთეკას არ სჭირდება ექსპლიციტური ინიციალიზაცია. თუმცა, შესაძლოა საჭირო გახდეს ზოგიერთი კონფიგურაციის ნაგულისხმევი პარამეტრის შეცვლა მთავარი ხაზის რედაქტირების ფუნქციის გამოძახებამდე.

linenoiseClearScreen
Clear terminal screen using an escape sequence and position the cursor at the top left corner.
linenoiseSetMultiLine
Switch between single line and multi line editing modes. In single line mode, if the length of the command exceeds the width of the terminal, the command text is scrolled within the line to show the end of the text. In this case the beginning of the text is hidden. Single line needs less data to be sent to refresh screen on each key press, so exhibits less glitching compared to the multi line mode. On the flip side, editing commands and copying command text from terminal in single line mode is harder. Default is single line mode.

მთავარი ციკლი¶

linenoise
In most cases, console applications have some form of read/eval loop. linenoise is the single function which handles user’s key presses and returns completed line once ‘enter’ key is pressed. As such, it handles the ‘read’ part of the loop.
linenoiseFree
This function must be called to release the command line buffer obtained from linenoise function.

მინიშნებები და დასრულებები¶

linenoiseSetCompletionCallback

როდესაც მომხმარებელი აჭერს „tab“ ღილაკს, linenoise ბიბლიოთეკა იწვევს დასრულების უკუკავშირს. უკუკავშირმა უნდა შეამოწმოს აქამდე აკრეფილი ბრძანების შინაარსი და მოგვაწოდოს შესაძლო დასრულებების სია გამოძახებების გამოყენებით. linenoiseAddCompletion ფუნქცია. linenoiseSetCompletionCallback თუ სასურველია დასრულების ფუნქცია, ფუნქცია უნდა გამოიძახოს ამ დასრულების უკუგამოძახების რეგისტრაციისთვის.

console კომპონენტი გთავაზობთ მზა ფუნქციას რეგისტრირებული ბრძანებების შესასრულებლად. esp_console_get_completion (იხილეთ ქვემოთ).

linenoiseAddCompletion
Function to be called by completion callback to inform the library about possible completions of the currently typed command.
linenoiseSetHintsCallback
Whenever user input changes, linenoise invokes hints callback. This callback can inspect the command line typed so far, and provide a string with hints (which can include list of command arguments, for example). The library then displays the hint text on the same line where editing happens, possibly with a different color.
linenoiseSetFreeHintsCallback
If hint string returned by hints callback is dynamically allocated or needs to be otherwise recycled, the function which performs such cleanup should be registered via linenoiseSetFreeHintsCallback.

ისტორია¶

linenoiseHistorySetMaxLen
This function sets the number of most recently typed commands to be kept in memory. Users can navigate the history using up/down arrows.
linenoiseHistoryAdd
Linenoise does not automatically add commands to history. Instead, applications need to call this function to add command strings to the history.
linenoiseHistorySave
Function saves command history from RAM to a text file, for example on an SD card or on a filesystem in flash memory.
linenoiseHistoryLoad
Counterpart to linenoiseHistorySave, loads history from a file.
linenoiseHistoryFree
Releases memory used to store command history. Call this function when done working with linenoise library.

ბრძანების ხაზის არგუმენტებად დაყოფა¶

console კომპონენტი ითვალისწინებს esp_console_split_argv ფუნქცია ბრძანების ხაზის სტრიქონის არგუმენტებად დაყოფისთვის. ფუნქცია აბრუნებს ნაპოვნი არგუმენტების რაოდენობას (argc) და ავსებს მაჩვენებლების მასივს, რომელთა გადაცემა შესაძლებელია როგორც argv არგუმენტი ნებისმიერი ფუნქციისთვის, რომელიც იღებს არგუმენტებს argc, argv ფორმატი.

ბრძანების ხაზი დაყოფილია არგუმენტებად შემდეგი წესების მიხედვით:

  • არგუმენტები გამოყოფილია ინტერვალებით
  • თუ არგუმენტებში სივრცეებია საჭირო, მათი გამორიცხვა შესაძლებელია გამოყენებით \ (უკუხაზი) სიმბოლო.
  • სხვა ამოცნობილი გაქცევის თანმიმდევრობებია \\ (რაც რეალურ უკუხაზს წარმოქმნის) და \", რომელიც ორმაგ ბრჭყალს წარმოქმნის.
  • არგუმენტები შეიძლება მოთავსდეს ორმაგი ბრჭყალების გამოყენებით. ბრჭყალები შეიძლება გამოჩნდეს მხოლოდ არგუმენტის დასაწყისში და ბოლოს. არგუმენტში ბრჭყალები უნდა გამოირიცხოს ზემოთ აღნიშნულის შესაბამისად. არგუმენტის გარშემო ბრჭყალები იშლება შემდეგი გზით: esp_console_split_argv ფუნქცია.

მაგალითები:

  • abc def 1 20 .3 ⟶ [ abc, def, 1, 20, .3 ]
  • abc "123 456" def ⟶ [ abc, 123 456, def ]
  • `a\ b\\c\" ⟶ [ a b\c" ]

არგუმენტის დამუშავება¶

არგუმენტის დამუშავებისთვის, console კომპონენტი შედის argtable3 ბიბლიოთეკა. გთხოვთ იხილოთ სახელმძღვანელო შესავლისთვის argtable3Github-ის საცავი ასევე მოიცავს მაგალითები.

სარდლობის რეგისტრაცია და გაგზავნა¶

console კომპონენტი მოიცავს სასარგებლო ფუნქციებს, რომლებიც ამუშავებენ ბრძანებების რეგისტრაციას, მომხმარებლის მიერ აკრეფილი ბრძანებების რეგისტრირებულ ბრძანებების შესაბამისობას და ამ ბრძანებების გამოძახებას ბრძანების ხაზზე მოცემული არგუმენტებით.

აპლიკაცია თავდაპირველად ბრძანების რეგისტრაციის მოდულს ინიციალიზაციას უკეთებს ზარის გამოყენებით. esp_console_initდა ზარები esp_console_cmd_register ფუნქცია ბრძანების დამმუშავებლების რეგისტრაციისთვის.

თითოეული ბრძანებისთვის, აპლიკაცია უზრუნველყოფს შემდეგ ინფორმაციას (ფორმით esp_console_cmd_t სტრუქტურა):

  • ბრძანების სახელი (სტრიქონი სივრცეების გარეშე)
  • დამხმარე ტექსტი, რომელიც განმარტავს ბრძანების მოქმედებას
  • ბრძანების არგუმენტების ჩამონათვალის სურვილისამებრ მინიშნების ტექსტი. თუ აპლიკაცია არგუმენტების ანალიზისთვის Argtable3-ს იყენებს, მინიშნების ტექსტის გენერირება შესაძლებელია ავტომატურად, argtable-ის არგუმენტების განმარტებების სტრუქტურისკენ მითითების მიწოდებით.
  • ბრძანების დამმუშავებლის ფუნქცია.

ბრძანების რეგისტრაციის მოდული კიდევ რამდენიმე ფუნქციას ასრულებს:

esp_console_run
This function takes the command line string, splits it into argc/argv argument list using esp_console_split_argv, looks up the command in the list of registered components, and if it is found, executes its handler.
esp_console_register_help_command
Adds help command to the list of registered commands. This command prints the list of all the registered commands, along with their arguments and help texts.
esp_console_get_completion
Callback function to be used with linenoiseSetCompletionCallback from linenoise library. Provides completions to linenoise based on the list of registered commands.
esp_console_get_hint
Callback function to be used with linenoiseSetHintsCallback from linenoise library. Provides argument hints for registered commands to linenoise.

მაგალითი¶

აპლიკაციის მაგალითი, რომელიც ასახავს მის გამოყენებას console კომპონენტი ხელმისაწვდომია examples/system/console დირექტორია. ეს მაგალითი გვიჩვენებს, თუ როგორ უნდა მოხდეს UART და VFS ფუნქციების ინიციალიზაცია, linenoise ბიბლიოთეკის დაყენება, UART-დან ბრძანებების წაკითხვა და დამუშავება და ბრძანებების ისტორიის შენახვა Flash-ში. დამატებითი ინფორმაციისთვის იხილეთ README.md ფაილი მაგალითის დირექტორიაში.

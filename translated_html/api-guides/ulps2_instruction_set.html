ESP32-S2 ULP კოპროცესორის ინსტრუქციების ნაკრები¶

ეს დოკუმენტი დეტალურ ინფორმაციას გვაწვდის ESP32-S2 ULP კოპროცესორ-ასამბლერის მიერ გამოყენებული ინსტრუქციების შესახებ.

ULP კოპროცესორს აქვს 4 16-ბიტიანი ზოგადი დანიშნულების რეგისტრი, რომლებიც აღნიშნულია როგორც R0, R1, R2, R3. მას ასევე აქვს 8-ბიტიანი მრიცხველის რეგისტრი (stage_cnt), რომლის გამოყენებაც შესაძლებელია ციკლების განსახორციელებლად. ეტაპების მრიცხველის რეგისტრზე წვდომა ხორციელდება სპეციალური ინსტრუქციების გამოყენებით.

ULP კოპროცესორს შეუძლია წვდომა RTC_SLOW_MEM მეხსიერების რეგიონის 8 ათას ბაიტზე. მეხსიერება დამისამართებულია 32-ბიტიანი სიტყვიერი ერთეულებით. მას ასევე შეუძლია წვდომა RTC_CNTL, RTC_IO და SENS პერიფერიულ მოწყობილობებში არსებულ პერიფერიულ რეგისტრებზე.

ყველა ინსტრუქცია 32-ბიტიანია. ნახტომის ინსტრუქციები, ALU ინსტრუქციები, პერიფერიული მოწყობილობების რეგისტრის და მეხსიერებაზე წვდომის ინსტრუქციები სრულდება 1 ციკლში. პერიფერიულ მოწყობილობებთან (TSENS, ADC, I2C) მომუშავე ინსტრუქციებს სჭირდებათ ციკლების ცვლადი რაოდენობა, პერიფერიული მოწყობილობების მუშაობის მიხედვით.

ინსტრუქციის სინტაქსი არ არის მგრძნობიარე რეგისტრისა და დიდი ასოების მიმართ. დიდი და პატარა ასოების გამოყენება და ერთმანეთში შერევა შესაძლებელია თვითნებურად. ეს ეხება როგორც რეგისტრის სახელებს, ასევე ინსტრუქციის სახელებს.

შენიშვნა მისამართის შესახებ¶

ESP32-S2 ULP თანაპროცესორის JUMP, ST, LD ინსტრუქციები, რომლებიც რეგისტრს არგუმენტად (ნახტომის მისამართი, შენახვის/ჩატვირთვის ბაზის მისამართი) იღებენ, მოელიან, რომ არგუმენტი 32-ბიტიანი სიტყვებით იქნება გამოხატული.

განვიხილოთ შემდეგი პროგრამის მაგალითი:

entry:
        NOP
        NOP
        NOP
        NOP
loop:
        MOVE R1, loop
        JUMP R1

როდესაც ეს პროგრამა აწყობილი და დაკავშირებულია, ეტიკეტის მისამართი loop 16-ის ტოლი იქნება (გამოსახული ბაიტებში). თუმცა JUMP ინსტრუქცია ელის, რომ რეგისტრში შენახული მისამართი გამოისახება 32-ბიტიანი სიტყვებით. ამ გავრცელებული შემთხვევის გათვალისწინებით, ასამბლერი გარდაქმნის ეტიკეტის მისამართს. loop ბაიტებიდან სიტყვებამდე, გენერირებისას MOVE ინსტრუქცია, ამიტომ კოდის გენერირებული კოდი ეკვივალენტური იქნება:

0000    NOP
0004    NOP
0008    NOP
000c    NOP
0010    MOVE R1, 4
0014    JUMP R1

მეორე შემთხვევაა, როდესაც არგუმენტი MOVE ინსტრუქცია არ არის იარლიყი, არამედ კონსტანტაა. ამ შემთხვევაში, ასამბლერი გამოიყენებს მნიშვნელობას ისე, როგორც არის, ყოველგვარი კონვერტაციის გარეშე:

.set        val, 0x10
MOVE        R1, val

ამ შემთხვევაში, R1-ში ჩატვირთული მნიშვნელობა იქნება 0x10.

მსგავსი მოსაზრებები ვრცელდება LD და ST ინსტრუქციები. განვიხილოთ შემდეგი კოდი:

        .global array
array:  .long 0
        .long 0
        .long 0
        .long 0

        MOVE R1, array
        MOVE R2, 0x1234
        ST R2, R1, 0      // write value of R2 into the first array element,
                          // i.e. array[0]

        ST R2, R1, 4      // write value of R2 into the second array element
                          // (4 byte offset), i.e. array[1]

        ADD R1, R1, 2     // this increments address by 2 words (8 bytes)
        ST R2, R1, 0      // write value of R2 into the third array element,
                          // i.e. array[2]

შენიშვნა ინსტრუქციის შესრულების დროის შესახებ¶

ULP კოპროცესორის ტაქტური რეჟიმი იმართება RTC_FAST_CLK-დან, რომელიც ჩვეულებრივ მიიღება შიდა 8MHz ოსცილატორიდან. აპლიკაციებს, რომლებსაც სჭირდებათ ULP ტაქტური სიხშირის ზუსტი ცოდნა, შეუძლიათ მისი დაკალიბრება XTAL-ის მთავარ ტაქტთან:

#include "soc/rtc.h"

// calibrate 8M/256 clock against XTAL, get 8M/256 clock period
uint32_t rtc_8md256_period = rtc_clk_cal(RTC_CAL_8MD256, 100);
uint32_t rtc_fast_freq_hz = 1000000ULL * (1 << RTC_CLK_CAL_FRACT) * 256 / rtc_8md256_period;

ULP კოპროცესორს თითოეული ინსტრუქციის მისაღებად სჭირდება საათის ციკლების გარკვეული რაოდენობა, პლუს მის შესასრულებლად ციკლების გარკვეული რაოდენობა, ინსტრუქციის მიხედვით. შესრულების დროის შესახებ დეტალებისთვის იხილეთ თითოეული ინსტრუქციის აღწერა ქვემოთ.

ინსტრუქციის მიღების დროა:

  • 2 საათის ციკლი — ALU-ს და განშტოების ინსტრუქციების შემდეგ ინსტრუქციებისთვის.
  • 4 საათის ციკლი — სხვა შემთხვევებში.

გაითვალისწინეთ, რომ RTC მეხსიერებებსა და RTC რეგისტრებზე წვდომისას, ULP კოპროცესორს უფრო დაბალი პრიორიტეტი აქვს, ვიდრე მთავარ CPU-ებს. ეს ნიშნავს, რომ ULP კოპროცესორის შესრულება შეიძლება შეჩერდეს, სანამ მთავარი CPU-ები იმავე მეხსიერების რეგიონზე წვდომას მიიღებენ, რასაც ULP.

განსხვავება ESP32 ULP-სა და ESP32-S2 ULP ინსტრუქციების ნაკრებებს შორის¶

ESP32 ULP კოპროცესორთან შედარებით, ESP-S2 ULP კოპროცესორს აქვს გაფართოებული ინსტრუქციების ნაკრები. ESP32-S2 ULP-ზე დამატებული ახალი ინსტრუქციების სიაა: LDL, LDH, STO, ST32, STI32. ამ ბრძანებების დეტალური აღწერა იხილეთ ქვემოთ.

  • მდგომარეობა:
    • ეკვალაიზერი (თანაბარი) – ნახტომი, თუ მნიშვნელობა R0-ში == ზღურბლი

    • დიდი ტევადობა (მეტია ვიდრე) – გადახტომა, თუ R0-ში მნიშვნელობა &gt; ზღურბლი
    • GE (მეტი ან ტოლი) – გადახტომა, თუ R0-ში მნიშვნელობა &gt;= ზღურბლი
ციკლები

პირობები ეკვალაიზერი, დიდი ტევადობა და LT: 2 ციკლი შესასრულებლად, 2 ციკლი შემდეგი ინსტრუქციის მისაღებად

პირობები ლე და GE ასამბლერში ხორციელდება ორი ფუნქციის გამოყენებით ჯუმპრი ინსტრუქციები:

         JUMPR target, threshold, EQ
         JUMPR target, threshold, LT

// JUMPR target, threshold, GE is implemented as:

         JUMPR target, threshold, EQ
         JUMPR target, threshold, GT

ამგვარად, შესრულების დრო დამოკიდებული იქნება აღებულ ტოტებზე: ან 2 ციკლი შესასრულებლად + 2 ციკლი მოძიებისთვის, ან 4 ციკლი შესასრულებლად + 4 ციკლი მოძიებისთვის.

აღწერა
The instruction makes a jump to a relative address if condition is true. Condition is the result of comparison of R0 register value and the threshold value.

მაგალითები:

1:pos:    JUMPR       16, 20, GE   // Jump to address (position + 16 bytes) if value in R0 >= 20

2:        // Down counting loop using R0 register
          MOVE        R0, 16       // load 16 into R0
  label:  SUB         R0, R0, 1    // R0--
          NOP                      // do something
          JUMPR       label, 1, GE // jump to label if R0 >= 1

          STAGE_INC  16            // increment stage_cnt to 16
  label:  STAGE_DEC  1             // stage_cnt--;
          NOP                      // do something
          JUMPS      label, 0, GT  // jump to label if stage_cnt > 0

სინტაქსი
შეჩერება
ოპერანდები
No operands
ციკლები

The instruction halts the ULP coprocessor and restarts ULP wakeup timer, if it is enabled.

WAKE - გააღვიძე ჩიპი¶

სინტაქსი
WAKE
ოპერანდები
No operands
ციკლები
2 cycles to execute, 4 cycles to fetch next instruction
აღწერა

ინსტრუქცია ULP-დან RTC კონტროლერს შეწყვეტას უგზავნის.

  • თუ SoC ღრმა ძილის რეჟიმშია და ULP გამოღვიძება ჩართულია, ეს გამოიწვევს SoC-ის გამოღვიძებას.
  • თუ SoC არ არის ღრმა ძილის რეჟიმში და RTC_CNTL_INT_ENA_REG რეგისტრში დაყენებულია ULP შეწყვეტის ბიტი (RTC_CNTL_ULP_CP_INT_ENA), ამოქმედდება RTC შეწყვეტა.

გაითვალისწინეთ, რომ WAKE ინსტრუქციის გამოყენებამდე, ULP პროგრამას შეიძლება დასჭირდეს ლოდინი, სანამ RTC კონტროლერი მზად იქნება მთავარი CPU-ს გასაღვიძებლად. ეს მითითებულია RTC_CNTL_LOW_POWER_ST_REG რეგისტრის RTC_CNTL_RDY_FOR_WAKEUP ბიტის გამოყენებით. თუ WAKE ინსტრუქცია შესრულდება მაშინ, როდესაც RTC_CNTL_RDY_FOR_WAKEUP ნულის ტოლია, მას არანაირი ეფექტი არ ექნება (გაღვიძება არ ხდება).

მაგალითები:

1: is_rdy_for_wakeup:                   // Read RTC_CNTL_RDY_FOR_WAKEUP bit
          READ_RTC_FIELD(RTC_CNTL_LOW_POWER_ST_REG, RTC_CNTL_RDY_FOR_WAKEUP)
          AND r0, r0, 1
          JUMP is_rdy_for_wakeup, eq    // Retry until the bit is set
          WAKE                          // Trigger wake up
          REG_WR 0x006, 24, 24, 0       // Stop ULP timer (clear RTC_CNTL_ULP_CP_SLP_TIMER_EN)
          HALT                          // Stop the ULP program
          // After these instructions, SoC will wake up,
          // and ULP will not run again until started by the main program.

I2C_RD - წაიკითხეთ ერთი ბაიტი I2C მონიდან¶

სინტაქსი
  • I2C_RD Sub_addr, მაღალი, დაბალი, Slave_sel
ოპერანდები
  • ქვე-მისამართი – წასაკითხი I2C მონა-მონაწილის მისამართი.
  • მაღალი, დაბალი — განსაზღვრეთ წასაკითხი ბიტების დიაპაზონი. [მაღალი, დაბალი] დიაპაზონის მიღმა არსებული ბიტები ნიღბიანია.
  • მონა_სელ - გამოსაყენებელი I2C მონა მისამართის ინდექსი.
ციკლები
Execution time mostly depends on I2C communication time. 4 cycles to fetch next instruction.
აღწერა
I2C_RD instruction reads one byte from I2C slave with index Slave_sel. Slave address (in 7-bit format) has to be set in advance into SENS_I2C_SLAVE_ADDRx register field, where x == Slave_sel. 8 bits of read result is stored into R0 register.

მაგალითები:

1:        I2C_RD      0x10, 7, 0, 0      // Read byte from sub-address 0x10 of slave with address set in SENS_I2C_SLAVE_ADDR0

ოპერანდები
  • ქვე-მისამართი – მისამართი I2C მონაში ჩასაწერად.
  • ღირებულება – ჩასაწერი 8-ბიტიანი მნიშვნელობა.
  • მაღალი, დაბალი — განსაზღვრეთ ჩასაწერი ბიტების დიაპაზონი. [მაღალი, დაბალი] დიაპაზონის მიღმა არსებული ბიტები ნიღბიანია.
  • მონა_სელ - გამოსაყენებელი I2C მონა მისამართის ინდექსი.

REG_RD – პერიფერიული რეგისტრიდან წაკითხვა¶

სინტაქსი
REG_RD მისამართი, მაღალი, დაბალი
ოპერანდები
  • მისამართი – რეგისტრაციის მისამართი, 32-ბიტიანი სიტყვებით
  • მაღალი – R0-ის მაღალი ნაწილი
  • დაბალი – R0-ის დაბალი ნაწილი
ციკლები
4 cycles to execute, 4 cycles to fetch next instruction
აღწერა

ინსტრუქცია პერიფერიული რეგისტრიდან ზოგადი დანიშნულების რეგისტრში კითხულობს 16 ბიტამდე ინფორმაციას: R0 = REG[Addr][High:Low].

ამ ინსტრუქციას შეუძლია წვდომა ჰქონდეს RTC_CNTL, RTC_IO, SENS და RTC_I2C პერიფერიულ მოწყობილობებში არსებულ რეგისტრებზე. რეგისტრის მისამართი, როგორც ეს ULP-დან ჩანს, შეიძლება გამოითვალოს DPORT ავტობუსზე იმავე რეგისტრის მისამართიდან შემდეგნაირად:

addr_ulp = (addr_dport - DR_REG_RTCCNTL_BASE) / 4

მაგალითები:

1:        REG_RD      0x120, 2, 0     // load 4 bits: R0 = {12'b0, REG[0x120][7:4]}

REG_WR – პერიფერიულ რეგისტრში ჩაწერა¶

სინტაქსი
REG_WR მისამართი, მაღალი, დაბალი, მონაცემები
ოპერანდები
  • მისამართი – რეგისტრაციის მისამართი, 32-ბიტიანი სიტყვებით.
  • მაღალი – R0-ის მაღალი ნაწილი
  • დაბალი – R0-ის დაბალი ნაწილი
  • მონაცემები – ჩასაწერი მნიშვნელობა, 8 ბიტი
ციკლები
8 cycles to execute, 4 cycles to fetch next instruction
აღწერა

ინსტრუქცია ზოგადი დანიშნულების რეგისტრიდან 8 ბიტამდე ინფორმაციას პერიფერიულ რეგისტრში წერს. REG[Addr][High:Low] = data

ამ ინსტრუქციას შეუძლია წვდომა ჰქონდეს RTC_CNTL, RTC_IO, SENS და RTC_I2C პერიფერიულ მოწყობილობებში არსებულ რეგისტრებზე. რეგისტრის მისამართი, როგორც ეს ULP-დან ჩანს, შეიძლება გამოითვალოს DPORT ავტობუსზე იმავე რეგისტრის მისამართიდან შემდეგნაირად:

addr_ulp = (addr_dport - DR_REG_RTCCNTL_BASE) / 4

მაგალითები:

1:        REG_WR      0x120, 7, 0, 0x10   // set 8 bits: REG[0x120][7:0] = 0x10

მოსახერხებელი მაკროები პერიფერიული რეგისტრების წვდომისთვის¶

ULP საწყისი ფაილები ასამბლერამდე C წინასწარი პროცესორის გავლით გადის. ეს საშუალებას იძლევა გარკვეული მაკროების გამოყენების პერიფერიულ რეგისტრებზე წვდომის გასაადვილებლად.

ზოგიერთი არსებული მაკრო განსაზღვრულია soc/soc_ulp.h სათაურის ფაილი. ეს მაკროები პერიფერიული რეგისტრების ველებზე მათი სახელებით წვდომის საშუალებას იძლევა. პერიფერიული რეგისტრების სახელები, რომელთა გამოყენებაც ამ მაკროებთან შეიძლება, განსაზღვრულია soc/rtc_cntl_reg.h, soc/rtc_io_reg.h, soc/sens_reg.hდა soc/rtc_i2c_reg.h.

READ_RTC_REG(rtc_reg, low_bit, bit_width)

rtc_reg[low_bit + bit_width - 1 : low_bit]-დან R0-ში 16 ბიტამდე ინფორმაციის წაკითხვა. მაგალითად:

#include "soc/soc_ulp.h"
#include "soc/rtc_cntl_reg.h"

/* Read 16 lower bits of RTC_CNTL_TIME0_REG into R0 */
READ_RTC_REG(RTC_CNTL_TIME0_REG, 0, 16)
READ_RTC_FIELD(rtc_reg, field)

rtc_reg-ის ველიდან R0-ში წაკითხვა, მაქსიმუმ 16 ბიტი. მაგალითად:

#include "soc/soc_ulp.h"
#include "soc/sens_reg.h"

/* Read 8-bit SENS_TSENS_OUT field of SENS_SAR_SLAVE_ADDR3_REG into R0 */
READ_RTC_FIELD(SENS_SAR_SLAVE_ADDR3_REG, SENS_TSENS_OUT)
WRITE_RTC_REG(rtc_reg, low_bit, bit_width, value)

ჩაწერეთ დაუყოვნებელი მნიშვნელობა rtc_reg[low_bit + bit_width - 1 : low_bit]-ში, bit_width &lt;= 8. მაგალითად:

#include "soc/soc_ulp.h"
#include "soc/rtc_io_reg.h"

/* Set BIT(2) of RTC_GPIO_OUT_DATA_W1TS field in RTC_GPIO_OUT_W1TS_REG */
WRITE_RTC_REG(RTC_GPIO_OUT_W1TS_REG, RTC_GPIO_OUT_DATA_W1TS_S + 2, 1, 1)
WRITE_RTC_FIELD(rtc_reg, field, value)

ჩაწერეთ დაუყოვნებელი მნიშვნელობა rtc_reg ფორმატის ველში, მაქსიმუმ 8 ბიტი. მაგალითად:

#include "soc/soc_ulp.h"
#include "soc/rtc_cntl_reg.h"

/* Set RTC_CNTL_ULP_CP_SLP_TIMER_EN field of RTC_CNTL_STATE0_REG to 0 */
WRITE_RTC_FIELD(RTC_CNTL_STATE0_REG, RTC_CNTL_ULP_CP_SLP_TIMER_EN, 0)

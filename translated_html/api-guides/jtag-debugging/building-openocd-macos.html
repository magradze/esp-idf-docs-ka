OpenOCD-ის შექმნა MacOS-ისთვის წყაროებიდან¶

[中文]

შემდეგი ინსტრუქციები ორობითი OpenOCD-ის ჩამოტვირთვის ალტერნატივაა. Espressif GitHubორობითი OpenOCD ფაილის სწრაფად დასაყენებლად, დამოუკიდებლად კომპილაციის ნაცვლად, შექმენით სარეზერვო ასლი და გადადით განყოფილებაზე. OpenOCD-ის დაყენება.

OpenOCD-ის წყაროების ჩამოტვირთვა¶

OpenOCD-ის ESP32-თან თავსებადი ვარიანტის წყაროები ხელმისაწვდომია Espressif GitHub-დან, შემდეგ ბმულზე: https://github.com/espressif/openocd-esp32წყაროების ჩამოსატვირთად გამოიყენეთ შემდეგი ბრძანებები:

cd ~/esp
git clone --recursive https://github.com/espressif/openocd-esp32.git

წყაროების კლონი ახლა უნდა იყოს შენახული ~/esp/openocd-esp32 დირექტორია.

ინსტალაციის დამოკიდებულებები¶

დააინსტალირეთ პაკეტები, რომლებიც საჭიროა OpenOCD-ის კომპილაცია Homebrew-ის გამოყენებით:

brew install automake libtool libusb wget [email protected] pkg-config

OpenOCD-ის შექმნა¶

გააგრძელეთ OpenOCD-ის კონფიგურაცია და აწყობა:

cd ~/esp/openocd-esp32
./bootstrap
./configure
make

სურვილისამებრ, შეგიძლიათ დაამატოთ sudo make install ბოლო ნაბიჯი. გამოტოვეთ ის, თუ უკვე გაქვთ OpenOCD (მაგ. სხვა განვითარების პლატფორმიდან), რადგან შესაძლოა მისი გადაწერა მოხდეს.

შენიშვნა

  • შეცდომის შემთხვევაში, გამოასწორეთ იგი და სცადეთ ხელახლა ბრძანების დასრულებამდე. make მუშაობს.
  • თუ OpenOCD-დან ქვემოდულის პრობლემაა, გთხოვთ cd რომ openocd-esp32 დირექტორია და შეყვანა git submodule update --init.
  • თუ ./configure წარმატებით გაშვების შემთხვევაში, ჩართული JTAG-ის შესახებ ინფორმაცია დაიბეჭდება OpenOCD configuration summary.
  • თუ თქვენი მოწყობილობის ინფორმაცია ჟურნალში არ არის ნაჩვენები, გამოიყენეთ ./configure ჩართეთ ის, როგორც აღწერილია ../openocd-esp32/doc/INSTALL.txt.
  • OpenOCD-ის კომპილაციის შესახებ დეტალებისთვის იხილეთ openocd-esp32/README.OSX.

ერთხელ make პროცესი წარმატებით დასრულდა, OpenOCD-ის შესრულებადი ფაილი შეინახება ~/esp/openocd-esp32/src/openocd დირექტორია.

შემდეგი ნაბიჯები¶

გამართვა ging გარემოს დაყენების გასაგრძელებლად, გადადით განყოფილებაში ESP32 Target-ის კონფიგურაცია.

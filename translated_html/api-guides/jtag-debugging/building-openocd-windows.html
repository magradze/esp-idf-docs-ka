OpenOCD-ის შექმნა Windows-ის წყაროებიდან¶

[中文]

შემდეგი ინსტრუქციები ორობითი OpenOCD-ის ჩამოტვირთვის ალტერნატივაა. Espressif GitHubორობითი OpenOCD ფაილის სწრაფად დასაყენებლად, დამოუკიდებლად კომპილაციის ნაცვლად, შექმენით სარეზერვო ასლი და გადადით განყოფილებაზე. OpenOCD-ის დაყენება.

შენიშვნა

შემდეგი ინსტრუქციები, სავარაუდოდ, შესრულდება MSYS2 გარემოში MINGW32 ქვესისტემით!

ინსტალაციის დამოკიდებულებები¶

დააინსტალირეთ პაკეტები, რომლებიც საჭიროა კომპილაცია OpenOCD-ისთვის:

pacman -S --noconfirm --needed autoconf automake git make \
mingw-w64-i686-gcc \
mingw-w64-i686-toolchain \
mingw-w64-i686-libtool \
mingw-w64-i686-pkg-config \
mingw-w64-cross-winpthreads-git \
p7zip

OpenOCD-ის წყაროების ჩამოტვირთვა¶

OpenOCD-ის ESP32-თან თავსებადი ვარიანტის წყაროები ხელმისაწვდომია Espressif GitHub-დან, შემდეგ ბმულზე: https://github.com/espressif/openocd-esp32წყაროების ჩამოსატვირთად გამოიყენეთ შემდეგი ბრძანებები:

cd ~/esp
git clone --recursive https://github.com/espressif/openocd-esp32.git

წყაროების კლონი ახლა უნდა იყოს შენახული ~/esp/openocd-esp32 დირექტორია.

libusb-ის ჩამოტვირთვა¶

შემდეგი OpenOCD კომპილაციისთვის ცვლადების შექმნა და ექსპორტი:

wget https://github.com/libusb/libusb/releases/download/v1.0.22/libusb-1.0.22.7z
7z x -olibusb ./libusb-1.0.22.7z
export CPPFLAGS="$CPPFLAGS -I${PWD}/libusb/include/libusb-1.0"
export LDFLAGS="$LDFLAGS -L${PWD}/libusb/MinGW32/.libs/dll"

OpenOCD-ის შექმნა¶

გააგრძელეთ OpenOCD-ის კონფიგურაცია და აწყობა:

cd ~/esp/openocd-esp32
export CPPFLAGS="$CPPFLAGS -D__USE_MINGW_ANSI_STDIO=1 -Wno-error"; export CFLAGS="$CFLAGS -Wno-error"
./bootstrap
./configure --disable-doxygen-pdf --enable-ftdi --enable-jlink --enable-ulink --build=i686-w64-mingw32 --host=i686-w64-mingw32
make
cp ../libusb/MinGW32/dll/libusb-1.0.dll ./src
cp /opt/i686-w64-mingw32/bin/libwinpthread-1.dll ./src

სურვილისამებრ, შეგიძლიათ დაამატოთ make install ბოლო ნაბიჯი. გამოტოვეთ, თუ უკვე გაქვთ OpenOCD (მაგ. სხვა განვითარების პლატფორმიდან), რადგან შესაძლოა მისი გადაწერა მოხდეს. ასევე შეგიძლიათ გამოიყენოთ export DESTDIR="/custom/install/dir"; make install.

შენიშვნა

  • შეცდომის შემთხვევაში, გამოასწორეთ იგი და სცადეთ ხელახლა ბრძანების დასრულებამდე. make მუშაობს.
  • თუ OpenOCD-დან ქვემოდულის პრობლემაა, გთხოვთ cd რომ openocd-esp32 დირექტორია და შეყვანა git submodule update --init.
  • თუ ./configure წარმატებით გაშვების შემთხვევაში, ჩართული JTAG-ის შესახებ ინფორმაცია დაიბეჭდება OpenOCD configuration summary.
  • თუ თქვენი მოწყობილობის ინფორმაცია ჟურნალში არ არის ნაჩვენები, გამოიყენეთ ./configure ჩართეთ ის, როგორც აღწერილია ../openocd-esp32/doc/INSTALL.txt.
  • OpenOCD-ის კომპილაციის შესახებ დეტალებისთვის იხილეთ openocd-esp32/README.Windows.
  • არ დაგავიწყდეთ კოპირება libusb-1.0.dll და libwinpthread-1.dll შიგნით OOCD_INSTALLDIR/bin -დან ~/esp/openocd-esp32/src.

ერთხელ make პროცესი წარმატებით დასრულდა, OpenOCD-ის შესრულებადი ფაილი შეინახება ~/esp/openocd-esp32/src დირექტორია.

სრული სია¶

ქვემოთ მოცემულია სრულად აღწერილი პროცესი, რომელიც უფრო სწრაფი შესრულებისთვისაა მოცემული, მაგ., როგორც shell სკრიპტი:

pacman -S --noconfirm --needed autoconf automake git make mingw-w64-i686-gcc mingw-w64-i686-toolchain mingw-w64-i686-libtool mingw-w64-i686-pkg-config mingw-w64-cross-winpthreads-git p7zip
cd ~/esp
git clone --recursive https://github.com/espressif/openocd-esp32.git

wget https://github.com/libusb/libusb/releases/download/v1.0.22/libusb-1.0.22.7z
7z x -olibusb ./libusb-1.0.22.7z
export CPPFLAGS="$CPPFLAGS -I${PWD}/libusb/include/libusb-1.0"; export LDFLAGS="$LDFLAGS -L${PWD}/libusb/MinGW32/.libs/dll"

export CPPFLAGS="$CPPFLAGS -D__USE_MINGW_ANSI_STDIO=1 -Wno-error"; export CFLAGS="$CFLAGS -Wno-error"
cd ~/esp/openocd-esp32
./bootstrap
./configure --disable-doxygen-pdf --enable-ftdi --enable-jlink --enable-ulink --build=i686-w64-mingw32 --host=i686-w64-mingw32
make
cp ../libusb/MinGW32/dll/libusb-1.0.dll ./src
cp /opt/i686-w64-mingw32/bin/libwinpthread-1.dll ./src

JTAG-ის გამართვა¶

[中文]

ეს დოკუმენტი წარმოადგენს GDB-ის გამოყენებით ESP32-ისა და გამართვა ging-ისთვის OpenOCD-ის ინსტალაციის სახელმძღვანელოს. დოკუმენტი სტრუქტურირებულია შემდეგნაირად:

შესავალი¶

ESP32-ს აქვს ორი მძლავრი Xtensa ბირთვი, რაც პროგრამული არქიტექტურის მრავალფეროვნების საშუალებას იძლევა. ESP-IDF თან ერთად მოწოდებული უფასო RTOS ოპერაციული სისტემა მრავალბირთვიან პრეემპტიურ მრავალძაფიან პროგრამირებას უზრუნველყოფს, რაც პროგრამული უზრუნველყოფის წერის ინტუიციურ გზას უზრუნველყოფს.

პროგრამირების სიმარტივის უარყოფითი მხარე ის არის, რომ შესაბამისი ხელსაწყოების გარეშე გამართვა ging-ის გამოყენება უფრო რთულია: ორი ნაკადით გამოწვეული შეცდომის გარკვევას, რომლებიც ერთდროულად მუშაობენ ორ სხვადასხვა CPU ბირთვზე, შეიძლება დიდი დრო დასჭირდეს, როდესაც მხოლოდ printf ოპერატორები გაქვთ. ასეთი პრობლემების გამართვა უკეთესი და ბევრ შემთხვევაში უფრო სწრაფი გზაა გამართვა ger-ის გამოყენება, რომელიც პროცესორებთან დაკავშირებულია გამართვა პორტის საშუალებით.

Espressif-მა OpenOCD პორტირება გაუკეთა ESP32 პროცესორისა და მრავალბირთვიანი Free RTOS მხარდასაჭერად, რომელიც ESP32 აპლიკაციების უმეტესობის საფუძველი იქნება და დაწერა რამდენიმე ინსტრუმენტი იმ ფუნქციების დასახმარებლად, რომლებსაც OpenOCD მშობლიურად არ უჭერს მხარს.

ეს დოკუმენტი წარმოადგენს Linux-ში, Windows-სა და MacOS- გამართვა GDB-ის გამოყენებით OpenOCD-ის ინსტალაციის სახელმძღვანელოს. ოპერაციული სისტემის სპეციფიკური ინსტალაციის პროცედურების გარდა, პროგრამული უზრუნველყოფის ინტერფეისი და გამოყენების პროცედურები ყველა მხარდაჭერილ ოპერაციულ სისტემაში ერთნაირია.

შენიშვნა

ამ დოკუმენტში წარმოდგენილი ეკრანის ანაბეჭდები შექმნილია Ubuntu 16.04 LTE-ზე მომუშავე Eclipse Neon 3-ისთვის. შესაძლოა, გარკვეული მომხმარებლის ინტერფეისის იერსახეში მცირედი განსხვავებები იყოს, იმისდა მიხედვით, იყენებთ თუ არა Windows-ს, MacOS-ს თუ Linux-ს და/ან Eclipse-ის სხვა ვერსიას.

როგორ მუშაობს?¶

ESP32-ის OpenOCD-ით JTAG (Joint Test Action Group) ინტერფეისით გამართვა ging-ის შესასრულებლად ძირითადი პროგრამული და აპარატურული უზრუნველყოფა წარმოდგენილია ქვემოთ და მოიცავს: xtensa-esp32-elf-gdb გამართვა ger, OpenOCD ჩიპზე გამართვა ger და JTAG ადაპტერი დაკავშირებულია ESP32 სამიზნე.

JTAG გამართვა ging - მიმოხილვის დიაგრამა

„აპლიკაციის ჩატვირთვისა და მონიტორინგის“ ქვეშ არის კიდევ ერთი პროგრამული და აპარატურული უზრუნველყოფა კომპილაცია ისთვის, ფლეშ მეხსიერება აპლიკაციის ESP32-ისთვის შესაქმნელად, ასევე ESP32-დან დიაგნოსტიკური შეტყობინებების მონიტორინგის საშუალებების უზრუნველსაყოფად.

JTAG-ის გამოყენებით გამართვა და აპლიკაციის ჩატვირთვა/მონიტორინგი ინტეგრირებულია დაბნელება გარემო, რათა უზრუნველყოფილი იყოს სწრაფი და მარტივი გადასვლა კოდის დაწერიდან, კომპილაციიდან და ჩატვირთვიდან გამართვა ging-ზე, შემდეგ კი ისევ კოდის დაწერაზე და ა.შ. ყველა პროგრამული უზრუნველყოფა ხელმისაწვდომია Windows, Linux და MacOS პლატფორმებისთვის.

თუ ESP-WROVER-KIT როდესაც გამოიყენება, კომპიუტერიდან ESP32-ზე დაკავშირება ეფექტურად ხორციელდება ერთი USB კაბელის გამოყენებით, WROVER-ზე დაინსტალირებული FT2232H ჩიპის წყალობით, რომელიც უზრუნველყოფს ორ USB არხს, ერთს JTAG-ისთვის და მეორეს UART კავშირისთვის.

მომხმარებლის პრეფერენციებიდან გამომდინარე, ორივე debugger და idf.py build მისი მართვა შესაძლებელია პირდაპირ ტერმინალიდან/ბრძანების ხაზიდან, Eclipse-ის ნაცვლად.

JTAG ადაპტერის შერჩევა¶

JTAG გამართვა ging-ის გამოყენების ყველაზე სწრაფი და მოსახერხებელი გზაა ESP-WROVER-KITამ განვითარების დაფის თითოეულ ვერსიას უკვე აქვს ჩაშენებული JTAG ინტერფეისი. JTAG-ის ESP32-თან დასაკავშირებლად არ არის საჭირო გარე JTAG ადაპტერი და დამატებითი გაყვანილობა/კაბელი. WROVER KIT იყენებს FT2232H JTAG ინტერფეისს, რომელიც მუშაობს 20 MHz საათის სიხშირით, რაც რთულია გარე ადაპტერით მიღწევა.

თუ გადაწყვეტთ ცალკე JTAG ადაპტერის გამოყენებას, მოძებნეთ ისეთი, რომელიც თავსებადია როგორც ESP32-ის ძაბვის დონეებთან, ასევე OpenOCD პროგრამულ უზრუნველყოფასთან. ESP32-ის JTAG პორტი არის ინდუსტრიის სტანდარტული JTAG პორტი, რომელსაც არ აქვს (და არც სჭირდება) TRST პინი. JTAG-ის შემავალი/გამომავალი პინები იკვებება VDD_3P3_RTC პინიდან (რომელიც ჩვეულებრივ იკვებება 3.3 ვოლტიანი რელსით), ამიტომ JTAG ადაპტერს უნდა შეეძლოს JTAG პინებთან მუშაობა ამ ძაბვის დიაპაზონში.

პროგრამული უზრუნველყოფის მხრივ, OpenOCD მხარს უჭერს JTAG ადაპტერების საკმაოდ დიდ რაოდენობას. იხილეთ http://openocd.org/doc/html/Debug-Adapter-Hardware.html OpenOCD-ის მიერ თავსებადი ადაპტერების (სამწუხაროდ, ოდნავ არასრული) სიისთვის. ამ გვერდზე ასევე ჩამოთვლილია SWD-თან თავსებადი ადაპტერები; გაითვალისწინეთ, რომ ESP32 არ უჭერს მხარს SWD-ს. JTAG ადაპტერები, რომლებიც კოდირებულია კონკრეტული პროდუქტის ხაზზე, მაგ. ST-LINK გამართვა ging ადაპტერები STM32 ოჯახებისთვის, არ იმუშავებს.

მოქმედი JTAG კავშირის მისაღებად მინიმალური სიგნალიზაციაა TDI, TDO, TCK, TMS და GND. ზოგიერთ JTAG გამართვა გერს ასევე სჭირდება კავშირი ESP32 ელექტროგადამცემი ხაზიდან, მაგალითად, Vtar-ის სახელწოდების ხაზთან, სამუშაო ძაბვის დასაყენებლად. SRST-ის დამატებით შესაძლებელია ESP32-ის CH_PD-თან დაკავშირება, თუმცა ამ ეტაპზე OpenOCD-ში ამ ხაზის მხარდაჭერა საკმაოდ მინიმალურია.

OpenOCD-ის დაყენება¶

თუ უკვე დააყენეთ ESP-IDF CMake build სისტემის გამოყენებით შემდეგის მიხედვით: დაწყების სახელმძღვანელო, მაშინ OpenOCD უკვე დაინსტალირებულია. გარემოს შექმნა თქვენს ტერმინალში უნდა შეძლოთ OpenOCD-ის გაშვება. ამის შესამოწმებლად, შეასრულეთ შემდეგი ბრძანება:

openocd --version

გამომავალი უნდა იყოს შემდეგი (თუმცა ვერსია შეიძლება უფრო ახალი იყოს, ვიდრე აქ ჩამოთვლილია):

ასევე შეგიძლიათ დაადასტუროთ, რომ OpenOCD-მა იცის, სად მდებარეობს მისი კონფიგურაციის სკრიპტები მნიშვნელობის დაბეჭდვით. OPENOCD_SCRIPTS გარემოს ცვლადი, აკრეფით echo $OPENOCD_SCRIPTS (Linux-ისა და macOS-ისთვის) ან echo %OPENOCD_SCRIPTS% (Windows-ისთვის). თუ სწორი გზა დაბეჭდილია, მაშინ OpenOCD სწორად არის დაყენებული.

თუ ამ ნაბიჯებიდან რომელიმე არ მუშაობს, გთხოვთ, დაბრუნდეთ ხელსაწყოების დაყენება დაწყების სახელმძღვანელოს განყოფილება.

შენიშვნა

ასევე შესაძლებელია OpenOCD-ის აწყობა წყაროდან. იხილეთ OpenOCD-ის შექმნა წყაროებიდან დეტალებისთვის იხილეთ განყოფილება.

ESP32 Target-ის კონფიგურაცია¶

OpenOCD-ის ინსტალაციის შემდეგ, გადადით ESP32 სამიზნის კონფიგურაციაზე (ანუ ESP32 დაფა JTAG ინტერფეისით). ამას გააკეთებთ შემდეგი სამი ნაბიჯით:

  • JTAG ინტერფეისის კონფიგურაცია და დაკავშირება
  • OpenOCD-ის გაშვება
  • ატვირთეთ განაცხადი გამართვა კურსისთვის

JTAG ინტერფეისის კონფიგურაცია და დაკავშირება¶

ეს ნაბიჯი დამოკიდებულია თქვენს მიერ გამოყენებულ JTAG-სა და ESP32 დაფაზე - იხილეთ ქვემოთ აღწერილი ორი შემთხვევა.

  • WROVER JTAG ინტერფეისის კონფიგურაცია
  • სხვა JTAG ინტერფეისის კონფიგურაცია

OpenOCD-ის გაშვება¶

სამიზნე კონფიგურაციისა და კომპიუტერთან დაკავშირების შემდეგ, თქვენ მზად ხართ OpenOCD-ის გასაშვებად.

გახსენით ტერმინალი და დააყენეთ ის ESP-IDF გამოსაყენებლად, როგორც ეს აღწერილია გარემოს შექმნა „დაწყების სახელმძღვანელოს“ განყოფილება. შემდეგ გაუშვით OpenOCD (ეს ბრძანება მუშაობს Windows-ზე, Linux-სა და macOS-ზე):

openocd -f board/esp32-wrover-kit-3.3v.cfg

შენიშვნა

ფაილები, რომლებიც მოწოდებულია შემდეგ -f ზემოთ მოცემული ინფორმაცია სპეციფიკურია ESP-WROVER-KIT-ისთვის ESP32-WROOM-32 მოდული. შესაძლოა დაგჭირდეთ სხვადასხვა ფაილების მიწოდება გამოყენებული აპარატურიდან გამომდინარე. ინსტრუქციისთვის იხილეთ OpenOCD-ის კონფიგურაცია კონკრეტული სამიზნისთვის.

ახლა თქვენ უნდა ნახოთ მსგავსი შედეგი (ეს ჟურნალი ESP-WROVER-KIT-ისთვისაა):

user-name@computer-name:~/esp/esp-idf$ openocd -f board/esp32-wrover-kit-3.3v.cfg

ატვირთეთ განაცხადი გამართვა კურსისთვის¶

შექმენით და ატვირთვა თქვენი განაცხადი ESP32-ზე, როგორც ყოველთვის, იხილეთ ნაბიჯი 8. პროექტის შექმნა.

კიდევ ერთი ვარიანტია აპლიკაციის სურათის ფლეშ მეხსიერება ში ჩაწერა OpenOCD-ის გამოყენებით JTAG-ის მეშვეობით შემდეგი ბრძანებების გამოყენებით:

openocd -f board/esp32-wrover-kit-3.3v.cfg -c "program_esp filename.bin 0x10000 verify exit"

OpenOCD ფლეშ მეხსიერება ing ბრძანება program_esp აქვს შემდეგი ფორმატი:

program_esp <image_file> <offset> [verify] [reset] [exit]

  • image_file - პროგრამის სურათის ფაილისკენ მიმავალი გზა.
  • offset - გამოსახულების ჩასაწერად ფლეშ მეხსიერება ბანკში ოფსეტის გაკეთება.
  • verify - არასავალდებულო. დაწერის შემდეგ გადაამოწმეთ ფლეშ მეხსიერება შინაარსი.
  • reset - არასავალდებულო. პროგრამირების შემდეგ სამიზნის გადატვირთვა.
  • exit - არასავალდებულო. საბოლოოდ, OpenOCD-დან გამოდით.

ახლა თქვენ მზად ხართ დაიწყოთ განაცხადის გამართვა განმავლობაში. მიჰყევით ქვემოთ მოცემულ ნაწილში აღწერილ ნაბიჯებს.

დებაგერის გაშვება¶

ESP32-ისთვის განკუთვნილი ინსტრუმენტების ნაკრები შეიცავს GNU Debugger-ს, შემოკლებით GDB-ს. ის ხელმისაწვდომია სხვა ინსტრუმენტების ნაკრები პროგრამებთან ერთად ფაილის სახელით xtensa-esp32-elf-gdbGDB-ის გამოძახება და მართვა შესაძლებელია პირდაპირ ტერმინალში ბრძანების ხაზიდან. კიდევ ერთი ვარიანტია მისი გამოძახება IDE-დან (მაგალითად, Eclipse, Visual Studio Code და ა.შ.) და ირიბად მუშაობა GUI-ის დახმარებით, ტერმინალში ბრძანებების აკრეფის ნაცვლად.

გამართვა ger-ის გამოყენების ორივე ვარიანტი განხილულია ქვემოთ მოცემულ ბმულებში.

  • დაბნელება
  • ბრძანების ხაზი

რეკომენდებულია პირველ რიგში შეამოწმოთ, მუშაობს თუ არა გამართვა ger ბრძანების ხაზი და შემდეგ გადადით გამოყენებაზე დაბნელება.

გამართვის მაგალითები¶

ეს განყოფილება განკუთვნილია იმ მომხმარებლებისთვის, რომლებიც არ იცნობენ GDB-ს. იგი წარმოადგენს გამართვა ging სესიის მაგალითს დაბნელება მარტივი აპლიკაციის გამოყენებით, რომელიც ხელმისაწვდომია ქვემოთ დაწყება/თვალის დახამხამება და მოიცავს შემდეგ გამართვა , რომელიც მოიცავს შემდეგ ქმედებებს:

  1. კოდში, გამოძახების დასტასა და ძაფებში ნავიგაცია
  2. წყვეტის წერტილების დაყენება და გასუფთავება
  3. სამიზნის ხელით გაჩერება
  4. კოდის გაცნობა
  5. მეხსიერების შემოწმება და დაყენება
  6. პროგრამის ცვლადების ყურება და დაყენება
  7. პირობითი შესვენების წერტილების დაყენება

მსგავსი გამართვა ging მოქმედებები მოცემულია GDB-ის გამოყენებით ბრძანების ხაზი.

მაგალითებზე გადასვლამდე, დააყენეთ თქვენი ESP32 სამიზნე და ჩატვირთეთ იგი დაწყება/თვალის დახამხამება.

OpenOCD-ის შექმნა წყაროებიდან¶

გთხოვთ, იხილოთ ქვემოთ ჩამოთვლილი ცალკეული დოკუმენტები, რომლებიც აღწერს შექმნის პროცესს.

  • Windows
  • Linux
  • MacOS

ამ დოკუმენტში OpenOCD-ის გამოძახების მაგალითები გულისხმობს წინასწარ აშენებული ორობითი დისტრიბუციის გამოყენებას, რომელიც აღწერილია ნაწილში OpenOCD-ის დაყენება.

წყაროებიდან ლოკალურად აწყობილი ბინარული ფაილების გამოსაყენებლად, შეცვალეთ OpenOCD შესრულებადი ფაილის გზა შემდეგით: src/openocd და დააყენეთ OPENOCD_SCRIPTS გარემოს ცვლადი, რათა OpenOCD-მა შეძლოს კონფიგურაციის ფაილების პოვნა. Linux-ისა და macOS-ისთვის:

cd ~/esp/openocd-esp32
export OPENOCD_SCRIPTS=$PWD/tcl

Windows-ისთვის:

cd %USERPROFILE%\esp\openocd-esp32
set "OPENOCD_SCRIPTS=%CD%\tcl"

Linux-ისა და macOS-ისთვის OpenOCD build-ის ლოკალურად წყაროებიდან გამოძახების მაგალითი:

src/openocd -f board/esp32-wrover-kit-3.3v.cfg

და Windows:

src\openocd -f board\esp32-wrover-kit-3.3v.cfg

რჩევები და უცნაურობები¶

ეს განყოფილება გთავაზობთ ბმულების კრებულს ყველა რჩევასა და უცნაურობაზე, რომლებიც მოხსენიებულია ამ სახელმძღვანელოს სხვადასხვა ნაწილიდან.

  • ხელმისაწვდომია შემაკავებელი და სათვალთვალო წერტილები
  • კიდევ რა უნდა ვიცოდე შესვენების წერტილების შესახებ?
  • ფლეშის შესაბამისობები პროგრამული უზრუნველყოფის ფლეშის შესვენების წერტილებთან შედარებით
  • რატომ არ გვერდს უვლის „შემდეგი“-ს გამოყენებით ქვერუტინის გამოძახებებს?
  • OpenOCD-ის მხარდაჭერის ვარიანტები კომპილაცია დროს
  • უფასო RTOS მხარდაჭერა
  • რატომ უნდა დავაყენოთ SPI ფლეშ მეხსიერება ძაბვა OpenOCD კონფიგურაციაში?
  • JTAG სიჩქარის ოპტიმიზაცია
  • რას ნიშნავს გამართვა ger-ის გაშვების ბრძანებები?
  • OpenOCD-ის კონფიგურაცია კონკრეტული სამიზნისთვის
  • როგორ აღადგენს გამართვა ger ESP32-ს?
  • არ გამოიყენოთ JTAG პინები სხვა რამისთვის
  • JTAG-ისა და ESP32-WROOM-32 AT პროგრამული უზრუნველყოფა თავსებადობის პრობლემა
  • OpenOCD / GDB-თან დაკავშირებული პრობლემების შესახებ შეტყობინება

ULP კოპროცესორის პროგრამირება C მაკროების გამოყენებით¶

ESP32 ULP კოპროცესორისთვის არსებული binutils პორტის გარდა, შესაძლებელია ULP-სთვის პროგრამების გენერირება ESP32 აპლიკაციაში ასამბლეის მსგავსი მაკროების ჩასმით. აქ მოცემულია მაგალითი, თუ როგორ შეიძლება ამის გაკეთება:

const ulp_insn_t program[] = {
    I_MOVI(R3, 16),         // R3 <- 16
    I_LD(R0, R3, 0),        // R0 <- RTC_SLOW_MEM[R3 + 0]
    I_LD(R1, R3, 1),        // R1 <- RTC_SLOW_MEM[R3 + 1]
    I_ADDR(R2, R0, R1),     // R2 <- R0 + R1
    I_ST(R2, R3, 2),        // R2 -> RTC_SLOW_MEM[R2 + 2]
    I_HALT()
};
size_t load_addr = 0;
size_t size = sizeof(program)/sizeof(ulp_insn_t);
ulp_process_macros_and_load(load_addr, program, &size);
ulp_run(load_addr);

ის program მასივი არის მასივი ulp_insn_t, ანუ ULP თანაპროცესორის ინსტრუქციები. თითოეული I_XXX წინასწარი პროცესორის განსაზღვრა ითარგმნება ერთ 32-ბიტიან ინსტრუქციად. ამ წინასწარი პროცესორის განსაზღვრებების არგუმენტები შეიძლება იყოს რეგისტრის ნომრები (R0 — R3) და ლიტერალური კონსტანტები. იხილეთ ULP კოპროცესორის ინსტრუქცია განსაზღვრავს განყოფილება ინსტრუქციებისა და მათ მიერ აღებული არგუმენტების აღწერილობებისთვის.

შენიშვნა

რადგან ზოგიერთი ინსტრუქციის მაკრო ფუნქციური ზარების ხაზში გაფართოვდება, ასეთი მასივის გლობალურ დიაპაზონში განსაზღვრა კომპილაცია r-ის მიერ „ინიციალიზატორის ელემენტი მუდმივი არ არის“ შეცდომის წარმოქმნას გამოიწვევს. ამ შეცდომის გამოსასწორებლად, ინსტრუქციების მასივის განმარტება ლოკალურ დიაპაზონში გადაიტანეთ.

ინსტრუქციების ჩატვირთვისა და შენახვისას გამოიყენება 32-ბიტიანი სიტყვებით გამოხატული მისამართები. მისამართი 0 შეესაბამება პირველ სიტყვას. RTC_SLOW_MEM (რომელიც არის მისამართი 0x50000000, როგორც ეს მთავარი პროცესორების მიერ ჩანს).

განშტოების ინსტრუქციების გენერირებისთვის, სპეციალური M_ გამოიყენება წინასწარი პროცესორის განსაზღვრებები. M_LABEL define-ის გამოყენება შესაძლებელია განშტოების სამიზნის განსაზღვრისთვის. ეტიკეტის იდენტიფიკატორი არის 16-ბიტიანი მთელი რიცხვი. M_Bxxx defines-ის გამოყენება შესაძლებელია განშტოების ინსტრუქციების გენერირებისთვის, რომელთა სამიზნე კონკრეტული იარლიყია.

განხორციელების შენიშვნა: ესენი M_ წინასწარი პროცესორის განმარტებები ითარგმნება ორად ulp_insn_t მნიშვნელობები: ერთი არის ტოკენის მნიშვნელობა, რომელიც შეიცავს ეტიკეტის ნომერს, ხოლო მეორე არის თავად ინსტრუქცია. ulp_process_macros_and_load ფუნქცია ეტიკეტის ნომერს მისამართად ანაწილებს, ცვლის განშტოების ინსტრუქციას სწორი მისამართის გამოსაყენებლად და შლის ზედმეტს. ulp_insn_t ტოკენი, რომელიც შეიცავს ეტიკეტის ნომერს.

აქ მოცემულია ეტიკეტებისა და განშტოებების გამოყენების მაგალითი:

const ulp_insn_t program[] = {
    I_MOVI(R0, 34),         // R0 <- 34
    M_LABEL(1),             // label_1
    I_MOVI(R1, 32),         // R1 <- 32
    I_LD(R1, R1, 0),        // R1 <- RTC_SLOW_MEM[R1]
    I_MOVI(R2, 33),         // R2 <- 33
    I_LD(R2, R2, 0),        // R2 <- RTC_SLOW_MEM[R2]
    I_SUBR(R3, R1, R2),     // R3 <- R1 - R2
    I_ST(R3, R0, 0),        // R3 -> RTC_SLOW_MEM[R0 + 0]
    I_ADDI(R0, R0, 1),      // R0++
    M_BL(1, 64),            // if (R0 < 64) goto label_1
    I_HALT(),
};
RTC_SLOW_MEM[32] = 42;
RTC_SLOW_MEM[33] = 18;
size_t load_addr = 0;
size_t size = sizeof(program)/sizeof(ulp_insn_t);
ulp_process_macros_and_load(load_addr, program, &size);
ulp_run(load_addr);

ფუნქციები¶

esp_err_t ulp_process_macros_and_load(uint32_t load_addr, კონსტ ulp_insn_t *პროგრამა, size_t *ზომა)¶

პროგრამაში არსებული ყველა მაკრო მითითების ამოხსნა და RTC მეხსიერებაში ჩატვირთვა.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_NO_MEM თუ დამხმარე დროებითი სტრუქტურის გამოყოფა შეუძლებელია
  • ESP_ERR_ULP_xxx-დან ერთ-ერთი, თუ პროგრამა არასწორია ან მისი ჩატვირთვა შეუძლებელია
პარამეტრები
  • load_addrმისამართი, სადაც პროგრამა უნდა ჩაიტვირთოს, გამოხატული 32-ბიტიანი სიტყვებით
  • program: ulp_insn_t მასივი პროგრამასთან ერთად
  • psizeპროგრამის ზომა, გამოხატული 32-ბიტიანი სიტყვებით

esp_err_t ulp_run(uint32_t შესვლის_პუნქტი)

გაუშვით RTC მეხსიერებაში ჩატვირთული პროგრამა.

დაბრუნება
ESP_OK on success
პარამეტრები
  • entry_point: შესვლის წერტილი, გამოხატული 32-ბიტიანი სიტყვებით

შეცდომის კოდები¶

ESP_ERR_ULP_BASE¶

ULP-თან დაკავშირებული შეცდომის კოდების ოფსეტი

ESP_ERR_ULP_SIZE_TOO_BIG¶

პროგრამა არ ჯდება ULP-სთვის დაჯავშნილ RTC მეხსიერებაში.

ESP_ERR_ULP_INVALID_LOAD_ADDR¶

ჩატვირთვის მისამართი ULP-სთვის დაჯავშნილი RTC მეხსიერების გარეთაა.

ESP_ERR_ULP_DUPLICATE_LABEL¶

განსაზღვრული იყო ერთზე მეტი იარლიყი ერთი და იგივე ნომრით

ESP_ERR_ULP_UNDEFINED_LABEL¶

განშტოების ინსტრუქციები მიუთითებს განუსაზღვრელ ლეიბლზე

ESP_ERR_ULP_BRANCH_OUT_OF_RANGE¶

განშტოების სამიზნე B ინსტრუქციის დიაპაზონის გარეთაა (სცადეთ BX-ით ჩანაცვლება)

ULP კოპროცესორის რეგისტრები¶

ULP თანაპროცესორს აქვს 4 16-ბიტიანი ზოგადი დანიშნულების რეგისტრი. ყველა რეგისტრს აქვს ერთნაირი ფუნქციონალი, ერთი გამონაკლისის გარდა. R0 რეგისტრი გამოიყენება ზოგიერთი შედარებისა და განშტოების ინსტრუქციის მიერ, როგორც საწყისი რეგისტრი.

ეს განმარტებები შეიძლება გამოყენებულ იქნას ყველა ინსტრუქციისთვის, რომელიც მოითხოვს რეგისტრს.

R0¶

ზოგადი დანიშნულების რეგისტრი 0

R1¶

ზოგადი დანიშნულების რეგისტრი 1

R2¶

ზოგადი დანიშნულების რეგისტრი 2

R3¶

ზოგადი დანიშნულების რეგისტრი 3

ULP კოპროცესორის ინსტრუქცია განსაზღვრავს¶

I_DELAY(cycles_)¶

შეფერხება (nop) მოცემული რაოდენობის ციკლებისთვის

I_HALT()¶

კოპროცესორის გაჩერება.

ეს ინსტრუქცია აჩერებს კოპროცესორს, მაგრამ ULP ტაიმერს აქტიურად ინარჩუნებს. ამგვარად, ULP პროგრამა ხელახლა ჩაირთვება ტაიმერის მიერ. პროგრამის შესაჩერებლად და ტაიმერის მიერ პროგრამის გადატვირთვის თავიდან ასაცილებლად, გამოიყენეთ I_END(0) ინსტრუქცია.

I_END()¶

ULP პროგრამის ტაიმერის გაჩერება.

ეს არის მოსახერხებელი მაკრო, რომელიც გამორთავს ULP პროგრამის ტაიმერს. ამ ინსტრუქციის გამოყენების შემდეგ, ULP პროგრამა აღარ გადაიტვირთება ulp_run ფუნქციის გამოძახებამდე.

ULP პროგრამა ამ ინსტრუქციის შემდეგ გააგრძელებს მუშაობას. ამჟამად გაშვებული პროგრამის შესაჩერებლად გამოიყენეთ I_HALT().

I_ST(reg_val, reg_addr, offset_)¶

შეინახეთ მნიშვნელობა reg_val რეგისტრიდან RTC მეხსიერებაში.

მნიშვნელობა იწერება ოფსეტში, რომელიც გამოითვლება reg_addr რეგისტრისა და offset_ ველის მნიშვნელობის შეკრებით (ეს ოფსეტი გამოიხატება 32-ბიტიანი სიტყვებით). RTC მეხსიერებაში ჩაწერილი 32 ბიტი აგებულია შემდეგნაირად:

  • ბიტები [31:21] ინახავენ მიმდინარე ინსტრუქციის PC-ს, რომელიც გამოხატულია 32-ბიტიანი სიტყვებით
  • ბიტები [20:16] = 5&#39;b1
  • ბიტებს [15:0] ენიჭებათ reg_val-ის შინაარსი

RTC_SLOW_MEM[addr + offset_] = { 5&#39;b0, insn_PC[10:0], val[15:0] }

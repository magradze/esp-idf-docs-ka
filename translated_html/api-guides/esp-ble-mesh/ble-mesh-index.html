ESP-BLE-MESH¶

[中文]

Bluetooth ® mesh ქსელი უზრუნველყოფს მრავალ-მრავალ (m:m) მოწყობილობათა კომუნიკაციას და ოპტიმიზებულია მასშტაბური მოწყობილობების ქსელების შესაქმნელად.

მოწყობილობებმა შეიძლება გადასცენ მონაცემები სხვა მოწყობილობებს, რომლებიც არ იმყოფებიან წყაროს მოწყობილობის პირდაპირ რადიო დიაპაზონში. ამ გზით, mesh ქსელებს შეუძლიათ მოიცავონ ძალიან დიდ ფიზიკურ არეალს და შეიცავდეს მოწყობილობების დიდ რაოდენობას. ის იდეალურად შეეფერება შენობების ავტომატიზაციის, სენსორული ქსელების და სხვა ნივთების ინტერნეტის გადაწყვეტილებებს, სადაც ათობით, ასობით ან ათასობით მოწყობილობას სჭირდება ერთმანეთთან საიმედო და უსაფრთხო კომუნიკაცია.

Bluetooth mesh არ არის უკაბელო საკომუნიკაციო ტექნოლოგია, არამედ ქსელური ტექნოლოგიაა. ეს ტექნოლოგია დამოკიდებულია Bluetooth Low Energy (BLE)-ზე - უკაბელო საკომუნიკაციო პროტოკოლების დასტაზე.

Zephyr Bluetooth Mesh სტეკის საფუძველზე აგებული ESP-BLE-MESH იმპლემენტაცია მხარს უჭერს მოწყობილობების უზრუნველყოფას და კვანძის კონტროლს. ის ასევე მხარს უჭერს კვანძის ისეთ ფუნქციებს, როგორიცაა Proxy, Relay, Low power და Friend.

გთხოვთ, იხილოთ ESP-BLE-MESH არქიტექტურა ESP-BLE-MESH არქიტექტურის იმპლემენტაციის შესახებ ინფორმაციისთვის და ESP-BLE-MESH API მითითება შესაბამისი API შესახებ ინფორმაციისთვის 8.

ESP-BLE-MESH დანერგილი და სერტიფიცირებულია უახლესი Mesh Profile v1.0.1-ის საფუძველზე, მომხმარებლებს შეუძლიათ მიმართონ აქ ESP-BLE-MESH-ის სერტიფიცირების დეტალებისთვის.

შენიშვნა

თუ ESP32-ისთვის mesh-ის Wi-Fi ზე დაფუძნებულ იმპლემენტაციას ეძებთ, გთხოვთ, გაეცნოთ Espressif-ის სხვა პროდუქტს, სახელწოდებით ESP-MESH. დამატებითი ინფორმაციისა და დოკუმენტაციისთვის იხილეთ ESP-MESH.

ESP-BLE-MESH-ის გამოყენების დაწყება¶

ეს განყოფილება განკუთვნილია იმისთვის, რომ დაგეხმაროთ ESP-BLE-MESH-ის გამოყენებაში Espressif-ის ESP32 ჩიპზე დაფუძნებული აპარატურისთვის.

ჩვენ ვაპირებთ სამი კვანძისგან შემდგარი მცირე ESP-BLE-MESH ქსელის დაყენებისა და მუშაობის პროცესის დემონსტრირებას. ეს პროცესი მოიცავს მოწყობილობების უზრუნველყოფას და კვანძების კონფიგურაციას, შემდეგ კი კონკრეტულ კვანძებზე არსებული ზოგადი OnOff სერვერის მოდელებისთვის ჩართვის/გამორთვის ბრძანებების გაგზავნას.

თუ ESP-IDF ში ახალი ხართ, გთხოვთ, ჯერ დააყენოთ განვითარების გარემო, კომპილაცია , ფლეშ მეხსიერება და გაუშვათ მაგალითის აპლიკაცია, რომელიც შეესაბამება ESP-IDF ზედა დონეს. დაწყება დოკუმენტაცია.

რა გჭირდებათ¶

აპარატურა:

  • სამი ESP32 დაფა, იხ. ვარიანტები.
  • USB კაბელები პლატების დასაკავშირებლად.
  • კომპიუტერი კონფიგურირებულია ESP-IDF ით.
  • მობილური ტელეფონი ან პლანშეტი Android-ის ან iOS-ის ოპერაციული სისტემით.

პროგრამული უზრუნველყოფა:

  • განაცხადის მაგალითი bluetooth/esp_ble_mesh/ble_mesh_node/onoff_server კოდი ESP32 დაფებზე ჩასატვირთად.

  • მობილური აპლიკაცია: nRF ბადე Android-ის ან iOS-ისთვის. სურვილისამებრ, შეგიძლიათ გამოიყენოთ სხვა აპლიკაციები:

    • EspBleMesh Android აპლიკაცია
    • Silicon Labs-ის Android ან iOS აპლიკაცია

ინსტალაცია ეტაპობრივად¶

ეს არის დეტალური გზამკვლევი, რომელიც ინსტალაციის პროცესს გაგიძღვებათ.

ნაბიჯი 1. აპარატურის შემოწმება¶

ორივე ESP32-DevKitC და ESP-WROVER-KIT ESP-BLE-MESH იმპლემენტაციისთვის მხარდაჭერილია დეველოპერული დაფები. კონკრეტული დაფის არჩევა შეგიძლიათ menuconfig მეშვეობით: idf.py menuconfig &gt; Example Configuration &gt; Board selection for ESP-BLE-MESH

შენიშვნა

თუ გამოყენებას გეგმავთ ESP32-DevKitC, შეაერთეთ RGB LED GPIO 25, 26 და 27 პინებთან.

ნაბიჯი 2. პროგრამული უზრუნველყოფის კონფიგურაცია¶

შეიყვანეთ bluetooth/esp_ble_mesh/ble_mesh_node/onoff_server მაგალითი დირექტორია, გაშვება idf.py menuconfig თქვენი დაფის ასარჩევად და შემდეგ გასაშვებად idf.py build მაგალითისთვის კომპილაცია .

ნაბიჯი 3. აპლიკაციის ატვირთვა Nodes-ში¶

შემდეგ bluetooth/esp_ble_mesh/ble_mesh_node/onoff_server მაგალითია კომპილაცია d წარმატებით, მომხმარებლებს შეუძლიათ გაუშვან idf.py flash ატვირთვა ში იგივე გენერირებული ორობითი ფაილები სამივე დეველოპერულ დაფაზე უნდა განთავსდეს.

როგორც კი დაფები ჩაირთვება, თითოეულ დაფაზე RGB LED უნდა ჩაირთოს. მწვანე.

ESP-BLE-MESH მოწყობილობების ჩართვა

ნაბიჯი 4. უზრუნველყოფის კვანძები¶

ამ განყოფილებაში ჩვენ გამოვიყენებთ nRF Mesh Android აპლიკაცია, რომელიც აჩვენებს, თუ როგორ უნდა მოამზადოთ მოწყობილობა, რომელიც არ არის მომარაგებული. მომხმარებლებს ასევე შეუძლიათ მისი iOS ვერსიის ჩამოტვირთვა App Store-დან.

4.1 სკანერი¶

სკანერი აპლიკაციის ფუნქციაა, რომელიც გამოიყენება დიაპაზონში არსებული გაუმართავი მოწყობილობების მოსაძებნად. გახსენით აპლიკაცია, დააჭირეთ სკანერი ქვედა ნაწილში და დაიწყება ძებნა. ცოტა ხნის შემდეგ უნდა დავინახოთ სამი გაუმართავი მოწყობილობა.

nRF Mesh - სკანერი

4.2 იდენტიფიცირება¶

მომხმარებლებს შეუძლიათ აირჩიონ ნებისმიერი არამოწოდებული მოწყობილობა, შემდეგ აპლიკაცია შეეცდება კავშირის დამყარებას არჩეულ მოწყობილობასთან. BLE კავშირის წარმატებით დამყარების შემდეგ (ზოგჯერ მომხმარებლებს რამდენჯერმე სჭირდებათ კავშირის დასამყარებლად) და შესაბამისი ESP-BLE-MESH GATT სერვისის აღმოჩენის შემდეგ, მომხმარებლებს შეეძლებათ ნახონ იდენტიფიცირება ინტერფეისის ღილაკი ეკრანზე. IDENTIFY ოპერაციის გამოყენება შესაძლებელია იმის სათქმელად, თუ რომელი მოწყობილობა იქნება უზრუნველყოფილი.

შენიშვნა

IDENTIFY ოპერაციას ასევე სჭირდება გარკვეული თანამშრომლობა მოწყობილობის მხრიდან, რის შემდეგაც მომხმარებლებს შეეძლებათ ნახონ, რომელი მოწყობილობაა დანერგვის პროცესში. ამჟამად, როდესაც დააჭირეთ იდენტიფიცირება ინტერფეისის ღილაკის შემთხვევაში, მოწყობილობიდან არანაირი ნიშანი არ ჩანს, სერიული მონიტორის ჟურნალის გარდა.

შემდეგ იდენტიფიცირება ინტერფეისის ღილაკზე დაჭერის შემდეგ, მომხმარებლებს შეუძლიათ ნახონ დებულება ინტერფეისის ღილაკი.

nRF Mesh - IDENTIFY - PROVISION

4.3 დებულება¶

შემდეგ, აპლიკაცია შეეცდება გაუმართავი მოწყობილობის მიწოდებას. მოწყობილობის წარმატებით მიწოდების შემდეგ, დაფაზე RGB LED ინდიკატორი გამოირთვება და აპლიკაცია განახორციელებს შემდეგ პროცედურებს:

  1. კვანძთან კავშირის გაწყვეტა
  2. სცადეთ კვანძთან ხელახლა დაკავშირება
  3. წარმატებით დაუკავშირდით და აღმოაჩინეთ ESP-BLE-MESH GATT სერვისი
  4. მიიღეთ კვანძის კომპოზიციის მონაცემები და დაამატეთ მას AppKey

როდესაც ყველა პროცედურა დასრულდება, კვანძი სწორად იქნება კონფიგურირებული. და დაჭერის შემდეგ კარგიმომხმარებლებს შეუძლიათ დაინახონ, რომ უნიქასთ მისამართი მინიჭებულია და კვანძის კომპოზიციის მონაცემები წარმატებით არის გაშიფრული.

nRF Mesh - კონფიგურაცია დასრულებულია

ზოგჯერ პროცედურა 2-ში, აპლიკაციამ შეიძლება ვერ შეძლოს კვანძთან ხელახლა დაკავშირება. ამ შემთხვევაში, ღილაკზე დაჭერის შემდეგ კარგი, მომხმარებლებს შეუძლიათ დაინახონ, რომ მხოლოდ კვანძის unicast მისამართია მინიჭებული, მაგრამ კომპოზიციის მონაცემები არ არის მიღებული. შემდეგ მომხმარებლებს უნდა დააჭირონ დაკავშირება ზედა მარჯვენა კუთხეში, ეკრანზე გამოჩნდება ადრე მოწოდებული კვანძი, რომლის არჩევაც მომხმარებლებმა და კვანძთან დაკავშირება უნდა სცადონ.

nRF Mesh - საწყისი კონფიგურაცია ვერ მოხერხდა

წარმატებით დაკავშირების შემდეგ, აპლიკაცია აჩვენებს ინტერფეისის ღილაკებს, რომელთა გამოყენება შესაძლებელია კომპოზიციის მონაცემების მისაღებად და AppKey-ს დასამატებლად.

nRF Mesh - ხელახლა დაკავშირება - საწყისი კონფიგურაცია

თუ მოწყობილობა მეორე ან მესამეა, რომელიც აპლიკაციით არის უზრუნველყოფილი და ღილაკზე დაჭერის შემდეგ დაკავშირებამომხმარებლებს ეკრანზე ორი ან სამი კვანძის დანახვა შეუძლიათ. ამ სიტუაციაში, მომხმარებლებს შეუძლიათ აირჩიონ ნებისმიერი მოწყობილობა დასაკავშირებლად და წარმატების შემდეგ დაბრუნდნენ მთავარ ეკრანზე, რათა აირჩიონ კონფიგურირებადი კვანძი.

აქ მოცემულია სამი მოწყობილობის მაგალითი, რომლებიც ჩამოთვლილია.

  • მარცხენა სურათზე ჩანს, რომ მესამე მოწყობილობა წარმატებით არის უზრუნველყოფილი, მაგრამ აპლიკაციამ ვერ შეძლო მასთან დაკავშირება. როდესაც ის ცდილობს მესამე კვანძთან ხელახლა დაკავშირებას, აპლიკაციაში სამი კვანძი ჩანს.
  • მარჯვენა სურათზე ჩანს, რომ ნებისმიერ კვანძთან წარმატებით დაკავშირების შემდეგ, აპლიკაცია აჩვენებს სამივე კვანძის ინფორმაციას. მომხმარებლებს შეუძლიათ დაინახონ, რომ აპლიკაციას აქვს პირველი და მეორე კვანძების შემადგენლობის მონაცემები, მაგრამ მესამესთვის მას მხოლოდ უნიქასტის მისამართი აქვს მინიჭებული, ხოლო შემადგენლობის მონაცემები უცნობია.

nRF Mesh - ხელახლა დაკავშირება - სამი კვანძი

4.4 კონფიგურაცია¶

როდესაც უზრუნველყოფა და საწყისი კონფიგურაცია დასრულდება, მომხმარებლებს შეუძლიათ დაიწყონ კვანძის კონფიგურაცია, მაგალითად, AppKey-ის ელემენტებით თითოეულ მოდელთან დაკავშირება, მასზე პუბლიკაციის ინფორმაციის დაყენება და ა.შ.

ქვემოთ მოცემული მაგალითი გვიჩვენებს, თუ როგორ უნდა დააკავშიროთ AppKey Generic OnOff Server Model-თან პირველადი ელემენტის ფარგლებში.

nRF Mesh - მოდელის შეკავშირების AppKey

შენიშვნა

AppKey-ის კონფიგურაციის სერვერის მოდელთან დაკავშირება საჭირო არ არის, რადგან ის DevKey-ს მხოლოდ ზედა ტრანსპორტის დონეზე შეტყობინებების დაშიფვრისთვის იყენებს.

ნაბიჯი 5. ქსელის მართვა¶

მას შემდეგ, რაც სამი ელემენტის ფარგლებში არსებული ყველა ზოგადი OnOff სერვერის მოდელი შესაბამისი AppKey-ით იქნება დაკავშირებული, მომხმარებლებს შეუძლიათ აპლიკაციის გამოყენება RGB LED-ის ჩასართავად/გამოსართავად.

ში bluetooth/esp_ble_mesh/ble_mesh_node/onoff_server მაგალითად, პირველი ზოგადი OnOff სერვერის მოდელი გამოიყენება კონტროლისთვის წითელი ფერი, მეორე გამოიყენება კონტროლისთვის მწვანე ფერი და მესამე გამოიყენება კონტროლისთვის ლურჯი ფერი

nRF Mesh - ზოგადი OnOff კონტროლი

ქვემოთ მოცემულ ეკრანის სურათზე ნაჩვენებია სხვადასხვა ფერის სხვადასხვა დაფა.

სამი ESP-BLE-MESH კვანძი ჩართულია

შენიშვნა

ამისთვის nRF ბადე iOS აპლიკაციაში [ვერსია 1.0.4], როდესაც კვანძი შეიცავს ერთზე მეტ ელემენტს, აპლიკაცია არ მუშაობს სწორად. თუ მომხმარებლები ცდილობენ მეორე ან მესამე ზოგადი OnOff სერვერის მოდელის ჩართვას/გამორთვას, აპლიკაციის მიერ გაგზავნილი შეტყობინება მიმართული იქნება პირველადი ელემენტის ფარგლებში არსებულ პირველ ზოგად OnOff სერვერის მოდელზე.

ESP-BLE-MESH მაგალითები¶

  • ESP-BLE-MESH კვანძის OnOff სერვერი - ნაჩვენებია ESP-BLE-MESH-ის გამოყენება, როგორც კვანძისა, რომელსაც აქვს კონფიგურაციის სერვერის მოდელი და ზოგადი OnOff სერვერის მოდელი. ESP-BLE-MESH პროვიზერს შემდეგ შეუძლია მოამზადოს დაუმონტაჟებელი მოწყობილობა და აკონტროლოს RGB LED, რომელიც წარმოადგენს ჩართვას/გამორთვის მდგომარეობას, იხილეთ მაგალითის კოდი .
  • ESP-BLE-MESH კვანძის OnOff კლიენტი - გვიჩვენებს, თუ როგორ მუშაობს კვანძში Generic OnOff Client მოდელი. კვანძს აქვს კონფიგურაციის სერვერის მოდელი და Generic OnOff Client მოდელი, იხილეთ მაგალითის კოდი .
  • ESP-BLE-MESH პროვაიდერი - გვიჩვენებს, თუ როგორ შეუძლია მოწყობილობას იმოქმედოს, როგორც ESP-BLE-MESH პროვაიდერმა მოწყობილობების უზრუნველყოფისთვის. პროვაიდერს აქვს კონფიგურაციის სერვერის მოდელი, კონფიგურაციის კლიენტის მოდელი და ზოგადი OnOff კლიენტის მოდელი, იხილეთ მაგალითის კოდი .
  • ESP-BLE-MESH სწრაფი უზრუნველყოფა - კლიენტი და სერვერი - ეს მაგალითი გამოიყენება იმის საჩვენებლად, თუ რამდენად სწრაფია ბადისებრი ქსელის შექმნა. 100 მოწყობილობის უზრუნველყოფას არაუმეტეს 60 წამი სჭირდება, იხილეთ კლიენტის კოდის მაგალითი და სერვერის კოდის მაგალითი .
  • ESP-BLE-MESH და Wi-Fi თანაარსებობა - მაგალითი, რომელიც აჩვენებს ESP32-ის Wi-Fi და Bluetooth (BLE/BR/EDR) თანაარსებობის ფუნქციას. მარტივად რომ ვთქვათ, მომხმარებლებს შეუძლიათ გამოიყენონ Wi-Fi Bluetooth გამოყენებისას, იხილეთ მაგალითის კოდი .
  • ESP-BLE-MESH კვანძის კონსოლი - მაგალითი, რომელიც ახორციელებს BLE Mesh კვანძის ძირითად ფუნქციებს. ამ მაგალითში კვანძის სკანირება და უზრუნველყოფა შესაძლებელია Provisioner-ის მიერ და პასუხის გაცემა Provisioner-ისგან get/set შეტყობინებაზე, იხილეთ კვანძის კოდის მაგალითი და პროვიზერის კოდის მაგალითი .

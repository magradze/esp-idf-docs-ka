ESP-BLE-MESH არქიტექტურა¶

[中文]

ეს დოკუმენტი წარმოგიდგენთ ESP-BLE-MESH არქიტექტურის მიმოხილვას, ESP-BLE-MESH არქიტექტურის იმპლემენტაციას, ასევე ESP-BLE-MESH დამხმარე რუტინებს.

  • ESP-BLE-MESH არქიტექტურის მიმოხილვა
    • აღწერს ESP-BLE-MESH არქიტექტურის ხუთ ძირითად ნაწილს და თითოეული ნაწილის ფუნქციონალურობას.
  • ESP-BLE-MESH არქიტექტურის იმპლემენტაცია
    • აღწერს ESP-BLE-MESH ფაილების ძირითად ფუნქციებს, ფაილებსა და ESP-BLE-MESH არქიტექტურას შორის შესაბამისობას და ფაილებს შორის გამოძახების ინტერფეისს.
  • ESP-BLE-MESH დამხმარე რუტინები
    • აღწერეთ ESP-BLE-MESH-ის დამხმარე რუტინები, როგორიცაა Mesh ქსელის მართვა, Mesh-ის ფუნქციები და ა.შ.

1. ESP-BLE-MESH არქიტექტურის მიმოხილვა¶

ამჟამად ESP-BLE-MESH-მა დანერგა Mesh Profile-ის ფუნქციების უმეტესობა და Mesh Model სპეციფიკაციაში განსაზღვრული ყველა კლიენტის მოდელი. დაკარგული ფუნქციები/მოდელები დამუშავების პროცესშია და მალე იქნება ხელმისაწვდომი. ESP-BLE-MESH არქიტექტურას მიენიჭა ოფიციალური Bluetooth სერტიფიკაცია.

სურათი 1.1 ESP-BLE-MESH არქიტექტურის დიაგრამა

ESP-BLE-MESH არქიტექტურა ხუთ ძირითად ნაწილს მოიცავს:

  • Mesh Protocol Stack
    • Mesh Networking პასუხისმგებელია ESP-BLE-MESH კვანძების შეტყობინებების დამუშავებაზე.
    • Mesh Provisioning პასუხისმგებელია ESP-BLE-MESH მოწყობილობების ნაკადის უზრუნველყოფაზე.
    • Mesh Models პასუხისმგებელია SIG-ის მიერ განსაზღვრული მოდელების დანერგვაზე.
  • Network Management
    • ახორციელებს ქსელის მართვის რამდენიმე პროცედურას, მათ შორის კვანძის წაშლის პროცედურას, IV ინდექსის აღდგენის პროცედურას და ა.შ.
  • Features
    • მოიცავს რამდენიმე ESP-BLE-MESH ფუნქციას, მაგ. დაბალი სიმძლავრის ფუნქციას, მეგობრის ფუნქციას, რელეს ფუნქციას და ა.შ.
  • Mesh Bearer Layer
    • მოიცავს Advertising Bearer და GATT Bearerმატარებელი ფენა გადამწყვეტია ESP-BLE-MESH პროტოკოლის დასტისთვის, რომელიც აგებულია Bluetooth Low-Energy ტექნოლოგიაზე, რადგან პროტოკოლის დასტამ უნდა გამოიყენოს მატარებელი ფენა მონაცემების BLE სარეკლამო არხისა და კავშირის არხის მეშვეობით გადასაცემად.
  • Applications
    • ESP-BLE-MESH პროტოკოლის დასტაზე დაყრდნობით და Mesh Models.
    • API დარეკვით და Event-ის დამუშავებით, Applications ურთიერთქმედება Mesh Networking და Mesh Provisioning ESP-BLE-MESH პროტოკოლის დასტაში, ასევე მოდელების სერიაში, რომელიც მოწოდებულია Mesh Models.

1.1 Mesh პროტოკოლის დასტა¶

1.1.1 ბადისებრი ქსელი¶

Mesh Networking პროტოკოლის დასტის არქიტექტურაში ახორციელებს შემდეგ ფუნქციებს:

  • Mesh ქსელში კვანძებს შორის კომუნიკაცია.
  • შეტყობინებების დაშიფვრა და გაშიფვრა Mesh ქსელში.
  • Mesh ქსელის რესურსების მართვა (ქსელის გასაღები, IV ინდექსი და ა.შ.).
  • Mesh ქსელის შეტყობინებების სეგმენტაცია და ხელახლა აწყობა.
  • შეტყობინებების მოდელის შესაბამისობა სხვადასხვა მოდელებს შორის.
  • მეტი ფუნქციისთვის, გთხოვთ, იხილოთ ESP-BLE-MESH ფუნქციების სია.

განხორციელება Mesh Networking ფუნქციები იერარქიულ სტრუქტურაზეა დაფუძნებული. თითოეული ფენის ფუნქციები ნაჩვენებია ცხრილში 1.1:

ცხრილი 1.1. ბადისებრი ქსელის არქიტექტურის აღწერა¶
ფენა ფუნქცია
წვდომის ფენა წვდომის დონე არა მხოლოდ განსაზღვრავს აპლიკაციის მონაცემების ფორმატს, არამედ განსაზღვრავს და აკონტროლებს ზედა სატრანსპორტო დონის მიერ განხორციელებულ მონაცემთა პაკეტების დაშიფვრასა და გაშიფვრას.
ზედა სატრანსპორტო ფენა ზედა სატრანსპორტო ფენა შიფრავს, გაშიფრავს და ავთენტიფიკაციას უკეთებს აპლიკაციის მონაცემებს წვდომის დონიდან და მისგან; ის ასევე ამუშავებს სპეციალურ შეტყობინებებს, რომლებსაც „ტრანსპორტის კონტროლის შეტყობინებები“ ეწოდება, მათ შორის „მეგობრობასთან“ და „გულისცემის“ შეტყობინებებთან დაკავშირებულ შეტყობინებებს.
ქვედა სატრანსპორტო ფენა ქვედა სატრანსპორტო ფენა ახორციელებს PDU-ს სეგმენტაციას და ხელახლა აწყობას.
ქსელის ფენა ქსელის დონე განსაზღვრავს ქსელური შეტყობინებების მისამართის ტიპსა და ფორმატს და ახორციელებს მოწყობილობის რელეის ფუნქციას.

1.1.2 ბადისებრი ქსელის უზრუნველყოფა¶

Mesh Provisioning პროტოკოლის დასტის არქიტექტურაში ახორციელებს შემდეგ ფუნქციებს:

  • დაუმონტაჟებელი მოწყობილობების უზრუნველყოფა.
  • Mesh ქსელის რესურსების განაწილება (unicast მისამართი, IV ინდექსი, NetKey და ა.შ.).
  • უზრუნველყოფის დროს მხარდაჭერილია ოთხი ავტორიზაციის მეთოდი.
  • მეტი ფუნქციისთვის, გთხოვთ, იხილოთ ESP-BLE-MESH ფუნქციების სია.

განხორციელება Mesh Provisioning ფუნქციები იერარქიულ სტრუქტურაზეა დაფუძნებული. თითოეული ფენის ფუნქციები ნაჩვენებია ცხრილში 1.2:

ცხრილი 1.2 ბადისებრი რესურსების უზრუნველყოფის არქიტექტურის აღწერა¶
ფენა ფუნქცია
PDU-ების უზრუნველყოფა სხვადასხვა ფენიდან PDU-ების უზრუნველყოფა ხორციელდება უზრუნველყოფის პროტოკოლის გამოყენებით.
ზოგადი უზრუნველყოფის PDU/Proxy PDU უზრუნველყოფის PDU-ები გადაეცემა არაუზრუნველყოფილ მოწყობილობას ზოგადი უზრუნველყოფის ფენის ან პროქსი პროტოკოლის ფენის გამოყენებით.
PB-ADV/PB-GATT ეს ფენები განსაზღვრავს, თუ როგორ გადაიცემა პროვიზირების PDU-ები ტრანზაქციების სახით, რომელთა სეგმენტირება და ხელახლა აწყობა შესაძლებელია.
სარეკლამო/მომსახურების სერვისი უზრუნველყოფის მატარებლები განსაზღვრავენ, თუ როგორ იქმნება სესიები ისე, რომ ზოგადი უზრუნველყოფის ფენიდან ტრანზაქციები გადაეცეს ერთ მოწყობილობას.

1.1.3 ბადისებრი მოდელები¶

Mesh Models პროტოკოლის დასტის არქიტექტურაში ახორციელებს შემდეგ ფუნქციებს:

  • კონფიგურაციის კლიენტი/სერვერის მოდელები
  • კლიენტის/სერვერის ჯანმრთელობის მოდელები
  • ზოგადი კლიენტის/სერვერის მოდელები
  • სენსორის კლიენტი/სერვერის მოდელები
  • დროისა და სცენების კლიენტ/სერვერის მოდელები
  • განათების კლიენტი/სერვერის მოდელები

თითოეული ფენის ფუნქციები ნაჩვენებია ცხრილში 1.3:

ცხრილი 1.3 ბადისებრი მოდელების არქიტექტურის აღწერა¶
ფენა ფუნქცია
მოდელის ფენა მოდელის დონე ნერგავს მოდელებს, რომლებიც გამოიყენება ტიპიური მომხმარებლის სცენარების მუშაობის სტანდარტიზაციისთვის, მათ შორის ზოგადი კლიენტ/სერვერის მოდელები, სენსორების კლიენტ/სერვერის მოდელები, დროისა და სცენების კლიენტ/სერვერის მოდელები, განათების კლიენტ/სერვერის მოდელები და რამდენიმე მომწოდებლის მოდელი.
საძირკვლის მოდელის ფენა საძირკვლის მოდელის ფენა ახორციელებს ESP-BLE-MESH კონფიგურაციასთან, მართვასთან, თვითდიაგნოსტიკასთან და ა.შ. დაკავშირებულ მოდელებს.

1.2 ბადისებრი ქსელის მართვა¶

Network Management ახორციელებს შემდეგ ფუნქციებს:

  • კვანძის ქსელიდან მოსაშორებლად გამოიყენება კვანძის ამოღების პროცედურა.
  • IV ინდექსის აღდგენის პროცედურა გამოიყენება კვანძის IV ინდექსის აღსადგენად.
  • კვანძების IV ინდექსის განახლებისთვის გამოიყენება IV განახლების პროცედურა.
  • გასაღების განახლების პროცედურა გამოიყენება კვანძების NetKey-ის, AppKey-ის და ა.შ. განახლებისთვის.
  • ქსელის შექმნის პროცედურა გამოიყენება mesh ქსელის შესაქმნელად.
  • NVS საცავი გამოიყენება კვანძის ქსელური ინფორმაციის შესანახად.

1.3 ბადისებრი მახასიათებლები¶

Features მოიცავს შემდეგ ვარიანტებს:

  • დაბალი ენერგიის ფუნქცია გამოიყენება კვანძის ენერგომოხმარების შესამცირებლად.
  • მეგობრობის ფუნქცია გამოიყენება დაბალი სიმძლავრის კვანძებისთვის შეტყობინებების შესანახად.
  • რელეის ფუნქცია გამოიყენება კვანძის მიერ სარეკლამო მატარებლის მეშვეობით მიღებული ქსელის PDU-ების რელევანტურ/გადასამისამართებლად.
  • პროქსი სერვერი/კლიენტი პროქსი პროტოკოლში ორი კვანძის როლია, რომლებიც კვანძებს საშუალებას აძლევს გაგზავნონ და მიიღონ ქსელის PDU-ები, mesh beacon-ები, პროქსი კონფიგურაციის შეტყობინებები და უზრუნველყონ PDU-ები კავშირზე ორიენტირებული მატარებლის მეშვეობით.

2. ESP-BLE-MESH არქიტექტურის იმპლემენტაცია¶

ESP-BLE-MESH არქიტექტურის დიზაინი და იმპლემენტაცია დაფუძნებულია ფენებსა და მოდულებზე. დეტალურად, 2.1 ნაწილი (Mesh Networking-ის იმპლემენტაცია), 2.2 ნაწილი (Mesh Provisioning-ის იმპლემენტაცია) და 2.3 ნაწილი (Mesh Bearers-ის იმპლემენტაცია) დაფუძნებულია ფენებზე, ხოლო 2.4 ნაწილი (Mesh Models-ის იმპლემენტაცია) - მოდულებზე.

  • ფენებზე დაფუძნებული მიდგომაფენებზე დაფუძნებული მიდგომით, არქიტექტურა შექმნილია Mesh Profile Specification-ში მითითებული ფენების მიხედვით. თითოეულ ფენას აქვს თავისი უნიკალური ფაილები, რომლებიც მოიცავს ამ ფენის API -ს და ა.შ. კონკრეტული დიზაინი ნაჩვენებია ნახაზ 2.1-ში.
  • მოდულზე დაფუძნებული მიდგომაყველა ფაილი ახორციელებს დამოუკიდებელ ფუნქციას, რომლის გამოძახებაც სხვა პროგრამების მიერ არის შესაძლებელი.

სურათი 2.1 ESP-BLE-MESH არქიტექტურის იმპლემენტაციის დიაგრამა

ESP-BLE-MESH არქიტექტურის დიზაინი იყენებს ფენებზე დაფუძნებულ მიდგომას. ფენების თანმიმდევრობა, რომლითაც მონაცემთა პაკეტები მუშავდება, ფიქსირებულია, ანუ პაკეტების დამუშავება შექმნის message flowამგვარად, პროტოკოლის დასტის ინტერფეისის დიაგრამიდან შეტყობინებების ნაკადები ნაჩვენებია ნახაზ 2.1-ში.

2.2 ბადისებრი მატარებლების იმპლემენტაცია¶

Mesh Bearers-ის იმპლემენტაციისას სრულად გათვალისწინებულია პორტაბელურობა. როდესაც ESP-BLE-MESH პროტოკოლის დასტა სხვა პლატფორმებზე პორტირდება, მომხმარებლებს მხოლოდ მისი შეცვლა სჭირდებათ. mesh_bearer_adapt.c .

ცხრილი 2.5 ბადის მატარებლების ფაილის აღწერა¶
ფაილი ფუნქციონალურობა
mesh_bearer_adapt.c ESP-BLE-MESH Bearer Layer ადაპტერი, ეს ფაილი უზრუნველყოფს ინტერფეისებს, რომლებიც გამოიყენება ESP-BLE-MESH ADV და GATT-თან დაკავშირებული პაკეტების მისაღებად და გასაგზავნად.

mesh_bearer_adapt.c არის განხორციელება Advertising Bearer და GATT Bearer Mesh Networking-ის ჩარჩო .

2.3 Mesh აპლიკაციების იმპლემენტაცია¶

ჩვენ მოგაწოდეთ აპლიკაციის მაგალითების სერია მომხმარებლის განვითარებისთვის, ხოლო მომხმარებლებს შეუძლიათ შეიმუშაონ პროდუქტები მათ საფუძველზე. ESP-BLE-MESH მაგალითები.

3. დამხმარე რუტინა¶

დამხმარე რუტინა გულისხმობს ESP-BLE-MESH პროტოკოლის დასტაში არსებულ დამატებით ფუნქციებს. დამხმარე რუტინის დიზაინი, როგორც წესი, კოდის შეკვეცვას ახორციელებს CONFIG_BLE_MESH.

3.1 მახასიათებლები¶

  • დაბალი სიმძლავრე
  • მეგობარი
  • ესტაფეტა
  • პროქსი კლიენტი/სერვერი

3.2 ქსელის მართვა¶

  • კვანძის მოცილების პროცედურა
  • IV ინდექსის აღდგენის პროცედურა
  • IV განახლების პროცედურა
  • გასაღების განახლების პროცედურა
  • ქსელის შექმნის პროცედურა
  • NVS საცავი

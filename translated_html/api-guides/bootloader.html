ჩატვირთვის სისტემა¶

[中文]

Bootloader ასრულებს შემდეგ ფუნქციებს:

  1. შიდა მოდულების მინიმალური საწყისი კონფიგურაცია;
  2. აირჩიეთ აპლიკაციის დანაყოფი ჩასატვირთად, დანაყოფების ცხრილი და ota_data-ს (ასეთის არსებობის შემთხვევაში) მიხედვით;
  3. ჩატვირთეთ ეს სურათი ოპერატიულ მეხსიერებაში (IRAM და DRAM) და გადაიტანეთ მენეჯმენტი მასზე.

Bootloader მდებარეობს მისამართზე 0x1000 ფლეშ მეხსიერება .

ქარხნული პარამეტრების აღდგენა¶

მომხმარებელს შეუძლია დაწეროს ძირითადი სამუშაო პროგრამული უზრუნველყოფა და ჩატვირთოს ის ქარხნულ დანაყოფში. შემდეგ, განაახლოს პროგრამული უზრუნველყოფა OTA-ს (ეთერში გადართვის) საშუალებით. განახლებული პროგრამული უზრუნველყოფა ჩაიტვირთება OTA აპლიკაციის დანაყოფში და OTA მონაცემთა დანაყოფი განახლდება ამ დანაყოფიდან ჩასატვირთად. თუ გსურთ, რომ შეძლოთ ქარხნულ პროგრამული უზრუნველყოფა ზე დაბრუნება და პარამეტრების წაშლა, მაშინ უნდა დააყენოთ CONFIG_BOOTLOADER_FACTORY_RESETქარხნული პარამეტრების აღდგენის მექანიზმი საშუალებას გაძლევთ, მოწყობილობა ქარხნულ პარამეტრებზე დააბრუნოთ:

  • ერთი ან მეტი მონაცემთა დანაყოფის გასუფთავება.
  • ჩატვირთვა „ქარხნული“ დანაყოფიდან.

CONFIG_BOOTLOADER_DATA_FACTORY_RESET მომხმარებლებს საშუალებას აძლევს აირჩიონ, რომელი მონაცემთა დანაყოფები წაიშლება ქარხნული პარამეტრების აღდგენისას. დანაყოფების სახელების მითითება შესაძლებელია მძიმით გამოყოფილი ინტერვალებით, წაკითხვადობისთვის (მაგალითად: „nvs, phy_init, nvs_custom, …“). დარწმუნდით, რომ დანაყოფების ცხრილი სა და აქ მითითებული სახელები ერთნაირია. აქ „app“ ტიპის დანაყოფების მითითება შეუძლებელია.

CONFIG_BOOTLOADER_OTA_DATA_ERASE - ქარხნული პარამეტრების აღდგენის შემდეგ მოწყობილობა ჩაიტვირთება „ქარხნული“ დანაყოფიდან. OTA მონაცემთა დანაყოფი წაიშლება.

CONFIG_BOOTLOADER_NUM_PIN_FACTORY_RESET- ქარხნული პარამეტრების აღდგენისთვის GPIO შეყვანის ნომერი გამოიყენება ქარხნული პარამეტრების აღდგენის გასააქტიურებლად, ამის გასააქტიურებლად ეს GPIO უნდა იყოს დაბალ დონეზე გადატვირთვისას.

CONFIG_BOOTLOADER_HOLD_TIME_ GPIO- ეს არის GPIO გადატვირთვის/ტესტირების რეჟიმის ლოდინის დრო (ნაგულისხმევად 5 წამი). გადატვირთვის შემდეგ, ამ პერიოდის განმავლობაში, GPIO მუდმივად დაბალ ტემპერატურაზე უნდა იყოს შენარჩუნებული, სანამ ქარხნული პარამეტრებიდან გადატვირთვა ან დანაყოფის ტესტური ჩატვირთვა (საჭიროების შემთხვევაში) შესრულდება.

დაყოფის ცხრილი.:

# Name,   Type, SubType, Offset,   Size, Flags
# Note: if you have increased the bootloader size, make sure to update the offsets to avoid overlap
nvs,      data, nvs,     0x9000,   0x4000
otadata,  data, ota,     0xd000,   0x2000
phy_init, data, phy,     0xf000,   0x1000
factory,  0,    0,       0x10000,  1M
test,     0,    test,    ,         512K
ota_0,    0,    ota_0,   ,         512K
ota_1,    0,    ota_1,   ,         512K

ჩატვირთვა TEST პროგრამული უზრუნველყოფა დან¶

მომხმარებელს შეუძლია დაწეროს სპეციალური პროგრამული უზრუნველყოფა წარმოებაში ტესტირებისთვის და საჭიროებისამებრ გაუშვას. დანაყოფების ცხრილი ასევე სჭირდება ამ ტესტირების პროგრამული უზრუნველყოფა ისთვის განკუთვნილი დანაყოფი (იხ. partition table). სატესტო აპლიკაციის გასააქტიურებლად, თქვენ უნდა დააყენოთ CONFIG_BOOTLOADER_APP_TEST.

CONFIG_BOOTLOADER_NUM_PIN_APP_TEST - GPIO ნომერი სატესტო დანაყოფის ჩასატვირთად. არჩეული GPIO კონფიგურირებული იქნება, როგორც შემავალი სიგნალი ჩართული შიდა pull-up ფუნქციით. სატესტო აპლიკაციის გასააქტიურებლად, ეს GPIO უნდა იყოს დაბალი მნიშვნელობის გადატვირთვისას. GPIO შეყვანის გამორთვისა და მოწყობილობის გადატვირთვის შემდეგ, ჩაირთვება ჩვეულებრივ კონფიგურირებული აპლიკაცია (ქარხნული ან ნებისმიერი OTA სლოტი).

CONFIG_BOOTLOADER_HOLD_TIME_ GPIO - ეს არის GPIO გადატვირთვის/ტესტირების რეჟიმის ლოდინის დრო (ნაგულისხმევად 5 წამი). გადატვირთვის შემდეგ, ამ პერიოდის განმავლობაში, GPIO მუდმივად დაბალ ტემპერატურაზე უნდა იყოს შენარჩუნებული, სანამ ქარხნული პარამეტრებიდან გადატვირთვა ან დანაყოფის ტესტური ჩატვირთვა (საჭიროების შემთხვევაში) შესრულდება.

სწრაფი ჩატვირთვა Deep Sleep-დან¶

ჩამტვირთავი აქვს CONFIG_BOOTLOADER_SKIP_VALIDATE_IN_DEEP_SLEEP ვარიანტი, რომელიც საშუალებას იძლევა შემცირდეს გაღვიძების დრო (სასარგებლოა მოხმარების შესამცირებლად). ეს ვარიანტი ხელმისაწვდომია, როდესაც CONFIG_SECURE_BOOT_ENABLED ოფცია გამორთულია. დროის შემცირება მიიღწევა სურათის ვერიფიკაციის არარსებობის გამო. პირველი ჩატვირთვისას, ჩამტვირთავი ინახავს გაშვებული აპლიკაციის მისამართს RTC FAST მეხსიერებაში. ხოლო გამოღვიძებისას, ეს მისამართი გამოიყენება ჩატვირთვისთვის ყოველგვარი შემოწმების გარეშე, რითაც მიიღწევა სწრაფი ჩატვირთვა.

მომხმარებლის ჩამტვირთავი¶

მიმდინარე ჩამტვირთავი იმპლემენტაცია მომხმარებელს საშუალებას აძლევს, შეცვალოს იგი. ამისათვის თქვენ უნდა დააკოპიროთ საქაღალდე. /esp-idf/components/bootloader და შემდეგ რედაქტირება /your_project/components/bootloader/subproject/main/bootloader_main.cჩამტვირთავი სივრცეში თქვენ არ შეგიძლიათ გამოიყენოთ დრაივერები და ფუნქციები სხვა კომპონენტი დან. საჭიროების შემთხვევაში, საჭირო ფუნქციონალი უნდა განთავსდეს ჩამტვირთავი საქაღალდეში (გაითვალისწინეთ, რომ ეს გაზრდის მის ზომას). აუცილებელია მისი ზომის მონიტორინგი, რადგან მეხსიერებაში შეიძლება იყოს გადაფარვები დანაყოფების ცხრილი ით, რაც დაზიანებას გამოიწვევს. ამ ეტაპზე ჩამტვირთავი შემოიფარგლება მისამართიდან დანაყოფების ცხრილი ით. 0x8000.

ESP32 ROM კონსოლი¶

როდესაც ESP32 ვერ ჩაიტვირთება ფლეშ მეხსიერება ROM-დან (და მისი გამომრთველი ფიუზერი არ არის გაფუჭებული), ის ჩაიტვირთება ROM კონსოლში. კონსოლი დაფუძნებულია TinyBasic-ზე და შეყვანილი ინსტრუქციები უნდა იყოს BASIC ინსტრუქციების სახით. როგორც ეს ჩვეულებრივ ხდება BASIC ენაში, წინა ხაზის ნომრის გარეშე, შეყვანილი ბრძანებები დაუყოვნებლივ სრულდება; პრეფიქსირებული ხაზის ნომრის მქონე ხაზები ინახება პროგრამის ნაწილად.

მხარდაჭერილი ოპერატორებისა და ფუნქციების სრული სია¶

სისტემა¶

  • ნახვამდის - Basic-დან გამოსვლა, ESP32-ის გადატვირთვა, ფლეშ მეხსიერება დან ჩატვირთვის ხელახლა მცდელობა
  • დასასრული - აჩერებს პროგრამის შესრულებას, ასევე „STOP“-ს.
  • მემ - აჩვენებს მეხსიერების გამოყენების სტატისტიკას
  • ახალი - მიმდინარე პროგრამის წაშლა
  • სირბილი - ახორციელებს მიმდინარე პროგრამას

IO, დოკუმენტაცია¶

  • PEEK (მისამართი) - მეხსიერების მისამართიდან 32-ბიტიანი მნიშვნელობის მიღება
  • პოკე - 32-ბიტიანი მნიშვნელობის მეხსიერებაში ჩაწერა
  • USR(მისამართი, arg1, ..) - შეასრულეთ მანქანური ენის ფუნქცია
  • PRINT გამოსახულება - დაბეჭდეთ გამოსახულება, ასევე „?“
  • PHEX გამოსახულება - გამოთქმის დაბეჭდვა თექვსმეტობითი რიცხვის სახით
  • REM მასალა - შენიშვნა/კომენტარი, ასევე „&#39;“

გამოთქმები, მათემატიკა¶

  • A=V, LET A=V - ცვლადისთვის მნიშვნელობის მინიჭება
  • +, -, *, / - მათემატიკა
  • &lt;,&lt;=,=,&lt;&gt;,!=,&gt;=,&gt; - შედარებები
  • ABS(გამოსახულება) - აბრუნებს გამოსახულების აბსოლუტურ მნიშვნელობას
  • RSEED(v) - შემთხვევითი თესლის v-ზე დაყენება
  • RND(მ) - აბრუნებს შემთხვევით რიცხვს 0-დან m-მდე
  • A=1234 - * ათობითი მნიშვნელობის მინიჭება*
  • A=&amp;h1A2 - * მინიჭება თექვსმეტობითი მნიშვნელობა*
  • A=&amp;b1001 - ორობითი მნიშვნელობის მინიჭება

კონტროლი¶

  • IF გამოთქმის ოპერატორი - შეასრულეთ ინსტრუქცია, თუ გამოსახულება მართალია
  • FOR ცვლადი = დასაწყისიდან დასასრულამდე - ბლოკის დაწყება
  • FOR ცვლადი = დასაწყისიდან დასასრულამდე STEP მნიშვნელობა - ბლოკის დაწყება ნაბიჯით
  • შემდეგი - ბლოკის დასასრული
  • GOTO ხაზის ნომერი - განაგრძეთ შესრულება ამ ხაზის ნომერზე
  • GOSUB ხაზის ნომერი - გამოიძახეთ ქვერუტინა ამ ხაზის ნომერზე
  • დაბრუნება - ქვერუტინიდან დაბრუნება
  • შეფერხება - მოცემული რაოდენობის მილიწამების დაყოვნება

პინის IO¶

  • იოდირი - დააყენეთ GPIO -pin, როგორც გამომავალი (1) ან შესასვლელი (0)
  • IOSET - დააყენეთ GPIO პინიანი, რომელიც კონფიგურირებულია როგორც გამომავალი, მაღალ (1) ან დაბალ (0) პარამეტრებზე.
  • იოგეტი - მიიღეთ GPIO ის მნიშვნელობა -pin

პროგრამების მაგალითები¶

აქ მოცემულია რამდენიმე ბრძანებისა და პროგრამის მაგალითი, რომელიც დაგეხმარებათ დაწყებაში…

uart0-ის UART_DATE რეგისტრის წაკითხვა¶

> PHEX PEEK(&h3FF40078)
15122500

GPIO 2-ის დაყენება მეხსიერების მიერ GPIO _OUT_REG-ში ჩაწერის გამოყენებით¶

შენიშვნა: ამის გაკეთება უფრო ადვილია IOSET ბრძანების გამოყენებით.

> POKE &h3FF44004,PEEK(&h3FF44004) OR &b100

მიიღეთ GPIO 0-ის მნიშვნელობა¶

> IODIR 0,0
> PRINT IOGET(0)
0

ციმციმის LED¶

შეაერთეთ LED GPIO 7-სა და დამიწებას შორის. პროგრამის გაშვებისას, LED 10-ჯერ უნდა აციმციმდეს.

10 IODIR 2,1
20 FOR A=1 TO 10
30 IOSET 2,1
40 DELAY 250
50 IOSET 2,0
60 DELAY 250
70 NEXT A
RUN

კრედიტები¶

ROM კონსოლი დაფუძნებულია მაიკ ფილდისა და სკოტ ლოურენსის „TinyBasicPlus“-ზე, რომელიც გორდონ ბრენდლის „68000 TinyBasic“-ზეა დაფუძნებული.

IDF Docker-ის სურათი¶

IDF Docker-ის სურათი (espressif/idf) განკუთვნილია ESP-IDF ის კონკრეტული ვერსიებით აპლიკაციებისა და ბიბლიოთეკების შესაქმნელად, ავტომატური აწყობის დროს.

სურათი შეიცავს:

  • ისეთი გავრცელებული უტილიტები, როგორიცაა git, wget, curl, zip.
  • Python 3.6 ან უფრო ახალი.
  • ESP-IDF ის კონკრეტული ვერსიის ასლი (ვერსიების შესახებ ინფორმაციისთვის იხილეთ ქვემოთ). IDF_PATH გარემოს ცვლადი დაყენებულია და მიუთითებს ESP-IDF მდებარეობაზე კონტეინერში.
  • ESP-IDF ის კონკრეტული ვერსიისთვის საჭირო ყველა აწყობის ინსტრუმენტი: CMake, make, ninja, cross- კომპილაცია , r, ინსტრუმენტების ნაკრები , s და ა.შ.
  • ESP-IDF მიერ მოთხოვნილი ყველა Python პაკეტი ვირტუალურ გარემოშია დაინსტალირებული.

სურათის შეყვანის წერტილი იქმნება PATH გარემოს ცვლადი, რომელიც მიუთითებს ინსტრუმენტების სწორ ვერსიაზე და ააქტიურებს Python-ის ვირტუალურ გარემოს. შედეგად, გარემო მზადაა ESP-IDF build სისტემის გამოსაყენებლად.

სურათი ასევე შეიძლება გამოყენებულ იქნას, როგორც ბაზა საკუთარი სურათებისთვის, თუ საჭიროა დამატებითი უტილიტები.

თეგები¶

ამ სურათის მრავალი თეგი შენარჩუნებულია:

  • latest: ტრეკები master ESP-IDF ის განშტოება
  • vX.Y: შეესაბამება ESP-IDF გამოშვებას vX.Y
  • release-vX.Y: ტრეკები release/vX.Y ESP-IDF ის განშტოება

შენიშვნა

ამ ფუნქციის დანერგვამდე გამოშვებულ ESP-IDF ის ვერსიებს არ აქვთ შესაბამისი Docker-ის გამოსახულების ვერსიები. ხელმისაწვდომი თეგების განახლებული სიის შემოწმება შეგიძლიათ შემდეგ ბმულზე: https://hub.docker.com/r/espressif/idf/tags.

გამოყენება¶

Docker-ის დაყენება¶

გამოყენებამდე espressif/idf Docker-ის იმიჯი ლოკალურად, დარწმუნდით, რომ Docker დაინსტალირებული გაქვთ. მიჰყევით ინსტრუქციას https://docs.docker.com/install/, თუ ის ჯერ არ არის დაინსტალირებული.

თუ გამოსახულებას CI გარემოში იყენებთ, იხილეთ თქვენი CI სერვისის დოკუმენტაცია იმის შესახებ, თუ როგორ მიუთითოთ შექმნის პროცესისთვის გამოყენებული გამოსახულება.

პროექტის შექმნა CMake-ის გამოყენებით¶

პროექტის დირექტორიაში, გაუშვით:

docker run --rm -v $PWD:/project -w /project espressif/idf idf.py build

ზემოთ მოცემული ბრძანება განმარტავდა:

  • docker run: ასრულებს Docker-ის გამოსახულებას. ეს ბრძანების შემოკლებული ფორმაა. docker container run.
  • --rm: შლის კონტეინერს აწყობის დასრულების შემდეგ
  • -v $PWD:/project: ახდენს მიმდინარე დირექტორიის მონტაჟს ჰოსტზე ($PWD) როგორც /project დირექტორია კონტეინერში
  • espressif/idf: იყენებს Docker-ის სურათს espressif/idf თეგით latest (Docker-ის მიერ ირიბად დამატებულია, როდესაც ტეგი არ არის მითითებული)
  • idf.py build: ასრულებს ამ ბრძანებას კონტეინერში

კონკრეტული docker-ის სურათის თეგით ასაწყობად, მიუთითეთ ის, როგორც espressif/idf:TAG:

docker run --rm -v $PWD:/project -w /project espressif/idf:v4.0 idf.py build

შენიშვნა

ამ წერის მომენტისთვის, ESP-IDF ის v4.0 ვერსია ჯერ არ არსებობს, ამიტომ ზემოთ მოცემული ბრძანება არ იმუშავებს. ხელმისაწვდომი თეგების განახლებული სიის შემოწმება შეგიძლიათ შემდეგ ბმულზე: https://hub.docker.com/r/espressif/idf/tags.

პროექტის შექმნა GNU Make-ის გამოყენებით¶

იგივეა, რაც CMake-სთვის, გარდა იმისა, რომ build ბრძანება განსხვავებულია:

docker run --rm -v $PWD:/project -w /project espressif/idf make defconfig all -j4

შენიშვნა

თუ sdkconfig ფაილი არ არსებობს, GNU Make build სისტემის ნაგულისხმევი ქცევაა menuconfig UI-ის გახსნა. ეს შეიძლება არ იყოს სასურველი ავტომატიზირებულ build გარემოში. იმის უზრუნველსაყოფად, რომ sdkconfig ფაილი არსებობს, defconfig სამიზნე დამატებულია ადრე all.

სურათის ინტერაქტიულად გამოყენება¶

ასევე შესაძლებელია ინტერაქტიულად აწყობა, გამართვა აწყობის პრობლემების მოგვარება ან ავტომატური აწყობის სკრიპტების ტესტირება. კონტეინერი დაიწყეთ -i -t დროშები:

docker run --rm -v $PWD:/project -w /project -it espressif/idf

შემდეგ კონტეინერში გამოიყენეთ idf.py როგორც ყოველთვის:

idf.py menuconfig
idf.py build

შენიშვნა

ბრძანებები, რომლებიც ურთიერთობენ დეველოპერის დაფასთან, როგორიცაა idf.py flash და idf.py monitor კონტეინერში არ იმუშავებს, თუ სერიული პორტი არ გადაეცემა კონტეინერს. თუმცა, ამჟამად ეს შეუძლებელია Docker for Windows-ით (https://github.com/docker/for-win/issues/1018) და Docker Mac-ისთვის (https://github.com/docker/for-mac/issues/900).

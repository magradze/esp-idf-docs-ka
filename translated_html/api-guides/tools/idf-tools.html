ჩამოსატვირთი ინსტრუმენტები¶

ESP-IDF აწყობის პროცესი ეყრდნობა რიგ ინსტრუმენტებს: cross- კომპილაცია r, ინსტრუმენტების ნაკრები , CMake აწყობის სისტემას და სხვა.

ხელსაწყოების ინსტალაცია ოპერაციული სისტემისთვის სპეციფიკური პაკეტების მენეჯერის (მაგალითად, apt, yum, brew და ა.შ.) გამოყენებით სასურველი მეთოდია, როდესაც ხელსაწყოს საჭირო ვერსია ხელმისაწვდომია. ეს რეკომენდაცია ასახულია „დაწყების სახელმძღვანელოში“. მაგალითად, Linux-სა და macOS-ზე რეკომენდებულია CMake-ის ინსტალაცია ოპერაციული სისტემის პაკეტების მენეჯერის გამოყენებით.

თუმცა, ზოგიერთი ინსტრუმენტი IDF-სპეციფიკურია და არ არის ხელმისაწვდომი ოპერაციული სისტემის პაკეტების საცავებში. გარდა ამისა, ESP-IDF ის სხვადასხვა ვერსიას სწორად მუშაობისთვის ინსტრუმენტების სხვადასხვა ვერსია სჭირდება. ამ ორი პრობლემის გადასაჭრელად, ESP-IDF გთავაზობთ სკრიპტების ნაკრებს ინსტრუმენტების სწორი ვერსიების ჩამოსატვირთად და ინსტალაციისთვის, ასევე მათი გარემოში გამოსავლენად.

დოკუმენტის დანარჩენ ნაწილში ეს ჩამოსატვირთი ინსტრუმენტები უბრალოდ „ინსტრუმენტებად“ არის მოხსენიებული. ESP-IDF ში გამოყენებული სხვა სახის ინსტრუმენტებია:

  • ESP-IDF თან ერთად შეფუთული Python სკრიპტები (მაგალითად, idf.py)
  • PyPI-დან დაინსტალირებული Python-ის პაკეტები.

შემდეგ ნაწილებში ახსნილია ინსტალაციის მეთოდი და მოცემულია თითოეულ პლატფორმაზე დაინსტალირებული ინსტრუმენტების სია.

შენიშვნა

ეს დოკუმენტი განკუთვნილია მოწინავე მომხმარებლებისთვის, რომლებსაც სჭირდებათ ინსტალაციის პერსონალიზება, მომხმარებლებისთვის, რომელთაც სურთ ინსტალაციის პროცესის გაგება და ESP-IDF დეველოპერებისთვის.

თუ ხელსაწყოების დაყენების ინსტრუქციებს ეძებთ, იხილეთ დაწყების სახელმძღვანელო.

ხელსაწყოების მეტამონაცემების ფაილი¶

თითოეული პლატფორმისთვის საჭირო ხელსაწყოებისა და ვერსიების სია მოცემულია შემდეგ ბმულზე: ინსტრუმენტები/ინსტრუმენტები.jsonამ ფაილის სქემა განისაზღვრება შემდეგით: ინსტრუმენტები/tools_schema.json.

ეს ფაილი გამოიყენება ინსტრუმენტები/idf_tools.py სკრიპტი ინსტრუმენტების ინსტალაციის ან გარემოს ცვლადების დაყენებისას.

ინსტრუმენტების ინსტალაციის დირექტორია¶

IDF_TOOLS_PATH გარემოს ცვლადი განსაზღვრავს ადგილმდებარეობას, სადაც ინსტრუმენტები უნდა ჩამოიტვირთოს და დაინსტალირდეს. თუ არ არის დაყენებული, IDF_TOOLS_PATH ნაგულისხმევად HOME/.espressif Linux-სა და macOS-ზე, და %USER_PROFILE%\.espressif Windows-ზე.

შიგნით IDF_TOOLS_PATH, ინსტრუმენტების ინსტალაციის შემსრულებელი სკრიპტები ქმნიან შემდეგ დირექტორიებს:

  • dist — სადაც ინსტრუმენტების არქივები იტვირთება.
  • tools — სადაც ინსტრუმენტები ამოღებულია. ინსტრუმენტები ამოღებულია ქვედირექტორიებში: tools/TOOL_NAME/VERSION/ეს განლაგება საშუალებას იძლევა, რომ ხელსაწყოების სხვადასხვა ვერსია გვერდიგვერდ დამონტაჟდეს.

idf_tools.py სცენარი¶

ინსტრუმენტები/idf_tools.py ESP-IDF თან ერთად შეფუთული სკრიპტი ასრულებს რამდენიმე ფუნქციას:

  • install: ჩამოტვირთეთ ინსტრუმენტი ${IDF_TOOLS_PATH}/dist დირექტორიაში, ამოიღეთ იგი ${IDF_TOOLS_PATH}/tools/TOOL_NAME/VERSION.

    install ბრძანება იღებს ინსტალაციისთვის საჭირო ინსტრუმენტების სიას, TOOL_NAME ან TOOL_NAME@VERSION ფორმატი. თუ all მოცემულია, ყველა ინსტრუმენტი (სავალდებულო და არასავალდებულო) დაინსტალირებულია. თუ არგუმენტი ან required მოცემულია, დამონტაჟებულია მხოლოდ საჭირო ხელსაწყოები.

  • download: მსგავსი install მაგრამ არ იღებს ხელსაწყოებს. სურვილისამებრ --platform არგუმენტი შეიძლება გამოყენებულ იქნას კონკრეტული პლატფორმისთვის ინსტრუმენტების ჩამოსატვირთად.

  • export: ჩამოთვლის გარემოს ცვლადებს, რომელთა დაყენებაც საჭიროა დაინსტალირებული ხელსაწყოების გამოსაყენებლად. ხელსაწყოების უმეტესობისთვის, დაყენება PATH გარემოს ცვლადი საკმარისია, მაგრამ ზოგიერთ ინსტრუმენტს დამატებითი გარემოს ცვლადები სჭირდება.

    გარემოს ცვლადები შეიძლება ჩამოთვლილი იყოს შემდეგიდან რომელიმეში: shell ან key-value ფორმატები, დადგენილი --format პარამეტრი:

    • shell აწარმოებს შელიში შეფასებისთვის შესაფერის შედეგს. მაგალითად,

      export PATH="/home/user/.espressif/tools/tool/v1.0.0/bin:$PATH"

      Linux-სა და macOS-ზე, და

      set "PATH=C:\Users\user\.espressif\tools\v1.0.0\bin;%PATH%"

      Windows-ზე.

      შენიშვნა

      გარემოს ცვლადების Powershell ფორმატში ექსპორტი ამჟამად არ არის მხარდაჭერილი. key-value ფორმატის გამოყენება შესაძლებელია მის ნაცვლად.

      ამ ბრძანების გამომავალი შეიძლება გამოყენებულ იქნას გარემოს ცვლადების განახლებისთვის, თუ shell მხარს უჭერს ამას. მაგალითად:

      eval $($IDF_PATH/tools/idf_tools.py export)

    • key-value აწარმოებს გამომავალს VARIABLE=VALUE ფორმატი, შესაფერისი სხვა სკრიპტებით დამუშავებისთვის:

      PATH=/home/user/.espressif/tools/tool/v1.0.0:$PATH

      გაითვალისწინეთ, რომ ამ გამომავალს მომხმარი სკრიპტი უნდა ასრულებდეს გაფართოებას $VAR ან %VAR% გამომავალში ნაპოვნი ნიმუშები.

  • list: აჩვენებს ინსტრუმენტების ცნობილ ვერსიებს და მიუთითებს, რომელია დაინსტალირებული.

  • checkთითოეული ხელსაწყოსთვის ამოწმებს, ხელმისაწვდომია თუ არა ინსტრუმენტი სისტემის გზაზე და IDF_TOOLS_PATH.

სკრიპტების ინსტალაცია¶

ESP-IDF რეპოზიტორიის root-ში მოცემულია shell-სპეციფიკური მომხმარებლისთვის განკუთვნილი სკრიპტები, რაც ხელსაწყოების ინსტალაციას უწყობს ხელს. ესენია:

  • install.bat Windows-ის ბრძანების ხაზისთვის
  • install.ps1 Powershell-ისთვის
  • install.sh Bash-ისთვის

ინსტრუმენტების ჩამოტვირთვისა და ინსტალაციის გარდა, IDF_TOOLS_PATH, ეს სკრიპტები ამზადებენ Python-ის ვირტუალურ გარემოს და აინსტალირებენ საჭირო პაკეტებს ამ გარემოში.

სკრიპტების ექსპორტი¶

რადგან დაინსტალირებული ინსტრუმენტები მუდმივად არ არის დამატებული მომხმარებელში ან სისტემაში PATH გარემოს ცვლადის გამოყენებისას, ბრძანების ხაზში მათი გამოსაყენებლად დამატებითი ნაბიჯია საჭირო. შემდეგი სკრიპტები ცვლის მიმდინარე გარსის გარემოს ცვლადებს, რათა ხელმისაწვდომი იყოს ინსტრუმენტების სწორი ვერსიები:

  • export.bat Windows-ის ბრძანების ხაზისთვის
  • export.ps1 Powershell-ისთვის
  • export.sh Bash-ისთვის

შენიშვნა

Bash-ში shell გარემოს შესაცვლელად, export.sh უნდა იყოს „წყაროებით მოძიებული“: . ./export.sh (ყურადღება მიაქციეთ წამყვან წერტილს და ინტერვალს).

export.sh შეიძლება გამოყენებულ იქნას Bash-ის გარდა სხვა shell-ებთან ერთად (მაგალითად, zsh). თუმცა, ამ შემთხვევაში IDF_PATH სკრიპტის გაშვებამდე უნდა დაყენდეს გარემოს ცვლადი. Bash-ში გამოყენებისას, სკრიპტი გამოიცნობს IDF_PATH ღირებულება საკუთარი მდებარეობიდან.

დარეკვის გარდა idf_tools.py, ეს სკრიპტები ჩამოთვლის დირექტორიებს, რომლებიც დაემატა PATH.

სხვა ინსტალაციის მეთოდები¶

გარემოდან გამომდინარე, უფრო მოსახერხებელი შესაფუთი საშუალებები idf_tools.py მოწოდებულია:

  • IDF Tools-ის ინსტალერი Windows-ისთვის შეუძლია ინსტრუმენტების ჩამოტვირთვა და ინსტალაცია. ინსტალერი შიდა რეჟიმში იყენებს idf_tools.py.
  • Eclipse-ის მოდული ESP-IDF სთვის მოიცავს მენიუს ელემენტს ხელსაწყოების დასაყენებლად. შიდა რეჟიმში, დანამატი იძახებს idf_tools.py.
  • Visual Studio-ს კოდის გაფართოება ESP-IDF ისთვის მოიცავს ინტეგრაციის ნაკადს. ეს ნაკადი ხელს უწყობს ინსტრუმენტების დაყენებას. თუმცა გაფართოება არ არის დამოკიდებული idf_tools.py, იგივე ინსტალაციის მეთოდი გამოიყენება.

ინდივიდუალური ინსტალაცია¶

მიუხედავად იმისა, რომ ზემოთ ჩამოთვლილი მეთოდები რეკომენდებულია ESP-IDF მომხმარებლებისთვის, ისინი არ არის აუცილებელი ESP-IDF აპლიკაციების შესაქმნელად. ESP-IDF შექმნის სისტემა ელის, რომ ყველა საჭირო ინსტრუმენტი სადმე იქნება დაინსტალირებული და ხელმისაწვდომი იქნება... PATH.

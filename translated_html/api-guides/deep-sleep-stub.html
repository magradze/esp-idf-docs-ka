ღრმა ძილის გაღვიძების სტატიები¶

ESP32 ღრმა ძილიდან გამოსვლისას მხარს უჭერს „ღრმა ძილის გამოღვიძების სტაბის“ გაშვებას. ეს ფუნქცია მაშინვე ირთვება, როგორც კი ჩიპი გაიღვიძებს - ნებისმიერი ნორმალური ინიციალიზაციის, ჩამტვირთავი ან ESP-IDF კოდის გაშვებამდე. გამოღვიძების სტაბის გაშვების შემდეგ, SoC-ს შეუძლია დაუბრუნდეს ძილის რეჟიმს ან გააგრძელოს ESP-IDF ჩვეულებრივად გაშვება.

ღრმა ძილის დროს გაღვიძების სტუბ კოდი იტვირთება „RTC სწრაფ მეხსიერებაში“ და მის მიერ გამოყენებული ნებისმიერი მონაცემი ასევე უნდა ჩაიტვირთოს RTC მეხსიერებაში. RTC მეხსიერების რეგიონები ინახავენ თავიანთ შინაარსს ღრმა ძილის დროს.

Wake Stubs-ის წესები¶

Wake stub კოდი ფრთხილად უნდა დაიწეროს:

  • რადგანაც SoC ახლახან გამოვიდა ძილის რეჟიმიდან, პერიფერიული მოწყობილობების უმეტესობა გადატვირთვის მდგომარეობაშია. SPI ფლეშ მეხსიერება არ არის მიმაგრებული.
  • Wake stub კოდს მხოლოდ ROM-ში დანერგილი ან RTC Fast Memory-ში ჩატვირთული ფუნქციების გამოძახება შეუძლია (იხილეთ ქვემოთ).
  • გაღვიძების სტუბის კოდს მხოლოდ RTC მეხსიერებაში ჩატვირთულ მონაცემებზე წვდომა შეუძლია. ყველა სხვა ოპერატიული მეხსიერება არ იქნება ინიციალიზებული და მას შემთხვევითი შინაარსი ექნება. გაღვიძების სტუბს შეუძლია სხვა ოპერატიული მეხსიერების გამოყენება დროებითი შენახვისთვის, მაგრამ შინაარსი გადაიწერება, როდესაც SoC ისევ ძილის რეჟიმში გადავა ან ESP-IDF დაიწყებს.
  • RTC მეხსიერება უნდა შეიცავდეს stub-ის მიერ გამოყენებულ ნებისმიერ მხოლოდ წასაკითხ მონაცემებს (.rodata).
  • RTC მეხსიერებაში არსებული მონაცემები ინიციალიზაციას განიცდის SoC-ის გადატვირთვისას, გარდა ღრმა ძილიდან გამოღვიძების შემთხვევებისა. ღრმა ძილიდან გამოღვიძებისას, დაძინებამდე არსებული მნიშვნელობები შენარჩუნებულია.
  • Wake stub კოდი esp-idf აპლიკაციის ძირითადი ნაწილია. esp-idf-ის ნორმალური გაშვებისას, ფუნქციებს შეუძლიათ Wake stub ფუნქციების გამოძახება ან RTC მეხსიერებაზე წვდომა. თითქოს ესენი აპლიკაციის ჩვეულებრივი ნაწილები იყვნენ.

Stub-ის იმპლემენტაცია¶

esp-idf-ში Wake stub-ს ეწოდება esp_wake_deep_sleep()ეს ფუნქცია მუშაობს ყოველთვის, როდესაც SoC ღრმა ძილის რეჟიმიდან გამოდის. esp-idf-ში მოცემულია ამ ფუნქციის ნაგულისხმევი ვერსია, მაგრამ ნაგულისხმევი ფუნქცია სუსტი კავშირის მქონეა, ამიტომ თუ თქვენი აპლიკაცია შეიცავს ფუნქციას სახელწოდებით esp_wake_deep_sleep() შემდეგ ეს გადაფარავს ნაგულისხმევ პარამეტრს.

თუ მორგებულ გაღვიძების სტატუსს აწვდით, პირველი, რასაც ის გააკეთებს, არის დარეკვა. esp_default_wake_deep_sleep().

მისი განხორციელება აუცილებელი არ არის esp_wake_deep_sleep() ღრმა ძილის რეჟიმის გამოსაყენებლად თქვენს აპლიკაციაში. ეს მხოლოდ იმ შემთხვევაშია საჭირო, თუ გსურთ, რომ გაღვიძებისთანავე განსაკუთრებული ქცევა გქონდეთ.

თუ გსურთ გაშვების დროს სხვადასხვა ღრმა ძილის ფაილებს შორის გადართვა, ამის გაკეთება ასევე შესაძლებელია დარეკვით. esp_set_deep_sleep_wake_stub() ფუნქცია. ეს აუცილებელი არ არის, თუ მხოლოდ ნაგულისხმევ ფუნქციას იყენებთ esp_wake_deep_sleep() ფუნქცია.

ყველა ეს ფუნქცია გამოცხადებულია esp_deepsleep.h სათაური კომპონენტი s/esp32-ის ქვეშ.

კოდის RTC მეხსიერებაში ჩატვირთვა¶

გაღვიძების stub კოდი უნდა იყოს RTC Fast Memory-ში. ეს შეიძლება გაკეთდეს ორიდან ერთ-ერთი გზით.

პირველი გზა არის გამოყენება RTC_IRAM_ATTR ატრიბუტი ფუნქციის RTC მეხსიერებაში განსათავსებლად:

void RTC_IRAM_ATTR esp_wake_deep_sleep(void) {
    esp_default_wake_deep_sleep();
    // Add additional functionality here
}

მეორე გზაა ფუნქციის განთავსება ნებისმიერ საწყის ფაილში, რომლის სახელიც იწყება rtc_wake_stubფაილების სახელები rtc_wake_stub* მათი შინაარსი ლინკერის მიერ ავტომატურად შეინახება RTC მეხსიერებაში.

პირველი გზა უფრო მარტივია ძალიან მოკლე და მარტივი კოდისთვის, ან საწყისი ფაილებისთვის, სადაც გსურთ „ჩვეულებრივი“ და „RTC“ კოდის შერევა. მეორე გზა უფრო მარტივია, როდესაც გსურთ RTC მეხსიერებისთვის კოდის უფრო გრძელი ნაწილების დაწერა.

მონაცემების RTC მეხსიერებაში ჩატვირთვა¶

stub კოდის მიერ გამოყენებული მონაცემები უნდა იყოს RTC მეხსიერებაში. მონაცემები შეიძლება განთავსდეს RTC Fast მეხსიერებაში ან RTC Slow მეხსიერებაში, რომელსაც ასევე იყენებს ULP.

ამ მონაცემების მითითება შესაძლებელია ორიდან ერთი გზით:

პირველი გზა არის გამოყენება RTC_DATA_ATTR და RTC_RODATA_ATTR RTC მეხსიერებაში ჩასატვირთი ნებისმიერი მონაცემის მითითებისთვის (შესაბამისად, ჩასაწერი ან მხოლოდ წასაკითხი):

RTC_DATA_ATTR int wake_count;

void RTC_IRAM_ATTR esp_wake_deep_sleep(void) {
    esp_default_wake_deep_sleep();
    static RTC_RODATA_ATTR const char fmt_str[] = "Wake count %d\n";
    ets_printf(fmt_str, wake_count++);
}

RTC მეხსიერების არე, სადაც ეს მონაცემები განთავსდება, შეიძლება კონფიგურირებული იყოს menuconfig ოფციის მეშვეობით, სახელწოდებით CONFIG_ESP32_RTCDATA_IN_FAST_MEMეს პარამეტრი საშუალებას იძლევა ULP პროგრამებისთვის შეინარჩუნოთ ნელი მეხსიერების არეალი და მისი ჩართვის შემდეგ მონაცემები მონიშნულია . RTC_DATA_ATTR და RTC_RODATA_ATTR მოთავსებულია RTC სწრაფი მეხსიერების სეგმენტში, წინააღმდეგ შემთხვევაში ის გადადის RTC ნელ მეხსიერებაში (ნაგულისხმევი ვარიანტი). ეს ვარიანტი დამოკიდებულია CONFIG_FREERTOS_UNICORE რადგან RTC სწრაფ მეხსიერებაზე წვდომა მხოლოდ PRO_CPU-ს შეუძლია.

მსგავსი ატრიბუტები RTC_FAST_ATTR და RTC_SLOW_ATTR შეიძლება გამოყენებულ იქნას მონაცემების მითითებისთვის, რომლებიც იძულებით განთავსდება შესაბამისად RTC_FAST და RTC_SLOW მეხსიერებაში. ნებისმიერი წვდომა მონაცემებზე, რომლებიც მონიშნულია RTC_FAST_ATTR დაშვებულია მხოლოდ PRO_CPU-ს მიერ და მომხმარებლის პასუხისმგებლობაა ამის დადასტურება.

სამწუხაროდ, ამ გზით გამოყენებული ნებისმიერი სტრიქონული კონსტანტა უნდა გამოცხადდეს მასივებად და მონიშნული იყოს RTC_RODATA_ATTR-ით, როგორც ეს ზემოთ მოცემულ მაგალითშია ნაჩვენები.

მეორე გზაა მონაცემების განთავსება ნებისმიერ საწყის ფაილში, რომლის სახელიც იწყება rtc_wake_stub.

მაგალითად, ექვივალენტური მაგალითია rtc_wake_stub_counter.c:

int wake_count;

void RTC_IRAM_ATTR esp_wake_deep_sleep(void) {
    esp_default_wake_deep_sleep();
    ets_printf("Wake count %d\n", wake_count++);
}

მეორე გზა უკეთესი ვარიანტია, თუ გჭირდებათ სტრიქონების გამოყენება ან სხვა, უფრო რთული კოდის დაწერა.

გაღვიძების დროის შესამცირებლად გამოიყენეთ CONFIG_BOOTLOADER_SKIP_VALIDATE_IN_DEEP_SLEEP Kconfig ვარიანტი, დამატებითი ინფორმაციისთვის იხილეთ სწრაფი ჩატვირთვა ღრმა ძილისგან.

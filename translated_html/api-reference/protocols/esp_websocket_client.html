ESP WebSocket კლიენტი¶

მიმოხილვა¶

ESP WebSocket კლიენტი არის იმპლემენტაცია WebSocket პროტოკოლის კლიენტი ESP32-ისთვის

მახასიათებლები¶

  • მხარს უჭერს WebSocket-ს TCP-ზე, SSL-ს mbedtls-ით
  • მარტივი დაყენება URI-ით
  • მრავალი ეგზემპლარი (მრავალი კლიენტი ერთ აპლიკაციაში)

კონფიგურაცია¶

URI¶

  • მხარს უჭერს ws, wss სქემები
  • WebSocket-ის ნიმუშები:
    • ws://websocket.org: WebSocket TCP-ის საშუალებით, ნაგულისხმევი პორტი 80
    • wss://websocket.org: WebSocket SSL-ის საშუალებით, ნაგულისხმევი პორტი 443
  • მინიმალური კონფიგურაციები:
const esp_websocket_client_config_t ws_cfg = {
    .uri = "ws://websocket.org",
};
  • თუ არსებობს URI-სთან დაკავშირებული რაიმე ვარიანტი esp_websocket_client_config_t, URI-ით განსაზღვრული ოფცია გადაიწერება. ნიმუში:
const esp_websocket_client_config_t ws_cfg = {
    .uri = "ws://websocket.org:123",
    .port = 4567,
};
//WebSocket client will connect to websocket.org using port 4567

SSL¶

  • მიიღეთ სერტიფიკატი სერვერიდან, მაგალითად: websocket.org openssl s_client -showcerts -connect websocket.org:443 </dev/null 2>/dev/null|openssl x509 -outform PEM >websocket_org.pem
  • კონფიგურაცია:
const esp_websocket_client_config_t ws_cfg = {
    .uri = "wss://websocket.org",
    .cert_pem = (const char *)websocket_org_pem_start,
};

მეტი ვარიანტისთვის esp_websocket_client_config_t, გთხოვთ, იხილოთ ქვემოთ მოცემული API ბმული

განაცხადის მაგალითი¶

მარტივი WebSocket მაგალითი, რომელიც იყენებს esp_websocket_client-ს websocket კავშირის დასამყარებლად და მონაცემების გასაგზავნად/მისაღებად შემდეგით: websocket.org სერვერი: პროტოკოლები/ვებ-სოკეტი.

esp_websocket_client_handle_t esp_websocket_client_init(კონსტ esp_websocket_client_config_t *კონფიგურაცია)¶

Websocket სესიის დაწყება ეს ფუნქცია უნდა იყოს პირველი გამოძახებული ფუნქცია და ის აბრუნებს esp_websocket_client_handle_t მნიშვნელობას, რომელიც უნდა გამოიყენოთ ინტერფეისში სხვა ფუნქციებისთვის შეყვანის სახით. ოპერაციის დასრულების შემდეგ, ამ გამოძახებას უნდა ჰქონდეს შესაბამისი გამოძახება esp_websocket_client_destroy-ზე.

დაბრუნება
  • esp_websocket_client_handle_t
  • NULL, თუ შეცდომები არსებობს
პარამეტრები
  • configკონფიგურაცია

esp_err_t esp_websocket_client_destroy(esp_websocket_client_handle_t კლიენტი)¶

გაანადგურეთ WebSocket კავშირი და გაათავისუფლეთ ყველა რესურსი. ეს ფუნქცია უნდა იყოს ბოლო ფუნქცია, რომელიც გამოიძახებს სესიას. ის esp_websocket_client_init ფუნქციის საპირისპიროა და უნდა გამოიძახოთ იმავე სახელურით, როგორც შეყვანის სახელი, რომელიც დააბრუნა esp_websocket_client_init გამოძახებით. ამან შეიძლება დახუროს ყველა კავშირი, რომელიც ამ სახელურმა გამოიყენა.

  • clientკლიენტი

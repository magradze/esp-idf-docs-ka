ESP HTTP კლიენტი¶

მიმოხილვა¶

esp_http_client ESP-IDF პროგრამებიდან HTTP/S მოთხოვნების განსახორციელებლად API იძლევა. HTTP მოთხოვნისთვის ამ API გამოყენების ნაბიჯებია:

  • esp_http_client_init()HTTP კლიენტის გამოსაყენებლად, პირველი, რაც უნდა გავაკეთოთ, არის შექმნათ esp_http_client ამ ფუნქციაში გადაცემით esp_http_client_config_t კონფიგურაციები. რომელ კონფიგურაციის მნიშვნელობებსაც ჩვენ არ განვსაზღვრავთ, ბიბლიოთეკა გამოიყენებს ნაგულისხმევს.
  • esp_http_client_perform(): ის esp_http_client საჭიროა init ფუნქციიდან შექმნილი არგუმენტი. ეს ფუნქცია ასრულებს esp_http_client-ის ყველა ოპერაციას, კავშირის გახსნიდან დაწყებული მონაცემების გაგზავნით, მონაცემების ჩამოტვირთვით და საჭიროების შემთხვევაში კავშირის დახურვით. ყველა დაკავშირებული მოვლენა გამოიძახება event_handle-ში (განსაზღვრულია esp_http_client_config_t). ეს ფუნქცია ასრულებს თავის დავალებას და ბლოკავს მიმდინარე დავალებას მის დასრულებამდე.
  • esp_http_client_cleanup(): ჩვენი დასრულების შემდეგ esp_http_კლიენტის დავალების შესრულებისას, ეს არის ბოლო ფუნქცია, რომელიც უნდა გამოიძახოთ. ის დახურავს კავშირს (ასეთის არსებობის შემთხვევაში) და გაათავისუფლებს HTTP კლიენტისთვის გამოყოფილ მთელ მეხსიერებას.

განაცხადის მაგალითი¶

esp_err_t _http_event_handle(esp_http_client_event_t *evt)
{
    switch(evt->event_id) {
        case HTTP_EVENT_ERROR:
            ESP_LOGI(TAG, "HTTP_EVENT_ERROR");
            break;
        case HTTP_EVENT_ON_CONNECTED:
            ESP_LOGI(TAG, "HTTP_EVENT_ON_CONNECTED");
            break;
        case HTTP_EVENT_HEADER_SENT:
            ESP_LOGI(TAG, "HTTP_EVENT_HEADER_SENT");
            break;
        case HTTP_EVENT_ON_HEADER:
            ESP_LOGI(TAG, "HTTP_EVENT_ON_HEADER");
            printf("%.*s", evt->data_len, (char*)evt->data);
            break;
        case HTTP_EVENT_ON_DATA:
            ESP_LOGI(TAG, "HTTP_EVENT_ON_DATA, len=%d", evt->data_len);
            if (!esp_http_client_is_chunked_response(evt->client)) {
                printf("%.*s", evt->data_len, (char*)evt->data);
            }

            break;
        case HTTP_EVENT_ON_FINISH:
            ESP_LOGI(TAG, "HTTP_EVENT_ON_FINISH");
            break;
        case HTTP_EVENT_DISCONNECTED:
            ESP_LOGI(TAG, "HTTP_EVENT_DISCONNECTED");
            break;
    }
    return ESP_OK;
}

esp_http_client_config_t config = {
   .url = "http://httpbin.org/redirect/2",
   .event_handler = _http_event_handle,
};
esp_http_client_handle_t client = esp_http_client_init(&config);
esp_err_t err = esp_http_client_perform(client);

if (err == ESP_OK) {
   ESP_LOGI(TAG, "Status = %d, content_length = %d",
           esp_http_client_get_status_code(client),
           esp_http_client_get_content_length(client));
}
esp_http_client_cleanup(client);

მუდმივი კავშირები¶

მუდმივი კავშირები ნიშნავს, რომ HTTP კლიენტს შეუძლია იგივე კავშირის ხელახლა გამოყენება რამდენიმე გადაცემისთვის. თუ სერვერი არ მოითხოვს კავშირის დახურვას Connection: close სათაური, ახალი გადაცემა ნიმუშის IP მისამართით, პორტით და პროტოკოლით.

იმისათვის, რომ HTTP კლიენტმა სრულად ისარგებლოს მუდმივი კავშირებით, რაც შეიძლება მეტი ფაილის გადაცემა ერთი და იგივე სახელურის გამოყენებით უნდა განახორციელოთ.

მუდმივი კავშირების მაგალითი¶

esp_err_t err;
esp_http_client_config_t config = {
    .url = "http://httpbin.org/get",
};
esp_http_client_handle_t client = esp_http_client_init(&config);
// first request
err = esp_http_client_perform(client);

// second request
esp_http_client_set_url(client, "http://httpbin.org/anything")
esp_http_client_set_method(client, HTTP_METHOD_DELETE);
esp_http_client_set_header(client, "HeaderKey", "HeaderValue");
err = esp_http_client_perform(client);

esp_http_client_cleanup(client);

HTTPS¶

HTTP კლიენტი მხარს უჭერს SSL კავშირებს mbedtls, -თან ერთად URL კონფიგურაცია, რომელიც იწყება https სქემა (ან transport_type = HTTP_TRANSPORT_OVER_SSL). HTTPS მხარდაჭერის კონფიგურაცია შესაძლებელია :ref:CONFIG_ENABLE_HTTPS-ის მეშვეობით (ჩართულია ნაგულისხმევად)..

შენიშვნა

HTTPS-ის გამოყენებით ინფორმაციის მიწოდებით, ბიბლიოთეკა სერვერთან დასაკავშირებლად SSL ტრანსპორტის ტიპს გამოიყენებს. თუ გსურთ სერვერის დადასტურება, მაშინ საჭიროა დამატებითი სერტიფიკატის მიწოდება PEM ფორმატში და cert_pem ში esp_http_client_config_t

HTTPS-ის მაგალითი¶

static void https()
{
    esp_http_client_config_t config = {
        .url = "https://www.howsmyssl.com",
        .cert_pem = howsmyssl_com_root_cert_pem_start,
    };
    esp_http_client_handle_t client = esp_http_client_init(&config);
    esp_err_t err = esp_http_client_perform(client);

    if (err == ESP_OK) {
        ESP_LOGI(TAG, "Status = %d, content_length = %d",
                esp_http_client_get_status_code(client),
                esp_http_client_get_content_length(client));
    }
    esp_http_client_cleanup(client);
}

HTTP ნაკადი¶

ზოგიერთ აპლიკაციას სჭირდება კავშირის გახსნა და მონაცემების წაკითხვის აქტიური კონტროლი. HTTP კლიენტი მხარს უჭერს ზოგიერთ ფუნქციას ამის გასაადვილებლად, რა თქმა უნდა, ამ ფუნქციების გამოყენების შემდეგ არ უნდა გამოიყენოთ esp_http_client_perform() ფუნქცია ამ სახელურით და esp_http_client_init() სახელურის მისაღებად ყოველთვის პირველი უნდა გამოიძახოთ. შეასრულეთ ეს ფუნქციები ქვემოთ მოცემული თანმიმდევრობით:

  • esp_http_client_init(): შექმნა და დამუშავება
  • esp_http_client_set_* ან esp_http_client_delete_*: http კავშირის ინფორმაციის შესაცვლელად (არასავალდებულო)
  • esp_http_client_open()გახსენით http კავშირი write_len პარამეტრი, write_len=0 თუ მხოლოდ წაკითხვა გვჭირდება
  • esp_http_client_write(): მონაცემების ატვირთვა, მაქსიმალური სიგრძე ტოლია write_len -ის esp_http_client_open() ფუნქცია. შესაძლოა მისი გამოძახება არ დაგვჭირდეს, თუ write_len=0
  • esp_http_client_fetch_headers()სათაურების სერვერზე გაგზავნისა და მონაცემების (ასეთის არსებობის შემთხვევაში) ჩაწერის შემდეგ, ეს ფუნქცია წაიკითხავს HTTP სერვერის პასუხის სათაურებს. ამ ფუნქციის გამოძახება დააბრუნებს content-length სერვერიდან და შეგვიძლია დავრეკოთ esp_http_client_get_status_code() კავშირის HTTP სტატუსისთვის.
  • esp_http_client_read()ახლა, ამ ფუნქციით შეგვიძლია HTTP ნაკადის წაკითხვა.
  • esp_http_client_close(): დასრულების შემდეგ უნდა დავამყაროთ კავშირი
  • esp_http_client_cleanup()და გაათავისუფლეთ რესურსები

HTTP მოთხოვნის შესრულება, როგორც ნაკადის წამკითხველი¶

შეამოწმეთ ფუნქციის მაგალითი http_perform_as_stream_reader დროს პროტოკოლები/esp_http_კლიენტი.

HTTP ავთენტიფიკაცია¶

HTTP კლიენტი მხარს უჭერს ორივეს ძირითადი და დაიჯესტი ავთენტიფიკაცია. მომხმარებლის სახელებისა და პაროლების მითითებით url ან username, password კონფიგურაციის ჩანაწერის. და auth_type = HTTP_AUTH_TYPE_BASIC, HTTP კლიენტს ავტორიზაციის პროცესის გასავლელად მხოლოდ 1 შესრულება სჭირდება. თუ auth_type = HTTP_AUTH_TYPE_NONE, მაგრამ არსებობენ username და password კონფიგურაციაში HTTP კლიენტი ასრულებს 2 შესრულებას. პირველად ის უკავშირდება სერვერს და იღებს არაავტორიზებული სათაურის ველს. ამ ინფორმაციის საფუძველზე, ის იცის, რომელი ავტორიზაციის მეთოდი აირჩიოს და შეასრულებს მას მეორე ჯერზე.

URI-ით ავტორიზაციის კონფიგურაციის მაგალითი¶

esp_http_client_config_t config = {
    .url = "http://user:[email protected]/basic-auth/user/passwd",
    .auth_type = HTTP_AUTH_TYPE_BASIC,
};

კონფიგურაციის ავტორიზაციის მაგალითი მომხმარებლის სახელით და პაროლის შეყვანით¶

esp_http_client_config_t config = {
    .url = "http://httpbin.org/basic-auth/user/passwd",
    .username = "user",
    .password = "passwd",
    .auth_type = HTTP_AUTH_TYPE_BASIC,
};

HTTP კლიენტის მაგალითი: პროტოკოლები/esp_http_კლიენტი.

esp_http_client_handle_t esp_http_client_init(კონსტ esp_http_client_config_t *კონფიგურაცია)¶

HTTP სესიის დაწყება ეს ფუნქცია უნდა იყოს პირველი გამოძახებული ფუნქცია და ის აბრუნებს esp_http_client_handle_t-ს, რომელიც უნდა გამოიყენოთ ინტერფეისში სხვა ფუნქციებისთვის შეყვანის სახით. ოპერაციის დასრულების შემდეგ, ამ გამოძახებას უნდა ჰქონდეს შესაბამისი გამოძახება esp_http_client_cleanup-ზე.

პარამეტრები

esp_err_t esp_http_client_perform(esp_http_client_handle_t კლიენტი)¶

ამ ფუნქციის გამოძახების შემდეგ esp_http_client_init და ყველა ოფციის გამოძახება განხორციელდება და გადაცემა შესრულდება ოფციებში აღწერილი წესით. ის უნდა გამოიძახოს იმავე esp_http_client_handle_t შეყვანით, როგორც esp_http_client_init გამოძახებისას. esp_http_client_perform ასრულებს მთელ მოთხოვნას დაბლოკვით ან არადაბლოკვით. ნაგულისხმევად, API ასრულებს მოთხოვნას დაბლოკვით და აბრუნებს დასრულების შემდეგ, ან თუ ის ვერ მოხერხდა, და არადაბლოკვით, ის აბრუნებს, თუ გვხვდება EAGAIN/EWOULDBLOCK ან EINPROGRESS, ან თუ ის ვერ მოხერხდა. ხოლო არადაბლოკვითი მოთხოვნის შემთხვევაში, მომხმარებელს შეუძლია გამოიძახოს ეს API რამდენჯერმე, თუ მოთხოვნა და პასუხი არ არის დასრულებული ან არ არის შეცდომა. არადაბლოკვითი esp_http_client_perform()-ის ჩასართავად, is_async წევრი esp_http_client_config_t უნდა დაყენდეს esp_http_client_init() API გამოძახებისას. თქვენ შეგიძლიათ განახორციელოთ ნებისმიერი რაოდენობის გამოძახება esp_http_client_perform-ზე იმავე esp_http_client_handle_t-ის გამოყენებით. ძირითადი კავშირი შეიძლება ღიად დარჩეს, თუ სერვერი ამის საშუალებას იძლევა. თუ აპირებთ ერთზე მეტი ფაილის გადატანას, ამის გაკეთებაც კი რეკომენდებულია. შემდეგ esp_http_client შეეცდება იგივე კავშირის ხელახლა გამოყენებას შემდეგი გადაცემებისთვის, რითაც ოპერაციები უფრო სწრაფი, ნაკლებად დატვირთული და ნაკლები ქსელის რესურსების გამოყენებით მოხდება. უბრალოდ გაითვალისწინეთ, რომ მოგიწევთ გამოიყენოთ esp_http_client_set_** გამოძახებებს შორის შემდეგი esp_http_client_perform-ის პარამეტრების დასაყენებლად.

შენიშვნა
You must never call this function simultaneously from two places using the same client handle. Let the function return first before invoking it another time. If you want parallel transfers, you must use several esp_http_client_handle_t. This function include esp_http_client_open -> esp_http_client_write -> esp_http_client_fetch_headers -> esp_http_client_read (and option) esp_http_client_close.

ეს ფუნქცია უნდა იყოს სესიისთვის გამოძახებული ბოლო ფუნქცია. ის esp_http_client_init ფუნქციის საპირისპიროა და უნდა გამოიძახოს იმავე სახელურით, როგორც შეყვანის შედეგად მიღებული esp_http_client_init გამოძახებით. ამან შეიძლება დახუროს ყველა კავშირი, რომელიც ამ სახელურმა გამოიყენა და შესაძლოა აქამდე ღია შეინარჩუნა. არ გამოიძახოთ ეს ფუნქცია, თუ მეტი ფაილის გადატანას აპირებთ, სახელურების ხელახლა გამოყენება esp_http_client-ის კარგი მუშაობის გასაღებია.

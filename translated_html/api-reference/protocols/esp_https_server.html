HTTPS სერვერი¶

მიმოხილვა¶

ეს კომპონენტი აგებულია შემდეგზე: esp_http_serverHTTPS სერვერი იყენებს ჩვეულებრივ HTTP სერვერში არსებულ ჰუკებსა და ფუნქციების გადაფარვას, რათა უზრუნველყოს დაშიფვრა OpenSSL-ის გამოყენებით.

ყველა დოკუმენტაცია esp_http_server ეს ასევე ეხება თქვენს მიერ ამ გზით შექმნილ სერვერს.

გამოყენებული API წმ¶

შემდეგი API 8 esp_http_server არ უნდა იქნას გამოყენებული esp_https_server, რადგან ისინი გამოიყენება შიდა სესიების მართვისა და შიდა მდგომარეობის შესანარჩუნებლად:

  • „გაგზავნა“, „მიღება“ და „მოლოდინში მყოფი“ ფუნქციების გადაფარვა - უსაფრთხო სოკეტის დამუშავება
    • httpd_sess_set_send_override()
    • httpd_sess_set_recv_override()
    • httpd_sess_set_pending_override()
  • „ტრანსპორტის კონტექსტი“ - როგორც გლობალური, ასევე სესიის
    • httpd_sess_get_transport_ctx() - აბრუნებს სესიისთვის გამოყენებულ SSL-ს
    • httpd_sess_set_transport_ctx()
    • httpd_get_global_transport_ctx() - აბრუნებს გაზიარებულ SSL კონტექსტს
    • httpd_config_t.global_transport_ctx
    • httpd_config_t.global_transport_ctx_free_fn
    • httpd_config_t.open_fn - გამოიყენება უსაფრთხო სოკეტების დასაყენებლად

ყველაფერი დანარჩენი შეიძლება გამოყენებულ იქნას შეზღუდვების გარეშე.

გამოყენება¶

გთხოვთ, იხილოთ მაგალითი პროტოკოლები/https_სერვერი უსაფრთხო სერვერის დაყენების შესასწავლად.

არსებითად, თქვენ მხოლოდ სერტიფიკატის გენერირება, მისი პროგრამული უზრუნველყოფა ში ჩასმა და მისი მაჩვენებლებისა და სიგრძის start ფუნქციას init სტრუქტურის მეშვეობით მიაწოდოთ.

სერვერის გაშვება შესაძლებელია SSL-ით ან მის გარეშე init სტრუქტურაში დროშის შეცვლით - httpd_ssl_config.transport_modeეს შეიძლება გამოყენებულ იქნას, მაგალითად, ტესტირებისთვის ან სანდო გარემოში, სადაც უსაფრთხოებაზე მეტად სიჩქარეს ანიჭებთ უპირატესობას.

შესრულება¶

საწყისი სესიის დაყენებას შეიძლება დაახლოებით ორი წამი ან მეტი დასჭირდეს უფრო ნელი საათის სიჩქარით ან უფრო დეტალური ჟურნალირებით. შემდგომი მოთხოვნები ღია უსაფრთხო სოკეტის მეშვეობით გაცილებით სწრაფია (100 მილიწამზე ნაკლებ დროში).

მაკროები¶

HTTPD_SSL_CONFIG_DEFAULT()¶

ნაგულისხმევი კონფიგურაციის სტრუქტურა init

(მორგებისთვის საჭირო იყო http_server-ის ნაგულისხმევი კონფიგურაციის კოპირება)

შენიშვნები:

  • პორტი დაყენებულია სერვერის გაშვებისას, &#39;transport_mode&#39;-ის მიხედვით
  • ერთი სოკეტი იყენებს ~ 40 კბ ოპერატიულ მეხსიერებას SSL-ით, ჩვენ ვამცირებთ სოკეტების ნაგულისხმევ რაოდენობას 4-მდე
  • SSL სოკეტები, როგორც წესი, დიდხანს ძლებს, LRU-ს დახურვა ხელს უშლის პულის ამოწურვას DOS-ში.
  • დასტის ზომას შეიძლება დასჭირდეს კორექტირება მომხმარებლის აპლიკაციის მიხედვით

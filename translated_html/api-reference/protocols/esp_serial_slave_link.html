ESP სერიული მონა ბმული¶

მიმოხილვა¶

Espressif-ი რამდენიმე ჩიპს გვთავაზობს, რომლებსაც შეუძლიათ „დამნაშავე“ ჩიპებად მუშაობა. ეს დამნაშავე მოწყობილობები რამდენიმე საერთო ავტობუსზეა დამოკიდებული და ამ ავტობუსებზე საკუთარი საკომუნიკაციო პროტოკოლები აქვთ. esp_serial_slave_link კომპონენტი შექმნილია იმისთვის, რომ მასტერ-ოპერატორმა დაუკავშირდეს ESP მონა მოწყობილობებს ამ პროტოკოლების მეშვეობით ავტობუსის მძღოლების მეშვეობით.

შემდეგ esp_serial_slave_link მოწყობილობის სწორად ინიციალიზაციის შემდეგ, აპლიკაციას შეუძლია გამოიყენოს იგი ESP მონა მოწყობილობებთან მოსახერხებელი კომუნიკაციისთვის.

ESP32 SDIO მონა პროტოკოლის შესახებ დამატებითი ინფორმაციისთვის იხილეთ დოკუმენტი. კომუნიკაცია ESP SDIO Slave-თან.

ტერმინოლოგია¶

  • ESSL: ESP სერიული მონა-კავშირის აბრევიატურა, კომპონენტი რომელიც აღწერილია ამ დოკუმენტში.

  • მასტერი: მოწყობილობა, რომელიც მუშაობს esp_serial_slave_link კომპონენტი .

  • ESSL მოწყობილობა: ვირტუალური მოწყობილობა მთავარ მოწყობილობაზე, რომელიც დაკავშირებულია ESP დაქვემდებარებულ მოწყობილობასთან. მოწყობილობის კონტექსტი ფლობს ავტობუსზე ზემოთ მდებარე დაქვემდებარებული პროტოკოლის შესახებ ცოდნას და დაქვემდებარებულ მოწყობილობასთან კომუნიკაციისთვის ზოგიერთ ავტობუსის დრაივერს ეყრდნობა.

  • ESSL მოწყობილობის სახელური: ESSL მოწყობილობის კონტექსტის სახელური, რომელიც შეიცავს ESSL კომპონენტი მოთხოვნილ კონფიგურაციას, სტატუსს და მონაცემებს.19 კონტექსტი ინახავს დრაივერის კონფიგურაციებს, კომუნიკაციის მდგომარეობას, მასტერ და მონად მოქმედი მოწყობილობების მიერ გაზიარებულ მონაცემებს და ა.შ.

    კონტექსტი გამოყენებამდე უნდა იყოს ინიციალიზებული, ხოლო თუ აღარ გამოიყენება, უნდა მოხდეს მისი დეინიციალიზაცია. მთავარი აპლიკაცია ESSL მოწყობილობაზე ამ სახელურის მეშვეობით მუშაობს.

  • ESP მონა: მონა მოწყობილობა, რომელიც დაკავშირებულია ავტობუსთან, რომელთანაც ESSL კომპონენტი კომუნიკაციისთვისაა განკუთვნილი.

  • ავტობუსი: ავტობუსი, რომლის მეშვეობითაც ოსტატი და მონა ერთმანეთთან ურთიერთობენ.

  • მონა პროტოკოლი: Espressif HW/SW-ის მიერ ავტობუსის მეშვეობით განსაზღვრული სპეციალური საკომუნიკაციო პროტოკოლი.

  • TX ბუფერის ნომერი: მრიცხველი, რომელიც განთავსებულია მონაზე და მისი წაკითხვა მასტერს შეუძლია, მიუთითებს დაგროვილ ბუფერულ რიცხვებზე, რომლებიც მონამ ჩატვირთა აპარატურაში მასტერისგან მონაცემების მისაღებად.

  • RX მონაცემთა ზომა: მრიცხველი, რომელიც განთავსებულია მონა-მოწყობილობაზე და მისი წაკითხვა მასტერ-მოწყობილობას შეუძლია, მიუთითებს დაგროვილი მონაცემების ზომას, რომელიც მონამ ჩატვირთა აპარატურაში მასტერ-მოწყობილობისთვის გასაგზავნად.

ESP მონა-მონაწილის მიერ მოწოდებული სერვისები¶

Espressif-ის მონების მიერ შემოთავაზებული რამდენიმე საერთო სერვისი არსებობს:

  1. Tohost-ის შეფერხებები: მონა სერვერს შეუძლია აცნობოს მასტერს გარკვეული მოვლენების შესახებ შეფერხების ხაზის საშუალებით.

  2. Frhost-ის შეწყვეტა: მასტერს შეუძლია მონას აცნობოს გარკვეული მოვლენების შესახებ.

  3. Tx FIFO (master-დან slave-მდე): slave-ს შეუძლია მონაცემების ნაკადის სახით გაგზავნა მასტერისთვის. SDIO slave-ს ასევე შეუძლია მიუთითოს, რომ მას აქვს ახალი მონაცემები გასაგზავნი მასტერისთვის შეწყვეტის ხაზით.

    მონა ოპერატორი აახლებს TX ბუფერის ნომერს, რათა აცნობოს მასტერს, თუ რამდენი მონაცემის მიღება შეუძლია, შემდეგ კი მასტერი კითხულობს TX ბუფერის ნომერს და ამოიღებს გამოყენებული ბუფერის ნომერს, რათა გაიგოს, რამდენი ბუფერი დარჩა.

  4. Rx FIFO (slave to master): მონას შეუძლია მონაცემების მიღება მასტერისგან მიმღები ბუფერების ერთეულებში.

    მონა პორტი აახლებს RX მონაცემთა ზომას, რათა აცნობოს მასტერს, თუ რამდენი მონაცემი აქვს მომზადებული გასაგზავნად, შემდეგ კი მასტერი კითხულობს მონაცემთა ზომას და ამცირებს უკვე მიღებული მონაცემების სიგრძეს, რათა იცოდეს, რამდენი მონაცემია დარჩენილი.

  5. გაზიარებული რეგისტრები: მასტერს შეუძლია წაიკითხოს მონაზე არსებული რეგისტრების გარკვეული ნაწილი და ასევე ჩაწეროს ეს რეგისტრები ისე, რომ მონამ წაიკითხოს.

ESP SDIO Slave Link-ის ინიციალიზაცია¶

ESP SDIO მონათმფლობელური კავშირის (ESSL SDIO) მოწყობილობები ეყრდნობა sdmmc კომპონენტი . ESSL მოწყობილობის ინიციალიზაცია უნდა მოხდეს ქვემოთ მოცემული მეთოდით:

  1. sdmmc ბარათის ინიციალიზაცია (იხილეთ :doc:` SDMMC დრაივერის დოკუმენტი</api-reference/storage/sdmmc> `) სტრუქტურა.
  2. ზარი sdmmc_card_init() ბარათის ინიციალიზაციისთვის.
  3. ESSL მოწყობილობის ინიციალიზაცია შემდეგი მეთოდით: essl_sdio_config_t. ის card წევრი უნდა იყოს sdmmc_card_t მე-2 ეტაპზე გადავედი და recv_buffer_size წევრი სწორად უნდა შეივსოს წინასწარ შეთანხმებული ღირებულების მიხედვით.
  4. ზარი essl_init() SDIO ნაწილის ინიციალიზაციისთვის.
  5. ზარი essl_wait_for_ready() რომ მონის მზადყოფნას დალოდებოდნენ.

API სთ¶

ზემოთ აღწერილი ინიციალიზაციის პროცესის დასრულების შემდეგ, შეგიძლიათ გამოიძახოთ ქვემოთ მოცემული API , რათა ისარგებლოთ მონა-მომსახურე მოწყობილობის მიერ მოწოდებული სერვისებით:

შეწყვეტები¶

  1. ზარი essl_get_intr_ena() იმის ცოდნა, თუ რომელი მოვლენები გამოიწვევს მთავარ სისტემებში შეფერხებებს.
  2. ზარი essl_set_intr_ena() იმ მოვლენების დასაყენებლად, რომლებიც მთავარ მოწყობილობაში შეწყვეტას გამოიწვევს.
  3. ზარი essl_wait_int() დაელოდოთ მონის მიერ შეწყვეტას ან ტაიმაუტს.
  4. შეფერხების გააქტიურებისას, დარეკეთ essl_get_intr() რომ იცოდეთ, რომელი ღონისძიებებია აქტიური და დარეკეთ essl_clear_intr() მათ გასასუფთავებლად.
  5. ზარი essl_send_slave_intr() მონა-მოთამაშის ზოგადი დანიშნულების შეწყვეტის გასააქტიურებლად.

ტეხასის FIFO¶

  1. ზარი essl_get_tx_buffer_num() იმის გასაგებად, თუ რამდენი ბუფერი აქვს მომზადებული მონას მასტერისგან მონაცემების მისაღებად. ეს არასავალდებულოა. მასტერ გამოკითხვას გამართავს tx_buffer_num როდესაც ის ცდილობს პაკეტების გაგზავნას მონისთვის, სანამ მონს არ ექნება საკმარისი ბუფერი ან ტაიმ-აუტი.
  2. ზარი essl_send_paket() მონაცემების მონასთვის გასაგზავნად.

RX FIFO¶

  1. ზარი essl_get_rx_data_size() იმის გასაგებად, თუ რამდენი მონაცემი აქვს მომზადებული მონას მთავარ მოწყობილობას გასაგზავნად. ეს არასავალდებულოა. როდესაც მასტერი ცდილობს მონაცემების მიღებას მონადან, ის განაახლებს rx_data_size ერთხელ, თუ მიმდინარე rx_data_size უფრო მოკლეა, ვიდრე ბუფერის ზომა, რომლის მისაღებადაც მასტერი იყო მომზადებული. და შესაძლოა, გამოკითხვა ჩაატაროს rx_data_size თუ rx_dat_size ინარჩუნებს 0-ს, ტაიმაუტამდე.
  2. ზარი essl_get_packet() მონასგან მონაცემების მისაღებად.

მრიცხველების გადატვირთვა (არასავალდებულო)¶

ზარი essl_reset_cnt() შიდა მრიცხველის გადასაყენებლად, თუ აღმოაჩენთ, რომ მონამ მრიცხველი გადაატვირთა.

განაცხადის მაგალითი¶

ქვემოთ მოცემული მაგალითი გვიჩვენებს, თუ როგორ ურთიერთობენ ერთმანეთთან ESP32 SDIO ჰოსტი და მონა. ჰოსტი იყენებს ESSL SDIO-ს.

პერიფერიული მოწყობილობები/სდიო.

დეტალებისთვის იხილეთ README.md ფაილის კონკრეტული მაგალითი.

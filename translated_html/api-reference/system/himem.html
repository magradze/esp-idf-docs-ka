ჰიემის განაწილების API¶

მიმოხილვა¶

ESP32-ს შეუძლია გარე SPI ოპერატიულ მეხსიერებაზე წვდომა გამჭვირვალედ, ამიტომ თქვენ შეგიძლიათ გამოიყენოთ იგი როგორც ჩვეულებრივი მეხსიერება თქვენს პროგრამულ კოდში. თუმცა, რადგან გარე მეხსიერების მისამართების სივრცე შეზღუდულია ზომით, მხოლოდ პირველი 4 მბ-ის გამოყენებაა შესაძლებელი. დარჩენილ მეხსიერებაზე წვდომა კვლავ შესაძლებელია, თუმცა ეს უნდა მოხდეს hiem API ით კონტროლირებადი ბანკების გადართვის სქემის მეშვეობით.

კერძოდ, himem API მიერ იმპლემენტირებულია ბანკების გადართვის სქემა. აპარატურის მხრივ, გარე SPI ოპერატიული მეხსიერების 4MiB რეგიონი CPU-ს მისამართების სივრცეში MMU-ს მეშვეობით არის მიმაგრებული, რომელიც გარე SPI ოპერატიული მეხსიერების კონფიგურირებად 32K ბანკს/გვერდს CPU-ს მიერ წვდომილი 4MiB რეგიონის თითოეულ 32K გვერდზე აკავშირებს. 4MiB-ზე ნაკლები გარე მეხსიერებისთვის, ეს MMU კონფიგურირებულია ერთიანობის მიმაგრებისთვის, რაც ეფექტურად აკავშირებს CPU-ს თითოეულ მისამართს 1-დან 1-მდე გარე SPI RAM მისამართთან.

himem API გამოსაყენებლად, ის menuconfig ში უნდა ჩართოთ, გამოიყენეთ CONFIG_SPIRAM_BANKSWITCH_ENABLE, ასევე დაადგინეთ ამისთვის დაჯავშნილი ბანკების რაოდენობა CONFIG_SPIRAM_BANKSWITCH_RESERVEეს ამცირებს ისეთი ფუნქციების მიერ გამოყოფილი გარე მეხსიერების რაოდენობას, როგორიცაა malloc(), მაგრამ ის საშუალებას გაძლევთ გამოიყენოთ himem api დარჩენილი მეხსიერების რეზერვირებულ ბანკებში გადასატანად.

himem API მეტ-ნაკლებად ბანკების გადართვის სქემის აბსტრაქციაა: ის საშუალებას გაძლევთ მოითხოვოთ მისამართების სივრცის ერთი ან მეტი ბანკი ( API ში „რეგიონები“ ეწოდება), ასევე მეხსიერების ერთი ან მეტი ბანკი დიაპაზონებში მისამაგრებლად.

მაგალითი¶

მაღალი მეხსიერების დიაპაზონის მარტივი მეხსიერების ტესტის ჩატარების მაგალითი ხელმისაწვდომია esp-idf-ში: სისტემა/ჰიმემი

API მითითება¶

სათაურის ფაილი¶

  • esp32/include/esp32/himem.h

ფუნქციები¶

esp_err_t esp_himem_alloc(size_t ზომა, esp_himem_handle_t *გაუმკლავება)¶

ბლოკის გამოყოფა მაღალი მოცულობის მეხსიერებაში.

დაბრუნება
- ESP_OK if succesful
  • ESP_ERR_NO_MEM, თუ მეხსიერება ამოწურულია
  • ESP_ERR_INVALID_SIZE თუ ზომა არ არის 32K-ის ჯერადი
პარამეტრები
  • size: გამოსაყოფი ბლოკის ზომა, ბაიტებში. გაითვალისწინეთ, რომ ეს უნდა იყოს გარე RAM mmu ბლოკის ზომის (32K) ჯერადი.
  • handle_out: დასაბრუნებელი სახელური

esp_err_t esp_himem_alloc_map_range(size_t ზომა, esp_himem_rangehandle_t *გაუმკლავება)¶

გამოყავით მეხსიერების რეგიონი ბლოკების განსათავსებლად.

ეს გამოყოფს CPU-ს მომიჯნავე მეხსიერების რეგიონს, რომლის გამოყენებაც შესაძლებელია ფიზიკური მეხსიერების ბლოკების შესაბამისობაში მოსაყვანად.

დაბრუნება
- ESP_OK if succesful
  • ESP_ERR_NO_MEM, თუ მეხსიერება ან მისამართის სივრცე ამოწურულია
  • ESP_ERR_INVALID_SIZE თუ ზომა არ არის 32K-ის ჯერადი
პარამეტრები
  • size: გამოსაყოფი დიაპაზონის ზომა. გაითვალისწინეთ, რომ ეს უნდა იყოს გარე RAM mmu ბლოკის ზომის (32K) ჯერადი.
  • handle_out: დასაბრუნებელი სახელური

esp_err_t esp_himem_map(esp_himem_handle_t სახელური, esp_himem_rangehandle_t დიაპაზონი, size_t ram_offset, size_t დიაპაზონის_ოფსეტი, size_t ლენ, int დროშები, void **გამომავალი_პტრ)¶

მაღალი დატვირთვის მეხსიერების ბლოკის CPU-ს მისამართების სივრცეში განთავსება.

ეს ეფექტურად ხდის ბლოკს ხელმისაწვდომს წაკითხვის/ჩაწერის ოპერაციებისთვის.

შენიშვნა
The region to be mapped needs to have offsets and sizes that are aligned to the SPI RAM MMU block size (32K)
დაბრუნება
- ESP_OK if the memory could be mapped
  • ESP_ERR_INVALID_ARG თუ ოფსეტი, დიაპაზონი ან ლინზა არ არის MMU-ს ბლოკთან გასწორებული (32K)
  • ESP_ERR_INVALID_SIZE თუ ოფსეტები/სიგრძეები არ ჯდება გამოყოფილ მეხსიერებაში ან დიაპაზონში
  • ESP_ERR_INVALID_STATE, თუ არჩეულ ოპერატიული მეხსიერების ოფსეტში/სიგრძეში ბლოკი უკვე მიმაგრებულია, ან თუ არჩეულ დიაპაზონში ოფსეტის/სიგრძის ბლოკს უკვე აქვს მიმაგრება.
პარამეტრები
  • handleმეხსიერების ბლოკის სახელური, როგორც ეს მოცემულია esp_himem_alloc-ის მიერ
  • rangeმეხსიერების შესაბამისობის დიაპაზონის სახელური
  • ram_offset: ბლოკის ფიზიკური მეხსიერების ბლოკში გადატანა შესაბამისობის დასადგენად
  • range_offset: ოფსეტი მისამართების დიაპაზონში, სადაც ბლოკი იქნება მიმაგრებული
  • lenრუკაზე დასატანი რეგიონის სიგრძე
  • flags: ერთ-ერთი ESP_HIMEM_MAPFLAG_*-დან
  • out_ptr: ცვლადის მაჩვენებელი, რომელშიც შედეგად მიღებული მაჩვენებელი ინახება მეხსიერებაში

esp_err_t esp_himem_free(esp_himem_handle_t სახელური)¶

ფიზიკური მეხსიერების ბლოკის გათავისუფლება.

ეს ასუფთავებს ასოცირებულ სახელურს და მეხსიერებას ხელახლა ხელმისაწვდომს ხდის ხელახალი განაწილებისთვის. ეს მხოლოდ იმ შემთხვევაში იქნება წარმატებული, თუ მეხსიერების არცერთ ბლოკს ამჟამად არ აქვს შესაბამისობა.

დაბრუნება
- ESP_OK if the memory is succesfully freed
  • ESP_ERR_INVALID_ARG, თუ სახელური კვლავ (ნაწილობრივ) მიმაგრებულია
პარამეტრები
  • handleმეხსიერების ბლოკის სახელური, როგორც ეს მოცემულია esp_himem_alloc-ის მიერ

esp_err_t esp_himem_free_map_range(esp_himem_rangehandle_t სახელური)¶

რუკების დიაპაზონის გათავისუფლება.

ეს ასუფთავებს ასოცირებულ სახელურს და დიაპაზონს ხელახლა ხელმისაწვდომს ხდის ხელახალი განაწილებისთვის. ეს მხოლოდ იმ შემთხვევაში იქნება წარმატებული, თუ დიაპაზონის არცერთი ბლოკი ამჟამად არ გამოიყენება შესაბამისობისთვის.

დაბრუნება
- ESP_OK if the memory is succesfully freed
  • ESP_ERR_INVALID_ARG, თუ სახელური კვლავ (ნაწილობრივ) მიმაგრებულია
პარამეტრები
  • handleდიაპაზონის ბლოკის სახელური, როგორც ეს მოცემულია esp_himem_alloc_map_range-ის მიერ

esp_err_t esp_himem_unmap(esp_himem_rangehandle_t დიაპაზონი, void *პტრ, size_t ლენ)¶

რეგიონის რუკის მოხსნა.

დაბრუნება
- ESP_OK if the memory is succesfully unmapped,
  • ESP_ERR_INVALID_ARG, თუ ptr ან len არასწორია.
პარამეტრები
  • rangeდიაპაზონის სახელური
  • ptresp_himem_map-ის მიერ დაბრუნებული მაჩვენებელი
  • len: ბლოკის სიგრძე, რომლის მაპინგიც უნდა მოხდეს. უნდა შეესაბამებოდეს SPI RAM MMU ბლოკის ზომას (32K).

size_t esp_himem_get_phys_size(void)¶

მიიღეთ hiem API ის კონტროლის ქვეშ არსებული მეხსიერების საერთო რაოდენობა.

დაბრუნება
Amount of memory, in bytes

size_t esp_himem_get_free_size(void)¶

მიიღეთ მეხსიერების თავისუფალი რაოდენობა hiem API ის კონტროლის ქვეშ.

დაბრუნება
Amount of free memory, in bytes

size_t esp_himem_reserved_area_size(void)¶

მიიღეთ ბანკების გადართვისთვის საჭირო SPI მეხსიერების მისამართის სივრცის რაოდენობა.

შენიშვნა
This is also weakly defined in esp32/spiram.c and returns 0 there, so if no other function in this file is used, no memory is reserved.
დაბრუნება
Amount of reserved area, in bytes

მაკროები¶

ESP_HIMEM_BLKSZ¶
ESP_HIMEM_MAPFLAG_RO¶

მიუთითებს, რომ შესაბამისობის წაკითხვა მხოლოდ აქედან მოხდება. გაითვალისწინეთ, რომ ის ამჟამად გამოუყენებელია.

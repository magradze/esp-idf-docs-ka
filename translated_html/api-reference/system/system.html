სხვადასხვა სისტემა, API წმ¶

პროგრამული უზრუნველყოფის გადატვირთვა¶

ჩიპის პროგრამული უზრუნველყოფის გადატვირთვის შესასრულებლად, esp_restart() ფუნქცია მოცემულია. ფუნქციის გამოძახებისას, პროგრამის შესრულება შეჩერდება, ორივე პროცესორი გადაიტვირთება, აპლიკაცია ჩაიტვირთება ჩამტვირთავი მიერ და ხელახლა ჩაირთვება.

გარდა ამისა, esp_register_shutdown_handler() ფუნქცია გათვალისწინებულია რუტინის რეგისტრაციისთვის, რომლის გამოძახებაც საჭიროა გადატვირთვამდე (როდესაც ეს კეთდება esp_restart()). ეს მსგავსია ფუნქციონალურობისა atexit POSIX ფუნქცია.

მიზეზის გადაყენება¶

ESP-IDF აპლიკაციის გაშვება ან გადატვირთვა შესაძლებელია სხვადასხვა მიზეზის გამო. ბოლო გადატვირთვის მიზეზის გასაგებად, დარეკეთ esp_reset_reason() ფუნქცია. იხილეთ აღწერა esp_reset_reason_t შესაძლო გადატვირთვის მიზეზების სიისთვის.

გროვის მეხსიერება¶

გათვალისწინებულია ორი გროვის მეხსიერებასთან დაკავშირებული ფუნქცია:

  • esp_get_free_heap_size() აბრუნებს თავისუფალი მეხსიერების მიმდინარე ზომას
  • esp_get_minimum_free_heap_size() აბრუნებს პროგრამის შესრულების დროს ხელმისაწვდომი თავისუფალი მეხსიერების მინიმალურ ზომას.

გაითვალისწინეთ, რომ ESP-IDF მხარს უჭერს სხვადასხვა შესაძლებლობების მქონე მრავალ გროვას. ამ განყოფილებაში ნახსენები ფუნქციები აბრუნებენ გროვის მეხსიერების ზომას, რომლის გამოყოფაც შესაძლებელია malloc ფუნქციების ოჯახი. გროვის მეხსიერების შესახებ დამატებითი ინფორმაციისთვის იხილეთ გროვის მეხსიერების განაწილება.

შემთხვევითი რიცხვების გენერირება¶

ESP32 შეიცავს აპარატურულ შემთხვევითი რიცხვების გენერატორს, მისგან მნიშვნელობების მიღება შესაძლებელია გამოყენებით esp_random().

როდესაც Wi-Fi ან Bluetooth ჩართულია, აპარატურული შემთხვევითი რიცხვების გენერატორის (RNG) მიერ დაბრუნებული რიცხვები შეიძლება ჩაითვალოს ნამდვილ შემთხვევით რიცხვებად. Wi-Fi ან Bluetooth ჩართვის გარეშე, აპარატურული RNG არის ფსევდოშემთხვევითი რიცხვების გენერატორი. გაშვებისას, ESP-IDF ჩამტვირთავი აპარატურულ RNG-ში ენტროპიას ნერგავს, მაგრამ სიფრთხილეა საჭირო შემთხვევითი მნიშვნელობების წაკითხვისას დასაწყისს შორის. app_main და Wi-Fi ან Bluetooth დრაივერების ინიციალიზაცია.

MAC მისამართი¶

ეს API ები საშუალებას იძლევა მოითხოვოთ და მორგოთ MAC მისამართები სხვადასხვა მხარდაჭერილი ქსელური ინტერფეისებისთვის (მაგ. Wi-Fi , Bluetooth , Ethernet).

ESP-IDF ში ეს მისამართები გამოითვლება შემდეგიდან: ბაზის MAC მისამართიბაზის MAC მისამართის ინიციალიზაცია შესაძლებელია ქარხნულად დაპროგრამებული მნიშვნელობით შიდა eFuse-დან ან მომხმარებლის მიერ განსაზღვრული მნიშვნელობით. ბაზის MAC მისამართის დაყენების გარდა, აპლიკაციებს შეუძლიათ მიუთითონ, თუ როგორ ენიჭება MAC მისამართები მოწყობილობებს. იხილეთ უნივერსალურად ადმინისტრირებული MAC მისამართების რაოდენობა განყოფილება დამატებითი დეტალებისთვის.

ESP32-ისთვის:

ინტერფეისი MAC მისამართი (4 უნივერსალურად ადმინისტრირებადი) MAC მისამართი (2 უნივერსალურად ადმინისტრირებადი)
Wi-Fi სადგური base_mac base_mac
Wi-Fi SoftAP base_mac, +1 ბოლო ოქტეტამდე base_mac, პირველი ოქტეტი შემთხვევითი თანმიმდევრობით
Bluetooth base_mac, +2 ბოლო ოქტეტამდე base_mac, +1 ბოლო ოქტეტამდე
Ethernet base_mac, +3 ბოლო ოქტეტამდე base_mac, +1 ბოლო ოქტეტამდე, პირველი ოქტეტი შემთხვევითი თანმიმდევრობით

ESP32-S2-ისთვის:

ინტერფეისი MAC მისამართი (2 უნივერსალურად ადმინისტრირებადი) MAC მისამართი (1 უნივერსალურად ადმინისტრირებადი)
Wi-Fi სადგური base_mac base_mac
Wi-Fi SoftAP base_mac, +1 ბოლო ოქტეტამდე base_mac, პირველი ოქტეტი შემთხვევითი თანმიმდევრობით

ბაზის MAC მისამართი¶

კონკრეტული ინტერფეისის MAC მისამართის მისაღებად (მაგ. Wi-Fi , Bluetooth , Ethernet), შეგიძლიათ უბრალოდ გამოიყენოთ esp_read_mac() ფუნქცია.

ნაგულისხმევად, ეს ფუნქცია საბაზისო MAC მისამართად იღებს წინასწარ განსაზღვრულ ბლოკში ჩაწერილ eFuse მნიშვნელობას (მაგ. BLK0 ESP32-ისთვის, BLK1 ESP32-S2-ისთვის). თითოეული ინტერფეისის MAC მისამართები გამოითვლება ზემოთ მოცემული ცხრილის მიხედვით.

აპლიკაციებმა, რომლებსაც სურთ ძირითადი MAC მისამართის (და არა Espressif-ის მიერ მოწოდებულის) პერსონალიზაცია, უნდა დარეკონ esp_base_mac_addr_set() ადრე esp_read_mac()მორგებული MAC მისამართის შენახვა შესაძლებელია ნებისმიერ მხარდაჭერილ მეხსიერების მოწყობილობაში (მაგ. Flash, NVS და ა.შ.).

გაითვალისწინეთ, რომ ზარები esp_base_mac_addr_set() უნდა მოხდეს ქსელური დასტის ინიციალიზაციამდე, მაგალითად, დასაწყისში app_main.

eFuse-ში მორგებული MAC მისამართი¶

მორგებული MAC მისამართების გამოყენების გასაადვილებლად, ESP-IDF უზრუნველყოფს esp_efuse_mac_get_custom() ფუნქცია, რომელიც იტვირთება MAC მისამართი შიდა წინასწარ განსაზღვრული eFuse ბლოკიდან (მაგ. BLK3 ESP32-ისთვის). ეს ფუნქცია ვარაუდობს, რომ მორგებული MAC მისამართი შენახულია შემდეგ ფორმატში:

ველი ბიტების რაოდენობა ბიტების დიაპაზონი შენიშვნები
ვერსია 8 191:184 0: არასწორი, სხვები — ვალიდური
დაჯავშნილია 128 183:56  
MAC მისამართი 48 55:8  
MAC მისამართი CRC 8 7:0 CRC-8-CCITT, პოლინომი 0x07

მას შემდეგ, რაც MAC მისამართი მიღებული იქნება გამოყენებით esp_efuse_mac_get_custom(), დარეკვა esp_base_mac_addr_set() ამ MAC მისამართის საბაზისო MAC მისამართად დასაყენებლად.

უნივერსალურად ადმინისტრირებული MAC მისამართების რაოდენობა¶

რამდენიმე MAC მისამართი (რომლებსაც IEEE უნივერსალურად ადმინისტრირებს) უნიკალურად ენიჭება ქსელურ ინტერფეისებს ( Wi-Fi /BT/Ethernet). თითოეული უნივერსალურად ადმინისტრირებული MAC მისამართის ბოლო ოქტეტი იზრდება ერთით. მათგან მხოლოდ პირველი (რომელსაც საბაზო MAC მისამართი ეწოდება) ინახება eFuse-ში ან გარე მეხსიერებაში, დანარჩენები კი მისგან გენერირდება. აქ „გენერირება“ ნიშნავს საბაზო MAC მისამართის ბოლო ოქტეტს 0, 1, 2 და 3-ის დამატებას (შესაბამისად).

თუ უნივერსალურად ადმინისტრირებული MAC მისამართები არ არის საკმარისი ყველა ქსელური ინტერფეისისთვის, ლოკალურად ადმინისტრირებული MAC მისამართები, რომლებიც მიიღება უნივერსალურად ადმინისტრირებული MAC მისამართებიდან, ენიჭება დანარჩენ ქსელურ ინტერფეისებს.

იხილეთ ეს სტატია ლოკალური და უნივერსალურად ადმინისტრირებული MAC მისამართების განსაზღვრისთვის.

უნივერსალურად ადმინისტრირებული MAC მისამართების რაოდენობის კონფიგურაცია შესაძლებელია გამოყენებით CONFIG_ESP32_UNIVERSAL_MAC_ADDRESSES ESP32-ისთვის.

თუ უნივერსალური MAC მისამართების რაოდენობა ორია, მხოლოდ ორი ინტერფეისი ( Wi-Fi სადგური და Bluetooth ) მიიღებს უნივერსალურად ადმინისტრირებულ MAC მისამართს. ეს მისამართები თანმიმდევრულად გენერირდება საბაზისო MAC მისამართზე 0-ის და 1-ის დამატებით (შესაბამისად). დარჩენილი ორი ინტერფეისი ( Wi-Fi SoftAP და Ethernet) იღებს ლოკალურ MAC მისამართებს. ეს მისამართები შესაბამისად მიიღება უნივერსალური Wi-Fi სადგურისა და Bluetooth MAC მისამართებიდან.

თუ უნივერსალური MAC მისამართების რაოდენობა ოთხია, ოთხივე ინტერფეისი ( Wi-Fi Station, Wi-Fi SoftAP, Bluetooth და Ethernet) მიიღებს უნივერსალურად ადმინისტრირებულ MAC მისამართს. ეს მისამართები თანმიმდევრულად გენერირდება საბაზო MAC მისამართის საბოლოო ოქტეტზე 0, 1, 2 და 3-ის დამატებით (შესაბამისად).

ნაგულისხმევი (Espressif-ის მიერ მინიჭებული) საბაზისო MAC მისამართის გამოყენებისას, შესაძლებელია ორივე პარამეტრის გამოყენება. უნივერსალური MAC მისამართების დიაპაზონის გამოყენებისას, სწორი პარამეტრი დამოკიდებული იქნება ამ დიაპაზონში MAC მისამართების განაწილებაზე (2 ან 4 მოწყობილობაზე).

ჩიპური ვერსია¶

esp_chip_info() ფუნქციის შევსება esp_chip_info_t სტრუქტურა ჩიპის შესახებ ინფორმაციით. ეს მოიცავს ჩიპის ვერსიას, CPU ბირთვების რაოდენობას და ჩიპში ჩართული ფუნქციების ბიტ ნიღაბს.

SDK ვერსია¶

esp_get_idf_version() აბრუნებს სტრიქონს, რომელიც აღწერს IDF ვერსიას, რომელიც გამოყენებული იყო აპლიკაციის კომპილაცია ისთვის. ეს იგივე მნიშვნელობაა, რაც ის, რაც ხელმისაწვდომია IDF_VER აწყობის სისტემის ცვლადი. ვერსიის სტრიქონს ზოგადად აქვს ფორმატი git describe გამომავალი.

ვერსიის აწყობის დროს მისაღებად, მოწოდებულია დამატებითი ვერსიის მაკროები. მათი გამოყენება შესაძლებელია პროგრამის ნაწილების ჩასართავად ან გამორთვისთვის, IDF ვერსიის მიხედვით.

  • ESP_IDF_VERSION_MAJOR, ESP_IDF_VERSION_MINOR, ESP_IDF_VERSION_PATCH განისაზღვრება მთელი რიცხვებით, რომლებიც წარმოადგენენ ძირითად, მცირე და პატჩ ვერსიებს.

  • ESP_IDF_VERSION_VAL და ESP_IDF_VERSION შეიძლება გამოყენებულ იქნას ვერსიის შემოწმების განხორციელებისას:

    #include "esp_idf_version.h"

#if ESP_IDF_VERSION >= ESP_IDF_VERSION_VAL(4, 0, 0)
    // enable functionality present in IDF v4.0
#endif

აპლიკაციის ვერსია¶

აპლიკაციის ვერსია შენახულია esp_app_desc_t სტრუქტურა. ის მდებარეობს DROM სექტორში და აქვს ფიქსირებული ოფსეტი ორობითი ფაილის დასაწყისიდან. სტრუქტურა მდებარეობს შემდეგ esp_image_header_t და esp_image_segment_header_t სტრუქტურები. ველის ვერსიას აქვს სტრიქონის ტიპი და მაქსიმალური სიგრძე 32 სიმბოლო.

თქვენს პროექტში ვერსიის ხელით დასაყენებლად, თქვენ უნდა დააყენოთ PROJECT_VER ცვლადი თქვენს პროექტში CMakeLists.txt/Makefile:

  • CMakeLists.txt აპლიკაციაში ატვირთულია set(PROJECT_VER "0.1.0.1") ჩართვამდე project.cmake.

(GNU Make-ის მემკვიდრეობითი სისტემისთვის: Makefile აპლიკაციაში put PROJECT_VER = "0.1.0.1" ჩართვამდე project.mk.)

თუ CONFIG_APP_PROJECT_VER_FROM_CONFIG ოფცია დაყენებულია, მნიშვნელობა CONFIG_APP_PROJECT_VER გამოყენებული იქნება. წინააღმდეგ შემთხვევაში, თუ PROJECT_VER თუ ცვლადი პროექტში არ არის დაყენებული, მაშინ ის ამოღებული იქნება შემდეგი სისტემიდან: $(PROJECT_PATH)/version.txt ფაილი (თუ არსებობს) სხვა ფაილი git ბრძანების გამოყენებით git describeთუ არცერთი არ არის ხელმისაწვდომი, მაშინ PROJECT_VER დაყენდება „1“-ზე. აპლიკაციას შეუძლია ამის გამოყენება დარეკვით esp_ota_get_app_description() ან esp_ota_get_partition_description() ფუნქციები.

uint32_t esp_random(void)¶

მიიღეთ ერთი შემთხვევითი 32-ბიტიანი სიტყვა აპარატურული RNG-დან.

აპარატურის RNG სრულად ფუნქციონირებს ყოველთვის, როდესაც RF ქვესისტემა მუშაობს (ანუ Bluetooth ან WiFi ჩართულია). შემთხვევითი მნიშვნელობებისთვის, გამოიძახეთ ეს ფუნქცია WiFi-ის ან Bluetooth გაშვების შემდეგ.

თუ პროგრამა არ იყენებს RF ქვესისტემას, ენტროპიის წყაროს ჩასართავად შესაძლებელია ფუნქციის ჩამტვირთავი _random_enable() გამოძახება. RF ქვესისტემის ან I2S პერიფერიული მოწყობილობის გამოყენებამდე უნდა გამოიძახოთ ჩამტვირთავი _random_disable(). დამატებითი ინფორმაციისთვის იხილეთ ამ ფუნქციების დოკუმენტაცია.

როდესაც აპლიკაცია მუშაობს RF ქვესისტემის (ან ჩამტვირთავი _random) ჩართვის გარეშე, RNG აპარატურა უნდა ჩაითვალოს PRNG-ად. IDF ჩამტვირთავი გაშვებისას წინასწარი დათესვის გამო ენტროპიის ძალიან მცირე რაოდენობაა ხელმისაწვდომი, მაგრამ ამაზე არ უნდა იყოს დამოკიდებული რაიმე მიზნით.

დაბრუნება
Random value between 0 and UINT32_MAX

esp_err_t esp_base_mac_addr_set(კონსტ uint8_t *მაკი)¶

დააყენეთ საბაზისო MAC მისამართი იმ MAC მისამართით, რომელიც შენახულია EFUSE-ს BLK3-ში ან გარე მეხსიერებაში, მაგ ფლეშ მეხსიერება და EEPROM.

ქსელის ინტერფეისების მიერ გამოყენებული MAC მისამართების გენერირებისთვის გამოიყენება საბაზო MAC მისამართი. თუ იყენებთ EFUSE-ს BLK3-ში ან გარე მეხსიერებაში შენახულ საბაზო MAC მისამართს, WiFi/BT/Ethernet-ის ინიციალიზაციამდე გამოიძახეთ ეს API , რათა საბაზო MAC მისამართი დააყენოთ EFUSE-ს BLK3-ში ან გარე მეხსიერებაში შენახულ MAC მისამართთან.

შენიშვნა
Base MAC must be a unicast MAC (least significant bit of first byte must be zero).
შენიშვნა
If not using a valid OUI, set the “locally administered” bit (bit value 0x02 in the first byte) to avoid collisions.
დაბრუნება
ESP_OK on success ESP_ERR_INVALID_ARG If mac is NULL or is not a unicast MAC
პარამეტრები
  • mac: ბაზის MAC მისამართი, სიგრძე: 6 ბაიტი.

esp_err_t esp_read_mac(uint8_t *მაკი, esp_mac_type_t ტიპი)¶

წაიკითხეთ ბაზის MAC მისამართი და დააყენეთ ინტერფეისის MAC მისამართი.

ეს ფუნქცია თავდაპირველად იღებს საბაზისო MAC მისამართს esp_base_mac_addr_get-ის გამოყენებით ან კითხულობს საბაზისო MAC მისამართს EFUSE-ს BLK0-დან. შემდეგ ადგენს ინტერფეისის MAC მისამართს, მათ შორის wifi სადგურს, wifi softap-ს, Bluetooth-ს და Ethernet-ს.

დაბრუნება
ESP_OK on success
პარამეტრები
  • macინტერფეისის MAC მისამართი, სიგრძე: 6 ბაიტი.
  • type: MAC მისამართის ტიპი, 0: Wi-Fi სადგური, 1: Wi-Fi SoftAp, 2: Bluetooth, 3: Ethernet.

esp_err_t esp_derive_local_mac(uint8_t *local_mac, კონსტ uint8_t *უნივერსალური_მაკი)¶

მიიღეთ ადგილობრივი MAC მისამართი უნივერსალური MAC მისამართიდან.

ეს ფუნქცია უნივერსალური MAC მისამართიდან იღებს ლოკალურ MAC მისამართს. A definition of local vs universal MAC address can be found on Wikipedia <>ESP32-ში უნივერსალური MAC მისამართი გენერირდება EFUSE-ში ან სხვა გარე მეხსიერებაში არსებული საბაზისო MAC მისამართიდან. ლოკალური MAC მისამართი მიიღება უნივერსალური MAC მისამართიდან.

დაბრუნება
ESP_OK on success
პარამეტრები
  • local_mac: მიღებული ლოკალური MAC მისამართი, სიგრძე: 6 ბაიტი.
  • universal_macწყაროს უნივერსალური MAC მისამართი, სიგრძე: 6 ბაიტი.

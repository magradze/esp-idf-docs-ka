ჟურნალირების ბიბლიოთეკა¶

მიმოხილვა¶

ჟურნალის ბიბლიოთეკა ჟურნალის დეტალიზაციის დაყენების ორ გზას გთავაზობთ:

  • კომპილაცია დროს: menuconfig ში, დააყენეთ დეტალიზაციის დონე ოფციის გამოყენებით CONFIG_LOG_DEFAULT_LEVELყველა ჟურნალირების ინსტრუქცია დეტალიზაციის დონეებისთვის, რომელიც უფრო მაღალია, ვიდრე CONFIG_LOG_DEFAULT_LEVEL წაიშლება წინასწარი პროცესორის მიერ.
  • გაშვების დროს: ყველა ჟურნალი უფრო დაბალია, ვიდრე CONFIG_LOG_DEFAULT_LEVEL ნაგულისხმევად ჩართულია. ფუნქცია esp_log_level_set() შეიძლება გამოყენებულ იქნას თითოეული მოდულის მიხედვით ჟურნალირების დონის დასაყენებლად. მოდულები იდენტიფიცირდება მათი თეგებით, რომლებიც ადამიანის მიერ წასაკითხი ASCII ნულოვანი დაბოლოებით დასრულებულ სტრიქონებს წარმოადგენს.

არსებობს შემდეგი სიზუსტის დონეები:

  • შეცდომა (ყველაზე დაბალი)
  • გაფრთხილება
  • ინფორმაცია
  • გამართვა
  • ვრცელი (ყველაზე მაღალი)

შენიშვნა

ფუნქცია esp_log_level_set() ჟურნალირების დონის დაყენება მითითებულზე მაღალი არ არის CONFIG_LOG_DEFAULT_LEVELკონკრეტული ფაილისთვის ლოგის დონის გასაზრდელად კომპილაცია დროს, გამოიყენეთ მაკრო LOG_LOCAL_LEVEL (დეტალები იხილეთ ქვემოთ).

როგორ გამოვიყენოთ ეს ბიბლიოთეკა¶

თითოეულ C ფაილში, რომელიც იყენებს ჟურნალირების ფუნქციონალს, განსაზღვრეთ TAG ცვლადი, როგორც ნაჩვენებია ქვემოთ:

static const char* TAG = "MyModule";

შემდეგ გამომავალი მონაცემების მისაღებად გამოიყენეთ ჟურნალირების ერთ-ერთი მაკრო, მაგალითად:

ESP_LOGW(TAG, "Baud rate error %.1f%%. Requested: %d baud, actual: %d baud", error * 100, baud_req, baud_real);

სხვადასხვა დეტალიზაციის დონისთვის ხელმისაწვდომია რამდენიმე მაკრო:

  • ESP_LOGE - შეცდომა (ყველაზე დაბალი)
  • ESP_LOGW - გაფრთხილება
  • ESP_LOGI - ინფორმაცია
  • ESP_LOGD - გამართვა
  • ESP_LOGV - ვრცელი (ყველაზე მაღალი)

გარდა ამისა, არსებობს ESP_EARLY_LOGx თითოეული ამ მაკროსის ვერსიები, მაგ., ESP_EARLY_LOGEეს ვერსიები უნდა იქნას გამოყენებული მხოლოდ ადრეული გაშვების კოდში, სანამ heap allocator და syscalls ინიციალიზაციას მოახდენს. ნორმალური ESP_LOGx მაკროების გამოყენება ასევე შესაძლებელია ჩამტვირთავი კომპილაციის დროს, მაგრამ ისინი იმავე იმპლემენტაციას გამოიყენებენ, როგორც ESP_EARLY_LOGx მაკროები.

ფაილში ან კომპონენტი ფარგლებში ნაგულისხმევი დეტალიზაციის დონის გადასაჭრელად, განსაზღვრეთ LOG_LOCAL_LEVEL მაკრო.

ფაილის მასშტაბში, განსაზღვრეთ ის ჩართვამდე esp_log.h, მაგ.:

#define LOG_LOCAL_LEVEL ESP_LOG_VERBOSE
#include "esp_log.h"

კომპონენტი სკოპის ფარგლებში, განსაზღვრეთ ის კომპონენტი makefile-ში:

CFLAGS += -D LOG_LOCAL_LEVEL=ESP_LOG_DEBUG

გაშვების დროს თითოეული მოდულის ლოგირების გამომავალი მონაცემების კონფიგურაციისთვის, დაამატეთ გამოძახებები ფუნქციას. esp_log_level_set() შემდეგნაირად:

esp_log_level_set("*", ESP_LOG_ERROR);        // set all components to ERROR level
esp_log_level_set("wifi", ESP_LOG_WARN);      // enable WARN logs from WiFi stack
esp_log_level_set("dhcpc", ESP_LOG_INFO);     // enable INFO logs from DHCP client

ჰოსტზე შესვლა JTAG-ის საშუალებით¶

ნაგულისხმევად, ჟურნალის ბიბლიოთეკა იყენებს vprintf-ის მსგავს ფუნქციას ფორმატირებული გამომავალი მონაცემების სპეციალურ UART-ზე ჩასაწერად. მარტივი API გამოძახებით, ჟურნალის ყველა გამომავალი შეიძლება გადამისამართდეს JTAG-ზე, რაც ჟურნალის შენახვას რამდენჯერმე დააჩქარებს. დეტალებისთვის იხილეთ განყოფილება ჰოსტზე შესვლა.

განაცხადის მაგალითი¶

ჟურნალის ბიბლიოთეკა ხშირად გამოიყენება esp-idf კომპონენტი ის უმეტესობის მიერ და მაგალითებით. ჟურნალის ფუნქციონალურობის დემონსტრირებისთვის, შეამოწმეთ ESP-IDF . მაგალითები დირექტორია. ჟურნალირებასთან დაკავშირებული ყველაზე მნიშვნელოვანი მაგალითებია შემდეგი:

  • სისტემა/ოტა
  • მეხსიერება/sd_ბარათი
  • პროტოკოლები/https_მოთხოვნა

API მითითება¶

სათაურის ფაილი¶

  • log/include/esp_log.h

ფუნქციები¶

void esp_log_level_set(კონსტ char *ტეგი, esp_log_level_t დონე)¶

მოცემული თეგისთვის ჟურნალის დონის დაყენება.

თუ მოცემული კომპონენტი ისთვის ჟურნალირება უკვე ჩართულია, იცვლება წინა პარამეტრი.

გაითვალისწინეთ, რომ ამ ფუნქციას არ შეუძლია ლოგარითმის დონის აწევა menuconfig ში CONFIG_LOG_DEFAULT_LEVEL პარამეტრის გამოყენებით დაყენებულ დონეზე მაღლა.

მოცემული ფაილისთვის ლოგის დონის ნაგულისხმევ დონეზე მაღლა ასაწევად, ამ ფაილში esp_log.h-ის ჩართვამდე, LOG_LOCAL_LEVEL განსაზღვრეთ ESP_LOG_* მნიშვნელობებიდან ერთ-ერთად.

პარამეტრები
  • tag: ჩასართავი ჟურნალის ჩანაწერების თეგი. უნდა იყოს არა-NULL ნულით დასრულებული სტრიქონი. მნიშვნელობა „*“ აღადგენს ყველა თეგის ჟურნალის დონეს მოცემულ მნიშვნელობაზე.
  • level: ირჩევს ჩასართავად ჟურნალის დონეს. ნაჩვენები იქნება მხოლოდ ამ და უფრო დაბალი დეტალური დონის ჟურნალები.

vprintf_like_t esp_log_set_vprintf(vprintf_like_t ფუნქცია)¶

ჟურნალის ჩანაწერების გამოსატანად გამოყენებული ფუნქციის დაყენება.

ნაგულისხმევად, ჟურნალის გამომავალი მონაცემები UART0-ზე გადადის. ეს ფუნქცია შეიძლება გამოყენებულ იქნას ჟურნალის გამომავალი მონაცემების სხვა დანიშნულების ადგილას, მაგალითად ფაილში ან ქსელში გადამისამართებლად. აბრუნებს ჟურნალის საწყის დამმუშავებელს, რომელიც შეიძლება საჭირო იყოს გამომავალი მონაცემების წინა დანიშნულების ადგილას დასაბრუნებლად.

დაბრუნება
func old Function used for output.
პარამეტრები
  • funcგამომავალი ვერსიისთვის გამოყენებული ახალი ფუნქცია. უნდა ჰქონდეს იგივე ხელმოწერა, რაც vprintf-ს.

uint32_t esp_log_timestamp(void)¶

ფუნქცია, რომელიც აბრუნებს დროის ნიშნულს ჟურნალის გამომავალში გამოსაყენებლად.

ეს ფუნქცია გამოიყენება ESP_LOGx მაკროების გაფართოებისას. მე-2 ეტაპზე ჩამტვირთავი და აპლიკაციის ადრეულ გაშვების ეტაპზე ეს ფუნქცია დროის წყაროდ იყენებს CPU ციკლის მრიცხველს. მოგვიანებით, როდესაც Free RTOS დამგეგმავი იწყებს მუშაობას, ის გადადის Free RTOS tick count-ზე.

ამჟამად, ჩვენ უგულებელვყოფთ მილიწამიანი მრიცხველის გადავსებას.

დაბრუნება
timestamp, in milliseconds

char *esp_log_system_timestamp(void)¶

ფუნქცია, რომელიც აბრუნებს სისტემის დროის ნიშნულს ჟურნალის გამომავალში გამოსაყენებლად.

ეს ფუნქცია გამოიყენება ESP_LOGx მაკროების გაფართოებისას, რათა სისტემის დრო დაიბეჭდოს, როგორც „HH:MM:SS.ss“. გაშვებისას სისტემის დრო ინიციალიზდება 0-ზე, მისი სწორ დროზე დაყენება შესაძლებელია SNTP სინქრონიზაციით ან ხელით სტანდარტული POSIX დროის ფუნქციებით.

ამჟამად ეს არ იქნება გამოყენებული ბინარული ბლობებიდან (მაგ. WiFi და Bluetooth ბიბლიოთეკებიდან) ჟურნალირებისთვის, ისინი მაინც დაბეჭდავენ RTOS ის ტიკის დროს.

დაბრუნება
timestamp, in “HH:MM:SS.sss”

uint32_t esp_log_early_timestamp(void)¶

ფუნქცია, რომელიც აბრუნებს დროის ნიშნულს ჟურნალის გამომავალში გამოსაყენებლად.

ეს ფუნქცია იყენებს აპარატურის ციკლის მრიცხველს და არ არის დამოკიდებული ოპერაციულ სისტემაზე, ამიტომ მისი უსაფრთხოდ გამოყენება შესაძლებელია აპლიკაციის გაფუჭების შემდეგ.

timestamp, in milliseconds

ჩაწერეთ შეტყობინება ჟურნალში.

ეს ფუნქცია არ არის განკუთვნილი პირდაპირი გამოყენებისთვის. ამის ნაცვლად, გამოიყენეთ ESP_LOGE, ESP_LOGW, ESP_LOGI, ESP_LOGD, ESP_LOGV მაკროებიდან ერთ-ერთი.

ეს ფუნქცია ან ეს მაკროები არ უნდა იქნას გამოყენებული შეწყვეტიდან.

მაკროები¶

ESP_LOG_BUFFER_HEX_LEVEL(tag, buffer, buff_len, level)¶

მითითებულ დონეზე თექვსმეტობითი ბაიტების ბუფერის ჟურნალირება, რომელიც თითოეულ ხაზზე 16 ბაიტად არის გამოყოფილი.

  • buff_len: ბუფერის სიგრძე ბაიტებში
  • level: ჟურნალის დონე

ჩამოთვლები¶

ჩამოთვლა esp_log_level_t¶

ჟურნალის დონე.

ღირებულებები:

ESP_LOG_NONE¶

ჟურნალის გამომავალი არ არის

ESP_LOG_ERROR¶

კრიტიკული შეცდომები, პროგრამული უზრუნველყოფის მოდული ვერ აღდგება თავისით

ESP_LOG_WARN¶

შეცდომის პირობები, რომელთა აღმოფხვრის ზომები იქნა მიღებული

ESP_LOG_INFO¶

საინფორმაციო შეტყობინებები, რომლებიც აღწერს მოვლენების ნორმალურ მიმდინარეობას

ESP_LOG_DEBUG¶

დამატებითი ინფორმაცია, რომელიც არ არის აუცილებელი ნორმალური გამოყენებისთვის (მნიშვნელობები, მაჩვენებლები, ზომები და ა.შ.).

ESP_LOG_VERBOSE¶

გამართვა ის ინფორმაციის უფრო დიდი ნაწილები ან ხშირი შეტყობინებები, რომლებმაც პოტენციურად შეიძლება დატბოროს გამომავალი.

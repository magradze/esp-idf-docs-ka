შეწყვეტის განაწილება¶

მიმოხილვა¶

ESP32-ს აქვს ორი ბირთვი, თითოეულში 32 შეფერხებით. თითოეულ შეფერხებას აქვს გარკვეული პრიორიტეტის დონე, შეფერხებების უმეტესობა (მაგრამ არა ყველა) დაკავშირებულია შეფერხების მულტიპლექსთან. იმის გამო, რომ შეფერხებების წყაროები უფრო მეტია, ვიდრე შეფერხებები, ზოგჯერ აზრი აქვს შეფერხების გაზიარებას რამდენიმე დრაივერში. esp_intr_alloc აბსტრაქცია არსებობს ყველა ამ იმპლემენტაციის დეტალის დასამალად.

დრაივერს შეუძლია გარკვეული პერიფერიული მოწყობილობისთვის შეფერხების გამოყოფა esp_intr_alloc (ან esp_intr_alloc_sintrstatus) გამოძახებით. მას შეუძლია გამოიყენოს ამ ფუნქციაზე გადაცემული დროშები გამოყოფილი შეფერხების ტიპის დასაყენებლად, კონკრეტული დონის ან ტრიგერის მეთოდის მითითებით. შემდეგ შეფერხების გამოყოფის კოდი იპოვის შესაბამის შეფერხებას, გამოიყენებს შეფერხების მულტიპლექსს პერიფერიულ მოწყობილობასთან დასაკავშირებლად და დააყენებს მასზე მოცემულ შეფერხების დამმუშავებელს და ISR-ს.

ამ კოდს აქვს ორი განსხვავებული ტიპის შეფერხება, რომლებსაც ის სხვადასხვაგვარად ამუშავებს: გაზიარებული შეფერხებები და არაგაზიარებული შეფერხებები. ორიდან ყველაზე მარტივი არის არაგაზიარებული შეფერხებები: ცალკე შეფერხება გამოიყოფა თითოეული esp_intr_alloc გამოძახებისთვის და ეს შეფერხება გამოიყენება მხოლოდ მასზე მიერთებული პერიფერიული მოწყობილობისთვის, მხოლოდ ერთი ISR-ით, რომელიც გამოიძახება. გაზიარებულ შეფერხებებს შეიძლება ჰქონდეთ მრავალი პერიფერიული მოწყობილობა, რომელიც ააქტიურებს მას, მრავალი ISR-ით გამოიძახება, როდესაც ერთ-ერთი მიერთებული პერიფერიული მოწყობილობა შეფერხების შესახებ სიგნალს გასცემს. ამრიგად, გაზიარებული შეფერხებებისთვის განკუთვნილი ISR-ები უნდა შეამოწმონ მათ მიერ მომსახურებული პერიფერიული მოწყობილობის შეფერხების სტატუსი, რათა ნახონ, საჭიროა თუ არა რაიმე ქმედება.

არაგაზიარებული შეფერხებები შეიძლება იყოს როგორც დონის, ასევე კიდის ტრიგერით. გაზიარებული შეფერხებები შეიძლება იყოს მხოლოდ დონის შეფერხებები (კიდის შეფერხებების გამოყენებისას შეფერხებების გამოტოვების ალბათობის გამო). (ამის ლოგიკა შემდეგია: DevA და DevB იზიარებენ ერთ მთელ რიცხვს. DevB აგზავნის სიგნალს მთელ რიცხვზე. Int ხაზი გადადის მაღალ ნიშნულზე. ISR დამმუშავებელი იძახებს DevA-ს კოდს -&gt; არაფერს აკეთებს. ISR დამმუშავებელი იძახებს DevB-ს კოდს, მაგრამ ამის გაკეთებისას DevA აგზავნის სიგნალს მთელ რიცხვზე. ISR DevB დასრულებულია, ასუფთავებს DevB-ს მთელ რიცხვს, წყვეტს შეფერხების კოდს. ახლა DevA-ს შეფერხება კვლავ მოლოდინშია, მაგრამ რადგან მთელი ხაზი არასდროს დაბლა არ დადიოდა (DevA ინარჩუნებდა მას მაღალ ნიშნულზე, მაშინაც კი, როდესაც DevB-ს მთელი რიცხვი წაშლილი იყო), შეფერხება არასდროს მოგემსახურებათ.)

მრავალბირთვიანი პრობლემები¶

პერიფერიული მოწყობილობები, რომლებსაც შეუძლიათ შეფერხებების გენერირება, შეიძლება დაიყოს ორ ტიპად:

  • გარე პერიფერიული მოწყობილობები, ESP32-ის შიგნით, მაგრამ თავად Xtensa-ს ბირთვების გარეთ. ESP32-ის პერიფერიული მოწყობილობების უმეტესობა ამ ტიპისაა.
  • შიდა პერიფერიული მოწყობილობები, თავად Xtensa CPU ბირთვების ნაწილი.

შეფერხებების დამუშავება ამ ორი ტიპის პერიფერიულ მოწყობილობას შორის ოდნავ განსხვავდება.

შიდა პერიფერიული შეფერხებები¶

Xtensa-ს თითოეულ პროცესორის ბირთვს აქვს ექვსი შიდა პერიფერიული მოწყობილობის საკუთარი ნაკრები:

  • სამი ტაიმერის შედარების მოწყობილობა
  • შესრულების მონიტორი
  • ორი პროგრამული შეფერხება.

შიდა შეფერხების წყაროები esp_intr_alloc.h-ში განსაზღვრულია, როგორც ETS_INTERNAL_*_INTR_SOURCE.

ამ პერიფერიული მოწყობილობების კონფიგურაცია შესაძლებელია მხოლოდ იმ ბირთვიდან, რომელთანაც ისინი ასოცირდება. შეფერხების გენერირებისას, მათ მიერ გენერირებული შეფერხება მჭიდროდ არის დაკავშირებული მათთან დაკავშირებულ ბირთვთან; შეუძლებელია, მაგალითად, ერთი ბირთვის შიდა ტაიმერის შედარებამ წარმოქმნას შეფერხება მეორე ბირთვზე. სწორედ ამიტომ, ამ წყაროების მართვა შესაძლებელია მხოლოდ ამ კონკრეტულ ბირთვზე გაშვებული დავალების გამოყენებით. შიდა შეფერხების წყაროები კვლავ გამოყოფადია esp_intr_alloc-ის გამოყენებით, როგორც წესი, მაგრამ მათი გაზიარება შეუძლებელია და ყოველთვის ექნებათ ფიქსირებული შეფერხების დონე (კერძოდ, ის, რაც აპარატურაში ასოცირდება პერიფერიულ მოწყობილობასთან).

გარე პერიფერიული შეფერხებები¶

დარჩენილი შეფერხების წყაროები გარე პერიფერიული მოწყობილობებიდან არის. ისინი soc/soc.h-ში განსაზღვრულია, როგორც ETS_*_INTR_SOURCE.

ორივე CPU ბირთვის არაშიდა შეფერხების სლოტები დაკავშირებულია შეფერხების მულტიპლექსორთან, რომლის გამოყენებაც შესაძლებელია ნებისმიერი გარე შეფერხების წყაროს ამ შეფერხების სლოტებიდან ნებისმიერზე გადასამისამართებლად.

  • გარე შეფერხების გამოყოფა ყოველთვის გამოყოფს მას იმ ბირთვზე, რომელიც ახორციელებს გამოყოფას.
  • გარე შეფერხების გათავისუფლება ყოველთვის უნდა მოხდეს იმავე ბირთვზე, რომელზეც ის იყო გამოყოფილი.
  • დაშვებულია სხვა ბირთვიდან გარე შეფერხებების გამორთვა და ჩართვა.
  • მრავალ გარე შეფერხების წყაროს შეუძლია შეფერხების სლოტის გაზიარება გავლის გზით. ESP_INTR_FLAG_SHARED როგორც დროშა esp_intr_alloc()-ისთვის.

სიფრთხილეა საჭირო esp_intr_alloc()-ის გამოძახებისას დავალებიდან, რომელიც არ არის მიმაგრებული ბირთვზე. დავალებების გადართვის დროს, ამ დავალებებს შეუძლიათ ბირთვებს შორის მიგრაცია. ამიტომ, შეუძლებელია იმის გარკვევა, თუ რომელ CPU-ზეა გამოყოფილი შეფერხება, რაც ართულებს შეფერხების სახელურის გათავისუფლებას და ასევე შეიძლება გამოიწვიოს გამართვა ის მუშაობის სირთულეები. რეკომენდებულია xTaskCreatePinnedToCore()-ის გამოყენება კონკრეტული CoreID არგუმენტით, რათა შეიქმნას დავალებები, რომლებიც გამოყოფენ შეფერხებებს. შიდა შეფერხების წყაროების შემთხვევაში, ეს აუცილებელია.

IRAM-უსაფრთხო შეფერხების დამმუშავებლები¶

ის ESP_INTR_FLAG_IRAM flag არეგისტრირებს შეწყვეტის დამმუშავებელს, რომელიც ყოველთვის მუშაობს IRAM-დან (და კითხულობს ყველა მონაცემს DRAM-დან) და შესაბამისად, არ საჭიროებს გამორთვას ფლეშ მეხსიერება წაშლისა და ჩაწერის ოპერაციების დროს.

ეს სასარგებლოა შეფერხებებისთვის, რომლებსაც გარანტირებული მინიმალური შესრულების შეყოვნება სჭირდებათ, რადგან ფლეშ მეხსიერება ჩაწერის და წაშლის ოპერაციები შეიძლება ნელი იყოს (წაშლას შეიძლება ათობით ან ასობით მილიწამი დასჭირდეს).

ასევე სასარგებლო შეიძლება იყოს IRAM-ში შეწყვეტის დამმუშავებლის შენარჩუნება, თუ ის ძალიან ხშირად გამოიძახება, რათა თავიდან იქნას აცილებული ფლეშ მეხსიერება ქეშის შეცდომები.

იხილეთ SPI-ის ფლეშ მეხსიერება API დოკუმენტაცია დამატებითი დეტალებისთვის.

მრავალი დამმუშავებელი, რომლებიც წყაროს იზიარებენ¶

ერთსა და იმავე წყაროს შეიძლება რამდენიმე დამმუშავებელი დაენიშნოს, იმის გათვალისწინებით, რომ ყველა დამმუშავებელი გამოყოფილია შემდეგი ფუნქციის გამოყენებით: ESP_INTR_FLAG_SHARED დროშა. ისინი ყველა მიენიჭება შეფერხებას, რომელსაც წყარო უკავშირდება და თანმიმდევრულად გამოიძახება, როდესაც წყარო აქტიურია. დამმუშავებლების გამორთვა და გათავისუფლება შესაძლებელია ინდივიდუალურად. წყარო მიმაგრებულია შეფერხებაზე (ჩართულია), თუ ერთი ან მეტი დამმუშავებელი ჩართულია, წინააღმდეგ შემთხვევაში გამორთულია. დამმუშავებელი არასდროს გამოიძახება გამორთვის შემთხვევაში, სანამ მისი წყარო შეიძლება კვლავ გააქტიურებული იყოს თუ მისი რომელიმე დამმუშავებელი ჩართულია.

არაგაზიარებულ შეწყვეტასთან დაკავშირებული წყაროები ამ ფუნქციას არ უჭერენ მხარს.

მიუხედავად იმისა, რომ ჩარჩო მხარს უჭერს ამ ფუნქციას, თქვენ უნდა გამოიყენოთ იგი ძალიან ფრთხილადროგორც წესი, შეფერხების თავიდან ასაცილებლად 2 გზა არსებობს: წყაროს გამორთვა ან პერიფერიული პერიფერიული შეფერხების სტატუსის შენიღბვაIDF მხოლოდ წყაროს ჩართვა-გამორთვას ახორციელებს, სტატუსისა და ნიღბის ბიტებს კი მომხმარებლები ამუშავებენ. სტატუსის ბიტები ყოველთვის უნდა იყოს ნიღბიანი, სანამ მათზე პასუხისმგებელი დამმუშავებელი გამორთული იქნება, ან სტატუსი სათანადოდ უნდა დამუშავდეს სხვა ჩართულ შეფერხებაში.თუ სტატუსის ბიტების შენიღბვის გარეშე გამორთავთ ყველა დამმუშავებლიდან ერთ-ერთს, შესაძლოა სტატუსის ზოგიერთი ბიტი დაუმუშავებელი დარჩეს, რაც გამოიწვევს შეფერხების უსასრულო გააქტიურებას, რაც სისტემის კრახს გამოიწვევს.

შეფერხების გამოსართავად და მასთან დაკავშირებული რესურსების გასათავისუფლებლად გამოიყენეთ შეფერხების სახელური. თუ მიმდინარე ბირთვი არ არის ის ბირთვი, რომელმაც ეს შეფერხება დაარეგისტრირა, ეს რუტინა მიენიჭება იმ ბირთვს, რომელმაც ეს შეფერხება გამოყო, დაბლოკავს და დაელოდება რესურსის წარმატებით გათავისუფლებას.

შენიშვნა
When the handler shares its source with other handlers, the interrupt status bits it’s responsible for should be managed properly before freeing it. see esp_intr_disable for more details. Please do not call this function in esp_ipc_call_blocking.
დაბრუნება
ESP_ERR_INVALID_ARG the handle is NULL ESP_FAIL failed to release this handle ESP_OK otherwise
პარამეტრები
  • handle: სახელური, რომელიც მიღებულია esp_intr_alloc-ის ან esp_intr_alloc_intrstatus-ის მიერ

esp_err_t esp_intr_disable(intr_handle_t სახელური)¶

გამორთეთ სახელურთან დაკავშირებული შეფერხება.

შენიშვნა
  1. ლოკალური შეფერხებების შემთხვევაში (ESP_INTERNAL_* წყაროები), ეს ფუნქცია უნდა გამოიძახოს იმ CPU-ზე, რომელზეც შეფერხებაა გამოყოფილი. სხვა შეფერხებებს ასეთი შეზღუდვა არ აქვთ.
  2. როდესაც რამდენიმე დამმუშავებელი იყენებს ერთი და იგივე შეფერხების წყაროს, დამმუშავებელში გამორთვისთვის დამუშავებული შეფერხების სტატუსის ბიტები უნდა დაიფაროს გამორთვამდე ან სათანადოდ დამუშავდეს სხვა ჩართულ შეფერხებებში. შეფერხების სტატუსის დამუშავების შეუსრულებლობა გამოიწვევს შეფერხების უსასრულო გამოძახებებს და საბოლოოდ სისტემის კრახს.
დაბრუნება
ESP_ERR_INVALID_ARG if the combination of arguments is invalid. ESP_OK otherwise
პარამეტრები
  • handle: სახელური, რომელიც მიღებულია esp_intr_alloc-ის ან esp_intr_alloc_intrstatus-ის მიერ

შეწყვეტის განაწილების დროშები.

ეს დროშები შეიძლება გამოყენებულ იქნას იმის დასადგენად, თუ რომელი შეფერხების თვისებები სჭირდება esp_intr_alloc*-ის გამომძახებელ კოდს. მიიღეთ 1 დონის შეფერხების ვექტორი (ყველაზე დაბალი პრიორიტეტით).

Xtensa ტაიმერი 0 შეწყვეტის წყარო.

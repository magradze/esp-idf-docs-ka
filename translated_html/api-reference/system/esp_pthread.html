ESP-pthread¶

მიმოხილვა¶

This module offers Espressif specific extensions to the pthread library that can be used to influence the behaviour of pthreads. Currently the following configuration can be tuned:
  • pthreads-ის დასტის ზომა
  • შექმნილი pthreads-ის პრიორიტეტი
  • ამ კონფიგურაციის მემკვიდრეობით გადაცემა თემებს შორის
  • თემის სახელი
  • ბირთვის აფინურობა / ბირთვის პინინგირება.

მაგალითი pthread-ის დასტის ზომის რეგულირებისთვის:

void * thread_func(void * p)
{
    printf("In thread_func\n");
    return NULL;
}

void app_main(void)
{
    pthread_t t1;

    esp_pthread_cfg_t cfg = esp_create_default_pthread_config();
    cfg.stack_size = (4 * 1024);
    esp_pthread_set_cfg(&cfg);

    pthread_create(&t1, NULL, thread_func);
}

API ასევე შეიძლება გამოყენებულ იქნას პარამეტრების მემკვიდრეობით გადაცემისთვის სხვადასხვა ნაკადებს შორის. მაგალითად:

void * my_thread2(void * p)
{
    /* This thread will inherit the stack size of 4K */
    printf("In my_thread2\n");

    return NULL;
}

void * my_thread1(void * p)
{
    printf("In my_thread1\n");
    pthread_t t2;
    pthread_create(&t2, NULL, my_thread2);

    return NULL;
}

void app_main(void)
{
    pthread_t t1;

    esp_pthread_cfg_t cfg = esp_create_default_pthread_config();
    cfg.stack_size = (4 * 1024);
    cfg.inherit_cfg = true;
    esp_pthread_set_cfg(&cfg);

    pthread_create(&t1, NULL, my_thread1);
}

ფუნქციები¶

esp_pthread_cfg_t esp_pthread_get_default_config(void)¶

menuconfig მეშვეობით დაყენებული მნიშვნელობების საფუძველზე ქმნის pthread-ის ნაგულისხმევ კონფიგურაციას.

დაბრუნება
A default configuration structure.

esp_err_t esp_pthread_set_cfg(კონსტ esp_pthread_cfg_t *cfg)¶

pthread-ის შესაქმნელად პარამეტრების კონფიგურაცია.

ეს API საშუალებას გაძლევთ დააკონფიგურიროთ, თუ როგორ იმოქმედებს შემდგომი pthread_create() გამოძახება. ამ გამოძახების გამოყენება შესაძლებელია კონფიგურაციის პარამეტრების დასაყენებლად, როგორიცაა დასტის ზომა, პრიორიტეტი, კონფიგურაციის მემკვიდრეობა და ა.შ.

თუ კონფიგურაციის სტრუქტურაში „მემკვიდრეობის“ დროშა ჩართულია, მაშინ იგივე კონფიგურაცია მემკვიდრეობით მიიღება ძაფების ქვეხეშიც.

შენიშვნა
Passing non-NULL attributes to pthread_create() will override the stack_size parameter set using this API
დაბრუნება
  • ESP_OK თუ კონფიგურაცია წარმატებით დაყენდა
  • ESP_ERR_NO_MEM, თუ მეხსიერება ამოწურულია
  • ESP_ERR_INVALID_ARG თუ stack_size ნაკლებია PTHREAD_STACK_MIN-ზე
პარამეტრები
  • cfgpthread კონფიგურაციის პარამეტრები

esp_err_t esp_pthread_get_cfg(esp_pthread_cfg_t *გვ)¶

მიიღეთ pthread-ის შექმნის მიმდინარე კონფიგურაცია.

ეს მოიძიებს მიმდინარე კონფიგურაციას, რომელიც გამოყენებული იქნება თემების შესაქმნელად.

დაბრუნება
  • ESP_OK, თუ კონფიგურაცია ხელმისაწვდომი იყო
  • ESP_ERR_NOT_FOUND თუ კონფიგურაცია ადრე არ იყო დაყენებული
პარამეტრები
  • p: pthread კონფიგურაციის სტრუქტურის მაჩვენებელი, რომელიც განახლდება ამჟამად კონფიგურირებული პარამეტრებით.

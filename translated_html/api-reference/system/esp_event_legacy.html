esp_err_t esp_event_send(სისტემის_მოვლენა_t *ღონისძიება)¶

მოვლენის გაგზავნა მოვლენის დავალებისთვის.

სხვა დავალებებს/მოდულებს, როგორიცაა tcpip_adapter, შეუძლიათ გამოიძახონ ეს API , რათა გაგზავნონ მოვლენა event task-ში.

შენიშვნა
This API is part of the legacy event system. New code should use event library API in esp_event.h

დაბრუნება
ESP_OK : succeed
დაბრუნება
others : fail
პარამეტრები
  • event: გასაგზავნი ღონისძიება

esp_err_t esp_event_send_internal(esp_event_base_t ღონისძიების_ბაზა, int32_t მოვლენის_იდენტიფიკატორი, void *მოვლენის_მონაცემები, size_t მოვლენის_მონაცემების_ზომა, TickType_t ლოდინის დრო)¶

მოვლენის გაგზავნა მოვლენის დავალებისთვის.

სხვა დავალებებს/მოდულებს, როგორიცაა tcpip_adapter, შეუძლიათ გამოიძახონ ეს API , რათა გაგზავნონ მოვლენა event task-ში.

შენიშვნა
This API is used by WiFi Driver only.

დაბრუნება
ESP_OK : succeed
დაბრუნება
others : fail
პარამეტრები
  • event_base: მოვლენის ბაზა, რომელიც განსაზღვრავს მოვლენას
  • event_id: მოვლენის ID, რომელიც განსაზღვრავს მოვლენას
  • event_data: მოვლენის შემთხვევისთვის სპეციფიკური მონაცემები, რომლებიც გადაეცემა დამმუშავებელს.
  • event_data_size: მოვლენის მონაცემების ზომა
  • ticks_to_wait: სრულ ღონისძიებათა რიგში დასაბლოკი ტიკების რაოდენობა

esp_err_t esp_event_process_default(სისტემის_მოვლენა_t *ღონისძიება)¶

სისტემის მოვლენების ნაგულისხმევი დამმუშავებელი.

ეს ფუნქცია სისტემური მოვლენების ნაგულისხმევ დამუშავებას ასრულებს. esp_event_loop API s-ის გამოყენებისას, ის ავტომატურად გამოიძახება მომხმარებლის მიერ მოწოდებული უკუკავშირის ფუნქციის გამოძახებამდე.

შენიშვნა
This API is part of the legacy event system. New code should use event library API in esp_event.h

აპლიკაციებმა, რომლებიც ახორციელებენ მორგებულ მოვლენათა ციკლს, უნდა გამოიძახონ ეს ფუნქცია მოვლენების დამუშავების ნაწილად.

დაბრუნება
ESP_OK if an event was handled successfully
პარამეტრები
  • event: დამუშავებადი მოვლენის მითითება

SYSTEM_EVENT_STA_GOT_IP¶

ESP32 სადგურმა მიიღო IP მისამართი დაკავშირებული წვდომის წერტილიდან.

SYSTEM_EVENT_STA_LOST_IP¶

ESP32 სადგურმა დაკარგა IP მისამართი და IP მისამართი 0-ზე გადავიდა.

SYSTEM_EVENT_STA_WPS_ER_SUCCESS¶

ESP32 სადგური wps წარმატებით რეგისტრირდება რეჟიმში

SYSTEM_EVENT_STA_WPS_ER_FAILED¶

ESP32 სადგურის wps შეცდომას იწვევს რეგისტრაციის რეჟიმში

SYSTEM_EVENT_STA_WPS_ER_TIMEOUT¶

ESP32 სადგურის wps დროის ამოწურვა რეგისტრაციის რეჟიმში

SYSTEM_EVENT_STA_WPS_ER_PIN¶

ESP32 სადგურის wps PIN კოდი რეგისტრაციის რეჟიმში

SYSTEM_EVENT_STA_WPS_ER_PBC_OVERLAP¶

ESP32 სადგურის wps გადაფარვა რეგისტრაციის რეჟიმში

SYSTEM_EVENT_AP_START¶

ESP32 რბილი AP დაწყება

SYSTEM_EVENT_AP_STOP¶

ESP32 რბილი AP გაჩერება

SYSTEM_EVENT_AP_STACONNECTED¶

სადგური, რომელიც დაკავშირებულია ESP32 soft-AP-თან

SYSTEM_EVENT_AP_STADISCONNECTED¶

სადგური გათიშულია ESP32 soft-AP-დან

SYSTEM_EVENT_AP_STAIPASSIGNED¶

ESP32 soft-AP-ის მიერ დაკავშირებულ სადგურზე IP მისამართის მინიჭება

SYSTEM_EVENT_AP_PROBEREQRECVED¶

მიიღეთ ზონდის მოთხოვნის პაკეტი soft-AP ინტერფეისში

SYSTEM_EVENT_GOT_IP6¶

სასურველია ESP32 სადგური ან AP ან Ethernet ინტერფეისი v6IP მისამართი

SYSTEM_EVENT_ETH_START¶

ESP32 Ethernet-ის დაწყება

SYSTEM_EVENT_ETH_STOP¶

ESP32 Ethernet stop

SYSTEM_EVENT_ETH_CONNECTED¶

ESP32 Ethernet ფიზიკური კავშირი

SYSTEM_EVENT_ETH_DISCONNECTED¶

ESP32 Ethernet ფიზიკური კავშირის გათიშვა

SYSTEM_EVENT_ETH_GOT_IP¶

ESP32 Ethernet-მა მიიღო IP მისამართი დაკავშირებული წვდომის წერტილიდან.

SYSTEM_EVENT_MAX¶

ამ სიაში წევრების რაოდენობა

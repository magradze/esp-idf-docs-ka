პროცესორებს შორის ზარი¶

მიმოხილვა¶

ESP32-ის ორბირთვიანი ბუნებიდან გამომდინარე, არსებობს შემთხვევები, როდესაც კონკრეტული ფუნქცია უნდა შესრულდეს კონკრეტული ბირთვის კონტექსტში (მაგ. ISR-ის გამოყოფა კონკრეტული ბირთვის შეფერხების წყაროსთვის). IPC (პროცესორთაშორისი ზარის) ფუნქცია საშუალებას იძლევა ფუნქციების შესრულება კონკრეტულ CPU-ზე.

მოცემული ფუნქციის შესრულება კონკრეტულ ბირთვზე შესაძლებელია გამოძახებით esp_ipc_call() ან esp_ipc_call_blocking()IPC იმპლემენტირებულია ორი მაღალი პრიორიტეტის Free RTOS დავალების მეშვეობით, რომლებიც მიმაგრებულია თითოეულ CPU-ზე და ცნობილია როგორც IPC ამოცანები. ორი IPC ამოცანა რჩება არააქტიური (დაბლოკილი) მანამ, სანამ esp_ipc_call() ან esp_ipc_call_blocking() გამოიძახება. როდესაც კონკრეტული ბირთვის IPC დავალება განბლოკილია, ის წინასწარ დაბლოკავს ამ ბირთვზე მიმდინარე დავალებას და შეასრულებს მოცემულ ფუნქციას.

გამოყენება¶

esp_ipc_call() კონკრეტული ფუნქციის შესასრულებლად კონკრეტულ ბირთვზე IPC დავალებას ხსნის განბლოკვას. დავალება, რომელიც გამოიძახებს esp_ipc_call() დაბლოკილი იქნება მანამ, სანამ IPC დავალება არ დაიწყებს მოცემული ფუნქციის შესრულებას. esp_ipc_call_blocking() მსგავსია, მაგრამ დაბლოკავს გამოძახების დავალებას მანამ, სანამ IPC დავალება არ დაასრულებს მოცემული ფუნქციის შესრულებას.

IPC-ების მიერ შესრულებული ფუნქციები უნდა იყოს შემდეგი ტიპის: void func(void *arg)უფრო რთული ფუნქციების გასაშვებად, რომლებიც უფრო დიდ დასტას მოითხოვს, IPC დავალებების დასტის ზომის კონფიგურაცია შესაძლებელია შეცვლით. CONFIG_ESP_IPC_TASK_STACK_SIZE ში menuconfigIPC API დაცულია mutex-ით, ამიტომ ერთდროული IPC გამოძახებები შეუძლებელია.

IPC-ის მიერ შესასრულებელი ფუნქციების ჩაწერისას სიფრთხილეა საჭირო ჩიხის თავიდან ასაცილებლად, განსაკუთრებით ფუნქციაში მუტექსის აღების მცდელობისას.

ფუნქციები¶

esp_err_t esp_ipc_call(uint32_t cpu_id, esp_ipc_func_t ფუნქცია, void *არგუმენტი)¶

შეასრულეთ ფუნქცია მოცემულ CPU-ზე.

მოცემული ფუნქციის გაშვება კონკრეტულ CPU-ზე. მოცემულმა ფუნქციამ უნდა მიიღოს void* არგუმენტი და დააბრუნოს void. მოცემული ფუნქცია შესრულდება CPU-ს IPC დავალების კონტექსტში, რომელიც მითითებულია cpu_id პარამეტრით. გამომძახებელი დავალება დაიბლოკება მანამ, სანამ IPC დავალება არ დაიწყებს მოცემული ფუნქციის შესრულებას. თუ სხვა IPC გამოძახება მიმდინარეობს, გამომძახებელი დავალება დაიბლოკება მანამ, სანამ სხვა IPC გამოძახება არ დასრულდება. IPC დავალებისთვის გამოყოფილი დასტის ზომის კონფიგურაცია შესაძლებელია menuconfig ში „პროცესორთაშორისი ზარის (IPC) დავალებების დასტის ზომის“ პარამეტრში. გაზარდეთ ეს პარამეტრი, თუ მოცემული ფუნქცია მოითხოვს ნაგულისხმევზე მეტ დასტას.

შენიშვნა
In single-core mode, returns ESP_ERR_INVALID_ARG for cpu_id 1.
დაბრუნება
  • ESP_ERR_INVALID_ARG თუ cpu_id არასწორია
  • ESP_ERR_INVALID_STATE თუ Free RTOS დამგეგმავი არ მუშაობს
  • სხვა შემთხვევაში ESP_OK
პარამეტრები
  • cpu_id: CPU, სადაც მოცემული ფუნქცია უნდა შესრულდეს (0 ან 1)
  • func: void func(void* arg) ტიპის ფუნქციის მაჩვენებელი შესასრულებლად
  • argფუნქციაში გადასაცემად void* ტიპის თვითნებური არგუმენტი

esp_err_t esp_ipc_call_blocking(uint32_t cpu_id, esp_ipc_func_t ფუნქცია, void *არგუმენტი)¶

ფუნქციის შესრულება მოცემულ CPU-ზე და ბლოკირება მის დასრულებამდე.

მოცემული ფუნქციის გაშვება კონკრეტულ CPU-ზე. მოცემულმა ფუნქციამ უნდა მიიღოს void* არგუმენტი და დააბრუნოს void. მოცემული ფუნქცია შესრულდება CPU-ს IPC დავალების კონტექსტში, რომელიც მითითებულია cpu_id პარამეტრით. გამომძახებელი დავალება დაიბლოკება მანამ, სანამ IPC დავალება არ დაასრულებს მოცემული ფუნქციის შესრულებას. თუ მიმდინარეობს სხვა IPC გამოძახება, გამომძახებელი დავალება დაიბლოკება მანამ, სანამ სხვა IPC გამოძახება არ დაასრულებს. IPC დავალებისთვის გამოყოფილი დასტის ზომის კონფიგურაცია შესაძლებელია menuconfig ში „პროცესორთაშორისი ზარის (IPC) დავალებების დასტის ზომის“ პარამეტრში. გაზარდეთ ეს პარამეტრი, თუ მოცემული ფუნქცია მოითხოვს ნაგულისხმევზე მეტ დასტას.

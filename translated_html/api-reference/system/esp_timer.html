მაღალი გარჩევადობის ტაიმერი¶

მიმოხილვა¶

მიუხედავად იმისა, რომ Free RTOS გთავაზობთ პროგრამულ ტაიმერებს, ამ ტაიმერებს აქვთ რამდენიმე შეზღუდვა:

  • მაქსიმალური გარჩევადობა უდრის RTOS ტიკის პერიოდს
  • ტაიმერის უკუკავშირი იგზავნება დაბალი პრიორიტეტის დავალებიდან

აპარატურული ტაიმერები ორივე შეზღუდვისგან თავისუფალია, მაგრამ ხშირად მათი გამოყენება ნაკლებად მოსახერხებელია. მაგალითად, აპლიკაციის კომპონენტი -ს შეიძლება დასჭირდეს ტაიმერის მოვლენები მომავალში გარკვეულ დროს გასააქტიურებლად, მაგრამ აპარატურული ტაიმერი შეიცავს მხოლოდ ერთ „შედარების“ მნიშვნელობას, რომელიც გამოიყენება შეფერხების გენერირებისთვის. ეს ნიშნავს, რომ აპარატურული ტაიმერის თავზე უნდა აშენდეს რაიმე ობიექტი, რომელიც მართავს მომლოდინე მოვლენების სიას და შეძლებს ამ მოვლენებისთვის უკუკავშირის გაგზავნას შესაბამისი აპარატურული შეფერხებების დროს.

esp_timer API s-ის ნაკრები უზრუნველყოფს ერთჯერადი და პერიოდული ტაიმერებს, მიკროწამიან დროის გარჩევადობას და 64-ბიტიან დიაპაზონს.

შინაგანად, esp_timer იყენებს 64-ბიტიან აპარატურულ ტაიმერს CONFIG_ESP_TIMER_IMPL:

  • LAC ტაიმერი (ESP32)
  • (მემკვიდრეობითი) FRC2 ტაიმერი (ESP32)
  • SYSTIMER (ESP32-S2)-ისთვის

ტაიმერის უკუკავშირი იგზავნება მაღალი პრიორიტეტიდან esp_timer დავალება. რადგან ყველა უკუკავშირი ერთი და იგივე დავალებიდან იგზავნება, რეკომენდებულია უკუკავშირიდან მხოლოდ მინიმალური სამუშაოს შესრულება, მოვლენის რიგის გამოყენებით დაბალი პრიორიტეტის მქონე დავალებაზე განთავსებით.

თუ სხვა ამოცანები უფრო მაღალი პრიორიტეტით ხასიათდება, ვიდრე esp_timer მიმდინარეობს, უკუკავშირის გაგზავნა გადაიდება esp_timer დავალებას აქვს გაშვების შანსი. მაგალითად, ეს მოხდება, თუ მიმდინარეობს SPI Flash ოპერაცია.

ტაიმერის შექმნას და გაშვებას, ასევე უკუკავშირის გაგზავნას გარკვეული დრო სჭირდება. ამიტომ, ერთჯერადი გასროლის დროის ამოწურვის მნიშვნელობას უფრო დაბალი ზღვარი აქვს. esp_timerთუ esp_timer_start_once() თუ გამოძახების დრო 20us-ზე ნაკლებია, უკუკავშირი გაიგზავნება მხოლოდ დაახლოებით 20us-ის შემდეგ.

პერიოდული esp_timer ასევე მინიმალურ ტაიმერზე 50 მიკროსქემის შეზღუდვას აწესებს. 50 მიკროსქემაზე ნაკლები პერიოდის მქონე პერიოდული პროგრამული ტაიმერები პრაქტიკული არ არის, რადგან ისინი პროცესორის დროის უმეტეს ნაწილს მოიხმარენ. თუ მცირე პერიოდის მქონე ტაიმერი გჭირდებათ, განიხილეთ სპეციალური აპარატურული პერიფერიული მოწყობილობების ან DMA ფუნქციების გამოყენება.

გამოყენება esp_timer API სთ¶

ერთი ტაიმერი წარმოდგენილია შემდეგნაირად esp_timer_handle_t ტიპი. ტაიმერს აქვს უკუკავშირის ფუნქცია, რომელიც დაკავშირებულია მასთან. ეს უკუკავშირის ფუნქცია გამოიძახება შემდეგი სქემიდან: esp_timer დავალება ყოველ ჯერზე, როდესაც ტაიმერი ამოიწურება.

  • ტაიმერის შესაქმნელად, დარეკეთ esp_timer_create().
  • ტაიმერის წასაშლელად, როდესაც ის აღარ დაგჭირდებათ, დარეკეთ esp_timer_delete().

ტაიმერის ჩართვა შესაძლებელია ერთჯერადი ან პერიოდული რეჟიმით.

  • ტაიმერის ერთჯერადი რეჟიმში დასაწყებად, დარეკეთ esp_timer_start_once(), დროის ინტერვალის გავლის შემდეგ, რომლის შემდეგაც უნდა მოხდეს უკუკავშირის გამოძახება. როდესაც უკუკავშირი გამოიძახება, ტაიმერი გაჩერებულად ითვლება.
  • ტაიმერის პერიოდულ რეჟიმში დასაწყებად, დარეკეთ esp_timer_start_periodic(), რომელიც გადის იმ პერიოდს, რომლითაც უნდა მოხდეს უკუკავშირის გამოძახება. ტაიმერი განაგრძობს მუშაობას მანამ, სანამ esp_timer_stop() ეწოდება.

გაითვალისწინეთ, რომ ტაიმერი არ უნდა იყოს ჩართული, როდესაც esp_timer_start_once() ან esp_timer_start_periodic() გამოძახებულია. ტაიმერის გადასატვირთად, დარეკეთ esp_timer_stop() ჯერ, შემდეგ კი გამოიძახეთ ერთ-ერთი დაწყების ფუნქცია.

მიმდინარე დროის მიღება¶

esp_timer ასევე უზრუნველყოფს მოხერხებულ ფუნქციას, რომელიც მიკროწამების სიზუსტით იძლევა ჩართვის შემდეგ გასული დროის გაზომვის საშუალებას: esp_timer_get_time()ეს ფუნქცია აბრუნებს მიკროწამების რაოდენობას მას შემდეგ, რაც esp_timer ინიციალიზაცია მოხდა, რაც, როგორც წესი, ცოტა ხნით ადრე ხდება app_main ფუნქციას უწოდებენ.

განსხვავებით gettimeofday ფუნქცია, მნიშვნელობები, რომლებიც დაბრუნებულია esp_timer_get_time():

  • ნულიდან დაწყება ჩიპის ღრმა ძილიდან გაღვიძების შემდეგ
  • დროის სარტყლის ან DST-ის კორექტირება არ არის გამოყენებული

განაცხადის მაგალითი¶

შემდეგი მაგალითი ასახავს -ის გამოყენებას esp_timer API : სისტემა/esp_timer.

API მითითება¶

სათაურის ფაილი¶

  • esp_timer/include/esp_timer.h

ფუნქციები¶

esp_err_t esp_timer_init(void)¶

esp_timer ბიბლიოთეკის ინიციალიზაცია.

შენიშვნა
This function is called from startup code. Applications do not need to call this function before using other esp_timer APIs.
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_NO_MEM, თუ გამოყოფა ვერ მოხერხდა
  • ESP_ERR_INVALID_STATE თუ უკვე ინიციალიზებულია
  • სხვა შეცდომები შეწყვეტის განაწილებისგან

esp_err_t esp_timer_deinit(void)¶

esp_timer ბიბლიოთეკის დეინიციალიზაცია.

შენიშვნა
Normally this function should not be called from applications
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_STATE, თუ ჯერ არ არის ინიციალიზებული

esp_err_t esp_timer_create(კონსტ esp_timer_create_args_t *create_args, esp_timer_handle_t *გაუმკლავებელი)¶

შექმენით esp_timer ეგზემპლარი.

შენიშვნა
When done using the timer, delete it with esp_timer_delete function.
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_ARG, თუ create_args-ის ზოგიერთი ფუნქცია არასწორია.
  • ESP_ERR_INVALID_STATE თუ esp_timer ბიბლიოთეკა ჯერ არ არის ინიციალიზებული
  • ESP_ERR_NO_MEM, თუ მეხსიერების გამოყოფა ვერ მოხერხდა
პარამეტრები
  • create_args: მაჩვენებელი ტაიმერის შექმნის არგუმენტებით სტრუქტურაზე. არ ინახება ბიბლიოთეკის მიერ, შეიძლება გამოყოფილი იყოს სტეკზე.
  • out_handleგამომავალი, მაჩვენებელი esp_timer_handle_t ცვლადზე, რომელიც შეინახავს შექმნილ ტაიმერის სახელურს.

esp_err_t esp_timer_start_once(esp_timer_handle_t ტაიმერი, uint64_t ტაიმ-აუტი_us)¶

ერთჯერადი ტაიმერის ჩართვა.

ამ ფუნქციის გამოძახებისას ტაიმერი არ უნდა იყოს ჩართული.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_ARG თუ სახელური არასწორია
  • ESP_ERR_INVALID_STATE, თუ ტაიმერი უკვე მუშაობს
პარამეტრები
  • timer: ტაიმერის სახელური შექმნილია esp_timer_create-ის გამოყენებით
  • timeout_us: ტაიმერის დროის ამოწურვა, მიკროწამებში მიმდინარე მომენტთან მიმართებაში

esp_err_t esp_timer_start_periodic(esp_timer_handle_t ტაიმერი, uint64_t პერიოდი)¶

პერიოდული ტაიმერის ჩართვა.

ტაიმერი არ უნდა იყოს ჩართული ამ ფუნქციის გამოძახებისას. ეს ფუნქცია გაუშვებს ტაიმერს, რომელიც ყოველ „პერიოდულ“ მიკროწამს გაააქტიურებს.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_ARG თუ სახელური არასწორია
  • ESP_ERR_INVALID_STATE, თუ ტაიმერი უკვე მუშაობს
პარამეტრები
  • timer: ტაიმერის სახელური შექმნილია esp_timer_create-ის გამოყენებით
  • period: ტაიმერის პერიოდი, მიკროწამებში

esp_err_t esp_timer_stop(esp_timer_handle_t ტაიმერი)¶

ტაიმერის გაჩერება.

esp_err_t esp_timer_dump(FILE *ნაკადი)¶

ტაიმერების სიის ნაკადში გადატანა.

თუ CONFIG_ESP_TIMER_PROFILING ოფცია ჩართულია, ის ბეჭდავს ყველა არსებული ტაიმერის სიას. სხვა შემთხვევაში, იბეჭდება მხოლოდ აქტიური ტაიმერების სია.

ფორმატია:

სახელი პერიოდი განგაში times_arm times_triggered total_callback_run_time

სადაც:

სახელი — ტაიმერის სახელი (თუ განსაზღვრულია CONFIG_ESP_TIMER_PROFILING), ან ტაიმერის მაჩვენებლის პერიოდი — ტაიმერის პერიოდი, მიკროწამებში, ან 0 ერთჯერადი ტაიმერის განგაშისთვის - შემდეგი განგაშის დრო, მიკროწამებში ჩატვირთვის შემდეგ, ან 0, თუ ტაიმერი არ არის ჩართული

შემდეგი ველები იბეჭდება, თუ განსაზღვრულია CONFIG_ESP_TIMER_PROFILING:

times_armed — ტაიმერის გააქტიურების რაოდენობა esp_timer_start_X-ის მეშვეობით times_triggered - უკუკავშირის გამოძახების რაოდენობა total_callback_run_time - უკუკავშირის შესრულებისთვის საჭირო საერთო დრო, ყველა ზარისთვის

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_NO_MEM თუ გამომავალისთვის დროებითი ბუფერის გამოყოფა შეუძლებელია
პარამეტრები
  • stream: ნაკადი (მაგალითად, stdout) ინფორმაციის გადასაცემად

შესრულების მონიტორი¶

Performance Monitor კომპონენტი უზრუნველყოფს API ს ESP32 შიდა შესრულების მრიცხველების გამოყენებით ფუნქციებისა და აპლიკაციების პროფილირებისთვის.

განაცხადის მაგალითი¶

მაგალითი, რომელიც აერთიანებს შესრულების მონიტორს, მოცემულია შემდეგში: examples/system/perfmon დირექტორია. ეს მაგალითი ახდენს შესრულების მონიტორის სტრუქტურის ინიციალიზაციას და მათ სტატისტიკის დაბეჭდვით შესრულებას.

ფუნქციები¶

esp_err_t xtensa_perfmon_init(int ID, uint16_t შერჩევა, uint16_t ნიღაბი, int kernelcnt, int კვალის დონე)¶

Init-ის შესრულების მონიტორი.

შესრულების მონიტორის რეგისტრის ინიციალიზაცია განსაზღვრული მნიშვნელობებით

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_ARG, თუ ერთ-ერთი არგუმენტი არასწორია
პარამეტრები
  • id: შესრულების მრიცხველის ნომერი
  • select: აირჩიეთ მნიშვნელობა PMCTRLx რეგისტრიდან
  • mask: ნიღბის მნიშვნელობა PMCTRLx რეგისტრიდან
  • kernelcnt: kernelcnt მნიშვნელობა PMCTRLx რეგისტრიდან
  • tracelevel: PMCTRLx რეგისტრიდან tracelevel მნიშვნელობა

esp_err_t xtensa_perfmon_reset(int ID)¶

PM მრიცხველის გადატვირთვა.

PM მრიცხველის გადატვირთვა. PMx რეგისტრში 0-ის ჩაწერა.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_ARG თუ id დიაპაზონს სცილდება
პარამეტრები
  • id: შესრულების მრიცხველის ნომერი

void xtensa_perfmon_start(void)¶

PM მრიცხველების დაწყება.

ყველა PM მრიცხველის სინქრონულად გაშვება. PGM რეგისტრში 1-ის ჩაწერა

void xtensa_perfmon_stop(void)¶

შეაჩერე PM მთვლელები.

ყველა PM მრიცხველის სინქრონულად შეჩერება. PGM რეგისტრში 0-ის ჩაწერა.

uint32_t xtensa_perfmon_value(int ID)¶

წაიკითხეთ PM მრიცხველი.

განსაზღვრული PM მრიცხველის მნიშვნელობის წაკითხვა.

დაბრუნება
  • შესრულების მრიცხველის მნიშვნელობა
პარამეტრები
  • id: შესრულების მრიცხველის ნომერი

esp_err_t xtensa_perfmon_overflow(int ID)¶

წაიკითხეთ PM-ის გადავსების მდგომარეობა.

განსაზღვრული PM მრიცხველის გადავსების მნიშვნელობის წაკითხვა.

დაბრუნება
  • ESP_OK თუ არ არის გადავსება (overflow = 0)
  • ESP_FAIL თუ მოხდება გადავსება (overflow = 1)
პარამეტრები
  • id: შესრულების მრიცხველის ნომერი

void xtensa_perfmon_dump(void)¶

PM მნიშვნელობების გადატანა.

ყველა PM რეგისტრის გადატანა კონსოლში.

ფუნქციები¶

esp_err_t xtensa_perfmon_exec(კონსტ xtensa_perfmon_config_t *კონფიგურაცია)¶

PM-ის შესრულება.

განსაზღვრული პარამეტრებით გამოყოფილი ფუნქციის შესრულების მრიცხველის შესრულება

დაბრუნება
  • ESP_OK თუ შეცდომები არ არის
  • ESP_ERR_INVALID_ARG, თუ საჭირო პარამეტრებიდან ერთ-ერთი არ არის განსაზღვრული
  • ESP_FAIL - მრიცხველის გადავსება
პარამეტრები
  • config: კონფიგურაციის სტრუქტურის მაჩვენებელი

void xtensa_perfmon_view_cb(void *პარამეტრები, uint32_t შერჩევა, uint32_t ნიღაბი, uint32_t ღირებულება)¶

PM-ის შედეგების გადატანა.

უკუკავშირი perfmon-ის შედეგის FILE* ნაკადში გადასატანად, რომელიც მითითებულია perfmon_config_t::callback_params-ში. თუ callback_params დაყენებულია NULL-ზე, დაიბეჭდება stdout-ში.

პარამეტრები
  • params: გამოყენებული პარამეტრები, რომლებიც გადაცემულია კონფიგურაციიდან (callback_params). ეს პარამეტრი მოსალოდნელი იყო, როგორც FILE* hanle, სადაც მონაცემები შეინახება. თუ ეს პარამეტრი NULL-ია, მაშინ მონაცემები შეინახება stdout-ში.
  • select: აირჩიეთ მიმდინარე მრიცხველის მნიშვნელობა
  • mask: დენის მრიცხველის ნიღბის მნიშვნელობა
  • value: მიმდინარე მრიცხველის მრიცხველის მნიშვნელობა

სტრუქტურები¶

სტრუქტურა xtensa_perfmon_config¶

შესრულების მონიტორის კონფიგურაციის სტრუქტურა.

სტრუქტურა შესრულების მრიცხველის კონფიგურაციისთვის, რომელიც გაზომავს სპეციალურ ფუნქციას

საჯარო წევრები

int repeat_count¶

რამდენჯერ გამოიძახება ფუნქცია, სანამ calback გამეორდება

float max_deviation¶

მინიმალურ და მაქსიმალურ მრიცხველებს შორის სხვაობა 0..1, 0 - განსხვავება არ არის, 1 - არ გამოიყენება

void *call_params¶

ეს მაჩვენებელი გადაეცემა call_function-ს პარამეტრის სახით.

void (*call_function)(void *params)¶

მაჩვენებელი იმ ფუნქციისკენ, რომლის გამოძახებაც აუცილებელია

void (*callback)(void *params, uint32_t select, uint32_t mask, uint32_t value)¶

მაჩვენებელი იმ ფუნქციისკენ, რომელიც გამოიძახება შედეგის პარამეტრებით

void *callback_params¶

პარამეტრი, რომელიც გადაეცემა უკუკავშირს

int tracelevel¶

ყველა მრიცხველის კვალის დონე. უარყოფითი მნიშვნელობის შემთხვევაში, ფილტრი იგნორირებული იქნება. თუ ის &gt;=0-ია, მაშინ პერფმონი მხოლოდ მაშინ ჩაითვლება, როდესაც შეფერხების დონე &gt; კვალის დონეა. სასარგებლოა შეფერხებების მონიტორინგი.

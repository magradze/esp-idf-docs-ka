მეთვალყურე ძაღლები¶

მიმოხილვა¶

ESP-IDF აქვს ორი ტიპის მაკონტროლებელი ფუნქციის მხარდაჭერა: შეწყვეტის მაკონტროლებელი ტაიმერი და დავალებების მაკონტროლებელი ტაიმერი (TWDT). შეწყვეტის მაკონტროლებელი ტაიმერი და TWDT შეიძლება ჩაირთოს გამოყენებით პროექტის კონფიგურაციის მენიუთუმცა, TWDT-ის ჩართვა ასევე შესაძლებელია გაშვების დროს. Interrupt Watchdog პასუხისმგებელია იმ შემთხვევების აღმოჩენაზე, როდესაც Free RTOS დავალებების გადართვა დაბლოკილია ხანგრძლივი პერიოდის განმავლობაში. TWDT პასუხისმგებელია იმ დავალებების შემთხვევების აღმოჩენაზე, რომლებიც გაშვებულია შეუფერხებლად ხანგრძლივი პერიოდის განმავლობაში.

შეწყვეტის მცველი¶

შეწყვეტის მაკონტროლებელი უზრუნველყოფს, რომ Free RTOS დავალების გადართვის შეწყვეტა დიდი ხნის განმავლობაში არ დაიბლოკოს. ეს ცუდია, რადგან სხვა დავალებები, მათ შორის პოტენციურად მნიშვნელოვანი დავალებები, როგორიცაა WiFi დავალება და უმოქმედო დავალება, ვერ მიიღებენ CPU-ს გაშვების დროს. დაბლოკილი დავალების გადართვის შეწყვეტა შეიძლება მოხდეს იმის გამო, რომ პროგრამა შედის უსასრულო ციკლში გამორთული შეწყვეტებით ან იჭედება შეწყვეტაში.

შეფერხების დამკვირვებლის ნაგულისხმევი მოქმედებაა პანიკის დამმუშავებლის გამოძახება, რაც იწვევს რეგისტრის დემპს და პროგრამისტს აძლევს შესაძლებლობას, გაარკვიოს, OpenOCD-ის ან gdbstub-ის გამოყენებით, კოდის რომელი ნაწილია გაჭედილი გამორთული შეფერხებებით. პანიკის დამმუშავებლის კონფიგურაციიდან გამომდინარე, მას ასევე შეუძლია ბრმად გადატვირთოს CPU, რაც შეიძლება სასურველი იყოს საწარმოო გარემოში.

შეწყვეტის მცველი აგებულია ტაიმერის ჯგუფ 1-ში არსებული აპარატურის მცველის გარშემო. თუ ეს მცველი რაიმე მიზეზით ვერ შეასრულებს NMI დამმუშავებელს, რომელიც იწვევს პანიკის დამმუშავებელს (მაგალითად, იმიტომ, რომ IRAM გადაწერილია ნაგვის საშუალებით), ის მოახდენს SOC-ის მყარ გადატვირთვას.

დავალების მაკონტროლებელი ტაიმერი¶

დავალებების მაკონტროლებელი ტაიმერი (TWDT) პასუხისმგებელია ხანგრძლივი პერიოდის განმავლობაში შეუფერხებლად შესრულებული დავალებების შემთხვევების აღმოჩენაზე. ეს პროცესორის დეფიციტის სიმპტომია და, როგორც წესი, გამოწვეულია მაღალი პრიორიტეტის დავალების ციკლით, რომელიც არ ცვლის დაბალი პრიორიტეტის მქონე დავალებას, რითაც დაბალი პრიორიტეტის მქონე დავალებას დრო აკლია პროცესორს. ეს შეიძლება იყოს ცუდად დაწერილი კოდის მაჩვენებელი, რომელიც ბრუნავს ციკლურად პერიფერიულ მოწყობილობაზე, ან დავალების, რომელიც უსასრულო ციკლშია ჩარჩენილი.

ნაგულისხმევად, TWDT დააკვირდება თითოეული CPU-ს უმოქმედო დავალებებს, თუმცა ნებისმიერ დავალებას შეუძლია აირჩიოს TWDT-ის მიერ დაკვირვება. თითოეულ დაკვირვებულ დავალებას პერიოდულად უნდა „გადატვირთოს“ TWDT, რათა მიუთითოს, რომ მას გამოეყო CPU დრო. თუ დავალება არ გადაიტვირთება TWDT-ის ვადის ამოწურვის პერიოდში, დაიბეჭდება გაფრთხილება იმის შესახებ, თუ რომელი დავალებების შესრულება ვერ მოხერხდა TWDT-ის დროულად გადატვირთვაში და რომელი დავალებებია ამჟამად გაშვებული ESP32 CPU-ებზე. ასევე არსებობს ფუნქციის ხელახლა განსაზღვრის შესაძლებლობა. esp_task_wdt_isr_user_handler მომხმარებლის კოდში ამ მოვლენის მისაღებად.

TWDT აგებულია აპარატურის მაკონტროლებელი ტაიმერის გარშემო ტაიმერის 0 ჯგუფში. TWDT-ის ინიციალიზაცია შესაძლებელია გამოძახებით. esp_task_wdt_init() რომელიც დააკონფიგურირებს აპარატურულ ტაიმერს. შემდეგ დავალებას შეუძლია TWDT-ზე გამოწერა esp_task_wdt_add() თვალყურის დევნებისთვის. თითოეული გამოწერილი დავალება პერიოდულად უნდა გამოიძახოს esp_task_wdt_reset() TWDT-ის გადატვირთვისთვის. რომელიმე გამოწერილი დავალების პერიოდული გამოძახების შეუძლებლობა esp_task_wdt_reset() მიუთითებს, რომ ერთ ან მეტ დავალებას CPU დრო აკლია ან სადღაც ციკლშია გაჭედილი.

თვალყურადევნებული დავალების TWDT-დან გამოწერის გაუქმება შესაძლებელია შემდეგი მეთოდის გამოყენებით: esp_task_wdt_delete(). დავალება, რომლის გამოწერაც გაუქმდა, აღარ უნდა გამოიძახოს esp_task_wdt_reset()მას შემდეგ, რაც ყველა დავალება გაუქმდება TWDT-დან, TWDT-ის დეინიციალიზაცია შესაძლებელია გამოძახებით. esp_task_wdt_deinit().

ნაგულისხმევად CONFIG_ESP_TASK_WDT ში პროექტის კონფიგურაციის მენიუ ჩაირთვება, რაც გაშვებისას TWDT-ის ავტომატურ ინიციალიზაციას გამოიწვევს. ანალოგიურად CONFIG_ESP_TASK_WDT_CHECK_IDLE_TASK_CPU0 და CONFIG_ESP_TASK_WDT_CHECK_IDLE_TASK_CPU1 ასევე ნაგულისხმევად ჩართულია, რაც გაშვებისას იწვევს ორი უმოქმედო დავალების TWDT-ზე გამოწერას.

JTAG და მცველები¶

მიუხედავად იმისა, რომ გამართვა მუშაობს OpenOCD-ის გამოყენებით, პროცესორები შეჩერდება ყოველ ჯერზე, როდესაც შესვენების წერტილი მიიღწევა. თუმცა, თუ watchdog ტაიმერები გააგრძელებენ მუშაობას შესვენების წერტილის დადგომისას, ისინი საბოლოოდ გამოიწვებენ გადატვირთვას, რაც ძალიან გაართულებს გამართვა კოდის ამოღებას. ამიტომ, OpenOCD გამორთავს როგორც შეწყვეტის, ასევე დავალების watchdog-ების აპარატურულ ტაიმერებს ყოველ შესვენების წერტილში. გარდა ამისა, OpenOCD არ ჩართავს მათ ხელახლა შესვენების წერტილიდან გასვლის შემდეგ. ეს ნიშნავს, რომ შეწყვეტის watchdog-ის და დავალების watchdog-ის ფუნქციონირება არსებითად გამორთული იქნება. როდესაც ESP32 დაკავშირებულია OpenOCD-თან JTAG-ის საშუალებით, არცერთი watchdog-ისგან არ გენერირდება გაფრთხილებები ან პანიკა.

შეწყვეტის მეთვალყურე API მითითება¶

სათაურის ფაილი¶

  • esp32/include/esp_int_wdt.h

ფუნქციები¶

void esp_int_wdt_init(void)¶

ინტერფუტის მაკონტროლებელი ფუნქციის (interrupt watchdog) არაპროცესორული ნაწილების ინიციალიზაცია. ეს ფუნქცია გამოიძახება ინიციაციის კოდში, თუ interrupt watchdog ჩართულია menuconfig ში.

დავალების მეთვალყურე API მითითება¶

Task Watchdog-ის გამოყენების სრული მაგალითი ხელმისაწვდომია esp-idf-ში: სისტემა/დავალების_მეთვალყურე

სათაურის ფაილი¶

  • esp_common/include/esp_task_wdt.h

ფუნქციები¶

esp_err_t esp_task_wdt_init(uint32_t ტაიმ-აუტი, bool პანიკა)¶

დავალებების მაკონტროლებელი ტაიმერის (TWDT) ინიციალიზაცია

ეს ფუნქცია ახდენს TWDT-ის კონფიგურაციას და ინიციალიზაციას. თუ TWDT უკვე ინიციალიზებულია ამ ფუნქციის გამოძახებისას, ეს ფუნქცია განაახლებს TWDT-ის ვადის ამოწურვის პერიოდს და პანიკის კონფიგურაციებს. TWDT-ის ინიციალიზაციის შემდეგ, ნებისმიერ დავალებას შეუძლია აირჩიოს TWDT-ის მიერ მისი მონიტორინგი esp_task_wdt_add() ფუნქციის გამოყენებით გამოწერით.

დაბრუნება
  • ESP_OK: ინიციალიზაცია წარმატებით დასრულდა
  • ESP_ERR_NO_MEM: ინიციალიზაცია ვერ მოხერხდა მეხსიერების ნაკლებობის გამო.
შენიშვნა
esp_task_wdt_init() must only be called after the scheduler started
პარამეტრები
  • timeoutTWDT-ის ვადის ამოწურვის პერიოდი წამებში
  • panicდროშა, რომელიც აკონტროლებს, შესრულდება თუ არა პანიკის დამმუშავებელი TWDT-ის ვადის ამოწურვისას.

esp_err_t esp_task_wdt_deinit(void)¶

დავალების მაკონტროლებელი ტაიმერის (TWDT) დეინიციალიზაცია

ეს ფუნქცია TWDT-ს დეინიციალიზაციას მოახდენს. ამ ფუნქციის გამოძახება მაშინ, როდესაც დავალებები ჯერ კიდევ გამოწერილია TWDT-ზე, ან როდესაც TWDT უკვე დეინიციალიზებულია, გამოიწვევს შეცდომის კოდის დაბრუნებას.

დაბრუნება
  • ESP_OK: TWDT წარმატებით დეინიციალიზებულია
  • ESP_ERR_INVALID_STATE: შეცდომა, დავალებები კვლავ გამოწერილია TWDT-ზე.
  • ESP_ERR_NOT_FOUND: შეცდომა, TWDT უკვე დეინიციალიზებულია.

esp_err_t esp_task_wdt_add(TaskHandle_t სახელური)¶

დავალების გამოწერა დავალებების მაკონტროლებელ ტაიმერზე (TWDT)

ეს ფუნქცია TWDT-ზე დავალებას აწერს ხელს. თითოეული აწერილი დავალება პერიოდულად უნდა გამოიძახოს esp_task_wdt_reset() ფუნქციამ, რათა TWDT-მ არ გაავლოს თავისი ვადის ამოწურვის პერიოდი. წინააღმდეგ შემთხვევაში, TWDT-ის ვადის ამოწურვა მოხდება. თუ აწერილი დავალება Idle Tasks-ის ნაწილია, ეს ფუნქცია ავტომატურად ჩართავს esp_task_wdt_reset() ფუნქციის გამოძახებას Idle Task-ის Idle Hook-დან. ამ ფუნქციის გამოძახება, როდესაც TWDT არ არის ინიციალიზებული ან უკვე აწერილი დავალების აწერის მცდელობა გამოიწვევს შეცდომის კოდის დაბრუნებას.

დაბრუნება
  • ESP_OK: დავალება წარმატებით გამოიწერა TWDT-ში
  • ESP_ERR_INVALID_ARG: შეცდომა, დავალება უკვე გამოწერილია
  • ESP_ERR_NO_MEM: შეცდომა, დავალების გამოწერა ვერ მოხერხდა მეხსიერების ნაკლებობის გამო.
  • ESP_ERR_INVALID_STATE: შეცდომა, TWDT ჯერ არ არის ინიციალიზებული
პარამეტრები
  • handleდავალების სახელური. შეიყვანეთ NULL მიმდინარე გაშვებული დავალების TWDT-ზე გამოსაწერად.

esp_err_t esp_task_wdt_reset(void)¶

გადატვირთეთ დავალებების მაკონტროლებელი ტაიმერი (TWDT) მიმდინარე დავალების სახელით.

ეს ფუნქცია გადატვირთავს TWDT-ს მიმდინარე დავალების სახელით. თითოეულმა გამოწერილმა დავალებამ პერიოდულად უნდა გამოიძახოს ეს ფუნქცია, რათა თავიდან აიცილოს TWDT-ის ვადის გასვლა. თუ ერთი ან მეტი გამოწერილი დავალება ვერ შეძლებს TWDT-ის საკუთარი სახელით გადატვირთვას, მოხდება TWDT ვადის გასვლა. თუ IDLE დავალებები გამოწერილია TWDT-ზე, ისინი ავტომატურად გამოიძახებენ ამ ფუნქციას მათი უმოქმედო კაუჭებიდან. ამ ფუნქციის გამოძახება დავალებიდან, რომელიც არ არის გამოწერილი TWDT-ზე, ან როდესაც TWDT არ არის ინიციალიზებული, გამოიწვევს შეცდომის კოდის დაბრუნებას.

დაბრუნება
  • ESP_OK: წარმატებით გადატვირთულია TWDT მიმდინარე დავალების სახელით.
  • ESP_ERR_NOT_FOUND: შეცდომა, მიმდინარე გაშვებული დავალება არ არის გამოწერილი TWDT-ზე.
  • ESP_ERR_INVALID_STATE: შეცდომა, TWDT ჯერ არ არის ინიციალიზებული

esp_err_t esp_task_wdt_delete(TaskHandle_t სახელური)¶

დავალების გამოწერის გაუქმება Task Watchdog Timer-დან (TWDT)

ეს ფუნქცია გააუქმებს დავალებას TWDT-დან. გამოწერის გაუქმების შემდეგ, დავალებამ აღარ უნდა გამოიძახოს esp_task_wdt_reset(). თუ დავალება IDLE დავალებაა, ეს ფუნქცია ავტომატურად გამორთავს esp_task_wdt_reset()-ის გამოძახებას Idle Hook-დან. ამ ფუნქციის გამოძახება მაშინ, როდესაც TWDT არ არის ინიციალიზებული ან TWDT-დან უკვე გაუქმებული დავალების გაუქმების მცდელობა გამოიწვევს შეცდომის კოდის დაბრუნებას.

დაბრუნება
  • ESP_OK: დავალების TWDT-დან გამოწერა წარმატებით გაუქმდა.
  • ESP_ERR_INVALID_ARG: შეცდომა, დავალება უკვე გაუქმებულია
  • ESP_ERR_INVALID_STATE: შეცდომა, TWDT ჯერ არ არის ინიციალიზებული
პარამეტრები
  • handleდავალების სახელური. მიმდინარე გაშვებული დავალების გამოწერის გასაუქმებლად შეიყვანეთ NULL.

esp_err_t esp_task_wdt_status(TaskHandle_t სახელური)¶

მოითხოვეთ, გამოწერილია თუ არა დავალება დავალებების მაკონტროლებელ ტაიმერზე (TWDT)

ეს ფუნქცია გკითხავთ, ამჟამად დავალება გამოწერილია თუ არა TWDT-ზე, თუ TWDT ინიციალიზებულია.

დაბრუნება
:
  • ESP_OK: დავალება ამჟამად TWDT-ზეა გამოწერილი.
  • ESP_ERR_NOT_FOUND: დავალება ამჟამად არ არის გამოწერილი TWDT-ზე.
  • ESP_ERR_INVALID_STATE: TWDT არ არის ინიციალიზებული, ამიტომ დავალებების გამოწერა შეუძლებელია.
პარამეტრები
  • handleდავალების სახელური. მიმდინარე გაშვებული დავალების შესამოწმებლად შეიყვანეთ NULL.

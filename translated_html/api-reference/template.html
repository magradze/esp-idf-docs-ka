API დოკუმენტაციის შაბლონი¶

შენიშვნა

ინსტრუქციები

  1. გამოიყენეთ ეს ფაილი (docs/api-reference/template.rst) როგორც შაბლონი API დოკუმენტირებისთვის.

  2. შეცვალეთ ფაილის სახელი სათაურის ფაილის სახელით, რომელიც წარმოადგენს დოკუმენტირებულ API .

  3. API საქაღალდიდან შესაბამისი ფაილების აღწერილობებით ჩართვა ..include::

    • README.rst
    • example.rst
    • …

  4. სურვილისამებრ, აღწერილობა პირდაპირ ამ ფაილში შეგიძლიათ მოგვაწოდოთ.

  5. დასრულების შემდეგ, წაშალეთ ყველა მსგავსი ინსტრუქცია და ნებისმიერი ზედმეტი სათაური.

მიმოხილვა¶

შენიშვნა

ინსტრუქციები

  1. მოგვაწოდეთ მიმოხილვა, თუ სად და როგორ შეიძლება ამ API გამოყენება.

  2. სადაც შესაძლებელია, ჩართეთ კოდის ფრაგმენტები კონკრეტული ფუნქციების ფუნქციონალურობის საილუსტრაციოდ.

  3. სექციებს შორის განასხვავებლად, გამოიყენეთ შემდეგი სათაურების დონეები:

    • # ზედა ხაზით, ნაწილებისთვის
    • * თავებისთვის, ზედსტრიქონით
    • =, სექციებისთვის
    • -, ქვეგანყოფილებებისთვის
    • ^, ქვესექციებისთვის
    • ", აბზაცებისთვის

განაცხადის მაგალითი¶

შენიშვნა

ინსტრუქციები

  1. მოამზადეთ ერთი ან მეტი პრაქტიკული მაგალითი ამ API ფუნქციონალურობის დემონსტრირებისთვის8.
  2. თითოეული მაგალითი უნდა მიჰყვებოდეს პროექტების ნიმუშს, რომლებიც განთავსებულია esp-idf/examples/ საქაღალდე.
  3. მოათავსეთ მაგალითი ამ საქაღალდეში, შეავსეთ README.md ფაილი.
  4. წარმოადგინეთ დემონსტრირებული ფუნქციონალურობის მიმოხილვა README.md.
  5. კარგი მიმოხილვის შემთხვევაში, მკითხველს უნდა შეეძლოს იმის გაგება, თუ რას აკეთებს მაგალითი საწყისი კოდის გახსნის გარეშე.
  6. მაგალითის სირთულიდან გამომდინარე, კოდის აღწერა ნაწილებად დაყავით და თითოეული ნაწილის ფუნქციონალურობის მიმოხილვა მოგვაწოდეთ.
  7. საჭიროების შემთხვევაში, დაურთეთ ნაკადის დიაგრამა და აპლიკაციის გამომავალი სკრინშოტები.
  8. და ბოლოს, ამ განყოფილებაში დაამატეთ თითოეული მაგალითის მოკლე შინაარსი შესაბამის საქაღალდეზე ბმულთან ერთად. esp-idf/examples/.

API მითითება¶

შენიშვნა

ინსტრუქციები

  1. ეს საცავი უზრუნველყოფს API საცნობარო დოკუმენტაციის ავტომატურ განახლებას კოდის მარკირება ამოღებულია Doxygen-ის მიერ სათაურის ფაილებიდან.

  2. განახლება ხდება თითოეული დოკუმენტაციის აწყობაზე სკრიპტის გამოძახებით. docs/gen-dxd.py ყველა სათაურის ფაილისთვის, რომელიც ჩამოთვლილია INPUT განცხადება docs/Doxyfile.

  3. თითოეული ხაზი INPUT განცხადება (გარდა კომენტარისა, რომელიც იწყება ##) შეიცავს სათაურის ფაილისკენ მიმავალ გზას *.h რომელიც გამოყენებული იქნება შესაბამისის შესაქმნელად *.inc ფაილები:

    ##
## Wi-Fi - API Reference
##
../components/esp32/include/esp_wifi.h \
../components/esp32/include/esp_smartconfig.h \

  4. ის *.inc ფაილები შეიცავს API წევრების ფორმატირებულ მითითებებს, რომლებიც ავტომატურად გენერირდება დოკუმენტაციის თითოეულ აწყობაზე. ყველა *.inc ფაილები განთავსებულია Sphinx-ში _build დირექტორია. მაგ. გენერირებული დირექტივების სანახავად esp_wifi.h, სირბილი python gen-dxd.py esp32/include/esp_wifi.h.

  5. შინაარსის საჩვენებლად *.inc ფაილი დოკუმენტაციაში, შეიტანეთ იგი შემდეგნაირად:

    .. include:: /_build/inc/esp_wifi.inc

    მაგალითად, იხილეთ docs/en/api-reference/wifi/esp_wifi.rst

  6. სურვილისამებრ, გამოყენების ნაცვლად *.inc ფაილები, შეიძლება დაგჭირდეთ API საკუთარი გზით აღწერა. იხილეთ docs/en/api-guides/ulp.rst მაგალითად.

    ქვემოთ მოცემულია საერთო სია .. doxygen...:: დირექტივები:

    • ფუნქციები - .. doxygenfunction:: name_of_function
    • პროფკავშირები -.. doxygenunion:: name_of_union
    • სტრუქტურები -.. doxygenstruct:: name_of_structure ერთად :members:
    • მაკროები - .. doxygendefine:: name_of_define
    • ტიპის განმარტებები - .. doxygentypedef:: name_of_type
    • ჩამოთვლები - .. doxygenenum:: name_of_enumeration

    იხილეთ Breathe-ის დოკუმენტაცია დამატებითი ინფორმაციისთვის.

    სათაურის ფაილზე ბმულის მისაწოდებლად გამოიყენეთ ბმული მორგებული როლი შემდეგნაირად:

    * :component_file:`path_to/header_file.h`

  7. ნებისმიერ შემთხვევაში, API მითითების გენერირებისთვის, ფაილი docs/Doxyfile უნდა განახლდეს ბილიკებით *.h სათაურები, რომლებიც დოკუმენტირებულია.

  8. როდესაც ცვლილებები განხორციელდება და დოკუმენტაცია შეიქმნება, შეამოწმეთ, თუ როგორ არის ეს სექცია რენდერირებული. სწორი ანოტაციები შესაბამის სათაურის ფაილებში, საჭიროების შემთხვევაში.

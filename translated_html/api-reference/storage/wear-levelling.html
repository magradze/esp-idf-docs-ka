ცვეთის გასწორების API¶

[中文]

მიმოხილვა¶

ფლეშ მეხსიერება მეხსიერების უმეტესი ნაწილი და განსაკუთრებით ESP32-ში გამოყენებული SPI ფლეშ მეხსიერება სექტორზე დაფუძნებული ორგანიზაციაა და ასევე მეხსიერების სექტორში წაშლის/მოდიფიკაციის ციკლების შეზღუდული რაოდენობა აქვს. ცვეთის გათანაბრების კომპონენტი ხელს უწყობს ცვეთის უფრო თანაბრად გადანაწილებას სექტორებს შორის, მომხმარებლის მხრიდან რაიმე ყურადღების მოთხოვნის გარეშე.

ცვეთის გასწორების კომპონენტი უზრუნველყოფს API ფუნქციებს, რომლებიც დაკავშირებულია მონაცემების წაკითხვასთან, ჩაწერასთან, წაშლასთან და მეხსიერების შესაბამისობაში მოყვანასთან გარე SPI ფლეშ მეხსიერება ში დანაყოფის კომპონენტი მეშვეობით. კომპონენტი ს ასევე აქვს უფრო მაღალი დონის API ფუნქციები, რომლებიც მუშაობენ FAT ფაილურ სისტემასთან, რომელიც განსაზღვრულია FAT ფაილური სისტემა.

ცვეთის გასწორების კომპონენტი , FAT FS კომპონენტი თან ერთად, იყენებს 4096 ბაიტის FAT FS სექტორებს, რაც ფლეშ მეხსიერება მეხსიერების სტანდარტული ზომაა. ამ ზომით, კომპონენტი აჩვენებს საუკეთესო მუშაობას, მაგრამ საჭიროებს დამატებით მეხსიერებას ოპერატიულ მეხსიერებაში.

შიდა მეხსიერების დაზოგვის მიზნით, კომპონენტი აქვს ორი დამატებითი რეჟიმი, რომლებიც ორივე 512 ბაიტის სექტორებს იყენებენ:

  • შესრულების რეჟიმი. წაშლის სექტორის ოპერაციის მონაცემები ინახება ოპერატიულ მეხსიერებაში, სექტორი იშლება და შემდეგ მონაცემები კოპირდება ფლეშ მეხსიერება მეხსიერებაში. თუმცა, თუ მოწყობილობა რაიმე მიზეზით გამოირთვება, მონაცემების ყველა 4096 ბაიტი იკარგება.
  • უსაფრთხოების რეჟიმი. მონაცემები თავდაპირველად ფლეშ მეხსიერება მეხსიერებაში ინახება და სექტორის წაშლის შემდეგ მონაცემები ისევ ინახება. თუ მოწყობილობა გამორთულია, მონაცემების აღდგენა მოწყობილობის ჩართვისთანავე იქნება შესაძლებელი.

ნაგულისხმევი პარამეტრებია შემდეგი: - სექტორის ზომაა 512 ბაიტი - შესრულების რეჟიმი

პარამეტრების შეცვლა შეგიძლიათ კონფიგურაციის მენიუს მეშვეობით.

ცვეთის გასწორების კომპონენტი არ ინახავს მონაცემებს ოპერატიულ მეხსიერებაში. ჩაწერის და წაშლის ფუნქციები პირდაპირ ცვლის ფლეშ მეხსიერება და ფლეშ მეხსიერება შინაარსი თანმიმდევრულია, როდესაც ფუნქცია აბრუნებს.

ცვეთის გასწორების წვდომის API ფუნქცია¶

ეს არის API ფუნქციების ნაკრები ფლეშ მეხსიერება ში მონაცემებთან სამუშაოდ:

  • wl_mount - ახდენს ცვეთის გასწორების მოდულის ინიციალიზაციას და მითითებულ დანაყოფს ამონტაჟებს
  • wl_unmount - ახდენს დანაყოფის მოდულის დეინიციალიზაციას და ცვეთის გასწორების მოდულის დეინიციალიზაციას
  • wl_erase_range - შლის მისამართების დიაპაზონს ფლეშ მეხსიერება ში
  • wl_write - მონაცემებს წერს დანაყოფზე
  • wl_read - კითხულობს მონაცემებს დანაყოფიდან
  • wl_size - აბრუნებს ხელმისაწვდომი მეხსიერების ზომას ბაიტებში
  • wl_sector_size - აბრუნებს ერთი სექტორის ზომას

როგორც წესი, ეცადეთ თავი აარიდოთ ნედლი ცვეთის გასწორების ფუნქციების გამოყენებას და ამის ნაცვლად გამოიყენოთ ფაილური სისტემისთვის სპეციფიკური ფუნქციები.

მეხსიერების ზომა¶

მეხსიერების ზომა გამოითვლება ცვეთის გასწორების მოდულში დანაყოფის პარამეტრების საფუძველზე. მოდული იყენებს ფლეშ მეხსიერება ის ზოგიერთ სექტორს შიდა მონაცემებისთვის.

აგრეთვე იხილეთ¶

  • FAT ფაილური სისტემა
  • დანაყოფების ცხრილის დოკუმენტაცია

განაცხადის მაგალითი¶

მაგალითი, რომელიც აერთიანებს ცვეთის გასწორების დრაივერს FATFS ბიბლიოთეკასთან, მოცემულია შემდეგში: შენახვა/ცვეთის გასწორება დირექტორია. ეს მაგალითი ახდენს ცვეთის გასწორების დრაივერის ინიციალიზაციას, FATFS დანაყოფს ამონტაჟებს, ასევე წერს და კითხულობს მონაცემებს მისგან POSIX-ისა და C ბიბლიოთეკის API ის გამოყენებით. იხილეთ შენახვა/wear_levelling/README.md ფაილი დამატებითი ინფორმაციისთვის.

სათაურის ფაილები¶

ფუნქციები¶

esp_err_t esp_vfs_fat_spiflash_mount(კონსტ char *ბაზისური_გზა, კონსტ char *დანაყოფის_ეტიკეტი, კონსტ esp_vfs_fat_mount_config_t *mount_config, wl_handle_t *wl_handle)¶

მოხერხებულობის ფუნქცია FAT ფაილური სისტემის SPI ფლეშ მეხსიერება ში ინიციალიზაციისთვის და VFS-ში რეგისტრაციისთვის.

ეს არის ყოვლისმომცველი ფუნქცია, რომელიც ასრულებს შემდეგ ფუნქციებს:

  • პოულობს დანაყოფს განსაზღვრული partition_label-ით. დანაყოფის ეტიკეტი უნდა იყოს კონფიგურირებული დანაყოფების ცხრილი ში.
  • მოცემული დანაყოფის თავზე ფლეშ მეხსიერება ცვეთის გასწორების ბიბლიოთეკის ინიციალიზაციას ახდენს
  • FAT დანაყოფს ამონტაჟებს FATFS ბიბლიოთეკის გამოყენებით ფლეშ მეხსიერება ცვეთის გასწორების ბიბლიოთეკის თავზე.
  • არეგისტრირებს FATFS ბიბლიოთეკას VFS-თან, პრეფიქსით, რომელიც მოცემულია base_prefix ცვლადის მიერ.

ეს ფუნქცია განკუთვნილია მაგალითის კოდის უფრო კომპაქტური გახადოს.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_NOT_FOUND თუ დანაყოფების ცხრილი არ შეიცავს მოცემული ლეიბლით FATFS დანაყოფს
  • ESP_ERR_INVALID_STATE თუ esp_vfs_fat_spi ფლეშ მეხსიერება _mount უკვე გამოძახებული იყო
  • ESP_ERR_NO_MEM თუ მეხსიერების გამოყოფა შეუძლებელია
  • ESP_FAIL თუ დანაყოფის დამონტაჟება შეუძლებელია
  • სხვა შეცდომის კოდები ცვეთის გასწორების ბიბლიოთეკიდან, SPI ფლეშ მეხსიერება დრაივერიდან ან FATFS დრაივერებიდან
პარამეტრები
  • base_path: გზა, სადაც უნდა იყოს დამონტაჟებული FATFS დანაყოფი (მაგ. „/spi ფლეშ მეხსიერება “)
  • partition_label: გამოყენებული დანაყოფის ეტიკეტი
  • mount_config: FATFS-ის დასამონტაჟებლად დამატებითი პარამეტრებით სტრუქტურის მითითება
  • wl_handle: გასწორების სახელურის ცვეთა

სტრუქტურა esp_vfs_fat_mount_config_t

კონფიგურაციის არგუმენტები esp_vfs_fat_sdmmc_mount და esp_vfs_fat_spi ფლეშ მეხსიერება _mount ფუნქციებისთვის.

საჯარო წევრები

bool format_if_mount_failed

თუ FAT დანაყოფის დამონტაჟება შეუძლებელია და ეს პარამეტრი true-ს ემთხვევა, შექმენით დანაყოფების ცხრილი და დააფორმატეთ ფაილური სისტემა.

int max_files

გახსნილი ფაილების მაქსიმალური რაოდენობა.

size_t allocation_unit_size

თუ დაყენებულია format_if_mount_failed და მთა ვერ ხერხდება, დააფორმატეთ ბარათი მოცემული განაწილების ერთეულის ზომით. უნდა იყოს 2-ის ხარისხი, სექტორის ზომასა და 128 * სექტორის ზომას შორის. SD ბარათებისთვის სექტორის ზომა ყოველთვის 512 ბაიტია. wear_levelling-ისთვის, სექტორის ზომა განისაზღვრება CONFIG_WL_SECTOR_SIZE ოფციით.

უფრო დიდი განაწილების ერთეულის ზომის გამოყენება გამოიწვევს წაკითხვის/ჩაწერის უფრო მაღალ შესრულებას და უფრო მეტ ზედნადებს მცირე ფაილების შენახვისას.

ამ ველის 0-ზე დაყენების შემთხვევაში, განაწილების ერთეული დაყენდება სექტორის ზომაზე.

esp_err_t esp_vfs_fat_spiflash_unmount(კონსტ char *ბაზისური_გზა, wl_handle_t wl_handle)¶

FAT ფაილური სისტემის გამორთვა და esp_vfs_fat_spi ფლეშ მეხსიერება _mount-ის გამოყენებით შეძენილი რესურსების გათავისუფლება.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში

esp_err_t wl_erase_range(wl_handle_t სახელური, size_t start_addr, size_t ზომა)¶

WL საცავის ნაწილის წაშლა.

დაბრუნება
  • ESP_OK, თუ დიაპაზონი წარმატებით წაიშალა;
  • ESP_ERR_INVALID_ARG, თუ იტერატორი ან dst არის NULL;
  • ESP_ERR_INVALID_SIZE, თუ წაშლა დანაყოფის საზღვრებს გასცდება;
  • ან ქვედა დონის ფლეშ მეხსიერება დრაივერის შეცდომის კოდიდან ერთ-ერთი.
პარამეტრები
  • handle: WL სახელური, რომელიც დაკავშირებულია დანაყოფთან
  • start_addrმისამართი, სადაც უნდა დაიწყოს წაშლის ოპერაცია. უნდა შეესაბამებოდეს wl_sector_size(…) ფუნქციის შედეგს.
  • size: წასაშლელი დიაპაზონის ზომა ბაიტებში. უნდა იყოფოდეს wl_sector_size(…) ფუნქციის შედეგზე.

esp_err_t wl_write(wl_handle_t სახელური, size_t dest_addr, კონსტ void *src, size_t ზომა)¶

მონაცემების ჩაწერა WL საცავში.

ფლეშ მეხსიერება ში მონაცემების ჩაწერამდე, ფლეშ მეხსიერება ის შესაბამისი რეგიონი უნდა წაიშალოს. ეს შეიძლება გაკეთდეს wl_erase_range ფუნქციის გამოყენებით.

შენიშვნა
Prior to writing to WL storage, make sure it has been erased with wl_erase_range call.
დაბრუნება
  • ESP_OK, თუ მონაცემები წარმატებით ჩაიწერა;
  • ESP_ERR_INVALID_ARG, თუ dst_offset აღემატება დანაყოფის ზომას;
  • ESP_ERR_INVALID_SIZE, თუ ჩაწერა დანაყოფის საზღვრებს გასცდება;
  • ან ქვედა დონის ფლეშ მეხსიერება დრაივერის შეცდომის კოდიდან ერთ-ერთი.
პარამეტრები
  • handle: WL სახელური, რომელიც დაკავშირებულია დანაყოფთან
  • dest_addrმისამართი, სადაც მონაცემები უნდა ჩაიწეროს, დანაყოფის დასაწყისთან მიმართებაში.
  • src: წყაროს ბუფერის მაჩვენებელი. მაჩვენებელი უნდა იყოს არა-NULL და ბუფერის სიგრძე უნდა იყოს მინიმუმ &#39;ზომა&#39; ბაიტები.
  • size: ჩასაწერი მონაცემების ზომა, ბაიტებში.

esp_err_t wl_read(wl_handle_t სახელური, size_t src_addr, void *დანიშნულების ადგილი, size_t ზომა)¶

  • ან ქვედა დონის ფლეშ მეხსიერება დრაივერის შეცდომის კოდიდან ერთ-ერთი.

  • dest: მაჩვენებელი ბუფერისკენ, სადაც მონაცემები უნდა იყოს შენახული. მაჩვენებელი უნდა იყოს არა-NULL და ბუფერის სიგრძე უნდა იყოს მინიმუმ &#39;ზომა&#39; ბაიტები.

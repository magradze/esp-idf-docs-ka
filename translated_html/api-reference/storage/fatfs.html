FAT ფაილური სისტემის მხარდაჭერა¶

[中文]

ESP-IDF იყენებს ცხიმოვანი მჟავები ბიბლიოთეკა FAT ფაილურ სისტემებთან სამუშაოდ. FatFs მდებარეობს fatfs კომპონენტი . მიუხედავად იმისა, რომ ბიბლიოთეკის პირდაპირ გამოყენება შესაძლებელია, მის ბევრ ფუნქციაზე წვდომა შესაძლებელია VFS-ის საშუალებით, C სტანდარტული ბიბლიოთეკისა და POSIX API ფუნქციების გამოყენებით.

გარდა ამისა, FatF-ები მოდიფიცირებულია გაშვების დროს ადაპტირებადი დისკის შეყვანის/გამოყვანის ფენის მხარდასაჭერად. ეს საშუალებას იძლევა FatF დისკების გაშვების დროს ფიზიკურ დისკებზე მიბმის.

FatF-ების გამოყენება VFS-თან ერთად¶

სათაურის ფაილი fatfs/vfs/esp_vfs_fat.h განსაზღვრავს FatF-ებისა და VFS-ების დამაკავშირებელ ფუნქციებს.

ფუნქცია esp_vfs_fat_register() გამოყოფს FATFS სტრუქტურას ქმნის და VFS-ში მოცემული გზის პრეფიქსს არეგისტრირებს. ამ პრეფიქსით დაწყებულ ფაილებზე შემდგომი ოპერაციები გადამისამართდება FatFs API s-ში. ფუნქცია esp_vfs_fat_unregister_path() შლის VFS-ში რეგისტრაციას და ათავისუფლებს FATFS სტრუქტურა.

აპლიკაციების უმეტესობა მუშაობისას იყენებს შემდეგ სამუშაო პროცესს: esp_vfs_fat_ ფუნქციები:

  1. Call esp_vfs_fat_register() to specify:
    • გზის პრეფიქსი, სადაც უნდა დამონტაჟდეს ფაილური სისტემა (მაგ. "/sdcard", "/spiflash")
    • FatFs დისკის ნომერი
    • ცვლადი, რომელიც მიიღებს მაჩვენებელს FATFS სტრუქტურა
  2. ზარი ff_diskio_register() დისკის შეყვანის/გამოყვანის დრაივერის რეგისტრაციისთვის პირველ ეტაპზე გამოყენებული დისკის ნომრისთვის.
  3. FatFs ფუნქციის გამოძახება f_mountდა სურვილისამებრ f_fdisk, f_mkfs, ფაილური სისტემის დასამონტაჟებლად იმავე დისკის ნომრის გამოყენებით, რომელიც გადაეცა esp_vfs_fat_register()დამატებითი ინფორმაციისთვის იხილეთ FatFs documentation <http://www.elm-chan.org/fsw/ff/doc/mount.html>.
  4. ფაილებზე ისეთი მოქმედებების შესასრულებლად, როგორიცაა გახსნა, წაკითხვა, ჩაწერა, წაშლა, კოპირება და ა.შ. გამოიძახეთ C სტანდარტული ბიბლიოთეკა და POSIX API ფუნქციები. გამოიყენეთ ბილიკები, რომლებიც იწყება გადაცემული ბილიკის პრეფიქსით. esp_vfs_register() (მაგალითად, "/sdcard/hello.txt").
  5. სურვილისამებრ, FatFs ბიბლიოთეკის ფუნქციების პირდაპირ გამოძახება. ამ შემთხვევაში, გამოიყენეთ VFS პრეფიქსის გარეშე არსებული გზები (მაგალითად, "/hello.txt").
  6. დახურეთ ყველა გახსნილი ფაილი.
  7. FatFs ფუნქციის გამოძახება f_mount იგივე დისკის ნომრისთვის, NULL-ით FATFS* არგუმენტი, ფაილური სისტემის გასათიშად.
  8. FatFs ფუნქციის გამოძახება ff_diskio_register() NULL-ით ff_diskio_impl_t* არგუმენტი და იგივე დისკის ნომერი დისკის შეყვანის/გამოყვანის დრაივერის რეგისტრაციის გასაუქმებლად.
  9. ზარი esp_vfs_fat_unregister_path() იმ ბილიკით, სადაც ფაილური სისტემაა დამონტაჟებული VFS-დან FatF-ების მოსაშორებლად და გასათავისუფლებლად. FATFS სტრუქტურა, რომელიც გამოყოფილია პირველ ეტაპზე.

მოხერხებულობის ფუნქციები; esp_vfs_fat_sdmmc_mount და esp_vfs_fat_sdmmc_unmount ზემოთ აღწერილი ნაბიჯების გადაფარვა და ასევე SD ბარათის ინიციალიზაციის დამუშავება. ეს ორი ფუნქცია აღწერილია შემდეგ ნაწილში.

esp_err_t esp_vfs_fat_register(კონსტ char *ბაზისური_გზა, კონსტ char *ცხიმის_ძრავა, size_t max_files, FATFS **out_fs)¶

დაარეგისტრირეთ FATFS VFS კომპონენტი ში.

ეს ფუნქცია არეგისტრირებს მოცემულ FAT დისკს VFS-ში, მითითებულ საბაზისო გზაზე. თუ გამოიყენება მხოლოდ ერთი დისკი, fat_drive არგუმენტი შეიძლება იყოს ცარიელი სტრიქონი. FAT დისკის მითითების შესახებ იხილეთ FATFS ბიბლიოთეკის დოკუმენტაცია. ეს ფუნქცია ასევე გამოყოფს FATFS სტრუქტურას, რომელიც უნდა იქნას გამოყენებული f_mount გამოძახებისთვის.

შენიშვნა
This function doesn’t mount the drive into FATFS, it just connects POSIX and C standard library IO function with FATFS. You need to mount desired drive into FATFS separately.
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_STATE თუ esp_vfs_fat_register უკვე გამოძახებული იყო
  • ESP_ERR_NO_MEM, თუ მეხსიერება არ არის საკმარისი ან უკვე ძალიან ბევრი VFS-ია რეგისტრირებული
პარამეტრები
  • base_path: გზის პრეფიქსი, სადაც FATFS უნდა დარეგისტრირდეს
  • fat_driveFATFS დისკის სპეციფიკაცია; თუ მხოლოდ ერთი დისკია გამოყენებული, შეიძლება იყოს ცარიელი სტრიქონი
  • max_files: ფაილების მაქსიმალური რაოდენობა, რომელთა გახსნაც შესაძლებელია ერთდროულად
  • out_fs: ამ არგუმენტის მეშვეობით ბრუნდება FATFS სტრუქტურის მაჩვენებელი, რომლის გამოყენებაც შესაძლებელია FATFS f_mount გამოძახებისთვის.

esp_err_t esp_vfs_fat_unregister_path(კონსტ char *ბაზისური_გზა)¶

FATFS-ის VFS-დან რეგისტრაციის გაუქმება.

შენიშვნა
FATFS structure returned by esp_vfs_fat_register is destroyed after this call. Make sure to call f_mount function to unmount it before calling esp_vfs_fat_unregister_ctx. Difference between this function and the one above is that this one will release the correct drive, while the one above will release the last registered one
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_STATE თუ FATFS არ არის რეგისტრირებული VFS-ში
პარამეტრები
  • base_path: გზის პრეფიქსი, სადაც FATFS რეგისტრირებულია. ეს იგივეა, რაც გამოიყენება esp_vfs_fat_register-ის გამოძახებისას.

FatF-ების გამოყენება VFS-თან და SD ბარათებთან ერთად¶

სათაურის ფაილი fatfs/vfs/esp_vfs_fat.h განსაზღვრავს მოხერხებულობის ფუნქციებს esp_vfs_fat_sdmmc_mount() და esp_vfs_fat_sdmmc_unmount()ეს ფუნქციები შესაბამისად ასრულებს 1–3 და 7–9 ნაბიჯებს და ამუშავებს SD ბარათის ინიციალიზაციას, თუმცა შეცდომების დამუშავებას მხოლოდ შეზღუდულად უზრუნველყოფს. დეველოპერებს ვურჩევთ, შეამოწმონ მისი საწყისი კოდი და უფრო მოწინავე ფუნქციები ჩართონ საწარმოო აპლიკაციებში.

მოხერხებულობის ფუნქცია; esp_vfs_fat_sdmmc_unmount() ხსნის ფაილურ სისტემას და ათავისუფლებს შეძენილ რესურსებს esp_vfs_fat_sdmmc_mount().

esp_err_t esp_vfs_fat_sdmmc_mount(კონსტ char *ბაზისური_გზა, კონსტ sdmmc_host_t *host_config, კონსტ void *სლოტის_კონფიგურაცია, კონსტ esp_vfs_fat_mount_config_t *mount_config, sdmmc_card_t **გასვლის_ბარათი)¶

მოხერხებულობის ფუნქცია SD ბარათზე არსებული FAT ფაილური სისტემის VFS-ში რეგისტრირებისთვის.

ეს არის ყოვლისმომცველი ფუნქცია, რომელიც ასრულებს შემდეგ ფუნქციებს:

  • ინიციალიზაციას უკეთებს SDMMC დრაივერს ან SPI დრაივერს host_config-ში კონფიგურაციით.
  • SD ბარათის ინიციალიზაცია slot_config-ში კონფიგურაციით
  • FAT დანაყოფის SD ბარათზე მონტაჟი FATFS ბიბლიოთეკის გამოყენებით, mount_config-ში კონფიგურაციით.
  • არეგისტრირებს FATFS ბიბლიოთეკას VFS-თან, პრეფიქსით, რომელიც მოცემულია base_prefix ცვლადის მიერ.

ეს ფუნქცია განკუთვნილია მაგალითის კოდის უფრო კომპაქტური გახდომისთვის. რეალური სამყაროს აპლიკაციებისთვის, დეველოპერებმა უნდა დანერგონ SD ბარათის შემოწმების, დანაყოფის ადგილმდებარეობისა და დამონტაჟების, ასევე VFS-ში FATFS-ის რეგისტრაციის ლოგიკა, სათანადო შეცდომების შემოწმებით და გამონაკლისი სიტუაციების დამუშავებით.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_STATE თუ esp_vfs_fat_sdmmc_mount უკვე გამოძახებულია
  • ESP_ERR_NO_MEM თუ მეხსიერების გამოყოფა შეუძლებელია
  • ESP_FAIL თუ დანაყოფის დამონტაჟება შეუძლებელია
  • SDMMC ან SPI დრაივერებიდან, SDMMC პროტოკოლიდან ან FATFS დრაივერებიდან მიღებული სხვა შეცდომის კოდები
პარამეტრები
  • base_path: გზა, სადაც უნდა დარეგისტრირდეს დანაყოფი (მაგ. „/sdcard“)
  • host_configSDMMC ჰოსტის აღმწერი სტრუქტურის მაჩვენებელი. SDMMC პერიფერიული მოწყობილობის გამოყენებისას, ამ სტრუქტურის ინიციალიზაცია შესაძლებელია SDMMC_HOST_DEFAULT() მაკროს გამოყენებით. SPI პერიფერიული მოწყობილობის გამოყენებისას, ამ სტრუქტურის ინიციალიზაცია შესაძლებელია SDSPI_HOST_DEFAULT() მაკროს გამოყენებით.
  • slot_config: სტრუქტურაზე სლოტის კონფიგურაციის მქონე მაჩვენებელი. SDMMC პერიფერიული მოწყობილობისთვის, გადაეცით მაჩვენებელი sdmmc_slot_config_t სტრუქტურა ინიციალიზებულია SDMMC_SLOT_CONFIG_DEFAULT-ის გამოყენებით. SPI პერიფერიულისთვის, გადაეცით მაჩვენებელი sdspi_slot_config_t სტრუქტურა ინიციალიზებულია SDSPI_SLOT_CONFIG_DEFAULT-ის გამოყენებით.
  • mount_config: FATFS-ის დასამონტაჟებლად დამატებითი პარამეტრებით სტრუქტურის მითითება
  • out_cardთუ არა NULL, ბარათის ინფორმაციის სტრუქტურის მაჩვენებელი დაბრუნდება ამ არგუმენტის მეშვეობით.

სტრუქტურა esp_vfs_fat_mount_config_t¶

კონფიგურაციის არგუმენტები esp_vfs_fat_sdmmc_mount და esp_vfs_fat_spi ფლეშ მეხსიერება _mount ფუნქციებისთვის.

საჯარო წევრები

bool format_if_mount_failed¶

თუ FAT დანაყოფის დამონტაჟება შეუძლებელია და ეს პარამეტრი true-ს ემთხვევა, შექმენით დანაყოფების ცხრილი და დააფორმატეთ ფაილური სისტემა.

int max_files¶

გახსნილი ფაილების მაქსიმალური რაოდენობა.

size_t allocation_unit_size¶

თუ დაყენებულია format_if_mount_failed და მთა ვერ ხერხდება, დააფორმატეთ ბარათი მოცემული განაწილების ერთეულის ზომით. უნდა იყოს 2-ის ხარისხი, სექტორის ზომასა და 128 * სექტორის ზომას შორის. SD ბარათებისთვის სექტორის ზომა ყოველთვის 512 ბაიტია. wear_levelling-ისთვის, სექტორის ზომა განისაზღვრება CONFIG_WL_SECTOR_SIZE ოფციით.

უფრო დიდი განაწილების ერთეულის ზომის გამოყენება გამოიწვევს წაკითხვის/ჩაწერის უფრო მაღალ შესრულებას და უფრო მეტ ზედნადებს მცირე ფაილების შენახვისას.

ამ ველის 0-ზე დაყენების შემთხვევაში, განაწილების ერთეული დაყენდება სექტორის ზომაზე.

esp_err_t esp_vfs_fat_sdmmc_unmount(void)¶

გამორთეთ FAT ფაილური სისტემა და გაათავისუფლეთ esp_vfs_fat_sdmmc_mount-ის გამოყენებით შეძენილი რესურსები.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_STATE თუ esp_vfs_fat_sdmmc_mount არ არის გამოძახებული

FatF-ების გამოყენება VFS-თან ერთად მხოლოდ წაკითხვის რეჟიმში¶

სათაურის ფაილი fatfs/vfs/esp_vfs_fat.h ასევე განსაზღვრავს მოხერხებულობის ფუნქციებს esp_vfs_fat_rawflash_mount() და esp_vfs_fat_rawflash_unmount()ეს ფუნქციები ასრულებენ 1-3 და 7-9 ნაბიჯებს შესაბამისად მხოლოდ წასაკითხი FAT დანაყოფებისთვის. ეს განსაკუთრებით სასარგებლოა მონაცემთა დანაყოფებისთვის, რომლებიც მხოლოდ ერთხელ იწერება ქარხნული უზრუნველყოფის დროს და რომლებიც არ შეიცვლება საწარმოო აპლიკაციის მიერ აპარატურის მთელი სიცოცხლის განმავლობაში.

esp_err_t esp_vfs_fat_rawflash_mount(კონსტ char *ბაზისური_გზა, კონსტ char *დანაყოფის_ეტიკეტი, კონსტ esp_vfs_fat_mount_config_t *mount_config)¶

მოხერხებულობის ფუნქცია მხოლოდ წასაკითხი FAT ფაილური სისტემის ინიციალიზაციისა და VFS-ში რეგისტრაციისთვის.

ეს არის ყოვლისმომცველი ფუნქცია, რომელიც ასრულებს შემდეგ ფუნქციებს:

  • პოულობს დანაყოფს განსაზღვრული partition_label-ით. დანაყოფის ეტიკეტი უნდა იყოს კონფიგურირებული დანაყოფების ცხრილი ში.
  • FAT დანაყოფის მონტაჟი FATFS ბიბლიოთეკის გამოყენებით
  • არეგისტრირებს FATFS ბიბლიოთეკას VFS-თან, პრეფიქსით, რომელიც მოცემულია base_prefix ცვლადის მიერ.

შენიშვნა
Wear levelling is not used when FAT is mounted in read-only mode using this function.
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_NOT_FOUND თუ დანაყოფების ცხრილი არ შეიცავს მოცემული ლეიბლით FATFS დანაყოფს
  • ESP_ERR_INVALID_STATE თუ esp_vfs_fat_raw ფლეშ მეხსიერება _mount უკვე გამოძახებული იყო იმავე დანაყოფისთვის.
  • ESP_ERR_NO_MEM თუ მეხსიერების გამოყოფა შეუძლებელია
  • ESP_FAIL თუ დანაყოფის დამონტაჟება შეუძლებელია
  • SPI ფლეშ მეხსიერება დრაივერის ან FATFS დრაივერებისგან მიღებული სხვა შეცდომის კოდები
პარამეტრები
  • base_path: გზა, სადაც უნდა იყოს დამონტაჟებული FATFS დანაყოფი (მაგ. „/spi ფლეშ მეხსიერება “)
  • partition_label: გამოყენებული დანაყოფის ეტიკეტი
  • mount_config: FATFS-ის დასამონტაჟებლად დამატებითი პარამეტრებით სტრუქტურის მითითება

FatFS დისკის IO ფენა¶

FatFs გაფართოვდა API ფუნქციებით, რომლებიც დისკის შეყვანის/გამოყვანის დრაივერს გაშვების დროს რეგისტრირებენ.

ისინი უზრუნველყოფენ SD/MMC ბარათებისთვის დისკის შეყვანის/გამოყვანის ფუნქციების განხორციელებას და მათი რეგისტრაცია შესაძლებელია მოცემულ FatFs დისკის ნომრისთვის ფუნქციის გამოყენებით. ff_diskio_register_sdmmc().

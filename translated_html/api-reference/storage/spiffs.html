SPIFFS ფაილური სისტემა¶

[中文]

მიმოხილვა¶

SPIFFS არის ფაილური სისტემა, რომელიც განკუთვნილია ჩაშენებულ სამიზნეებზე SPI NOR ფლეშ მეხსიერება მოწყობილობებისთვის. ის მხარს უჭერს ცვეთის გასწორებას, ფაილური სისტემის თანმიმდევრულობის შემოწმებას და სხვა.

შენიშვნები¶

  • ამჟამად, SPIFFS არ უჭერს მხარს დირექტორიებს, ის ქმნის ბრტყელ სტრუქტურას. თუ SPIFFS დამონტაჟებულია /spiffs, შემდეგ კი ფაილის შექმნა ბილიკით /spiffs/tmp/myfile.txt შექმნის ფაილს სახელწოდებით /tmp/myfile.txt SPIFFS-ში, ნაცვლად myfile.txt დირექტორიაში /spiffs/tmp.
  • ეს არ არის რეალურ დროში დასტა. ერთ ჩაწერის ოპერაციას შეიძლება გაცილებით მეტი დრო დასჭირდეს, ვიდრე მეორეს.
  • ამ ეტაპზე ის არ აფიქსირებს და არ ამუშავებს არასწორ ბლოკებს.

ხელსაწყოები¶

spiffsgen.py¶

spiffsgen.py არის მხოლოდ ჩაწერის უფლების მქონე Python SPIFFS იმპლემენტაცია, რომელიც გამოიყენება ფაილური სისტემის სურათების შესაქმნელად მასპინძელი საქაღალდის შიგთავსიდან. გამოსაყენებლად spiffsgen.py, გახსენით ტერმინალი და გაუშვით:

python spiffsgen.py <image_size> <base_dir> <output_file>

საჭირო არგუმენტები შემდეგია:

  • სურათის ზომა: იმ დანაყოფის ზომა, რომელზეც შექმნილი SPIFFS გამოსახულება განთავსდება ფლეშ მეხსიერება ed.
  • ბაზის_დირექტორიადირექტორია, რომლისთვისაც უნდა შეიქმნას SPIFFS სურათი.
  • გამომავალი_ფაილი: SPIFFS გამოსახულების გამომავალი ფაილი.

ასევე არსებობს სხვა არგუმენტები, რომლებიც აკონტროლებენ სურათების გენერირებას. ამ არგუმენტების დოკუმენტაცია შეგიძლიათ იხილოთ ინსტრუმენტის დახმარებაში:

python spiffsgen.py --help

ეს არასავალდებულო არგუმენტები შეესაბამება SPIFFS-ის შესაძლო აწყობის კონფიგურაციას. სწორი სურათის გენერირებისთვის, გთხოვთ, დარწმუნდეთ, რომ იყენებთ იგივე არგუმენტებს/კონფიგურაციას, რაც გამოყენებული იყო SPIFFS-ის ასაწყობად. სახელმძღვანელოდ, დახმარების შედეგი მიუთითებს SPIFFS-ის აწყობის კონფიგურაციას, რომელსაც შეესაბამება არგუმენტი. იმ შემთხვევებში, როდესაც ეს არგუმენტები არ არის მითითებული, გამოყენებული იქნება დახმარების შედეგს ნაჩვენები ნაგულისხმევი მნიშვნელობები.

როდესაც სურათი იქმნება, მისი ფლეშ მეხსიერება რედაქტირება შესაძლებელია გამოყენებით esptool.py ან parttool.py.

გარდა იმისა, რომ გამოძახებულია, spiffsgen.py დამოუკიდებელი, ბრძანების ხაზიდან ან სკრიპტიდან ხელით გაშვებით, ასევე შესაძლებელია მისი გამოძახება spiffsgen.py პირდაპირ აწყობის სისტემიდან დარეკვით spiffs_create_partition_image.

მწარმოებელი:

SPIFFS_IMAGE_FLASH_IN_PROJECT := ...
SPIFFS_IMAGE_DEPENDS := ...
$(eval $(call spiffs_create_partition_image,<partition>,<base_dir>))

CMake:

spiffs_create_partition_image(<partition> <base_dir> [FLASH_IN_PROJECT] [DEPENDS dep dep dep...])

ეს უფრო მოსახერხებელია, რადგან აწყობის კონფიგურაცია ავტომატურად გადაეცემა ინსტრუმენტს, რაც უზრუნველყოფს, რომ გენერირებული სურათი ვალიდურია ამ აწყობისთვის. ამის მაგალითია, როდესაც სურათის ზომა საჭიროა დამოუკიდებელი გამოძახებისთვის, მხოლოდ დანაყოფი გამოყენებისას სახელი აუცილებელია spiffs_create_partition_image – სურათის ზომა ავტომატურად მიიღება პროექტის დანაყოფების ცხრილი .

Make-სა და CMake-ს შორის სტრუქტურული განსხვავებების გამო, მნიშვნელოვანია აღინიშნოს, რომ: - Make-სთვის spiffs_create_partition_image უნდა გამოიძახოს პროექტიდან Makefile - CMake-სთვის spiffs_create_partition_image უნდა გამოიძახოს კომპონენტი CMakeLists.txt ფაილებიდან ერთ-ერთიდან

სურვილისამებრ, მომხმარებელს შეუძლია აირჩიოს სურათის ავტომატურად ფლეშ მეხსიერება რედაქტირება აპლიკაციის ბინარულ ფაილებთან, დანაყოფების ცხრილი თან და ა.შ. ერთად. idf.py flash ან make flash მითითებით FLASH_IN_PROJECTმაგალითად,

დამზადებაში:

SPIFFS_IMAGE_FLASH_IN_PROJECT := 1
$(eval $(call spiffs_create_partition_image,<partition>,<base_dir>))

CMake-ში:

spiffs_create_partition_image(my_spiffs_partition my_folder FLASH_IN_PROJECT)

თუ FLASH_IN_PROJECT/SPIFFS_IMAGE_FLASH_IN_PROJECT არ არის მითითებული, სურათი მაინც გენერირდება, მაგრამ მისი ხელით ფლეშ მეხსიერება მოგიწევთ esptool.py, parttool.py, ან მორგებული აწყობის სისტემის სამიზნე.

არსებობს შემთხვევები, როდესაც ბაზის დირექტორიის შინაარსი თავად გენერირდება შექმნის დროს. მომხმარებლებს შეუძლიათ გამოიყენონ DEPENDS/SPIFFS_IMAGE_DEPENDS იმ სამიზნეების მითითებისთვის, რომლებიც უნდა შესრულდეს გამოსახულების გენერირებამდე.

დამზადებაში:

dep:
    ...

SPIFFS_IMAGE_DEPENDS := dep
$(eval $(call spiffs_create_partition_image,<partition>,<base_dir>))

CMake-ში:

add_custom_target(dep COMMAND ...)

spiffs_create_partition_image(my_spiffs_partition my_folder DEPENDS dep)

+მაგალითისთვის იხილეთ შენახვა/სპიფსგენი.

მსპიფები¶

SPIFFS დანაყოფის სურათების შესაქმნელად კიდევ ერთი ინსტრუმენტია მსპიფებიმსგავსი spiffsgen.py, მისი გამოყენება შესაძლებელია მოცემული საქაღალდიდან სურათის შესაქმნელად და შემდეგ ფლეშ მეხსიერება გამოყენებით ამ სურათის შესაქმნელად esptool.py

ამისათვის თქვენ უნდა მიიღოთ შემდეგი პარამეტრები:

  • ბლოკის ზომა: 4096 (სტანდარტული SPI Flash-ისთვის)
  • გვერდის ზომა: 256 (სტანდარტული SPI Flash-ისთვის)
  • სურათის ზომა: დანაყოფის ზომა ბაიტებში (შეგიძლიათ მიიღოთ დანაყოფების ცხრილი დან)
  • დანაყოფის ოფსეტიდანაყოფის საწყისი მისამართი (შეგიძლიათ მიიღოთ დანაყოფების ცხრილი დან)

საქაღალდის 1 მეგაბაიტიან გამოსახულებაში ჩასართავად, გაუშვით:

mkspiffs -c [src_folder] -b 4096 -p 256 -s 0x100000 spiffs.bin

გამოსახულების ESP32-ზე 0x110000 ოფსეტით ფლეშ მეხსიერება , შეასრულეთ შემდეგი:

python esptool.py --chip esp32 --port [port] --baud [baud] write_flash -z 0x110000 spiffs.bin

შენიშვნები, თუ რომელი SPIFFS ინსტრუმენტი გამოვიყენოთ¶

ზემოთ წარმოდგენილი ორი ინსტრუმენტი ძალიან მსგავს ფუნქციონალურობას გვთავაზობს. თუმცა, არსებობს მიზეზები, რის გამოც ერთი მეორეზე უპირატესობა უნდა მიანიჭოთ, გამოყენების შემთხვევის მიხედვით.

გამოყენება spiffsgen.py შემდეგ შემთხვევებში: 1. თუ გსურთ უბრალოდ SPIFFS გამოსახულების გენერირება აწყობის დროს. spiffsgen.py ეს ძალიან მოსახერხებელია, რადგან ფუნქციები/ბრძანებები თავად აწყობის სისტემიდან არის მოწოდებული. 2. თუ ჰოსტს არ აქვს ხელმისაწვდომი C/C++ კომპილაცია r, რადგან spiffsgen.py კომპილაციას არ საჭიროებს.

გამოყენება mkspiffs შემდეგ შემთხვევებში: 1. თუ სურათების გენერირებასთან ერთად SPIFFS სურათების გახსნა გჭირდებათ. ამჟამად ეს შეუძლებელია spiffsgen.py. 2. თუ თქვენ გაქვთ გარემო, სადაც Python-ის ინტერპრეტატორი არ არის ხელმისაწვდომი, მაგრამ მასპინძელი კომპილაცია r ხელმისაწვდომია. წინააღმდეგ შემთხვევაში, ვადამდელი კომპილაცია d mkspiffs ბინარულ ფაილს შეუძლია ამ საქმის შესრულება. თუმცა, არ არსებობს სისტემის ინტეგრაცია mkspiffs და მომხმარებელმა უნდა შეასრულოს შესაბამისი სამუშაო: კომპილაცია mkspiffs აწყობის დროს (თუ pre- კომპილაცია d ორობითი ფაილი არ გამოიყენება), გამომავალი ფაილებისთვის აწყობის წესების/სამიზნეების შექმნა, ინსტრუმენტზე შესაბამისი პარამეტრების გადაცემა და ა.შ.

აგრეთვე იხილეთ¶

  • დანაყოფების ცხრილის დოკუმენტაცია

განაცხადის მაგალითი¶

SPIFFS-ის გამოყენების მაგალითი მოცემულია საწყობი/სპიფები დირექტორია. ეს მაგალითი ახდენს SPIFFS დანაყოფის ინიციალიზაციას და მონტაჟს, შემდეგ კი მონაცემებს იწერს და კითხულობს მისგან POSIX-ისა და C ბიბლიოთეკის API s-ის გამოყენებით. დამატებითი ინფორმაციისთვის იხილეთ README.md ფაილი მაგალითის დირექტორიაში.

სათაურის ფაილი¶

esp_err_t esp_vfs_spiffs_register(კონსტ esp_vfs_spiffs_conf_t *კონფესი)¶

SPIFFS-ის რეგისტრაცია და VFS-ზე დამონტაჟება მოცემული გზის პრეფიქსით.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_NO_MEM თუ ობიექტების გამოყოფა ვერ მოხერხდა
  • ESP_ERR_INVALID_STATE, თუ უკვე დამონტაჟებულია ან დანაყოფი დაშიფრულია
  • ESP_ERR_NOT_FOUND თუ SPIFFS-ის დანაყოფი ვერ მოიძებნა
  • ESP_FAIL, თუ მონტაჟი ან ფორმატირება ვერ ხერხდება
პარამეტრები
  • conf: მითითება esp_vfs_spiffs_conf_t კონფიგურაციის სტრუქტურა

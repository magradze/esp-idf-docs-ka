შეხების სენსორი¶

[中文]

შესავალი¶

შეხების სენსორული სისტემა აგებულია სუბსტრატზე, რომელიც დამცავი ბრტყელი ზედაპირის ქვეშ ატარებს ელექტროდებს და შესაბამის შეერთებებს. როდესაც მომხმარებელი ეხება ზედაპირს, შეხების სისწორის შესაფასებლად გამოიყენება ტევადობის ვარიაცია.

ESP32-ს შეუძლია 10-მდე ტევადური სენსორული პანელის / GPIO 7 წამის დამუშავება. სენსორული პანელების განლაგება შესაძლებელია სხვადასხვა კომბინაციით (მაგ., მატრიცა, სლაიდერი), ისე, რომ შესაძლებელი იყოს უფრო დიდი ფართობის ან მეტი წერტილის აღმოჩენა. სენსორული პანელის აღმოჩენის პროცესი კონტროლდება აპარატურულად დანერგილი სასრული მდგომარეობის მანქანის (FSM) მიერ, რომელიც ინიცირებულია პროგრამული უზრუნველყოფით ან სპეციალური აპარატურული ტაიმერით.

შეხების სენსორის დიზაინი, მუშაობა და მართვის რეგისტრები განხილულია ESP32 ტექნიკური საცნობარო სახელმძღვანელო (PDF). ამ ქვესისტემის მუშაობის შესახებ დამატებითი ინფორმაციისთვის, გთხოვთ, იხილოთ ეს სახელმძღვანელო.

შეხების სენსორების დეტალური დიზაინის დეტალები და ESP32-ისთვის პროგრამული უზრუნველყოფა შემუშავების სახელმძღვანელო მითითებები ხელმისაწვდომია შეხების სენსორის გამოყენების შენიშვნათუ გსურთ შეხების სენსორების ტესტირება სხვადასხვა კონფიგურაციაში მათი დამოუკიდებლად აწყობის გარეშე, შეამოწმეთ ESP32-Sense-ის შემუშავების ნაკრების სახელმძღვანელო.

ფუნქციონალურობის მიმოხილვა¶

API ის აღწერა დაყოფილია ფუნქციების ჯგუფებად, რათა უზრუნველყოს შემდეგი მახასიათებლების სწრაფი მიმოხილვა:

  • სენსორული პანელის დრაივერის ინიციალიზაცია
  • სენსორული პანელის GPIO პინის კონფიგურაცია
  • გაზომვების აღება
  • გაზომვების პარამეტრების რეგულირება
  • გაზომვების ფილტრაცია
  • შეხების ამოცნობის მეთოდები
  • შეხების ამოცნობის შესახებ შეტყობინების შეწყვეტის დაყენება
  • ძილის რეჟიმიდან გამოღვიძება შეფერხებისას

კონკრეტული ფუნქციის დეტალური აღწერილობისთვის, გთხოვთ, გადახვიდეთ განყოფილებაში API მითითებაამ API პრაქტიკული განხორციელება განხილულია ნაწილში 8 გამოყენების მაგალითები.

ინიციალიზაცია¶

სენსორული პანელის გამოყენებამდე, საჭიროა სენსორული პანელის დრაივერის ინიციალიზაცია ფუნქციის გამოძახებით. touch_pad_init()ეს ფუნქცია რამდენიმეს აყენებს .._DEFAULT დრაივერის პარამეტრები ჩამოთვლილია API მითითება ქვეშ მაკროებიის ასევე შლის ინფორმაციას იმის შესახებ, თუ რომელ პანელებს შეეხნენ ადრე, თუ საერთოდ შეხებიათ, და გამორთავს შეფერხებებს.

თუ დრაივერი აღარ არის საჭირო, მისი დეინიციალიზაცია დარეკვით touch_pad_deinit().

კონფიგურაცია¶

კონკრეტული GPIO შეხების სენსორის ფუნქციონალის ჩართვა7 ხორციელდება touch_pad_config().

გამოიყენეთ ფუნქცია touch_pad_set_fsm_mode() სენსორული პანელის გაზომვის (FSM-ის მიერ მართული) ავტომატურად დაწყება აპარატურული ტაიმერით თუ პროგრამული უზრუნველყოფით. თუ პროგრამული უზრუნველყოფის რეჟიმი არჩეულია, გამოიყენეთ touch_pad_sw_start() FSM-ის დასაწყებად.

შეხების მდგომარეობის გაზომვები¶

სენსორიდან ნედლი ან გაფილტრული გაზომვების წასაკითხად გამოგადგებათ შემდეგი ორი ფუნქცია:

  • touch_pad_read()
  • touch_pad_read_filtered()

მათი გამოყენება ასევე შესაძლებელია, მაგალითად, სენსორული პანელის კონკრეტული დიზაინის შესაფასებლად, სენსორის ჩვენებების დიაპაზონის შემოწმებით, როდესაც პანელს შეეხებით ან ათავისუფლებთ. ეს ინფორმაცია შემდეგ შეიძლება გამოყენებულ იქნას შეხების ზღურბლის დასადგენად.

შენიშვნა

გამოყენებამდე touch_pad_read_filtered(), თქვენ უნდა ინიციალიზაცია გაუკეთოთ და დააკონფიგურიროთ ფილტრი განყოფილებაში აღწერილი კონკრეტული ფილტრის ფუნქციების გამოძახებით. გაზომვების ფილტრაცია.

ორივე წაკითხვის ფუნქციის გამოყენების დემონსტრირებისთვის, იხილეთ აპლიკაციის მაგალითი პერიფერიული მოწყობილობები/სენსორული პანელის წაკითხვა.

გაზომვების ოპტიმიზაცია¶

სენსორულ სენსორს აქვს რამდენიმე კონფიგურირებადი პარამეტრი, რომელიც შეესაბამება კონკრეტული სენსორული პანელის დიზაინის მახასიათებლებს. მაგალითად, მცირე ტევადობის ცვლილებების აღსანიშნავად, შესაძლებელია შევიწროოთ საცნობარო ძაბვის დიაპაზონი, რომლის ფარგლებშიც სენსორული პანელები იტენება/განმუხტება. მაღალი და დაბალი საცნობარო ძაბვები დგინდება ფუნქციის გამოყენებით. touch_pad_set_voltage().

მცირე სიმძლავრის ცვლილებების გარჩევის უნარის გარდა, დადებითი გვერდითი ეფექტი დაბალი სიმძლავრის აპლიკაციებისთვის ენერგომოხმარების შემცირებაა. სავარაუდო უარყოფითი ეფექტი გაზომვის ხმაურის ზრდაა. თუ მიღებული ჩვენებების დინამიური დიაპაზონი კვლავ დამაკმაყოფილებელია, მაშინ ენერგომოხმარების შემდგომი შემცირება შესაძლებელია გაზომვის დროის შემცირებით. touch_pad_set_meas_time().

ქვემოთ მოცემულ სიაში შეჯამებულია ხელმისაწვდომი გაზომვის პარამეტრები და შესაბამისი „დაყენებული“ ფუნქციები:

  • სენსორული პანელის დატენვის / განმუხტვის პარამეტრები:

    • ძაბვის დიაპაზონი: touch_pad_set_voltage()
    • სიჩქარე (დახრილობა): touch_pad_set_cnt_mode()

  • გაზომვის დრო: touch_pad_set_meas_time()

ძაბვის დიაპაზონს (მაღალი/დაბალი საცნობარო ძაბვები), სიჩქარეს (დახრილობა) და გაზომვის დროს შორის დამოკიდებულება ნაჩვენებია ქვემოთ მოცემულ ფიგურაში.

სენსორული პანელი - გაზომვის პარამეტრებს შორის ურთიერთობა

ბოლო დიაგრამა გამომავალი წარმოადგენს შეხების სენსორის მაჩვენებელს, ანუ გაზომვის დროში შეგროვებული იმპულსების რაოდენობას.

ყველა ფუნქცია წყვილებშია მოცემული, ნაკრები კონკრეტული პარამეტრი და მიღება მიმდინარე პარამეტრის მნიშვნელობა, მაგ. touch_pad_set_voltage() და touch_pad_get_voltage().

გაზომვების ფილტრაცია¶

თუ გაზომვები ხმაურიანია, შეგიძლიათ მათი გაფილტვრა მოცემული API ფუნქციებით. ფილტრის გამოყენებამდე, გთხოვთ, ჩართოთ ის გამოძახებით. touch_pad_filter_start().

ფილტრის ტიპია IIR (უსასრულო იმპულსური პასუხი) და მას აქვს კონფიგურირებადი პერიოდი, რომლის დაყენებაც შესაძლებელია ფუნქციით. touch_pad_set_filter_period().

ფილტრის შეჩერება შეგიძლიათ touch_pad_filter_stop()თუ ფილტრი აღარ არის საჭირო, მისი წაშლა შესაძლებელია .-ის გამოძახებით. touch_pad_filter_delete().

შეხების ამოცნობა¶

შეხების ამოცნობა დანერგილია ESP32-ის აპარატურაში მომხმარებლის მიერ კონფიგურირებული ზღურბლისა და FSM-ის მიერ შესრულებული ნედლი გაზომვების საფუძველზე. გამოიყენეთ ფუნქციები touch_pad_get_status() იმის შესამოწმებლად, თუ რომელ ბალიშებს შეეხო და touch_pad_clear_status() შეხების სტატუსის ინფორმაციის გასასუფთავებლად.

აპარატურული შეხების დეტექციის სისტემა ასევე შეიძლება დაკავშირებული იყოს შეფერხებებთან. ეს აღწერილია შემდეგ ნაწილში.

თუ გაზომვები ხმაურიანია და სიმძლავრის ცვლილებები მცირეა, აპარატურული შეხებითი დეტექცია შესაძლოა არასანდო იყოს. ამ პრობლემის გადასაჭრელად, აპარატურული დეტექტირების / შეფერხებების გამოყენების ნაცვლად, დანერგეთ გაზომვის ფილტრაცია და შეასრულეთ შეხებითი დეტექცია თქვენს საკუთარ აპლიკაციაში. შეხებითი დეტექტირების ორივე მეთოდის ნიმუშის იმპლემენტაციისთვის იხილეთ პერიფერიული მოწყობილობები/სენსორული პანელის შეწყვეტა.

შეხებით გამოწვეული შეფერხებები¶

შეხების ამოცნობისას შეფერხების ჩართვამდე, უნდა დაადგინოთ შეხების ამოცნობის ზღვარი. გამოიყენეთ აღწერილი ფუნქციები შეხების მდგომარეობის გაზომვები სენსორის გაზომვების წაკითხვა და ჩვენება, როდესაც პანელს შეეხებით და აუშვებთ. გამოიყენეთ ფილტრი, თუ გაზომვები ხმაურიანია და ფარდობითი სიმძლავრის ცვლილებები მცირეა. თქვენი გამოყენებისა და გარემო პირობების მიხედვით, შეამოწმეთ ტემპერატურისა და კვების წყაროს ძაბვის ცვლილებების გავლენა გაზომილ მნიშვნელობებზე.

როგორც კი აღმოჩენის ზღვარი დადგინდება, მისი დაყენება ინიციალიზაციის დროს შესაძლებელია touch_pad_config() ან გაშვების დროს touch_pad_set_thresh().

შემდეგ ეტაპზე, დააკონფიგურირეთ, თუ როგორ ხდება შეფერხებების გააქტიურება. მათი გააქტიურება შესაძლებელია ზღურბლის ქვემოთ ან ზემოთ, რომელიც დაყენებულია ფუნქციით. touch_pad_set_trigger_mode().

და ბოლოს, დააკონფიგურირეთ და მართეთ შეწყვეტის ზარები შემდეგი ფუნქციების გამოყენებით:

  • touch_pad_isr_register() / touch_pad_isr_deregister()
  • touch_pad_intr_enable() / touch_pad_intr_disable()

როდესაც შეფერხებები ფუნქციონირებს, შეგიძლიათ მიიღოთ ინფორმაცია, თუ რომელი კონკრეტული პანელიდან მოხდა შეფერხება, გამოძახებით. touch_pad_get_status() და გაასუფთავეთ პანელის სტატუსი touch_pad_clear_status().

შენიშვნა

შეხების აღმოჩენის შეფერხებები მოქმედებს მომხმარებლის მიერ დადგენილ ზღურბლთან შემოწმებულ ნედლ/გაუფილტრავ გაზომვებზე და ხორციელდება აპარატურაში. პროგრამული უზრუნველყოფის ფილტრაციის API ჩართვა 8 (იხ. გაზომვების ფილტრაცია) ამ პროცესზე გავლენას არ ახდენს.

ძილის რეჟიმიდან გამოღვიძება¶

თუ სენსორული პანელის შეფერხებები გამოიყენება ჩიპის ძილის რეჟიმიდან გამოსაღვიძებლად, შეგიძლიათ აირჩიოთ პანელების გარკვეული კონფიგურაცია (SET1 ან ორივე SET1 და SET2), რომელსაც უნდა შეეხოთ შეფერხების გასააქტიურებლად და შემდგომი გამოღვიძებისთვის. ამისათვის გამოიყენეთ ფუნქცია touch_pad_set_trigger_source().

პანელების საჭირო ბიტური შაბლონების კონფიგურაციის მართვა შესაძლებელია თითოეული „SET“-ისთვის შემდეგნაირად:

  • touch_pad_set_group_mask() / touch_pad_get_group_mask()
  • touch_pad_clear_group_mask()

გამოყენების მაგალითები¶

  • შეხების სენსორის წაკითხვის მაგალითი: პერიფერიული მოწყობილობები/სენსორული პანელის წაკითხვა.
  • შეხების სენსორის შეფერხების მაგალითი: პერიფერიული მოწყობილობები/სენსორული პანელის შეწყვეტა.

MCPWM¶

ESP32-ს აქვს ორი MCPWM ბლოკი, რომელთა გამოყენება შესაძლებელია სხვადასხვა ტიპის ძრავების სამართავად. თითოეულ ბლოკს აქვს PWM გამომავალი სიგნალების სამი წყვილი.

MCPWM-ის მიმოხილვა

დოკუმენტაციაში დამატებით ერთი ერთეულის გამომავალი მონაცემები აღნიშნულია PWMxA / PWMxB.

MCPWM ბლოკის უფრო დეტალური ბლოკ-დიაგრამა ნაჩვენებია ქვემოთ. თითოეული A/B წყვილი შეიძლება დამუშავდეს სამი ტაიმერიდან ნებისმიერით - ტაიმერი 0, 1 და 2. იგივე ტაიმერი შეიძლება გამოყენებულ იქნას PWM გამომავალი სიგნალების ერთზე მეტი წყვილის დასაჭერად. თითოეულ ბლოკს ასევე შეუძლია შეაგროვოს ისეთი შეყვანის სიგნალები, როგორიცაა SYNC SIGNALS, აღმოაჩინოს FAULT SIGNALS როგორიცაა ძრავის ჭარბი დენი ან ძაბვა, ასევე უკუკავშირის მიღება CAPTURE SIGNALS მაგალითად, როტორის პოზიციაზე.

MCPWM ბლოკ-დიაგრამა

ამ API აღწერა იწყება MCPWM-ის კონფიგურაციით ტაიმერი და გენერატორი ქვემოდულები ძრავის მართვის ძირითადი ფუნქციონალურობის უზრუნველსაყოფად. შემდეგ განხილულია უფრო მოწინავე ქვემოდულები და ფუნქციონალურობა. შეცდომების დამმუშავებელი, სიგნალი გადაღება, გადამზიდავი და შეწყვეტები.

შინაარსი¶

  • კონფიგურაცია გამომავალი მონაცემების ძირითადი ფუნქციონირება
  • მართვა ძრავის მართვის გამომავალი სიგნალები
  • კორექტირება როგორ იმართება ძრავა
  • გადაღება გარე სიგნალები გამოსავალზე დამატებითი კონტროლის უზრუნველსაყოფად
  • გამოყენება შეცდომების დამმუშავებელი ხარვეზების აღმოსაჩენად და სამართავად
  • დაამატეთ უფრო მაღალი სიხშირე გადამზიდავი, თუ გამომავალი სიგნალები გადის იზოლაციის ტრანსფორმატორის მეშვეობით
  • კონფიგურაცია და დამუშავება შეწყვეტები.

კონფიგურაცია¶

კონფიგურაციის მასშტაბი დამოკიდებულია ძრავის ტიპზე, კერძოდ, რამდენი გამოსავალი და შემავალი სიგნალია საჭირო და როგორი იქნება სიგნალების თანმიმდევრობა ძრავის სამართავად.

ამ შემთხვევაში ჩვენ აღვწერთ მარტივ კონფიგურაციას ჯაგრისიანი დენის ძრავის სამართავად, რომელიც იყენებს მხოლოდ MCPWM-ის არსებული რესურსების ნაწილს. ქვემოთ ნაჩვენებია სქემატური მაგალითი. ის მოიცავს H-ხიდი ძრავზე (M) მიწოდებული ძაბვის პოლარიზაციის გადართვისა და მისი მართვისთვის საკმარისი დენის უზრუნველსაყოფად.

MCPWM-ით ჯაგრისიანი DC ძრავის მართვის მაგალითი

კონფიგურაცია მოიცავს შემდეგ ნაბიჯებს:

  1. ძრავის მართვისთვის გამოყენებული MPWn ერთეულის შერჩევა. ESP32-ის ბორტზე ხელმისაწვდომია ორი ერთეული, რომლებიც ჩამოთვლილია mcpwm_unit_t.
  2. ორი GPIO s-ის, როგორც გამომავალი სიგნალის, ინიციალიზაცია არჩეულ ერთეულში გამოძახებით mcpwm_gpio_init()ორი გამომავალი სიგნალი, როგორც წესი, გამოიყენება ძრავის მარჯვნივ ან მარცხნივ ბრუნვის ბრძანებისთვის. ყველა ხელმისაწვდომი სიგნალის ვარიანტი ჩამოთვლილია mcpwm_io_signals_tერთდროულად ერთზე მეტი პინის დასაყენებლად გამოიყენეთ ფუნქცია mcpwm_set_pin() ერთად mcpwm_pin_config_t.
  3. ტაიმერის შერჩევა. მოწყობილობაში სამი ტაიმერია ხელმისაწვდომი. ტაიმერები ჩამოთვლილია mcpwm_timer_t.
  4. ტაიმერის სიხშირის და საწყისი სამუშაო რეჟიმის დაყენება mcpwm_config_t სტრუქტურა.
  5. მოწოდება mcpwm_init() ზემოთ მოცემული პარამეტრებით, რათა კონფიგურაცია ეფექტური იყოს.

მართვა¶

MCPWM ბლოკთან დაკავშირებული ძრავის სამართავად, მაგალითად, მისი მარცხნივ ან მარჯვნივ მოსაბრუნებლად, ან სიჩქარის შესაცვლელად, ბლოკის გამოსავალზე უნდა მივმართოთ გარკვეული საკონტროლო სიგნალები. გამოსავლები ორგანიზებულია სამ წყვილად. წყვილში ისინი მონიშნულია როგორც „A“ და „B“ და თითოეული მათგანი იმართება ქვემოდულით, რომელსაც „გენერატორი“ ეწოდება. PWM სიგნალის უზრუნველსაყოფად, თავად ოპერატორი, რომელიც შეიცავს ორ გენერატორს, უნდა იყოს დათვლილი სამი ხელმისაწვდომი ტაიმერიდან ერთ-ერთით. API გასამარტივებლად, თითოეული ტაიმერი ავტომატურად ასოცირდება API თან, რათა იმართოს იგივე ინდექსის ოპერატორი, მაგალითად, ტაიმერი 0 ასოცირდება ოპერატორ 0-თან.

გამომავალი სიგნალების კონტროლის შემდეგი ძირითადი გზები არსებობს:

  • ჩვენ შეგვიძლია კონკრეტული სიგნალის სტაბილურად მაღალი ან სტაბილურად დაბალი მართვა ფუნქციით mcpwm_set_signal_high() ან mcpwm_set_signal_low()ეს ძრავას მაქსიმალური სიჩქარით აბრუნებს ან გაჩერდება. არჩეული გამომავალი A ან B სიჩქარიდან გამომდინარე, ძრავა მარჯვნივ ან მარცხნივ ბრუნავს.
  • კიდევ ერთი ვარიანტია გამომავალი მონაცემების PWM სიგნალით მართვა გამოძახებით. mcpwm_start() ან mcpwm_stop()ძრავის სიჩქარე პროპორციული იქნება PWM სამუშაო სიმძლავრის.
  • PWM-ის მოვალეობის შეცვლისთვის mcpwm_set_duty() და მიუთითეთ გადასახადის ღირებულება პროცენტებში. სურვილისამებრ, შეგიძლიათ დარეკოთ mcpwm_set_duty_in_us(), თუ გსურთ მოვალეობის მიკროწამებში დაყენება. მიმდინარე დაყენებული მნიშვნელობის შემოწმება შესაძლებელია გამოძახებით mcpwm_get_duty()PWM სიგნალის ფაზის შეცვლა შესაძლებელია გამოძახებით mcpwm_set_duty_type()მოვალეობა დაყენებულია ინდივიდუალურად თითოეული A და B გამოსავლისთვის, გამოყენებით mcpwm_generator_t კონკრეტული ფუნქციის გამოძახებებში. მოვალეობის მნიშვნელობა ეხება მაღალი ან დაბალი გამომავალი სიგნალის ხანგრძლივობას. ეს კონფიგურირებულია გამოძახებისას mcpwm_init(), როგორც განხილულია ნაწილში კონფიგურაციადა ერთ-ერთი ვარიანტის არჩევა mcpwm_duty_type_t.

შენიშვნა

გამოძახების ფუნქცია mcpwm_set_duty_type() ყოველ ჯერზე შემდეგ mcpwm_set_signal_high() ან mcpwm_set_signal_low() ადრე დაყენებული სამუშაო ციკლით გასაგრძელებლად.

კორექტირება¶

გამოსავალზე სიგნალის რეგულირებისა და ძრავის მუშაობის შეცვლის რამდენიმე გზა არსებობს.

  • დააყენეთ კონკრეტული PWM სიხშირე დარეკვით mcpwm_set_frequency()ეს შეიძლება საჭირო გახდეს კონკრეტული ძრავისა და დრაივერის ელექტრული ან მექანიკური მახასიათებლების დასარეგულირებლად. დაყენებული სიხშირის შესამოწმებლად გამოიყენეთ ფუნქცია mcpwm_get_frequency().
  • შემოიღეთ მკვდარი დრო A და B გამოსავალებს შორის, როდესაც ისინი ცვლიან მდგომარეობას ძრავის ბრუნვის საპირისპირო მიმართულებით. ეს კეთდება ძრავის დრაივერის FET-ების ჩართვა/გამორთვის გადართვის დაყოვნების კომპენსირებისთვის. მკვდარი დროის პარამეტრები განსაზღვრულია mcpwm_deadtime_type_t და გააქტიურებულია დარეკვით mcpwm_deadtime_enable()ამ ფუნქციონალის გამოსართავად დარეკეთ mcpwm_deadtime_disable().
  • ოპერატორის ქვემოდულების გამომავალი სიგნალების სინქრონიზაცია, მაგ. PWM0A/B და PWM1A/B აწევადი კიდის ზუსტად ერთსა და იმავე დროს დასაწყებად, ან მათი გადატანა ერთმანეთზე მოცემული ფაზით. სინქრონიზაცია გააქტიურებულია SYNC SIGNALS ნაჩვენებია ბლოკ-დიაგრამა ზემოთ მოცემული MCPWM-ის და განსაზღვრული mcpwm_sync_signal_tსიგნალის GPIO ზარზე დასაკავშირებლად mcpwm_gpio_init()შემდეგ შეგიძლიათ ჩართოთ სინქრონიზაცია ფუნქციასთან mcpwm_sync_enable()შეყვანის პარამეტრებად მოცემულია MCPWM ერთეული, სინქრონიზაციის ტაიმერი, სინქრონიზაციის სიგნალი და ფაზა ტაიმერის გადადების მიზნით.

შენიშვნა

სინქრონიზაციის სიგნალები აღინიშნება ორი განსხვავებული ჩამოთვლით. პირველი mcpwm_io_signals_t გამოიყენება ფუნქციასთან ერთად mcpwm_gpio_init() სიგნალის შეყვანის წყაროდ GPIO არჩევისას. მეორე mcpwm_sync_signal_t გამოიყენება სინქრონიზაციის ჩართვის ან გამორთვის დროს mcpwm_sync_enable() ან mcpwm_sync_disable().

  • A/B გამომავალი სიგნალების შაბლონის შეცვლა შესაძლებელია MCPWM მრიცხველების მიერ ზემოთ, ქვემოთ და ზემოთ/ქვემოთ დათვლის ინსტრუქციის დაყენებით (თვლის მიმართულების ავტომატური შეცვლა). შესაბამისი კონფიგურაცია ხდება გამოძახებისას. mcpwm_init(), როგორც განხილულია ნაწილში კონფიგურაციადა მრიცხველების ერთ-ერთი ტიპის არჩევა mcpwm_counter_type_tA/B PWM გამომავალი სიგნალების გენერირების ახსნისთვის იხილეთ ESP32 ტექნიკური საცნობარო სახელმძღვანელო.

გადაღება¶

BLDC (უსადნო დენის წყარო, იხილეთ ქვემოთ მოცემული სურათი) ძრავის მართვის ერთ-ერთი მოთხოვნა როტორის პოზიციის აღქმაა. ამ ამოცანის გასაადვილებლად, თითოეული MCPWM ბლოკი უზრუნველყოფს სამ აღქმის შესასვლელს სპეციალურ აპარატურასთან ერთად. აპარატურას შეუძლია შემავალი სიგნალის ზღვრის აღმოჩენა და სიგნალებს შორის დროის გაზომვა. შედეგად, მართვის პროგრამული უზრუნველყოფა უფრო მარტივია და CPU-ს სიმძლავრე შეიძლება გამოყენებულ იქნას სხვა ამოცანებისთვის.

MCPWM-ით უჯაგრისო DC ძრავის მართვის მაგალითი

აღბეჭდვის ფუნქცია შეიძლება გამოყენებულ იქნას სხვა ტიპის ძრავებისთვის ან ამოცანებისთვის. ფუნქციონალი ჩართულია ორ ეტაპად:

  1. GPIO ის კონფიგურაცია ფუნქციების გამოძახებით აღების სიგნალის შეყვანის ფუნქციაზე რეაგირებისთვის. mcpwm_gpio_init() ან mcpwm_set_pin(), რომლებიც აღწერილი იყო ნაწილში კონფიგურაცია.
  2. ფუნქციონალურობის ჩართვა თავად გამოძახებით mcpwm_capture_enable()სასურველი სიგნალის შეყვანის არჩევით mcpwm_capture_signal_t, სიგნალის კიდის დაყენება mcpwm_capture_on_edge_t და სიგნალის რაოდენობის წინასწარი სკალერი.

ზემოთ მოცემულ მეორე ეტაპზე ჩართულია 32-ბიტიანი ჩაწერის ტაიმერი. ტაიმერი უწყვეტად მუშაობს APB საათის მართვით. ტაქტური სიხშირე, როგორც წესი, 80 MHz-ია. თითოეულ ჩაწერის მოვლენაზე ჩაწერის ტაიმერის მნიშვნელობა ინახება დროის შტამპის რეგისტრში, რომლის შემოწმებაც შემდეგ შესაძლებელია გამოძახებით. mcpwm_capture_signal_get_value()ბოლო სიგნალის კიდის შემოწმება შესაძლებელია mcpwm_capture_signal_get_edge().

თუ აღარ არის საჭირო, აღების ფუნქცია შეიძლება გამორთული იყოს mcpwm_capture_disable().

შეცდომების დამმუშავებელი¶

MCPWM-ის თითოეულ ერთეულს შეუძლია გარე სიგნალების აღქმა, რომლებიც ინფორმაციას შეიცავს ძრავის, ძრავის დრაივერის ან MCPWM-თან დაკავშირებული ნებისმიერი სხვა მოწყობილობის გაუმართაობის შესახებ. თითოეულ ერთეულზე სამი ხარვეზის შეყვანაა, რომელთა გადამისამართება შესაძლებელია მომხმარებლის მიერ შერჩეულ GPIO 7. MCPWM შეიძლება კონფიგურირებული იყოს ისე, რომ ხარვეზის სიგნალის მიღებისას A/B გამოსავალზე შეასრულოს ოთხი წინასწარ განსაზღვრული მოქმედებიდან ერთ-ერთი:

  • გამომავალი სიგნალის მიმდინარე მდგომარეობის დაბლოკვა
  • გამომავალი სიმძლავრე დააწესეთ დაბალ დონეზე
  • გამომავალი სიმძლავრის მაღალი დაყენება
  • გამომავალი გადართვა

მომხმარებელმა უნდა განსაზღვროს ძრავის შესაძლო გაუმართაობის რეჟიმები და რა ქმედებები უნდა განხორციელდეს კონკრეტული გაუმართაობის აღმოჩენისას, მაგალითად, ჯაგრისიანი ძრავისთვის ყველა გამოსავლის დაბალ ძაბვაზე გადართვა, ან საფეხუროვანი ძრავისთვის დენის მდგომარეობის დაბლოკვა და ა.შ. ამ მოქმედების შედეგად, ძრავა უნდა გადავიდეს უსაფრთხო მდგომარეობაში, რათა შემცირდეს გაუმართაობით გამოწვეული დაზიანების ალბათობა.

შეცდომების დამმუშავებლის ფუნქცია ჩართულია ორ ეტაპად:

  1. GPIO 7-ის კონფიგურაცია ხარვეზის სიგნალის შესასვლელებად მოქმედებისთვის. ეს კეთდება ანალოგიურად, როგორც ეს ზემოთ მოცემულ ნაწილში აღინიშნა აღების სიგნალებისთვის. ეს მოიცავს სიგნალის დონის დაყენებას ხარვეზის გამოსაწვევად, როგორც ეს განსაზღვრულია mcpwm_fault_input_level_t.
  2. შეცდომების დამმუშავებლის ინიციალიზაცია რომელიმეს გამოძახებით mcpwm_fault_set_oneshot_mode() ან mcpwm_fault_set_cyc_mode()ეს ფუნქციები ადგენს რეჟიმს, რომლითაც MCPWM უნდა იმუშაოს გაუმართაობის სიგნალის არააქტიურობის შემდეგ. შესაძლებელია ორი რეჟიმი:
  • MCPWM ერთეულის მდგომარეობა დაბლოკილი იქნება გადატვირთვამდე - mcpwm_fault_set_oneshot_mode().
  • MCPWM განაახლებს მუშაობას, როგორც კი გაუმართაობის სიგნალი უმოქმედო გახდება - mcpwm_fault_set_cyc_mode().

ფუნქციის გამოძახების პარამეტრები მოიცავს სამი შეცდომის შეყვანიდან ერთ-ერთის შერჩევას, რომელიც განსაზღვრულია mcpwm_fault_signal_t და კონკრეტული მოქმედება A და B შედეგებზე, რომლებიც განსაზღვრულია mcpwm_action_on_pwmxa_t და mcpwm_action_on_pwmxb_t.

კონკრეტული შეცდომის სიგნალი შეიძლება გამორთული იყოს გაშვების დროს გამოძახებით. mcpwm_fault_deinit().

გადამზიდავი¶

MCPWM-ს აქვს გადამტანი ქვემოდული, რომელიც გამოიყენება, თუ საჭიროა ძრავის დრაივერიდან გალვანური იზოლაცია A/B გამომავალი სიგნალების ტრანსფორმატორების მეშვეობით გავლის გზით. A და B გამომავალი სიგნალებიდან ნებისმიერი შეიძლება იყოს 100%-იან დატვირთვით და არ შეიცვალოს, როდესაც ძრავას სრული დატვირთვით სტაბილურად მუშაობა სჭირდება. არაცვლადი სიგნალების ტრანსფორმატორთან შეერთება პრობლემურია, ამიტომ სიგნალები მოდულირდება გადამტანი ქვემოდულით ცვლადი დენის ტალღის ფორმის შესაქმნელად, რაც შეერთებას შესაძლებელს ხდის.

გადამზიდავი ქვემოდულის გამოსაყენებლად, ის ჯერ უნდა ინიციალიზებული იყოს გამოძახებით. mcpwm_carrier_init()მატარებლის პარამეტრები განსაზღვრულია mcpwm_carrier_config_t ფუნქციის გამოძახებისას გამოძახებული სტრუქტურა. შემდეგ გადამზიდავის ფუნქციონალურობის ჩართვა შესაძლებელია გამოძახებით mcpwm_carrier_enable().

გადამზიდავის პარამეტრების შესახებ შეტყობინება შეიძლება გაიგზავნოს გაშვების დროს, სპეციალური ფუნქციების გამოძახებით, რათა შეიცვალოს ცალკეული ველები. mcpwm_carrier_config_t სტრუქტურა, მაგალითად mcpwm_carrier_set_period(), mcpwm_carrier_set_duty_cycle(), mcpwm_carrier_output_invert()და ა.შ.

ეს მოიცავს კარიერის პირველი იმპულსის ხანგრძლივობის ჩართვას და დადგენას mcpwm_carrier_oneshot_mode_enable()დამატებითი ინფორმაციისთვის იხილეთ „PWM გადამზიდავი ქვემოდულის“ განყოფილება. ESP32 ტექნიკური საცნობარო სახელმძღვანელო.

ოპერატორის ფუნქციონალურობის გამოსართავად დარეკეთ mcpwm_carrier_disable().

შეწყვეტები¶

MCPWM შეფერხების დამმუშავებლის რეგისტრაცია შესაძლებელია გამოძახებით mcpwm_isr_register().

SD SPI ჰოსტის დრაივერი¶

მიმოხილვა¶

SD SPI ჰოსტის დრაივერი საშუალებას იძლევა SD ბარათებთან კომუნიკაციისთვის გამოვიყენოთ SPI2 (HSPI) ან SPI3 (VSPI) კონტროლერი. ამ დრაივერის დასახელების ნიმუში აღებულია SDMMC ჰოსტი მძღოლი მათი მსგავსების გამო. ანალოგიურად, ორივე მძღოლის API -იც ძალიან ჰგავს ერთმანეთს.

SD SPI ჰოსტის დრაივერს აქვს შემდეგი რეჟიმები:

  • SPI რეჟიმი: გთავაზობთ უფრო დაბალ გამტარუნარიანობას, მაგრამ პინების შერჩევას უფრო მოქნილს ხდის. GPIO მატრიცის დახმარებით, SPI პერიფერიული მოწყობილობის სიგნალები შეიძლება გადამისამართდეს ნებისმიერ ESP32 პინზე.
  • 1-ბიტიანი SD რეჟიმიგთავაზობთ უფრო მაღალ გამტარუნარიანობას, მაგრამ მოითხოვს სიგნალების მხოლოდ მათთვის განკუთვნილი IO_MUX პინების მეშვეობით გადამისამართებას.

SD SPI დრაივერი იყენებს პროგრამულად კონტროლირებად CS სიგნალს.

ამჟამად, SD SPI დრაივერს არ შეუძლია მრავალძაფიანი გარემოს დამუშავება, რადგან ის არ უჭერს მხარს დროის გაყოფის მულტიპლექსირებას იმავე SPI ავტობუსზე. ეს ნიშნავს, რომ თუ თქვენს აპლიკაციას სჭირდება SD ბარათთან და სხვა მოწყობილობებთან კომუნიკაცია იმავე SPI ავტობუსზე, თავად აპლიკაციამ უნდა უზრუნველყოს, რომ მისი სხვადასხვა დავალებები ერთდროულად არ ცდილობს SPI მონებზე წვდომას.

როგორ გამოვიყენოთ¶

SD SPI ჰოსტის დრაივერის მდგომარეობა და კონფიგურაციები ინახება sdmmc_host_t სტრუქტურა. ამ სტრუქტურის ინიციალიზაცია შესაძლებელია გამოყენებით SDSPI_HOST_DEFAULT მაკრო.

SD სლოტის მდგომარეობა და კონფიგურაციები ინახება sdmmc_slot_config_t სტრუქტურა. გამოიყენეთ მაკრო SDSPI_SLOT_CONFIG_DEFAULT სტრუქტურის ინიციალიზაციისა და ნაგულისხმევი პინების შესაბამისობების შესავსებად (SD რეჟიმის პინების შესაბამისობები).

აპლიკაციების უმეტესობა ჩვეულებრივ მხოლოდ შემდეგი დრაივერის API ფუნქციებს იყენებს:

  • sdspi_host_init()
  • sdspi_host_init_slot()
  • sdspi_host_deinit()

სხვა ფუნქციებს ძირითადად პროტოკოლის დონის SD/SDIO/MMC დრაივერი იყენებს ფუნქციის მაჩვენებლების მეშვეობით. sdmmc_host_t სტრუქტურა. დამატებითი ინფორმაციისთვის იხილეთ SD/SDIO/MMC დრაივერი.

შენიშვნა

SD SPI-ზე მეტი სიჩქარის მხარდაჭერა არ არის შესაძლებელი. SDMMC_FREQ_DEFAULT SPI დრაივერის შეზღუდვების გამო.

API მითითება¶

სათაურის ფაილი¶

  • დრაივერი/include/driver/sdspi_host.h

ფუნქციები¶

esp_err_t sdspi_host_init(void)¶

SD SPI დრაივერის ინიციალიზაცია.

შენიშვნა
This function is not thread safe
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • სხვა შეცდომის კოდები შეიძლება დაბრუნდეს მომავალ ვერსიებში

esp_err_t sdspi_host_init_slot(int სლოტი, კონსტ sdspi_slot_config_t *სლოტის_კონფიგურაცია)¶

კონკრეტული SPI კონტროლერისთვის SD SPI დრაივერის ინიციალიზაცია.

შენიშვნა
This function is not thread safe
შენიშვნა
The SDIO over sdspi needs an extra interrupt line. Call gpio_install_isr_service() before this function.
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_ARG თუ sdspi_init_slot-ს აქვს არასწორი არგუმენტები
  • ESP_ERR_NO_MEM თუ მეხსიერების გამოყოფა შეუძლებელია
  • სხვა შეცდომები ძირითადი spi_master-ისა და gpio დრაივერებიდან
პარამეტრები
  • slotგამოსაყენებელი SPI კონტროლერი (HSPI_HOST ან VSPI_HOST)
  • slot_config: მაჩვენებლის სლოტზე კონფიგურაციის სტრუქტურა

esp_err_t sdspi_host_do_transaction(int სლოტი, sdmmc_command_t *cmdinfo)¶

გაგზავნეთ ბრძანება ბარათზე და მიიღეთ პასუხი.

ეს ფუნქცია ბრუნდება ბრძანების გაგზავნისა და პასუხის მიღებისას, მონაცემების გადაცემისას ან დროის ამოწურვისას.

შენიშვნა
This function is not thread safe w.r.t. init/deinit functions, and bus width/clock speed configuration functions. Multiple tasks can call sdspi_host_do_transaction as long as other sdspi_host_* functions are not called.
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_TIMEOUT თუ პასუხის ან მონაცემთა გადაცემის დრო ამოიწურა
  • ESP_ERR_INVALID_CRC, თუ პასუხი ან მონაცემთა გადაცემის CRC შემოწმება ვერ მოხერხდა
  • ESP_ERR_INVALID_RESPONSE, თუ ბარათმა არასწორი პასუხი გაგზავნა
პარამეტრები
  • slotSPI კონტროლერი (HSPI_HOST ან VSPI_HOST)
  • cmdinfo: მაჩვენებელი სტრუქტურაზე, რომელიც აღწერს ბრძანებას და გადასაცემად მონაცემებს

esp_err_t sdspi_host_set_card_clk(int სლოტი, uint32_t freq_khz)¶

ბარათის საათის სიხშირის დაყენება.

ამჟამად, შესაძლებელია მხოლოდ 40 MHz საათის მთელი წილადების გამოყენება. მაღალი სიჩქარის ბარათებისთვის შესაძლებელია 40 MHz-ის გამოყენება. ნაგულისხმევი სიჩქარის ბარათებისთვის კი - 20 MHz-ის.

შენიშვნა
This function is not thread safe
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • მომავალში შესაძლოა სხვა შეცდომის კოდები დაბრუნდეს
პარამეტრები
  • slotSPI კონტროლერი (HSPI_HOST ან VSPI_HOST)
  • freq_khz: ბარათის საათის სიხშირე, kHz-ში

esp_err_t sdspi_host_deinit(void)¶

გაათავისუფლეთ sdspi_host_init-ის გამოყენებით გამოყოფილი რესურსები.

შენიშვნა
This function is not thread safe
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_STATE თუ sdspi_host_init ფუნქცია არ არის გამოძახებული.

esp_err_t sdspi_host_io_int_enable(int სლოტი)¶

SDIO შეფერხების ჩართვა.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
პარამეტრები
  • slotგამოსაყენებელი SPI კონტროლერი (HSPI_HOST ან VSPI_HOST)

esp_err_t sdspi_host_io_int_wait(int სლოტი, TickType_t ტაიმ-აუტის_წრეები)¶

დაელოდეთ SDIO შეწყვეტას დროის ამოწურვამდე.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
პარამეტრები
  • slotგამოსაყენებელი SPI კონტროლერი (HSPI_HOST ან VSPI_HOST)
  • timeout_ticks: ტაიმაუტის დასრულებამდე ლოდინის ტიკები.

მაკროები¶

SDSPI_HOST_DEFAULT()¶

ნაგულისხმევი sdmmc_host_t სტრუქტურის ინიციალიზატორი SD-ზე SPI დრაივერისთვის.

იყენებს SPI რეჟიმს და მაქსიმალური სიხშირე დაყენებულია 20 MHz-ზე

„slot“-ის დაყენება შესაძლებელია HSPI_HOST-ზე და VSPI_HOST-ზე.

SDSPI_SLOT_NO_CD¶

მიუთითებს, რომ ბარათის ამოცნობის ხაზი არ გამოიყენება

SDSPI_SLOT_NO_WP¶

მიუთითებს, რომ ჩაწერის დამცავი ხაზი არ გამოიყენება

SDSPI_SLOT_NO_INT¶

მიუთითებს, რომ შეწყვეტის ხაზი არ გამოიყენება

SDSPI_SLOT_CONFIG_DEFAULT()¶

მაკრო, რომელიც განსაზღვრავს SPI ჰოსტის ნაგულისხმევ კონფიგურაციას

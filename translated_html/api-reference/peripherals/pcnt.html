პულსის მრიცხველი¶

შესავალი¶

PCNT (პულსების მრიცხველი) მოდული შექმნილია შემავალი სიგნალის აღმავალი და/ან დაღმავალი კიდეების რაოდენობის დასათვლელად. თითოეულ იმპულსების მრიცხველს აქვს 16-ბიტიანი ხელმოწერილი მრიცხველის რეგისტრი და ორი არხი, რომელთა კონფიგურაციაც შესაძლებელია მრიცხველის გაზრდის ან შემცირებისთვის. თითოეულ არხს აქვს სიგნალის შესასვლელი, რომელიც იღებს სიგნალის კიდეების აღმოსაჩენად, ასევე საკონტროლო შესასვლელი, რომლის გამოყენებაც შესაძლებელია სიგნალის შესასვლელის ჩასართავად ან გამოსართავად. შესასვლელებს აქვთ დამატებითი ფილტრები, რომელთა გამოყენება შესაძლებელია სიგნალში არასასურველი ხარვეზების აღმოსაფხვრელად.

ფუნქციონალურობის მიმოხილვა¶

ამ API ფუნქციონალურობის აღწერა ოთხ ნაწილად იყოფა:

  • კონფიგურაცია - აღწერს მრიცხველის კონფიგურაციის პარამეტრებს და მრიცხველის დაყენების წესს.
  • მრიცხველის მართვა - იძლევა ინფორმაციას მრიცხველის პაუზის, გაზომვისა და გასუფთავების საკონტროლო ფუნქციების შესახებ.
  • ფილტრაციის იმპულსები - აღწერს იმპულსების ფილტრაციის ვარიანტებს და მრიცხველის მართვის სიგნალებს.
  • შეფერხებების გამოყენება - წარმოგიდგენთ, თუ როგორ უნდა გამოიწვიოთ შეფერხებები მრიცხველის კონკრეტულ მდგომარეობებზე.

კონფიგურაცია¶

PCNT მოდულს აქვს რვა დამოუკიდებელი დათვლის „ერთეული“, რომლებიც დანომრილია 0-დან 7-მდე. API ისინი მოხსენიებულია გამოყენებით pcnt_unit_tთითოეულ მოწყობილობას აქვს ორი დამოუკიდებელი არხი, რომლებიც დანომრილია 0-ით და 1-ით და მითითებულია pcnt_channel_t.

კონფიგურაცია მოცემულია ცალკე თითოეული მოწყობილობის არხისთვის, გამოყენებით pcnt_config_t და მოიცავს:

  • მოწყობილობა და არხის ნომერი, რომელზეც ეს კონფიგურაცია ვრცელდება.
  • GPIO იმპულსური შეყვანის და იმპულსური კარიბჭის შეყვანის ნომრები.
  • პარამეტრების ორი წყვილი: pcnt_ctrl_mode_t და pcnt_count_mode_t იმის დასადგენად, თუ როგორ რეაგირებს მრიცხველი საკონტროლო სიგნალის სტატუსის მიხედვით და როგორ ხდება დათვლა იმპულსების დადებით/უარყოფით კიდეზე.
  • ორი საზღვარი მნიშვნელობა (მინიმალური/მაქსიმალური), რომლებიც გამოიყენება საკონტროლო წერტილების დასადგენად და შეფერხებების გამოსაწვევად, როდესაც იმპულსების რაოდენობა აკმაყოფილებს კონკრეტულ ზღვარს.

კონკრეტული არხის დაყენება შემდეგ ხდება ფუნქციის გამოძახებით pcnt_unit_config() ზემოთ pcnt_config_t როგორც შეყვანის პარამეტრი.

კონფიგურაციაში იმპულსის ან მართვის შეყვანის პინის გამოსართავად, მიუთითეთ PCNT_PIN_NOT_USED GPIO ნომრის ნაცვლად.

მრიცხველის მართვა¶

დაყენების შემდეგ pcnt_unit_config(), მრიცხველი დაუყოვნებლივ იწყებს მუშაობას. დაგროვილი პულსის რაოდენობის შემოწმება შესაძლებელია ღილაკზე დარეკვით pcnt_get_counter_value().

არსებობს რამდენიმე ფუნქცია, რომელიც საშუალებას გაძლევთ აკონტროლოთ მრიცხველის მუშაობა: pcnt_counter_pause(), pcnt_counter_resume() და pcnt_counter_clear()

ასევე შესაძლებელია ადრე დაყენებული მრიცხველის რეჟიმების დინამიურად შეცვლა pcnt_unit_config() დარეკვით pcnt_set_mode().

სურვილის შემთხვევაში, იმპულსური შეყვანის პინი და საკონტროლო შეყვანის პინი შეიძლება შეიცვალოს „მომენტალურად“ გამოყენებით. pcnt_set_pin()კონკრეტული შეყვანის გამოსართავად, ფუნქციის პარამეტრად მიუთითეთ PCNT_PIN_NOT_USED GPIO ნომრის ნაცვლად.

შენიშვნა

იმისათვის, რომ მრიცხველმა არ გამოტოვოს არცერთი იმპულსი, იმპულსის ხანგრძლივობა უნდა იყოს ერთ APB_CLK ციკლზე (12.5 ns) მეტი. იმპულსები APB_CLK საათის კიდეებზეა შერჩეული და შესაძლოა გამოტოვდეს, თუ კიდეებს შორის მოხვდება. ეს ეხება მრიცხველის მუშაობას ან მის გარეშე. შემფუთველი.

ფილტრაციის იმპულსები¶

PCNT ბლოკს აქვს ფილტრები თითოეულ იმპულსურ და საკონტროლო შესასვლელზე, რაც სიგნალებში მოკლევადიანი ხარვეზების იგნორირების შესაძლებლობას იძლევა.

იგნორირებული იმპულსების ხანგრძლივობა მოცემულია APB_CLK საათის ციკლებში გამოძახებით pcnt_set_filter_value()მიმდინარე ფილტრის პარამეტრის შემოწმება შესაძლებელია pcnt_get_filter_value()APB_CLK საათი მუშაობს 80 MHz სიხშირეზე.

ფილტრი ამოქმედდება / შეჩერდება დარეკვით pcnt_filter_enable() / pcnt_filter_disable().

შეფერხებების გამოყენება¶

არსებობს სახელმწიფოს საწინააღმდეგო მეთვალყურეობის ხუთი ღონისძიება, რომლებიც განსაზღვრულია pcnt_evt_type_t, რომლებსაც შეუძლიათ შეფერხების გამოწვევა. მოვლენა ხდება იმპულსების მრიცხველზე, რომელიც აღწევს კონკრეტულ მნიშვნელობებს:

  • მინიმალური ან მაქსიმალური რაოდენობის მნიშვნელობები: counter_l_lim ან counter_h_lim მოწოდებულია pcnt_config_t როგორც განხილულია კონფიგურაცია
  • ფუნქციის გამოყენებით დაყენებულია 0 ან 1 ზღურბლის მნიშვნელობები pcnt_set_event_value().
  • პულსის რაოდენობა = 0

ზემოთ ჩამოთვლილი მოვლენების მომსახურების შეფერხების რეგისტრაციის, ჩართვის ან გამორთვისთვის დარეკეთ pcnt_isr_register(), pcnt_intr_enable(). და pcnt_intr_disable()ზღვრული მნიშვნელობების მიღწევისას მოვლენების ჩასართავად ან გამოსართავად, ასევე დაგჭირდებათ ფუნქციების გამოძახება. pcnt_event_enable() და pcnt_event_disable().

ამჟამად დაყენებული ზღვრული მნიშვნელობების შესამოწმებლად გამოიყენეთ ფუნქცია pcnt_get_event_value().

განაცხადის მაგალითი¶

პულსის მრიცხველი მართვის სიგნალით და მოვლენის შეწყვეტის მაგალითით: პერიფერიული მოწყობილობები/ცენტი.

API მითითება¶

  • დრაივერი/ინკლუზია/დრაივერი/pcnt.h

esp_err_t pcnt_unit_config(კონსტ pcnt_config_t *pcnt_config)¶

პულსის მრიცხველის ბლოკის კონფიგურაცია.

შენიშვნა
This function will disable three events: PCNT_EVT_L_LIM, PCNT_EVT_H_LIM, PCNT_EVT_ZERO.
დაბრუნება
  • ESP_OK წარმატება
  • ESP_ERR_INVALID_STATE pcnt დრაივერი უკვე ინიციალიზებულია
  • ESP_ERR_INVALID_ARG პარამეტრის შეცდომა
პარამეტრები
  • pcnt_configპულსის მრიცხველის ინდიკატორის კონფიგურაციის პარამეტრი

esp_err_t pcnt_get_counter_value(pcnt_unit_t pcnt_unite, int16_t *დათვლა)¶

დაბრუნება
  • ESP_OK წარმატება
  • ESP_ERR_INVALID_STATE pcnt დრაივერი არ არის ინიციალიზებული
  • ESP_ERR_INVALID_ARG პარამეტრის შეცდომა
პარამეტრები
  • pcnt_unitპულსის მრიცხველის ერთეულის ნომერი
  • count: მრიცხველის მნიშვნელობის მიღების მაჩვენებელი

  • ESP_ERR_INVALID_STATE pcnt დრაივერი არ არის ინიციალიზებული

პარამეტრები

  • ESP_ERR_INVALID_ARG პარამეტრის შეცდომა
პარამეტრები

esp_err_t pcnt_isr_register(void (*ფნ)(void *), void *არგუმენტი, int intr_alloc_flags, pcnt_isr_handle_t *სახელური, )¶

დარეგისტრირდით PCNT შეწყვეტის დამმუშავებელზე, დამმუშავებელი არის ISR. დამმუშავებელი მიმაგრებული იქნება იმავე CPU ბირთვზე, რომელზეც ეს ფუნქცია მუშაობს. გთხოვთ, არ გამოიყენოთ pcnt_isr_service_install, თუ ეს ფუნქცია გამოიძახეს.

დაბრუნება
  • ESP_OK წარმატება
  • ESP_ERR_NOT_FOUND ვერ პოულობს დროშებს შესაბამისი შეფერხებას.
  • ESP_ERR_INVALID_ARG ფუნქციის მაჩვენებლის შეცდომა.
პარამეტრები
  • fnშეფერხების დამმუშავებლის ფუნქცია.
  • arg: დამმუშავებლის ფუნქციის პარამეტრი
  • intr_alloc_flags: დროშები, რომლებიც გამოიყენება შეწყვეტის გამოსაყოფად. ერთი ან რამდენიმე (OR წითელი) ESP_INTR_FLAG_* მნიშვნელობა. დამატებითი ინფორმაციისთვის იხილეთ esp_intr_alloc.h.
  • handle: მაჩვენებელი დასაბრუნებელი სახელურისთვის. თუ ის არ არის NULL, შეწყვეტის სახელური დაბრუნდება აქ. საჭიროების შემთხვევაში, ამ ISR სერვისის რეგისტრაციის გასაუქმებლად გამოიძახება esp_intr_free, მაგრამ მხოლოდ იმ შემთხვევაში, თუ სახელური არ არის NULL.

esp_err_t pcnt_isr_handler_add(pcnt_unit_t ერთეული, void (*isr_handler)(void *), void *არგუმენტები, )¶

გამოიძახეთ ეს ფუნქცია pcnt_isr_service_install() ფუნქციის გამოყენების შემდეგ PCNT დრაივერის ISR დამმუშავებლის სერვისის დასაყენებლად.

ISR დამმუშავებლების IRAM_ATTR-ით გამოცხადება საჭირო არ არის, თუ pcnt_isr_service_install() ფუნქციაში ISR-ის გამოყოფისას არ გადასცემთ ESP_INTR_FLAG_IRAM ფლაგს.

ეს ISR დამმუშავებელი გამოიძახება ISR-დან. ამგვარად, არსებობს დასტის ზომის ლიმიტი (კონფიგურირებადია როგორც „ISR დასტის ზომა“ menuconfig ში). ეს ლიმიტი უფრო მცირეა გლობალურ PCNT შეწყვეტის დამმუშავებელთან შედარებით ირიბი შეფერხების დამატებითი დონის გამო.

დაბრუნება

  • ESP_ERR_INVALID_STATE pcnt დრაივერი არ არის ინიციალიზებული
  • ESP_ERR_INVALID_ARG პარამეტრის შეცდომა

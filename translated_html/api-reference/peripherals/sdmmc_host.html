SDMMC ჰოსტის დრაივერი¶

მიმოხილვა¶

ESP32-ის SDMMC მასპინძელ პერიფერიულ მოწყობილობას ორი სლოტი აქვს:

  • სლოტი 0 (SDMMC_HOST_SLOT_0) არის 8-ბიტიანი სლოტი. ის იყენებს HS1_* სიგნალები PIN MUX-ში.
  • სლოტი 1 (SDMMC_HOST_SLOT_1) არის 4-ბიტიანი სლოტი. ის იყენებს HS2_* სიგნალები PIN MUX-ში.

ამ სლოტების პინების შესაბამისობები მოცემულია ქვემოთ მოცემულ ცხრილში.

CMD GPIO GPIO 15-ე კურსი
CLK GPIO GPIO 14-ე კურსი
D0 GPIO 7 GPIO 2
D1 მე-7 8 GPIO GPIO 4
D2 მე-7 9 GPIO GPIO 12-ე კურსი
D3 GPIO 10 ტერმინალი GPIO 13-ე კურსი
D4 GPIO 16-ე კურსი  
D5 GPIO 17-ე კურსი

WP ნებისმიერი შეყვანა GPIO მატრიცის მეშვეობით

ბარათის აღმოჩენის და ჩაწერის დაცვის სიგნალები შეიძლება გადამისამართდეს ნებისმიერ პინებზე GPIO მატრიცის გამოყენებით. პინების დასაჯავშნად, დააყენეთ gpio_cd და gpio_wp წევრები sdmmc_slot_config_t სტრუქტურა დარეკვამდე sdmmc_host_init_slot()გთხოვთ გაითვალისწინოთ, რომ SDIO ბარათებთან მუშაობისას არ არის რეკომენდებული Card Detect PIN-ის მითითება, რადგან ESP32-ში ბარათის აღმოჩენის სიგნალმა ასევე შეიძლება გამოიწვიოს SDIO მონათმფლობელის შეფერხება.

გაფრთხილება

სლოტი 0-ის მიერ გამოყენებული პინები (HS1_*) ასევე გამოიყენება SPI ფლეშ მეხსიერება ჩიპის ESP32-WROOM და ESP32-WROVER მოდულებში შესაერთებლად. ეს პინები არ შეიძლება გაზიარდეს SD ბარათსა და SPI ფლეშ მეხსიერება შორის. თუ საჭიროა Slot 0-ის გამოყენება, შეაერთეთ SPI ფლეშ მეხსიერება სხვადასხვა პინებთან და შესაბამისად დააყენეთ eFuse-ები.

მხარდაჭერილი სიჩქარის რეჟიმები¶

SDMMC მასპინძლის დრაივერი მხარს უჭერს შემდეგ სიჩქარის რეჟიმებს:

  • ნაგულისხმევი სიჩქარე (20 MHz), 4 ხაზი/1 ხაზი (SD ბარათებით) და 8 ხაზი (3.3 V eMMC-ით)
  • მაღალი სიჩქარე (40 MHz), 4-ხაზიანი/1-ხაზიანი (SD ბარათებით) და 8-ხაზიანი (3.3 V eMMC-ით)
  • მაღალი სიჩქარის DDR (40 MHz), 4-ხაზიანი (3.3 V eMMC-ით)

სიჩქარის რეჟიმები ამჟამად არ არის მხარდაჭერილი:

  • მაღალი სიჩქარის DDR რეჟიმი, 8-ხაზიანი eMMC
  • UHS-I 1.8 V რეჟიმები, 4-ხაზიანი SD ბარათები

SDMMC მასპინძელი დრაივერის გამოყენება¶

ქვემოთ ჩამოთვლილი ყველა ფუნქციიდან, აპლიკაციების უმეტესობა პირდაპირ მხოლოდ შემდეგს გამოიყენებს:

  • sdmmc_host_init()
  • sdmmc_host_init_slot()
  • sdmmc_host_deinit()

სხვა ფუნქციები, როგორიცაა ქვემოთ მოცემული, გამოიძახება SD/MMC პროტოკოლის ფენის მიერ ფუნქციის მაჩვენებლების მეშვეობით. sdmmc_host_t სტრუქტურა:

  • sdmmc_host_set_bus_width()
  • sdmmc_host_set_card_clk()
  • sdmmc_host_do_transaction()

ავტობუსის სიგანისა და სიხშირის კონფიგურაცია¶

ნაგულისხმევი ინიციალიზატორებით sdmmc_host_t და sdmmc_slot_config_t (SDMMC_HOST_DEFAULT და SDMMC_SLOT_CONFIG_DEFAULT), SDMMC მასპინძელი დრაივერი შეეცდება გამოიყენოს ბარათის მიერ მხარდაჭერილი ყველაზე ფართო ავტობუსი (4 ხაზი SD-სთვის, 8 ხაზი eMMC-სთვის) და 20 MHz სიხშირე.

იმ დიზაინებში, სადაც კომუნიკაცია 40 MHz სიხშირეზეა შესაძლებელი, ავტობუსის სიხშირის გაზრდა შესაძლებელია შეცვლით. max_freq_khz სფერო sdmmc_host_t:

sdmmc_host_t host = SDMMC_HOST_DEFAULT();
host.max_freq_khz = SDMMC_FREQ_HIGHSPEED;

ავტობუსის სიგანის კონფიგურაციისთვის, დააყენეთ width სფერო sdmmc_slot_config_tმაგალითად, 1-ხაზიანი რეჟიმის დასაყენებლად:

sdmmc_slot_config_t slot = SDMMC_SLOT_CONFIG_DEFAULT();
slot.width = 1;

აგრეთვე იხილეთ¶

იხილეთ SD/SDIO/MMC დრაივერი უფრო მაღალი დონის დრაივერისთვის, რომელიც ახორციელებს პროტოკოლის ფენას.

იხილეთ SD SPI ჰოსტის დრაივერი მსგავსი დრაივერისთვის, რომელიც იყენებს SPI კონტროლერს და შემოიფარგლება SD პროტოკოლის SPI რეჟიმით.

იხილეთ SD Pull-up-ის მოთხოვნები Pull-up მხარდაჭერისა და მოდულებისა და განვითარების ნაკრებების თავსებადობისთვის.

API მითითება¶

სათაურის ფაილი¶

  • დრაივერი/include/driver/sdmmc_host.h

ფუნქციები¶

esp_err_t sdmmc_host_init(void)¶

SDMMC მასპინძელი პერიფერიული მოწყობილობის ინიციალიზაცია.

შენიშვნა
This function is not thread safe
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_STATE, თუ sdmmc_host_init უკვე გამოძახებულია
  • ESP_ERR_NO_MEM თუ მეხსიერების გამოყოფა შეუძლებელია

esp_err_t sdmmc_host_init_slot(int სლოტი, კონსტ sdmmc_slot_config_t *სლოტის_კონფიგურაცია)¶

SDMMC პერიფერიული მოწყობილობის მოცემული სლოტის ინიციალიზაცია.

ESP32-ზე, SDMMC პერიფერიულ მოწყობილობას ორი სლოტი აქვს:

  • სლოტი 0: 8 ბიტიანი, PIN MUX-ში HS1_* სიგნალებთან შესაბამისობაშია
  • სლოტი 1: 4 ბიტიანი, PIN MUX-ში HS2_* სიგნალებთან შესაბამისობაშია

ბარათის აღმოჩენის და ჩაწერის დაცვის სიგნალები შეიძლება გადამისამართდეს ნებისმიერ GPIO ზე GPIO მატრიცის გამოყენებით.

შენიშვნა
This function is not thread safe
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_STATE, თუ ჰოსტი არ არის ინიციალიზებული sdmmc_host_init-ის გამოყენებით.
პარამეტრები
  • slot: სლოტის ნომერი (SDMC_HOST_SLOT_0 ან SDMMC_HOST_SLOT_1)
  • slot_config: სლოტის დამატებითი კონფიგურაცია

esp_err_t sdmmc_host_set_bus_width(int სლოტი, size_t სიგანე)¶

აირჩიეთ მონაცემთა გადაცემისთვის გამოსაყენებელი ავტობუსის სიგანე.

SD/MMC ბარათი უნდა იყოს ინიციალიზებული ამ ბრძანებამდე და ბარათზე უნდა გაიგზავნოს ავტობუსის სიგანის დაყენების ბრძანება (მაგ. SD_APP_SET_BUS_WIDTH)

შენიშვნა
This function is not thread safe
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_INVALID_ARG თუ სლოტის ნომერი ან სიგანე არასწორია
პარამეტრები
  • slot: სლოტის ნომერი (SDMC_HOST_SLOT_0 ან SDMMC_HOST_SLOT_1)
  • width: ავტობუსის სიგანე (1, 4 ან 8 სლოტი 0-ისთვის; 1 ან 4 სლოტი 1-ისთვის)

size_t sdmmc_host_get_slot_width(int სლოტი)¶

ავტობუსის სიგანის კონფიგურაცია sdmmc_host_init_slot მონაცემთა გადაცემისთვის გამოსაყენებლად.

დაბრუნება
configured bus width of the specified slot.
პარამეტრები
  • slot: სლოტის ნომერი (SDMC_HOST_SLOT_0 ან SDMMC_HOST_SLOT_1)

esp_err_t sdmmc_host_set_card_clk(int სლოტი, uint32_t freq_khz)¶

ბარათის საათის სიხშირის დაყენება.

ამჟამად, შესაძლებელია მხოლოდ 40 MHz საათის მთელი წილადების გამოყენება. მაღალი სიჩქარის ბარათებისთვის შესაძლებელია 40 MHz-ის გამოყენება. ნაგულისხმევი სიჩქარის ბარათებისთვის კი - 20 MHz-ის.

შენიშვნა
This function is not thread safe
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • მომავალში შესაძლოა სხვა შეცდომის კოდები დაბრუნდეს
პარამეტრები
  • slot: სლოტის ნომერი (SDMC_HOST_SLOT_0 ან SDMMC_HOST_SLOT_1)
  • freq_khz: ბარათის საათის სიხშირე, kHz-ში

esp_err_t sdmmc_host_set_bus_ddr_mode(int სლოტი, bool ddr_ჩართულია)¶

SD ინტერფეისის DDR რეჟიმის ჩართვა ან გამორთვა.

დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_NOT_SUPPORTED თუ DDR რეჟიმი არ არის მხარდაჭერილი ამ სლოტზე
პარამეტრები
  • slot: სლოტის ნომერი (SDMC_HOST_SLOT_0 ან SDMMC_HOST_SLOT_1)
  • ddr_enabled: DDR რეჟიმის ჩართვა ან გამორთვა

esp_err_t sdmmc_host_do_transaction(int სლოტი, sdmmc_command_t *cmdinfo)¶

გაგზავნეთ ბრძანება ბარათზე და მიიღეთ პასუხი.

ეს ფუნქცია ბრუნდება ბრძანების გაგზავნისა და პასუხის მიღებისას, მონაცემების გადაცემისას ან დროის ამოწურვისას.

შენიშვნა
This function is not thread safe w.r.t. init/deinit functions, and bus width/clock speed configuration functions. Multiple tasks can call sdmmc_host_do_transaction as long as other sdmmc_host_* functions are not called.
ყურადღება
Data buffer passed in cmdinfo->data must be in DMA capable memory
დაბრუნება
  • ESP_OK წარმატების შემთხვევაში
  • ESP_ERR_TIMEOUT თუ პასუხის ან მონაცემთა გადაცემის დრო ამოიწურა
  • ESP_ERR_INVALID_CRC, თუ პასუხი ან მონაცემთა გადაცემის CRC შემოწმება ვერ მოხერხდა
  • ESP_ERR_INVALID_RESPONSE, თუ ბარათმა არასწორი პასუხი გაგზავნა
  • ESP_ERR_INVALID_SIZE თუ მონაცემთა გადაცემის ზომა SD პროტოკოლში არ არის სწორი
  • ESP_ERR_INVALID_ARG, თუ მონაცემთა ბუფერი არ არის DMA-სთან თავსებად მეხსიერებაში
პარამეტრები
  • slot: სლოტის ნომერი (SDMC_HOST_SLOT_0 ან SDMMC_HOST_SLOT_1)
  • cmdinfo: მაჩვენებელი სტრუქტურაზე, რომელიც აღწერს ბრძანებას და გადასაცემად მონაცემებს

esp_err_t sdmmc_host_pullup_en(int სლოტი, int სიგანე)¶

sd პინების აწევის ჩართვა.

შენიშვნა
You should always place actual pullups on the lines instead of using this function. Internal pullup resistance are high and not sufficient, may cause instability in products. This is for debug or examples only.

  • widthთქვენი კონფიგურაციის ბიტის სიგანე, 1 ან 4.

SDMMC_HOST_DEFAULT()¶

ნაგულისხმევი sdmmc_host_t SDMMC პერიფერიული მოწყობილობის სტრუქტურის ინიციალიზატორი.

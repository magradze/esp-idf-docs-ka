ტაიმერი¶

[中文]

შესავალი¶

ESP32 ჩიპი შეიცავს აპარატურული ტაიმერების ორ ჯგუფს. თითოეულ ჯგუფს აქვს ორი ზოგადი დანიშნულების აპარატურული ტაიმერი. ისინი ყველა 64-ბიტიანი ზოგადი ტაიმერებია, რომლებიც დაფუძნებულია 16-ბიტიან წინასწარ სკალერებსა და 64-ბიტიან აღმავალი/დაღმავალი მრიცხველებზე, რომელთა ავტომატური გადატვირთვა შესაძლებელია.

ფუნქციური მიმოხილვა¶

ამ დოკუმენტის შემდეგი ნაწილები მოიცავს ტაიმერის კონფიგურაციისა და მუშაობის ტიპურ ნაბიჯებს:

  • ტაიმერის ინიციალიზაცია - მოიცავს იმას, თუ რომელი პარამეტრების დაყენებაა საჭირო ტაიმერის მუშაობისთვის და ასევე, რა კონკრეტული ფუნქციონალია გათვალისწინებული ტაიმერის კონფიგურაციიდან გამომდინარე.
  • ტაიმერის კონტროლი - აღწერს, თუ როგორ წაიკითხოთ ტაიმერის მნიშვნელობა, შეაჩეროთ ან ჩართოთ ტაიმერი და შეცვალოთ მისი მუშაობის წესი.
  • სიგნალიზაცია - გვიჩვენებს, თუ როგორ დააყენოთ და გამოიყენოთ სიგნალიზაცია.
  • შეწყვეტები- განმარტავს, თუ როგორ უნდა ჩართოთ და გამოიყენოთ შეფერხებები.

ტაიმერის ინიციალიზაცია¶

ESP32 ტაიმერის ორი ჯგუფი, თითოეულში ორი ტაიმერით, გამოსაყენებლად სულ ოთხ ინდივიდუალურ ტაიმერს იძლევა. ESP32 ტაიმერის ჯგუფი უნდა განისაზღვროს შემდეგი გამოყენებით: timer_group_tჯგუფში ინდივიდუალური ტაიმერი უნდა იყოს იდენტიფიცირებული timer_idx_t.

პირველ რიგში, ტაიმერი უნდა ინიციალიზებული იყოს ფუნქციის გამოძახებით timer_init() და სტრუქტურის გავლა timer_config_t მასში განსაზღვრულია, თუ როგორ უნდა იმუშაოს ტაიმერი. კერძოდ, შესაძლებელია შემდეგი ტაიმერის პარამეტრების დაყენება:

  • გამყოფი: ადგენს, თუ რამდენად სწრაფად „წიკწიკებს“ ტაიმერის მრიცხველი. პარამეტრი divider გამოიყენება შემომავალი 80 MHz APB_CLK საათის გამყოფად.
  • რეჟიმი: ადგენს, უნდა იყოს თუ არა მრიცხველი მზარდი ან კლებადი. მისი განსაზღვრა შესაძლებელია გამოყენებით counter_dir ერთ-ერთი მნიშვნელობის არჩევით timer_count_dir_t.
  • მრიცხველის ჩართვათუ მრიცხველი ჩართულია, ის გამოძახებისთანავე დაიწყებს გაზრდას/კლებას timer_init(). თქვენ შეგიძლიათ შეცვალოთ ქცევა counter_en ერთ-ერთი მნიშვნელობის არჩევით timer_start_t.
  • სიგნალიზაციის ჩართვა: შეიძლება დაყენდეს გამოყენებით alarm_en.
  • ავტომატური გადატვირთვა: ადგენს, თუ მრიცხველი უნდა auto_reload ტაიმერის სიგნალიზაციაზე საწყისი მრიცხველის მნიშვნელობა ან გააგრძელეთ გაზრდა ან კლება.
  • შეწყვეტის ტიპი: აირჩიეთ, ტაიმერის სიგნალიზაციაზე რომელი შეფერხების ტიპი უნდა გააქტიურდეს. დააყენეთ განსაზღვრული მნიშვნელობა timer_intr_mode_t.

ტაიმერის პარამეტრების მიმდინარე მნიშვნელობების მისაღებად გამოიყენეთ ფუნქცია timer_get_config().

ტაიმერის კონტროლი¶

ტაიმერის ჩართვის შემდეგ, მისი მრიცხველი დაიწყებს მუშაობას. ტაიმერის ჩასართავად გამოიძახეთ ფუნქცია timer_init() თან counter_en დაყენებულია true, ან დარეკეთ timer_start()ტაიმერის საწყისი მრიცხველის მნიშვნელობის მითითება შეგიძლიათ გამოძახებით timer_set_counter_value()ტაიმერის მიმდინარე მნიშვნელობის შესამოწმებლად, დარეკეთ timer_get_counter_value() ან timer_get_counter_time_sec().

ტაიმერის ნებისმიერ დროს შესაჩერებლად, დარეკეთ timer_pause(). განახლებისთვის, დარეკეთ timer_start().

ტაიმერის ხელახლა კონფიგურაციისთვის შეგიძლიათ დარეკოთ timer_init()ეს ფუნქცია აღწერილია ნაწილში ტაიმერის ინიციალიზაცია.

ასევე შეგიძლიათ ტაიმერის ხელახლა კონფიგურაცია ცალკეული ფუნქციების გამოყენებით ინდივიდუალური პარამეტრების შესაცვლელად:

პარამეტრი სპეციალური ფუნქცია აღწერა
გამყოფი timer_set_divider() შეცვალეთ ტიკტიკის სიხშირე. არაპროგნოზირებადი შედეგების თავიდან ასაცილებლად, გამყოფის შეცვლისას ტაიმერი უნდა შეჩერდეს. თუ ტაიმერი მუშაობს, timer_set_divider() აჩერებს მას, ცვლის პარამეტრებს და ხელახლა ჩართავს ტაიმერს.
რეჟიმი timer_set_counter_mode() დააყენეთ, უნდა იყოს თუ არა მრიცხველი მზარდი ან კლებადი
ავტომატური გადატვირთვა timer_set_auto_reload() დააყენეთ, უნდა ჩაიტვირთოს თუ არა საწყისი მრიცხველის მნიშვნელობა ტაიმერის სიგნალიზაციაზე

სიგნალიზაცია¶

მაღვიძარას დასაყენებლად გამოიძახეთ ფუნქცია timer_set_alarm_value() და შემდეგ ჩართეთ სიგნალიზაცია timer_set_alarm()სიგნალიზაცია ასევე შეიძლება ჩაირთოს ტაიმერის ინიციალიზაციის ეტაპზე, როდესაც timer_init() ეწოდება.

სიგნალიზაციის ჩართვისა და ტაიმერის მიერ სიგნალიზაციის მნიშვნელობაზე მიღწევის შემდეგ, კონფიგურაციიდან გამომდინარე, შესაძლებელია შემდეგი ორი მოქმედების შესრულება:

  • შეფერხება გააქტიურდება, თუ ის წინასწარ არის კონფიგურირებული. იხილეთ განყოფილება შეწყვეტები შეფერხებების კონფიგურაციის შესახებ.
  • როდესაც auto_reload ჩართულია, ტაიმერის მრიცხველი ავტომატურად გადაიტვირთება, რათა თავიდან დაიწყოს დათვლა წინასწარ კონფიგურირებული მნიშვნელობიდან. ეს მნიშვნელობა წინასწარ უნდა იყოს დაყენებული timer_set_counter_value().

შენიშვნა

  • თუ სიგნალიზაციის მნიშვნელობა დაყენებულია და ტაიმერმა უკვე მიაღწია ამ მნიშვნელობას, სიგნალიზაცია დაუყოვნებლივ ჩაირთვება.
  • გააქტიურების შემდეგ, სიგნალიზაცია ავტომატურად გამორთულია და ხელახლა გასააქტიურებლად საჭიროა მისი ხელახლა ჩართვა.

მითითებული განგაშის მნიშვნელობის შესამოწმებლად, დარეკეთ timer_get_alarm_value().

შეწყვეტები¶

კონკრეტული ტაიმერის ან ტაიმერის ჯგუფის შეფერხების დამმუშავებლის რეგისტრაცია შესაძლებელია გამოძახებით. timer_isr_register().

ტაიმერის ჯგუფისთვის შეფერხებების ჩასართავად, დარეკეთ timer_group_intr_enable(), კონკრეტული ტაიმერის ზარისთვის timer_enable_intr()ტაიმერის ჯგუფისთვის შეფერხებების გამოსართავად, დარეკეთ timer_group_intr_disable(), მითითებული ტაიმერის მისაღებად, დარეკეთ timer_disable_intr().

შეფერხების სერვისის რუტინის (ISR) ფარგლებში შეფერხების დამუშავებისას, შეფერხების სტატუსის ბიტი უნდა იყოს ცალსახად გასუფთავებული. ამისათვის დააყენეთ TIMERGN.int_clr_timers.tM სტრუქტურა, განსაზღვრული soc/esp32/include/soc/timer_group_struct.hამ სტრუქტურაში, N არის ტაიმერის ჯგუფის ნომერი [0, 1], M არის ტაიმერის ნომერი [0, 1]. მაგალითად, ტაიმერის ჯგუფ 0-ში ტაიმერის 1-ისთვის შეფერხების სტატუსის ბიტის გასასუფთავებლად, გამოიძახეთ შემდეგი:

TIMERG0.int_clr_timers.t1 = 1

შეფერხებების გამოყენების შესახებ დამატებითი ინფორმაციისთვის იხილეთ ქვემოთ მოცემული აპლიკაციის მაგალითი.

განაცხადის მაგალითი¶

64-ბიტიანი აპარატურის ტაიმერის მაგალითი: პერიფერიული მოწყობილობები/ტაიმერის_ჯგუფი.

API მითითება¶

სათაურის ფაილი¶

  • დრაივერი/ინკლუზია/დრაივერი/ტიმერი.h

ფუნქციები¶

esp_err_t timer_get_counter_value(timer_group_t ჯგუფის_ნომერი, timer_idx_t ტაიმერის_ნომერი, uint64_t *ტაიმერის_მნიშვნელობა)¶

წაიკითხეთ აპარატურული ტაიმერის მრიცხველის მნიშვნელობა.

დაბრუნება
  • ESP_OK წარმატება
  • ESP_ERR_INVALID_ARG პარამეტრის შეცდომა
პარამეტრები
  • group_numტაიმერის ჯგუფი, 0 TIMERG0-სთვის ან 1 TIMERG1-ისთვის
  • timer_numტაიმერის ინდექსი, 0 hw_timer[0]-სთვის და 1 hw_timer[1]-სთვის

esp_err_t timer_isr_callback_add(timer_group_t ჯგუფის_ნომერი, timer_idx_t ტაიმერის_ნომერი, timer_isr_t isr_handler, void *არგუმენტი, int intr_alloc_flags)¶

დაამატეთ ISR სახელურის უკუკავშირი შესაბამისი ტაიმერისთვის.

თუ intr_alloc_flags-ის მნიშვნელობა ESP_INTR_FLAG_IRAM დაყენებულია, დამმუშავებელი ფუნქცია უნდა გამოცხადდეს IRAM_ATTR ატრიბუტით და მას მხოლოდ IRAM-ში ან ROM-ში არსებული ფუნქციების გამოძახება შეუძლია. მას არ შეუძლია სხვა ტაიმერის API 8-ის, გამოძახება.

შენიშვნა
This ISR handler will be called from an ISR. This ISR handler do not need to handle interrupt status, and should be kept short. If you want to realize some specific applications or write the whole ISR, you can call timer_isr_register(…) to register ISR.
პარამეტრები

  • isr_handlerშეწყვეტის დამმუშავებლის ფუნქცია, ეს არის უკუგამოძახების ფუნქცია.

  • intr_alloc_flags: დროშები, რომლებიც გამოიყენება შეწყვეტის გამოსაყოფად. ერთი ან რამდენიმე (OR წითელი) ESP_INTR_FLAG_* მნიშვნელობა. დამატებითი ინფორმაციისთვის იხილეთ esp_intr_alloc.h.

GATT სერვერი API¶

მიმოხილვა¶

ინსტრუქციები

განაცხადის მაგალითი¶

შემოწმება ბლუთუზი/ბლუდროიდი/ბლე ESP-IDF examples-ის საქაღალდეში, რომელიც შეიცავს შემდეგ დემო ვერსიებს და მათ სახელმძღვანელოებს:

  • ეს არის GATT სერვერის დემო ვერსია და მისი სახელმძღვანელო. ეს დემო ვერსია ქმნის GATT სერვისს ატრიბუტების ცხრილით, რომელიც მომხმარებელს ათავისუფლებს ატრიბუტების ერთმანეთის მიყოლებით დამატების ვალდებულებისგან. ეს არის ატრიბუტების დამატების რეკომენდებული მეთოდი.

    • bluetooth/bluedroid/ble/gatt_server_service_table
    • GATT სერვერის სერვისის ცხრილის მაგალითის დეტალური აღწერა

  • ეს არის GATT სერვერის დემო ვერსია და მისი სახელმძღვანელო. ეს დემო ვერსია ქმნის GATT სერვისს Bluedroid-ის მიერ განსაზღვრული ატრიბუტების ერთმანეთის მიყოლებით დამატებით. ატრიბუტების დამატების რეკომენდებული მეთოდი წარმოდგენილია ზემოთ მოცემულ მაგალითში.

    • Bluetooth/bluedroid/ble/gatt_server
    • GATT სერვერის მაგალითის მიმოხილვა

  • ეს არის BLE SPP-ის მსგავსი დემო ვერსია. ამ დემო ვერსიას, რომელიც GATT სერვერის ფუნქციას ასრულებს, შეუძლია მონაცემების მიღება UART-დან და შემდეგ მონაცემების ავტომატურად გაგზავნა შესაბამის მოწყობილობაზე.

    • Bluetooth/bluedroid/ble/ble_spp_server

esp_err_t esp_ble_gatts_create_service(esp_gatt_if_t გატსი_თუ, esp_gatt_srvc_id_t *სერვისის_იდენტიფიკატორი, uint16_t num_handle)¶

სერვისის შექმნა. სერვისის შექმნის დასრულების შემდეგ, სტატუსისა და სერვისის ID-ის პროფილში შესატყობინებლად გამოიძახება უკუკავშირის მოვლენა BTA_GATTS_CREATE_SRVC_EVT. უკუკავშირის ფუნქციაში მიღებული სერვისის ID უნდა იქნას გამოყენებული სერვისში ჩართული სერვისისა და მახასიათებლების/აღწერილობების დამატებისას.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: ვერ მოხერხდა
პარამეტრები
  • gatts_ifGATT სერვერის წვდომის ინტერფეისი
  • service_id: სერვისის ID.
  • num_handle: ამ სერვისისთვის მოთხოვნილი სახელურების რაოდენობა.

esp_err_t esp_ble_gatts_create_attr_tab(კონსტ esp_gatts_attr_db_t *gatts_attr_db, esp_gatt_if_t გატსი_თუ, uint8_t max_nb_attr, uint8_t srvc_inst_id)¶

დაბრუნება

  • სხვა: ვერ მოხერხდა
პარამეტრები

  • gatts_ifGATT სერვერის წვდომის ინტერფეისი
  • max_nb_attr: სერვისის მონაცემთა ბაზაში დასამატებელი ატრიბუტების რაოდენობა.
  • srvc_inst_id: სერვისის ეგზემპლარის იდენტიფიკატორი

esp_err_t esp_ble_gatts_add_included_service(uint16_t სერვისის_სახელური, uint16_t ჩართული_სერვისის_სახელური)¶

ეს ფუნქცია გამოიძახება ჩართული სერვისის დასამატებლად. ეს ფუნქცია უნდა გამოიძახოთ &#39;esp_ble_gatts_create_service&#39;-სა და &#39;esp_ble_gatts_add_char&#39;-ს შორის. ჩართული სერვისის ჩართვის შემდეგ, უკუკავშირის მოვლენა BTA_GATTS_ADD_INCL_SRVC_EVT მოხსენებული იქნება ჩართული სერვისის ID-ის სახით.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: ვერ მოხერხდა
პარამეტრები
  • service_handle: სერვისის სახელური, რომელსაც უნდა დაემატოს ეს ჩართული სერვისი.
  • included_service_handle: შესატანი სერვისის ID.

esp_err_t esp_ble_gatts_add_char(uint16_t სერვისის_სახელური, esp_bt_uuid_t *char_uuid, esp_gatt_perm_t პერმანენტი, esp_gatt_char_prop_t ქონება, esp_attr_value_t *char_val, esp_attr_control_t *კონტროლი)¶

ეს ფუნქცია გამოიძახება სერვისში მახასიათებლის დასამატებლად.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: ვერ მოხერხდა
პარამეტრები
  • service_handle: სერვისის სახელური, რომელსაც უნდა დაემატოს ეს ჩართული სერვისი.
  • char_uuidდამახასიათებელი UUID.
  • perm: : მახასიათებლის მნიშვნელობის დეკლარაციის ატრიბუტის ნებართვა.
  • propertyდამახასიათებელი თვისებები
  • char_val: : დამახასიათებელი მნიშვნელობა
  • control: : ატრიბუტის პასუხის კონტროლის ბაიტი

esp_err_t esp_ble_gatts_add_char_descr(uint16_t სერვისის_სახელური, esp_bt_uuid_t *აღწერილობა_uuid, esp_gatt_perm_t პერმანენტი, esp_attr_value_t *char_descr_val, esp_attr_control_t *კონტროლი)¶

ეს ფუნქცია გამოიძახება მახასიათებლის აღმწერის დასამატებლად. დასრულების შემდეგ, გამოიძახება უკუკავშირის მოვლენა BTA_GATTS_ADD_DESCR_EVT, რათა მოახდინოს ამ აღმწერის სტატუსისა და საიდენტიფიკაციო ნომრის შეტყობინება.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: ვერ მოხერხდა
პარამეტრები
  • service_handleსერვისის სახელური, რომელსაც უნდა დაემატოს ეს მახასიათებლის აღწერილობა.
  • perm: აღმწერის წვდომის ნებართვა.
  • descr_uuid: UUID-ის აღმწერი.
  • char_descr_val: : მახასიათებლის აღმწერის მნიშვნელობა
  • control: : ატრიბუტის პასუხის კონტროლის ბაიტი

esp_err_t esp_ble_gatts_send_indicate(esp_gatt_if_t გატსი_თუ, uint16_t conn_id, uint16_t attr_handle, uint16_t value_len, uint8_t *ღირებულება, bool საჭიროა_დადასტურება)¶

GATT კლიენტისთვის შეტყობინების ან მითითების გაგზავნა. need_confirm პარამეტრის დაყენებით, როგორც false, შეტყობინება გაიგზავნება, სხვა შემთხვევაში - მითითება.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: ვერ მოხერხდა
პარამეტრები
  • gatts_ifGATT სერვერის წვდომის ინტერფეისი
  • conn_id: - კავშირის ID-ის მითითება.
  • attr_handle: - ატრიბუტის სახელური, რომელიც მიუთითებს.
  • value_len: - მიუთითებს მნიშვნელობის სიგრძეზე.
  • value: მითითებული მნიშვნელობა.
  • need_confirm: - საჭიროა თუ არა დადასტურება. false აგზავნის GATT შეტყობინებას, true აგზავნის GATT მითითებას.

esp_err_t esp_ble_gatts_send_response(esp_gatt_if_t გატსი_თუ, uint16_t conn_id, uint32_t trans_id, esp_gatt_status_t სტატუსი, esp_gatt_rsp_t *rsp)¶

esp_err_t esp_bt_hf_register_callback(esp_hf_cb_t უკუკავშირი)¶

აპლიკაციის უკუკავშირის ფუნქციის HFP AG მოდულში რეგისტრაცია. ეს ფუნქცია უნდა გამოიძახოს მხოლოდ esp_bluedroid_enable() ფუნქციის წარმატებით დასრულების შემდეგ, რომელსაც HFP AG იყენებს.

დაბრუნება
  • ESP_OK: წარმატება
  • ESP_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: თუ უკუკავშირი არის NULL ფუნქციის მაჩვენებელი
პარამეტრები
  • callbackHFP AG მოვლენის უკუგამოძახების ფუნქცია

  • ESP_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: სხვები

typedef void (*esp_hf_incoming_data_cb_t)(კონსტ uint8_t *buf, uint32_t len)¶

AG შემომავალი მონაცემების უკუკავშირის ფუნქცია, უკუკავშირი სასარგებლოა Voice Over HCI-ის შემთხვევაში.

პარამეტრები
  • buf: : შემომავალი მონაცემების მაჩვენებელი (HCI სინქრონული მონაცემთა პაკეტის დატვირთვა), ბუფერი გამოყოფილია Bluetooth პროტოკოლის დასტაში და გათავისუფლდება უკუკავშირის გამოძახების დასრულების შემდეგ.
  • len: : ზომა (ბაიტებში) ბუფში

typedef uint32_t (*esp_hf_outgoing_data_cb_t)(uint8_t *buf, uint32_t len)¶

გამავალი მონაცემების უკუკავშირის ფუნქციის შემთხვევაში, უკუკავშირი სასარგებლოა Voice Over HCI-ის შემთხვევაში. აუდიო კავშირის დამყარების შემდეგ და აპლიკაციის დონე მოამზადებს მონაცემებს გასაგზავნად, ქვედა დონე გამოიძახებს ამ ფუნქციას მონაცემების წასაკითხად და შემდეგ გასაგზავნად. ეს უკუკავშირი უნდა იყოს დანერგილი არაბლოკირებად და თუ მონაცემები არ არის საკმარისი, დაბრუნებული მნიშვნელობა 0 იქნება.

პარამეტრები
  • buf: : შემომავალი მონაცემების მაჩვენებელი (HCI სინქრონული მონაცემთა პაკეტის დატვირთვა), ბუფერი გამოყოფილია Bluetooth პროტოკოლის დასტაში და გათავისუფლდება უკუკავშირის გამოძახების დასრულების შემდეგ.
  • len: : ზომა (ბაიტებში) ბუფში
  • length: წარმატებით წაკითხული მონაცემების

SPP API¶

მიმოხილვა¶

ინსტრუქციები

განაცხადის მაგალითი¶

შემოწმება ბლუთუზი/ბლუდროიდი/კლასიკური_ბტ ESP-IDF examples-ში არსებული საქაღალდე, რომელიც შეიცავს შემდეგ აპლიკაციას:

  • ეს არის SPP დემო ვერსია. ამ დემო ვერსიას შეუძლია სერვისის აღმოჩენა, დაკავშირება, SPP მონაცემების გაგზავნა და მიღება. bluetooth/bluedroid/classic_bt/bt_spp_acceptor, bluetooth/bluedroid/classic_bt/bt_spp_initiator

API მითითება¶

სათაურის ფაილი¶

  • bt/host/bluedroid/api/include/api/esp_spp_api.h

ფუნქციები¶

esp_err_t esp_spp_register_callback(esp_spp_cb_t უკუკავშირი)¶

ეს ფუნქცია გამოიძახება SPP მოდულის გამოყენებით უკუკავშირის ინიციალიზაციისთვის.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: წარუმატებელი
პარამეტრები
  • callback: მაჩვენებელი init უკუკავშირის ფუნქციისკენ.

esp_err_t esp_spp_init(esp_spp_mode_t რეჟიმი)¶

ეს ფუნქცია გამოიძახება SPP-ის ინიციალიზაციისთვის.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: წარუმატებელი
პარამეტრები
  • modeაირჩიეთ SPP, ESP_SPP_MODE_CB ან ESP_SPP_MODE_VFS რეჟიმი.

esp_err_t esp_spp_deinit(void)¶

ეს ფუნქცია გამოიძახება SPP-ის გასაუნიტირებლად.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: წარუმატებელი

esp_err_t esp_spp_start_discovery(esp_bd_addr_t bd_addr)¶

ეს ფუნქცია გამოიძახება მოცემული თანატოლი მოწყობილობის მიერ მოწოდებული სერვისებისთვის სერვისის აღმოჩენის შესასრულებლად. ოპერაციის დასრულების შემდეგ, უკუკავშირის ფუნქცია გამოიძახება ESP_SPP_DISCOVERY_COMP_EVT-ით.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: წარუმატებელი
პარამეტრები
  • bd_addrდისტანციური მოწყობილობის Bluetooth მოწყობილობის მისამართი.

esp_err_t esp_spp_connect(esp_spp_sec_t sec_mask, esp_spp_role_t როლი, uint8_t დისტანციური_სკნ, esp_bd_addr_t peer_bd_addr)¶

ეს ფუნქცია ამყარებს SPP კავშირს დისტანციურ BD მისამართთან. როდესაც კავშირი იწყება ან ვერ ხერხდება, უკუკავშირი გამოიძახება ESP_SPP_CL_INIT_EVT ფუნქციით. როდესაც კავშირი დამყარდება ან ვერ ხერხდება, უკუკავშირი გამოიძახება ESP_SPP_OPEN_EVT ფუნქციით.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: წარუმატებელი
პარამეტრები
  • sec_maskუსაფრთხოების პარამეტრების ნიღაბი. რეკომენდებულია მხოლოდ ESP_SPP_SEC_NONE, ESP_SPP_SEC_AUTHORIZE ან ESP_SPP_SEC_AUTHENTICATE-ის გამოყენება.
  • role: ბატონი თუ მონა.
  • remote_scnდისტანციური მოწყობილობა Bluetooth მოწყობილობა SCN.
  • peer_bd_addrდისტანციური მოწყობილობის Bluetooth მოწყობილობის მისამართი.

esp_err_t esp_spp_disconnect(uint32_t სახელური)¶

ეს ფუნქცია ხურავს SPP კავშირს.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: წარუმატებელი
პარამეტრები
  • handle: დაკავშირების სახელური.

esp_err_t esp_spp_start_srv(esp_spp_sec_t sec_mask, esp_spp_role_t როლი, uint8_t local_scn, კონსტ char *სახელი)¶

ეს ფუნქცია ქმნის SPP სერვერს და იწყებს SPP კავშირის მოთხოვნის მოსმენას დისტანციური Bluetooth მოწყობილობიდან. სერვერის წარმატებით გაშვების შემთხვევაში, უკუკავშირი გამოიძახება ESP_SPP_START_EVT ფუნქციით. კავშირის დამყარების შემთხვევაში, უკუკავშირი გამოიძახება ESP_SPP_SRV_OPEN_EVT ფუნქციით.

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: წარუმატებელი
პარამეტრები
  • sec_maskუსაფრთხოების პარამეტრების ნიღაბი. რეკომენდებულია მხოლოდ ESP_SPP_SEC_NONE, ESP_SPP_SEC_AUTHORIZE ან ESP_SPP_SEC_AUTHENTICATE-ის გამოყენება.
  • role: ბატონი თუ მონა.
  • local_scnკონკრეტული არხი, რომლის მიღებაც გსურთ. თუ არხი 0-ის ტოლია, ნიშნავს, რომ ნებისმიერი არხი უნდა მიიღოთ.
  • name: სერვერის სახელი.

esp_err_t esp_spp_write(uint32_t სახელური, int ლენ, uint8_t *p_მონაცემები)¶

დაბრუნება
  • ESP_OK: წარმატება
  • სხვა: წარუმატებელი
პარამეტრები
  • handle: დაკავშირების სახელური.
  • lenჩაწერილი მონაცემების სიგრძე.
  • p_data: ჩაწერილი მონაცემები.

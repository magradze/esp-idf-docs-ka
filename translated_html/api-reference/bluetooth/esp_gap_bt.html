esp_err_t esp_bt_gap_set_scan_mode(esp_bt_connection_mode_t c_mode, esp_bt_discovery_mode_t d_mode)¶

მემკვიდრეობითი Bluetooth-ისთვის აღმოჩენადობისა და დაკავშირების რეჟიმის დაყენება. ეს ფუნქცია უნდა გამოიძახოს esp_bluedroid_enable() ფუნქციის წარმატებით დასრულების შემდეგ.

დაბრუნება
  • ESP_OK: წარმატებით დასრულდა
  • ESP_ERR_INVALID_ARG: თუ არგუმენტი არასწორია
  • ESP_ERR_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: სხვები
პარამეტრები
  • c_mode: : esp_bt_connection_mode_t-ის ერთ-ერთი ჩამოთვლა
  • d_mode: : esp_bt_discovery_mode_t-ის ერთ-ერთი ჩამოთვლა

esp_err_t esp_bt_gap_start_discovery(esp_bt_inq_mode_t რეჟიმი, uint8_t inq_len, uint8_t num_rsps)¶

მოწყობილობის აღმოჩენის დაწყება. ეს ფუნქცია უნდა გამოიძახონ esp_bluedroid_enable() ფუნქციის წარმატებით დასრულების შემდეგ. თუ აღმოჩენა დაიწყო ან შეჩერდა, esp_bt_gap_cb_t გამოიძახება ESP_BT_GAP_DISC_STATE_CHANGED_EVT-ით. თუ აღმოჩენის შედეგი მიღებულია, esp_bt_gap_cb_t გამოიძახება ESP_BT_GAP_DISC_RES_EVT-ით.

დაბრუნება
  • ESP_OK: წარმატებით დასრულდა
  • ESP_ERR_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_ERR_INVALID_ARG: თუ მოწოდებულია არასწორი პარამეტრები
  • ESP_FAIL: სხვები
პარამეტრები
  • mode: - მოთხოვნის რეჟიმი
  • inq_len: - მოთხოვნის ხანგრძლივობა 1.28 წამიან ერთეულებში, 0x01-დან 0x30-მდე დიაპაზონში
  • num_rsps: - მოთხოვნაზე მიღებული პასუხების რაოდენობა, მნიშვნელობა 0 მიუთითებს პასუხების შეუზღუდავ რაოდენობაზე

esp_err_t esp_bt_gap_cancel_discovery(void)¶

მოწყობილობის აღმოჩენის გაუქმება. ეს ფუნქცია უნდა გამოიძახონ esp_bluedroid_enable() ფუნქციის წარმატებით დასრულების შემდეგ. თუ აღმოჩენა შეჩერებულია, esp_bt_gap_cb_t გამოიძახება ESP_BT_GAP_DISC_STATE_CHANGED_EVT-ით.

დაბრუნება
  • ESP_OK: წარმატებით დასრულდა
  • ESP_ERR_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: სხვები

esp_err_t esp_bt_gap_get_remote_service_record(esp_bd_addr_t remote_bda, esp_bt_uuid_t *uuid)¶

დაბრუნება
  • ESP_OK: წარმატებით დასრულდა
  • ESP_ERR_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_ERR_INVALID_ARG: თუ პარამეტრი არასწორია
  • ESP_FAIL: სხვები
პარამეტრები
  • cod: - მოწყობილობის კლასი

მოწყობილობის ნომერი მიიღეთ თანატოლი მოწყობილობის უსაფრთხოების მონაცემთა ბაზის სიიდან. ის დაუყოვნებლივ დააბრუნებს მოწყობილობის დაკავშირებულ ნომერს.

esp_err_t esp_bt_gap_get_bond_device_list(int *dev_num, esp_bd_addr_t *განვითარების სია)¶

მოწყობილობის მიღება თანატოლი მოწყობილობების უსაფრთხოების მონაცემთა ბაზის სიიდან. ის დაუყოვნებლივ დააბრუნებს მოწყობილობის შესახებ დაკავშირებულ ინფორმაციას.

დაბრუნება
  • ESP_OK: წარმატებით დასრულდა
  • ESP_ERR_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: სხვები
პარამეტრები
  • dev_num: შეყვანის სახით მიუთითეთ dev_list მასივის (ბუფერის) ზომა. თუ dev_num საკმარისად დიდია, ეს ნიშნავს, რომ გამომავალი რიცხვი რეალურია. ვარაუდობენ, რომ dev_num მნიშვნელობა ტოლია esp_ble_get_bond_device_num()-ის.
  • dev_list: მასივი (ბუფერი) esp_bd_addr_t ტიპი. გამოიყენება დაკავშირებული მოწყობილობების მისამართის შესანახად. dev_list უნდა გამოიყოს ამ API გამომძახებლის მიერ.

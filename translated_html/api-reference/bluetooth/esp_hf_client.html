esp_err_t esp_hf_client_register_callback(esp_hf_client_cb_t უკუკავშირი)¶

აპლიკაციის უკუკავშირის ფუნქციის HFP კლიენტის მოდულში რეგისტრაცია. ეს ფუნქცია უნდა გამოიძახოს მხოლოდ esp_bluedroid_enable() ფუნქციის წარმატებით დასრულების შემდეგ, რომელსაც იყენებს HFP კლიენტი.

დაბრუნება
  • ESP_OK: წარმატება
  • ESP_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: თუ უკუკავშირი არის NULL ფუნქციის მაჩვენებელი
პარამეტრები
  • callbackHFP კლიენტის მოვლენის უკუკავშირის ფუნქცია

esp_err_t esp_hf_client_init(void)¶

Bluetooth HFP კლიენტის მოდულის ინიციალიზაცია. ეს ფუნქცია უნდა გამოიძახოს esp_bluedroid_enable() ოპერაციის წარმატებით დასრულების შემდეგ.

დაბრუნება
  • ESP_OK: თუ ინიციალიზაციის მოთხოვნა წარმატებით გაიგზავნა
  • ESP_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: სხვები

esp_err_t esp_hf_client_deinit(void)¶

HFP კლიენტის მოდულის დეინიციალიზაცია. ეს ფუნქცია უნდა გამოიძახოს მხოლოდ esp_bluedroid_enable() ფუნქციის წარმატებით დასრულების შემდეგ.

დაბრუნება
  • ESP_OK: წარმატება
  • ESP_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: სხვები

esp_err_t esp_hf_client_send_chld_cmd(esp_hf_chld_type_t ბავშვი, int იდქსი)¶

გაგზავნეთ ზარის შეკავების და მრავალმხრივი ბრძანებები, ან გაძლიერებული ზარის კონტროლის ბრძანებები (გამოიყენეთ AT+CHLD). ამ API გამოყენების წინაპირობად, AG-სთან უნდა არსებობდეს მომსახურების დონის კავშირი.

დაბრუნება

  • ESP_INVALID_STATE: თუ Bluetooth სტეკი ჯერ არ არის ჩართული
  • ESP_FAIL: სხვები
პარამეტრები
  • chld: AT+CHLD ზარის დაკავება და მრავალმხრივი დამუშავება AT ბრძანება.
  • idx: გამოიყენება გაძლიერებული ზარის კონტროლის მექანიზმებში, გამოიყენება თუ chld არის ESP_HF_CHLD_TYPE_REL_X ან ESP_HF_CHLD_TYPE_PRIV_X

typedef void (*esp_hf_client_incoming_data_cb_t)(კონსტ uint8_t *buf, uint32_t len)¶

HFP კლიენტის შემომავალი მონაცემების უკუკავშირის ფუნქცია, უკუკავშირი სასარგებლოა Voice Over HCI-ის შემთხვევაში.

პარამეტრები
  • buf: : შემომავალი მონაცემების მაჩვენებელი (HCI სინქრონული მონაცემთა პაკეტის დატვირთვა), ბუფერი გამოყოფილია Bluetooth პროტოკოლის დასტაში და გათავისუფლდება უკუკავშირის გამოძახების დასრულების შემდეგ.
  • len: : ზომა (ბაიტებში) ბუფში

typedef uint32_t (*esp_hf_client_outgoing_data_cb_t)(uint8_t *buf, uint32_t len)¶

HFP კლიენტის გამავალი მონაცემების უკუკავშირის ფუნქცია სასარგებლოა Voice Over HCI-ის შემთხვევაში. აუდიო კავშირის დამყარების შემდეგ და აპლიკაციის დონე მოამზადებს მონაცემებს გასაგზავნად, ქვედა დონე გამოიძახებს ამ ფუნქციას მონაცემების წასაკითხად და შემდეგ გასაგზავნად. ეს უკუკავშირი უნდა იყოს დანერგილი არაბლოკირებად და თუ მონაცემები არ არის საკმარისი, დაბრუნებული მნიშვნელობა 0 იქნება.

პარამეტრები
  • buf: : შემომავალი მონაცემების მაჩვენებელი (HCI სინქრონული მონაცემთა პაკეტის დატვირთვა), ბუფერი გამოყოფილია Bluetooth პროტოკოლის დასტაში და გათავისუფლდება უკუკავშირის გამოძახების დასრულების შემდეგ.
  • len: : ზომა (ბაიტებში) ბუფში
  • length: წარმატებით წაკითხული მონაცემების

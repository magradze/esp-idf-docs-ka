esp_err_t esp_eth_driver_install(კონსტ esp_eth_config_t *კონფიგურაცია, esp_eth_handle_t *out_hdl)¶

დააინსტალირეთ Ethernet დრაივერი.

დაბრუნება
  • ESP_OK: esp_eth დრაივერი წარმატებით დაინსტალირდა
  • ESP_ERR_INVALID_ARG: esp_eth დრაივერის ინსტალაცია ვერ მოხერხდა არასწორი არგუმენტის გამო.
  • ESP_ERR_NO_MEM: esp_eth დრაივერის ინსტალაცია ვერ მოხერხდა, რადგან დრაივერის მეხსიერება არ არის.
  • ESP_FAIL: esp_eth დრაივერის ინსტალაცია ვერ მოხერხდა სხვა შეცდომის გამო.
პარამეტრები
  • configEthernet დრაივერის კონფიგურაცია
  • out_hdl: Ethernet დრაივერის სახელური

esp_err_t esp_eth_driver_uninstall(esp_eth_handle_t HDL)¶

Ethernet დრაივერის დეინსტალაცია.

შენიშვნა
It’s not recommended to uninstall Ethernet driver unless it won’t get used any more in application code. To uninstall Ethernet driver, you have to make sure, all references to the driver are released. Ethernet driver can only be uninstalled successfully when reference counter equals to one.
დაბრუნება
  • ESP_OK: esp_eth დრაივერის წარმატებით წაშლა
  • ESP_ERR_INVALID_ARG: esp_eth დრაივერის დეინსტალაცია ვერ მოხერხდა არასწორი არგუმენტის გამო.
  • ESP_ERR_INVALID_STATE: esp_eth დრაივერის დეინსტალაცია ვერ მოხერხდა, რადგან მას ერთზე მეტი მითითება აქვს.
  • ESP_FAIL: esp_eth დრაივერის დეინსტალაცია ვერ მოხერხდა სხვა შეცდომის გამო.
პარამეტრები
  • hdl: Ethernet დრაივერის სახელური

esp_err_t esp_eth_start(esp_eth_handle_t HDL)¶

Ethernet დრაივერის გაშვება მხოლოდ დამოუკიდებელ რეჟიმში (ანუ TCP/IP დასტის გარეშე)

შენიშვნა
This API will start driver state machine and internal software timer (for checking link status).
დაბრუნება
  • ESP_OK: esp_eth დრაივერის წარმატებით გაშვება
  • ESP_ERR_INVALID_ARG: esp_eth დრაივერის გაშვება ვერ მოხერხდა არასწორი არგუმენტის გამო.
  • ESP_ERR_INVALID_STATE: esp_eth დრაივერის გაშვება ვერ მოხერხდა, რადგან დრაივერი უკვე გაშვებულია.
  • ESP_FAIL: esp_eth დრაივერის გაშვება ვერ მოხერხდა სხვა შეცდომის გამო.
პარამეტრები
  • hdl: Ethernet დრაივერის სახელური

esp_err_t esp_eth_stop(esp_eth_handle_t HDL)¶

Ethernet დრაივერის შეჩერება.

შენიშვნა
This function does the oppsite operation of esp_eth_start.
დაბრუნება
  • ESP_OK: esp_eth დრაივერის წარმატებით შეჩერება
  • ESP_ERR_INVALID_ARG: esp_eth დრაივერის გაჩერება ვერ მოხერხდა არასწორი არგუმენტის გამო.
  • ESP_ERR_INVALID_STATE: esp_eth დრაივერის გაჩერება ვერ მოხერხდა, რადგან დრაივერი ჯერ არ არის ჩართული.
  • ESP_FAIL: esp_eth დრაივერის გაჩერება ვერ მოხერხდა სხვა შეცდომის გამო.
პარამეტრები
  • hdl: Ethernet დრაივერის სახელური

esp_err_t esp_eth_update_input_path(esp_eth_handle_t HDL, esp_err_t (*დასტის_შეყვანა)(esp_eth_handle_t hdl, uint8_t *buffer, uint32_t length, void *priv), void *პრივილეგია, )¶

Ethernet მონაცემთა შეყვანის გზის განახლება (ანუ მიუთითეთ, სად უნდა გადაეცეს შეყვანის ბუფერი)

შენიშვნა
After install driver, Ethernet still don’t know where to deliver the input buffer. In fact, this API registers a callback function which get invoked when Ethernet received new packets.
დაბრუნება
  • ESP_OK: შეყვანის გზის წარმატებით განახლება
  • ESP_ERR_INVALID_ARG: შეყვანის გზის განახლება ვერ მოხერხდა არასწორი არგუმენტის გამო.
  • ESP_FAIL: შეყვანის გზის განახლება ვერ მოხერხდა სხვა შეცდომის გამო
პარამეტრები
  • hdl: Ethernet დრაივერის სახელური
  • stack_input: ფუნქციის მაჩვენებელი, რომელიც ასრულებს რეალურ პროცესს შემომავალ პაკეტებზე
  • priv: კერძო რესურსი, რომელიც გადაეცემა stack_input უკუკავშირი ყოველგვარი ცვლილების გარეშე

esp_err_t esp_eth_receive(esp_eth_handle_t HDL, uint8_t *ბუფი, uint32_t *სიგრძე)¶

ზოგადი მიღება.

შენიშვნა
Before this function got invoked, the value of “length” should set by user, equals the size of buffer. After the function returned, the value of “length” means the real length of received data.
დაბრუნება
  • ESP_OK: ფრეიმ ბუფერის წარმატებით მიღება
  • ESP_ERR_INVALID_ARG: ჩარჩო ბუფერის მიღება ვერ მოხერხდა არასწორი არგუმენტის გამო.
  • ESP_ERR_INVALID_SIZE: შეყვანილი ბუფერის ზომა არ არის საკმარისი შემომავალი მონაცემების შესანახად. ამ შემთხვევაში, დაბრუნებული „სიგრძის“ მნიშვნელობა მიუთითებს შემომავალი მონაცემების რეალურ ზომაზე.
  • ESP_FAIL: ჩარჩოს ბუფერის მიღება ვერ მოხერხდა სხვა შეცდომის გამო
პარამეტრები
  • hdl: Ethernet დრაივერის სახელური

esp_err_t esp_eth_increase_reference(esp_eth_handle_t HDL)¶

Ethernet დრაივერის მითითების გაზრდა.

შენიშვნა
Ethernet driver handle can be obtained by os timer, netif, etc. It’s dangerous when thread A is using Ethernet but thread B uninstall the driver. Using reference counter can prevent such risk, but care should be taken, when you obtain Ethernet driver, this API must be invoked so that the driver won’t be uninstalled during your using time.
დაბრუნება
  • ESP_OK: მითითების წარმატებით გაზრდა
  • ESP_ERR_INVALID_ARG: მითითების გაზრდა ვერ მოხერხდა არასწორი არგუმენტის გამო.
პარამეტრები
  • hdl: Ethernet დრაივერის სახელური

  • ESP_FAIL: პაკეტის მიღება ვერ მოხერხდა სხვა შეცდომის გამო

ESP-NOW¶

[中文]

მიმოხილვა¶

ESP-NOW არის უკავშირო Wi-Fi საკომუნიკაციო პროტოკოლის სახეობა, რომელიც განისაზღვრება Espressif-ის მიერ. ESP-NOW-ში, აპლიკაციის მონაცემები კაფსულირებულია მომწოდებლის სპეციფიკურ სამოქმედო ჩარჩოში და შემდეგ გადაიცემა ერთი Wi-Fi მოწყობილობიდან მეორეზე კავშირის გარეშე. CBC-MAC პროტოკოლის (CCMP) გამოყენებით CTR გამოიყენება სამოქმედო ჩარჩოს უსაფრთხოების მიზნით დასაცავად. ESP-NOW ფართოდ გამოიყენება ჭკვიან განათებაში, დისტანციურ მართვაში, სენსორებში და ა.შ.

ჩარჩოს ფორმატი¶

ESP-NOW იყენებს მომწოდებლის სპეციფიკურ მოქმედების ჩარჩოს ESP-NOW მონაცემების გადასაცემად. ESP-NOW-ის ნაგულისხმევი ბიტური სიჩქარეა 1 Mbps. მომწოდებლის სპეციფიკური მოქმედების ჩარჩოს ფორმატი შემდეგია:

------------------------------------------------------------------------------------------------------------
| MAC Header | Category Code | Organization Identifier | Random Values | Vendor Specific Content |   FCS   |
------------------------------------------------------------------------------------------------------------
  24 bytes         1 byte              3 bytes               4 bytes             7~255 bytes        4 bytes
  • კატეგორიის კოდი: კატეგორიის კოდის ველში დაყენებულია მნიშვნელობა (127), რომელიც მიუთითებს მომწოდებლის სპეციფიკურ კატეგორიაზე.
  • ორგანიზაციის იდენტიფიკატორი: ორგანიზაციის იდენტიფიკატორი შეიცავს უნიკალურ იდენტიფიკატორს (0x18fe34), რომელიც წარმოადგენს Espressif-ის მიერ გამოყენებულ MAC მისამართის პირველ სამ ბაიტს.
  • შემთხვევითი მნიშვნელობა: შეყვანილი შემთხვევითი მნიშვნელობა გამოიყენება სარელეო შეტევების თავიდან ასაცილებლად.
  • მომწოდებლის სპეციფიკური კონტენტი: მომწოდებლის სპეციფიკური კონტენტი შეიცავს მომწოდებლის სპეციფიკურ ველებს შემდეგნაირად:
-------------------------------------------------------------------------------
| Element ID | Length | Organization Identifier | Type | Version |    Body    |
-------------------------------------------------------------------------------
    1 byte     1 byte            3 bytes         1 byte   1 byte   0~250 bytes
  • ელემენტის ID: ელემენტის ID ველში დაყენებულია მნიშვნელობა (221), რაც მიუთითებს მომწოდებლის სპეციფიკურ ელემენტზე.
  • სიგრძე: სიგრძე არის ორგანიზაციის იდენტიფიკატორის, ტიპის, ვერსიისა და ძირითადი ნაწილის მთლიანი სიგრძე.
  • ორგანიზაციის იდენტიფიკატორი: ორგანიზაციის იდენტიფიკატორი შეიცავს უნიკალურ იდენტიფიკატორს (0x18fe34), რომელიც წარმოადგენს Espressif-ის მიერ გამოყენებულ MAC მისამართის პირველ სამ ბაიტს.
  • ტიპი: ტიპის ველს აქვს მნიშვნელობა (4), რაც მიუთითებს ESP-NOW-ზე.
  • ვერსია: ველი „ვერსია“ დაყენებულია ESP-NOW-ის ვერსიაზე.
  • სხეული: სხეული შეიცავს ESP-NOW მონაცემებს.

რადგან ESP-NOW არ არის კავშირის გარეშე, MAC სათაური ოდნავ განსხვავდება სტანდარტული ჩარჩოების სათაურისგან. FrameControl ველის FromDS და ToDS ბიტები ორივე 0-ის ტოლია. პირველი მისამართის ველი დაყენებულია დანიშნულების მისამართზე. მეორე მისამართის ველი დაყენებულია წყაროს მისამართზე. მესამე მისამართის ველი დაყენებულია სამაუწყებლო მისამართზე (0xff:0xff:0xff:0xff:0xff:0xff).

უსაფრთხოება¶

ESP-NOW uses the CCMP method, which is described in IEEE Std. 802.11-2012, to protect the vendor-specific action frame. The Wi-Fi device maintains a Primary Master Key (PMK) and several Local Master Keys (LMK). The lengths of both PMK and LMk are 16 bytes.
  • PMK გამოიყენება LMK-ის AES-128 ალგორითმით დაშიფვრისთვის. ზარი esp_now_set_pmk() PMK-ის დასაყენებლად. თუ PMK არ არის დაყენებული, გამოყენებული იქნება ნაგულისხმევი PMK.
  • დაწყვილებული მოწყობილობის LMK გამოიყენება მომწოდებლის სპეციფიკური მოქმედების ჩარჩოს CCMP მეთოდით დაშიფვრისთვის. სხვადასხვა LMK-ების მაქსიმალური რაოდენობა ექვსია. თუ დაწყვილებული მოწყობილობის LMK არ არის დაყენებული, მომწოდებლის სპეციფიკური მოქმედების ჩარჩო არ დაიშიფრება.

მულტიკასტის მომწოდებლისთვის სპეციფიკური მოქმედების ჩარჩოს დაშიფვრა მხარდაჭერილი არ არის.

ინიციალიზაცია და დეინიციალიზაცია¶

ზარი esp_now_init() ESP-NOW-ის ინიციალიზაციისთვის და esp_now_deinit() ESP-NOW-ის დეინიციალიზაციისთვის. ESP-NOW მონაცემები უნდა გადაიცეს Wi-Fi გაშვების შემდეგ, ამიტომ რეკომენდებულია Wi-Fi გაშვება ESP-NOW-ის ინიციალიზაციამდე და Wi-Fi შეჩერება ESP-NOW-ის დეინიციალიზაციის შემდეგ. როდესაც esp_now_deinit() გამოძახების შემთხვევაში, დაწყვილებული მოწყობილობების ყველა ინფორმაცია წაიშლება.

დაწყვილებული მოწყობილობის დამატება¶

ზარი esp_now_add_peer() ამ მოწყობილობაზე მონაცემების გაგზავნამდე, მოწყობილობის დასამატებლად დაწყვილებული მოწყობილობების სიაში. დაწყვილებული მოწყობილობების მაქსიმალური რაოდენობა ოცი. თუ უსაფრთხოება ჩართულია, LMK უნდა იყოს დაყენებული. თქვენ შეგიძლიათ ESP-NOW მონაცემების გაგზავნა როგორც სადგურის, ასევე SoftAP ინტერფეისის საშუალებით. ESP-NOW მონაცემების გაგზავნამდე დარწმუნდით, რომ ინტერფეისი ჩართულია. მაუწყებლობის მონაცემების გაგზავნამდე უნდა დაემატოს მოწყობილობა, რომელსაც აქვს გადაცემის MAC მისამართი. დაწყვილებული მოწყობილობების არხის დიაპაზონი 0-დან 14-მდეა. თუ არხი დაყენებულია 0-ზე, მონაცემები გაიგზავნება მიმდინარე არხზე. წინააღმდეგ შემთხვევაში, არხი უნდა დაყენდეს, როგორც არხი, რომელზეც ლოკალური მოწყობილობაა.

ESP-NOW მონაცემების გაგზავნა¶

ზარი esp_now_send() ESP-NOW მონაცემების გასაგზავნად და esp_now_register_send_cb უკუკავშირის ფუნქციის რეგისტრაციისთვის. ის დააბრუნებს ESP_NOW_SEND_SUCCESS უკუკავშირის ფუნქციის გაგზავნისას, თუ მონაცემები წარმატებით მიიღება MAC დონეზე. წინააღმდეგ შემთხვევაში, ის დააბრუნებს ESP_NOW_SEND_FAILESP-NOW-ის მიერ მონაცემების გაგზავნის შეუძლებლობას რამდენიმე მიზეზი შეიძლება იწვევდეს. მაგალითად, დანიშნულების მოწყობილობა არ არსებობს; მოწყობილობების არხები არ არის იგივე; ეთერში გადაცემისას მოქმედების ჩარჩო იკარგება და ა.შ. არ არის გარანტირებული, რომ აპლიკაციის დონე მიიღებს მონაცემებს. საჭიროების შემთხვევაში, ESP-NOW მონაცემების მიღებისას, უკან გააგზავნეთ დადასტურების მონაცემები. თუ დადასტურების მონაცემების ვადის ამოწურვას იღებთ, ხელახლა გადაეცით ESP-NOW მონაცემები. დუბლიკატი მონაცემების წასაშლელად ESP-NOW მონაცემებს ასევე შეიძლება მიენიჭოს თანმიმდევრობის ნომერი.

თუ ESP-NOW მონაცემების დიდი რაოდენობაა გასაგზავნი, დარეკეთ esp_now_send() ერთდროულად 250 ბაიტზე ნაკლები ან ტოლი მონაცემების გაგზავნა. გაითვალისწინეთ, რომ ორ ESP-NOW მონაცემს შორის ძალიან მოკლე ინტერვალმა შეიძლება გამოიწვიოს უკუკავშირის ფუნქციის გაგზავნის დარღვევა. ამიტომ, რეკომენდებულია შემდეგი ESP-NOW მონაცემების გაგზავნა წინა გაგზავნის უკუკავშირის ფუნქციის გაგზავნის შემდეგ. გაგზავნის უკუკავშირის ფუნქცია მუშაობს მაღალი პრიორიტეტის მქონე Wi-Fi დავალებიდან. ამიტომ, ნუ შეასრულებთ ხანგრძლივ ოპერაციებს უკუკავშირის ფუნქციაში. ამის ნაცვლად, განათავსეთ საჭირო მონაცემები რიგში და დაამუშავეთ ისინი დაბალი პრიორიტეტის მქონე დავალებიდან.

ESP-NOW მონაცემების მიღება¶

ზარი esp_now_register_recv_cb მიმღები უკუკავშირის ფუნქციის რეგისტრაციისთვის. ESP-NOW-ის მიღებისას გამოიძახეთ მიმღები უკუკავშირის ფუნქცია. მიმღები უკუკავშირის ფუნქცია ასევე მუშაობს Wi-Fi დავალებიდან. ამიტომ, ნუ შეასრულებთ ხანგრძლივ ოპერაციებს უკუკავშირის ფუნქციაში. ამის ნაცვლად, განათავსეთ საჭირო მონაცემები რიგში და დაამუშავეთ ისინი დაბალი პრიორიტეტის დავალებიდან.

esp_err_t esp_now_send(კონსტ uint8_t *peer_addr, კონსტ uint8_t *მონაცემები, size_t ლენ)¶

ESPNOW-ის მონაცემების გაგზავნა.

ყურადღება
1. If peer_addr is not NULL, send data to the peer whose MAC address matches peer_addr
ყურადღება
2. If peer_addr is NULL, send data to all of the peers that are added to the peer list
ყურადღება
3. The maximum length of data must be less than ESP_NOW_MAX_DATA_LEN
ყურადღება
4. The buffer pointed to by data argument does not need to be valid after esp_now_send returns
დაბრუნება
  • ESP_OK: წარმატება
  • ESP_ERR_ESPNOW_NOT_INIT: ESPNOW არ არის ინიციალიზებული
  • ESP_ERR_ESPNOW_ARG: არასწორი არგუმენტი
  • ESP_ERR_ESPNOW_INTERNAL: შიდა შეცდომა
  • ESP_ERR_ESPNOW_NO_MEM: მეხსიერება ამოწურულია
  • ESP_ERR_ESPNOW_NOT_FOUND : ვერ მოიძებნა თანაბართან კავშირი
  • ESP_ERR_ESPNOW_IF: მიმდინარე WiFi ინტერფეისი არ ემთხვევა თანატოლის ინტერფეისს.
პარამეტრები
  • peer_addr: თანატოლის MAC მისამართი
  • data: გასაგზავნი მონაცემები
  • len: მონაცემების სიგრძე
